# 数据工程与 Tokenizer

> **前置知识**：[预训练语言模型演进](../AI学习_04_预训练语言模型演进/README.md)
>
> **本节目标**：理解数据质量和 Tokenizer 为什么会直接影响模型能力、成本和行为。

---

## 1. 数据决定模型会学什么

模型不是凭空聪明。训练数据里有什么，它就更可能学到什么。

高质量数据能带来：

- 更好的语言表达。
- 更可靠的知识模式。
- 更少重复和污染。
- 更强任务覆盖。

低质量数据会让模型学到噪声、偏见、错误格式甚至有害内容。

## 2. 数据清洗不是打扫卫生那么简单

数据工程要处理很多问题：

- 去重：避免模型过度记住重复内容。
- 过滤：去掉垃圾文本、乱码、低质量页面。
- 去污染：避免测试集答案混入训练集。
- 配比：不同来源数据如何混合。
- 安全处理：减少有害和敏感内容。

数据工程常常比模型结构更低调，但它直接影响上限。

## 3. Tokenizer：把文字切成模型能处理的块

模型不直接读汉字、单词或句子，而是读 token。

Tokenizer 会把文本切成一串 token，再映射成数字。

```text
大语言模型 → 大 / 语言 / 模型
unbelievable → un / believable
```

不同 Tokenizer 的切法会影响：

- 序列长度。
- 训练和推理成本。
- 多语言表现。
- 生僻词和代码处理能力。

## 4. Token 数就是成本

大模型按 token 处理信息。文本被切得越碎，同样内容就越长。

这会影响：

- 训练成本。
- 上下文窗口占用。
- 推理速度。
- API 费用。

所以 Tokenizer 不只是预处理工具，它直接影响系统效率。

## 5. 数据不是越多越好

更多数据通常有帮助，但前提是质量可控。

大量重复、错误、低价值内容可能浪费算力，甚至降低模型表现。

现代大模型越来越重视“数据质量、数据配比、去重和去污染”，而不是单纯堆数量。
