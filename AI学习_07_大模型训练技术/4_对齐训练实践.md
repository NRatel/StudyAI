## 直觉与概述

### 从"预训练"到"可用"

一个直觉：预训练完的 GPT 模型就像一个"读了整个互联网的人"——知道很多，但不会好好说话。你问它"法国的首都是哪里？"，它可能续写"法国的首都是哪里？这是一个地理题..."（因为它在训练数据中见过这样的考题格式）。

**对齐训练**（Alignment Training）就是让模型学会"好好说话"——按用户意图回答问题，拒绝有害请求，输出有帮助且安全的内容。

```
预训练模型 (Base Model)
    │ "我什么都知道，但不会好好回答"
    │
    ├─ SFT (Supervised Fine-Tuning)
    │   "教我怎么回答问题的格式"
    │   用 ~10K-100K 高质量对话数据微调
    │
    ├─ RLHF (Reinforcement Learning from Human Feedback)
    │   "用人类偏好进一步优化"
    │   让模型学会人类喜欢什么样的回答
    │
    └─ DPO (Direct Preference Optimization)
        "更简单的偏好学习"
        不需要单独训练奖励模型
```

### 三个阶段的效果

```
Base Model (预训练):
  用户: "写一首关于春天的诗"
  模型: "写一首关于春天的诗 题目：春风 作者：李明 春风吹过大地..."  ← 续写格式

SFT 之后:
  用户: "写一首关于春天的诗"
  模型: "春风拂柳绿意浓，细雨润物百花红。"  ← 会回答了，但质量一般

RLHF/DPO 之后:
  用户: "写一首关于春天的诗"
  模型: "春来万物换新颜，柳絮轻飞燕剪烟。
        细雨如丝润花径，暖风送暖入人间。"  ← 质量高、格式好、有创意
```

## 严谨定义与原理

### 1. SFT (Supervised Fine-Tuning)

**核心思想**：用人工编写的"指令-回复"对微调预训练模型，教会模型对话格式。

**数据格式**：

```json
{
  "conversations": [
    {"role": "system", "content": "你是一个有用的AI助手。"},
    {"role": "user", "content": "解释什么是梯度下降"},
    {"role": "assistant", "content": "梯度下降是一种优化算法..."}
  ]
}
```

**训练目标**：只对 **assistant 部分** 计算 loss（不对 user/system 的 token 计算 loss）。

$$L_{SFT} = -\sum_{t \in \text{assistant tokens}} \log P_\theta(x_t | x_{<t})$$

这叫 **Instruction Masking** 或 **Response-only Loss**——只训练模型生成回复，不训练它模仿用户提问。

**SFT 数据的关键属性**：

| 属性 | 说明 | 典型值 |
|------|------|--------|
| 数量 | 不需要太多 | 10K~100K 条 |
| 质量 | 极其重要 | 人工编写或精选 |
| 多样性 | 覆盖各种任务类型 | 编程、写作、问答、推理... |
| 格式 | 统一的对话模板 | ChatML, Llama-chat 格式 |

**LIMA 论文的发现**（2023）：仅用 **1000 条** 精心编写的 SFT 数据，就能让 65B 模型达到接近 GPT-4 的对话质量。**数据质量 >> 数据数量**。

### 2. RLHF 工程实践

**RLHF (Reinforcement Learning from Human Feedback)** 的核心流程：

```
Step 1: 收集人类偏好数据
  给定 prompt → 模型生成多个回复 → 人类排序
  "回复A > 回复B > 回复C"

Step 2: 训练奖励模型 (Reward Model, RM)
  学习人类偏好：给定 (prompt, response) → 输出分数

Step 3: 用 RL (PPO) 优化策略模型
  策略模型生成回复 → RM 打分 → PPO 更新参数
```

**PPO 四模型架构**：

这是 RLHF 的核心工程难点——同时需要 **4 个模型** 在 GPU 上运行：

```
┌──────────────────────────────────────────────────┐
│                 PPO 训练架构                       │
├──────────────┬───────────────────────────────────┤
│  Actor       │ 正在训练的策略模型                   │
│  (Policy)    │ 生成回复，接收梯度更新                │
│              │ 参数量 = 模型大小 (如 7B)             │
├──────────────┼───────────────────────────────────┤
│  Critic      │ 价值函数模型                        │
│  (Value)     │ 估计每个 token 位置的期望回报         │
│              │ 通常与 Actor 共享 backbone             │
│              │ 加一个 value head (Linear → 1)       │
├──────────────┼───────────────────────────────────┤
│  Reward      │ 奖励模型 (已训练好，冻结)            │
│  Model (RM)  │ 对完整回复打分                       │
│              │ 通常从 SFT 模型初始化 + 排序头        │
├──────────────┼───────────────────────────────────┤
│  Reference   │ 参考模型 (SFT 模型的冻结副本)        │
│  Model (Ref) │ 用于计算 KL 散度惩罚                 │
│              │ 防止策略偏离太远                      │
└──────────────┴───────────────────────────────────┘

显存需求 (7B 模型):
  Actor:  ~14GB (FP16)  + 优化器状态 ~42GB
  Critic: ~14GB (FP16)  + 优化器状态 ~42GB
  RM:     ~14GB (FP16)  (冻结，无优化器)
  Ref:    ~14GB (FP16)  (冻结，无优化器)
  ──────────────────────
  总计:   ~140GB+        → 7B 模型的 RLHF 至少需要 4×A100
```

**PPO 优化目标**：

$$\max_\theta \; \mathbb{E}_{x \sim D, y \sim \pi_\theta(y|x)} \left[ R(x, y) - \beta \cdot \text{KL}(\pi_\theta(y|x) \| \pi_{\text{ref}}(y|x)) \right]$$

其中：
- $R(x, y)$：奖励模型的打分
- $\beta \cdot \text{KL}(\cdot)$：KL 散度惩罚，防止偏离参考模型太远
- $\beta$：通常 0.01~0.1，越大越保守

**奖励模型训练**：

给定偏好数据 $(x, y_w, y_l)$（$y_w$ 优于 $y_l$），用 Bradley-Terry 模型：

$$L_{RM} = -\log \sigma(r_\phi(x, y_w) - r_\phi(x, y_l))$$

### 3. DPO (Direct Preference Optimization)

**核心思想**：绕过奖励模型和 RL，直接用偏好数据优化策略模型。

DPO 论文证明了一个关键等式：在 RLHF 中最优策略的奖励函数可以**用策略本身的对数概率表示**：

$$r^*(x, y) = \beta \log \frac{\pi_\theta(y|x)}{\pi_{\text{ref}}(y|x)} + \beta \log Z(x)$$

由此推导出 DPO 的 loss：

$$L_{DPO} = -\log \sigma \left( \beta \left[ \log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)} \right] \right)$$

**DPO vs RLHF 对比**：

| 特性 | RLHF (PPO) | DPO |
|------|-----------|-----|
| 需要奖励模型 | 是 | 否 |
| 需要 RL 训练 | 是 (PPO) | 否 (SFT-like) |
| 同时在线模型数 | 4 个 | 2 个 (policy + ref) |
| 显存需求 | 非常大 | 中等 |
| 实现难度 | 高 (PPO 调参困难) | 低 (标准微调) |
| 训练稳定性 | 低 (RL 不稳定) | 高 |
| 效果上限 | 高 (在线采样) | 中-高 (离线数据) |
| 工业应用 | GPT-4, Claude | Llama-2/3, Zephyr |

**DPO 的局限**：
- 数据必须预先收集好（离线），不能在线探索
- 对偏好数据的质量和分布非常敏感
- 可能存在"分布偏移"问题（训练数据与模型当前分布不匹配）

### 4. 数据质量的重要性

**InstructGPT 的关键发现**：

```
方案 A: 1.3B SFT 模型 + RLHF + 高质量数据
方案 B: 175B 预训练模型 (GPT-3)

人类评估结果: A > B  (虽然 A 只有 B 的 0.7% 参数!)

结论: 对齐训练的数据质量比模型大小更重要
```

**SFT 数据质量控制**：

```
高质量 SFT 数据的特征:
1. 准确性: 事实正确，无幻觉
2. 完整性: 回答全面，不遗漏关键信息
3. 格式规范: 适当使用列表、代码块、标题
4. 语气得体: 友好、专业、不卑不亢
5. 安全性: 拒绝有害请求，且拒绝方式得体

数据收集方式:
- 人工编写 (最贵但最好): 约 $10-50/条
- GPT-4 生成 + 人工审核: 约 $1-5/条
- 开源数据集: ShareGPT, OpenAssistant, LIMA
- 自我蒸馏: 用强模型生成，弱模型学习
```

**偏好数据质量控制**：

```
常见问题:
1. 标注者不一致 → 用多人标注 + 投票
2. 位置偏差 → 随机化回复顺序
3. 长度偏差 → 人类倾向于选更长的回复
4. 格式偏差 → 人类倾向于选用了列表/代码块的回复

解决方案:
- 详细的标注指南
- 标注者培训和考试
- 定期一致性检查
- 多维度标注 (有用性、安全性、准确性 分开打分)
```

### 示例 1：SFT 训练完整实现

```python
"""
SFT (Supervised Fine-Tuning) 完整训练示例
使用 HuggingFace Transformers
"""
import torch
import torch.nn as nn
from typing import Dict, List, Optional
from dataclasses import dataclass


@dataclass
class SFTConfig:
    """SFT 训练配置"""
    model_name: str = "meta-llama/Llama-2-7b-hf"
    max_seq_length: int = 2048
    learning_rate: float = 2e-5
    num_epochs: int = 3
    per_device_batch_size: int = 4
    gradient_accumulation_steps: int = 4
    warmup_ratio: float = 0.03
    weight_decay: float = 0.0
    max_grad_norm: float = 1.0
    bf16: bool = True


class ChatTemplate:
    """
    对话模板处理
    将对话格式化为模型输入，并生成 loss mask
    """
    # Llama-2 Chat 格式
    LLAMA2_TEMPLATE = {
        "system_start": "<<SYS>>\n",
        "system_end": "\n<</SYS>>\n\n",
        "user_start": "[INST] ",
        "user_end": " [/INST]",
        "assistant_start": " ",
        "assistant_end": " </s>",
    }

    # ChatML 格式 (Qwen, Yi 等使用)
    CHATML_TEMPLATE = {
        "system_start": "<|im_start|>system\n",
        "system_end": "<|im_end|>\n",
        "user_start": "<|im_start|>user\n",
        "user_end": "<|im_end|>\n",
        "assistant_start": "<|im_start|>assistant\n",
        "assistant_end": "<|im_end|>\n",
    }

    def __init__(self, template_type: str = "chatml"):
        if template_type == "chatml":
            self.template = self.CHATML_TEMPLATE
        else:
            self.template = self.LLAMA2_TEMPLATE

    def format_conversation(
        self, conversations: List[Dict[str, str]]
    ) -> tuple:
        """
        格式化对话并生成 loss mask

        返回:
            text: 格式化后的文本
            response_ranges: assistant 回复的字符范围列表
        """
        text = ""
        response_ranges = []

        for msg in conversations:
            role = msg["role"]
            content = msg["content"]

            if role == "system":
                text += self.template["system_start"] + content + self.template["system_end"]
            elif role == "user":
                text += self.template["user_start"] + content + self.template["user_end"]
            elif role == "assistant":
                start = len(text) + len(self.template["assistant_start"])
                text += self.template["assistant_start"] + content + self.template["assistant_end"]
                end = len(text)
                response_ranges.append((start, end))

        return text, response_ranges


def create_sft_dataset_example():
    """创建 SFT 训练数据示例"""
    # 高质量 SFT 数据示例
    sft_examples = [
        {
            "conversations": [
                {"role": "system", "content": "你是一个有用的AI助手。"},
                {"role": "user", "content": "什么是梯度下降？请用简单的语言解释。"},
                {"role": "assistant", "content": (
                    "梯度下降是一种优化算法，用于找到函数的最小值。\n\n"
                    "打个比方：想象你站在一座山上，被蒙住了眼睛，想要走到山谷最低处。"
                    "你的策略是：用脚感受地面的坡度，然后朝着最陡的下坡方向走一步。"
                    "重复这个过程，最终就能到达谷底。\n\n"
                    "在数学上：\n"
                    "1. **计算梯度**：算出当前位置的损失函数对每个参数的偏导数\n"
                    "2. **更新参数**：参数 = 参数 - 学习率 × 梯度\n"
                    "3. **重复**：直到损失不再显著下降\n\n"
                    "学习率控制每步走多远：太大会来回震荡，太小则下降太慢。"
                )},
            ]
        },
        {
            "conversations": [
                {"role": "system", "content": "你是一个有用的AI助手。"},
                {"role": "user", "content": "帮我写一个Python快速排序。"},
                {"role": "assistant", "content": (
                    "下面是一个清晰的 Python 快速排序实现：\n\n"
                    "```python\n"
                    "def quicksort(arr):\n"
                    "    if len(arr) <= 1:\n"
                    "        return arr\n"
                    "    pivot = arr[len(arr) // 2]\n"
                    "    left = [x for x in arr if x < pivot]\n"
                    "    middle = [x for x in arr if x == pivot]\n"
                    "    right = [x for x in arr if x > pivot]\n"
                    "    return quicksort(left) + middle + quicksort(right)\n"
                    "```\n\n"
                    "**时间复杂度**：\n"
                    "- 平均：O(n log n)\n"
                    "- 最坏（已排序数组）：O(n^2)\n\n"
                    "**示例**：\n"
                    "```python\n"
                    "print(quicksort([3, 6, 8, 10, 1, 2, 1]))\n"
                    "# 输出: [1, 1, 2, 3, 6, 8, 10]\n"
                    "```"
                )},
            ]
        },
    ]

    # 格式化
    template = ChatTemplate("chatml")
    for i, example in enumerate(sft_examples):
        text, response_ranges = template.format_conversation(example["conversations"])
        print(f"\n{'='*60}")
        print(f"示例 {i+1}:")
        print(f"{'='*60}")
        print(f"格式化文本 (前200字符):\n{text[:200]}...")
        print(f"\nResponse 范围: {response_ranges}")
        print(f"总长度: {len(text)} 字符")

    return sft_examples

### 示例 2：奖励模型训练

```python
"""
奖励模型 (Reward Model) 训练
从偏好数据中学习人类偏好
"""
import torch
import torch.nn as nn
import torch.nn.functional as F
from typing import List, Dict, Tuple
from dataclasses import dataclass


class RewardModel(nn.Module):
    """
    奖励模型: 基于预训练 LM + 一个标量头

    输入: (prompt, response) 拼接的 token 序列
    输出: 标量奖励分数
    """

    def __init__(self, base_model_dim: int = 1024, vocab_size: int = 32000,
                 n_layers: int = 6, n_heads: int = 16):
        super().__init__()
        # 简化: 用小 Transformer 模拟
        self.embed = nn.Embedding(vocab_size, base_model_dim)
        self.layers = nn.ModuleList([
            nn.TransformerEncoderLayer(
                d_model=base_model_dim, nhead=n_heads,
                dim_feedforward=4 * base_model_dim,
                batch_first=True, norm_first=True,
            ) for _ in range(n_layers)
        ])
        self.ln_f = nn.LayerNorm(base_model_dim)
        # 奖励头: 将最后一个 token 的隐藏状态映射到标量
        self.reward_head = nn.Linear(base_model_dim, 1, bias=False)

    def forward(self, input_ids: torch.Tensor) -> torch.Tensor:
        """
        输入: input_ids [batch, seq_len]
        输出: rewards [batch]
        """
        x = self.embed(input_ids)
        for layer in self.layers:
            x = layer(x)
        x = self.ln_f(x)
        # 取最后一个 token 的表示
        last_hidden = x[:, -1, :]  # [batch, dim]
        reward = self.reward_head(last_hidden).squeeze(-1)  # [batch]
        return reward


def train_reward_model():
    """训练奖励模型"""
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

    # 初始化奖励模型
    rm = RewardModel(base_model_dim=512, n_layers=4, n_heads=8).to(device)
    optimizer = torch.optim.AdamW(rm.parameters(), lr=1e-5, weight_decay=0.01)

    print(f"奖励模型参数量: {sum(p.numel() for p in rm.parameters())/1e6:.1f}M")

    # 模拟偏好数据
    # 实际中: chosen 是人类选择的更好回复, rejected 是较差的回复
    batch_size = 8
    seq_len = 128

    for step in range(200):
        # 模拟: chosen 和 rejected 的 token 序列
        chosen_ids = torch.randint(0, 32000, (batch_size, seq_len), device=device)
        rejected_ids = torch.randint(0, 32000, (batch_size, seq_len), device=device)

        # 计算奖励
        chosen_rewards = rm(chosen_ids)    # [batch]
        rejected_rewards = rm(rejected_ids)  # [batch]

        # Bradley-Terry 损失: chosen 的奖励应该高于 rejected
        # loss = -log(sigmoid(r_chosen - r_rejected))
        loss = -F.logsigmoid(chosen_rewards - rejected_rewards).mean()

        optimizer.zero_grad()
        loss.backward()
        torch.nn.utils.clip_grad_norm_(rm.parameters(), 1.0)
        optimizer.step()

        if step % 40 == 0:
            # 准确率: chosen 奖励 > rejected 奖励的比例
            accuracy = (chosen_rewards > rejected_rewards).float().mean()
            avg_chosen = chosen_rewards.mean().item()
            avg_rejected = rejected_rewards.mean().item()
            print(
                f"Step {step:3d} | Loss: {loss.item():.4f} | "
                f"Acc: {accuracy.item():.2%} | "
                f"R_chosen: {avg_chosen:.3f} | R_rejected: {avg_rejected:.3f} | "
                f"Margin: {avg_chosen - avg_rejected:.3f}"
            )

    return rm


# 实际工程中使用 trl 库:
RM_TRAINING_CODE = """

# 从 SFT 模型初始化奖励模型

    "path/to/sft_model",
    num_labels=1,           # 标量输出
    torch_dtype=torch.bfloat16,
)

# 偏好数据格式:
# {"chosen": "prompt + 好回复", "rejected": "prompt + 差回复"}

training_args = RewardConfig(
    output_dir="./rm_output",
    per_device_train_batch_size=8,
    learning_rate=1e-5,

### 示例 3：DPO 训练实现

```python
"""
DPO (Direct Preference Optimization) 训练
比 RLHF 简单得多，不需要奖励模型和 PPO
"""
import torch
import torch.nn as nn
import torch.nn.functional as F
from typing import Tuple
import math


class SimpleLM(nn.Module):
    """简化的语言模型"""
    def __init__(self, vocab_size=1000, d_model=256, n_layers=2, n_heads=4):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, d_model)
        self.layers = nn.ModuleList([
            nn.TransformerEncoderLayer(
                d_model=d_model, nhead=n_heads,
                dim_feedforward=4*d_model, batch_first=True, norm_first=True,
            ) for _ in range(n_layers)
        ])
        self.ln_f = nn.LayerNorm(d_model)
        self.head = nn.Linear(d_model, vocab_size, bias=False)

    def forward(self, input_ids: torch.Tensor) -> torch.Tensor:
        x = self.embed(input_ids)
        for layer in self.layers:
            x = layer(x)
        return self.head(self.ln_f(x))  # [batch, seq_len, vocab_size]

    def get_log_probs(self, input_ids: torch.Tensor, labels: torch.Tensor) -> torch.Tensor:
        """
        计算每个 token 的对数概率
        返回: [batch] (整个序列的对数概率之和)
        """
        logits = self.forward(input_ids)  # [B, T, V]
        log_probs = F.log_softmax(logits, dim=-1)  # [B, T, V]
        # 取对应 label 的对数概率
        token_log_probs = log_probs.gather(2, labels.unsqueeze(-1)).squeeze(-1)  # [B, T]
        # 对序列求和（或平均）
        return token_log_probs.sum(dim=-1)  # [B]


def dpo_loss(
    policy_chosen_logps: torch.Tensor,
    policy_rejected_logps: torch.Tensor,
    reference_chosen_logps: torch.Tensor,
    reference_rejected_logps: torch.Tensor,
    beta: float = 0.1,
) -> Tuple[torch.Tensor, torch.Tensor, torch.Tensor]:
    """
    计算 DPO 损失

    L_DPO = -log σ(β × [log π_θ(y_w|x)/π_ref(y_w|x) - log π_θ(y_l|x)/π_ref(y_l|x)])

    参数:
        policy_chosen_logps:    π_θ(y_w|x) 的对数概率
        policy_rejected_logps:  π_θ(y_l|x) 的对数概率
        reference_chosen_logps: π_ref(y_w|x) 的对数概率
        reference_rejected_logps: π_ref(y_l|x) 的对数概率
        beta: 温度参数（越大越保守）

    返回:
        loss, chosen_reward, rejected_reward
    """
    # 计算 log ratio
    chosen_logratios = policy_chosen_logps - reference_chosen_logps
    rejected_logratios = policy_rejected_logps - reference_rejected_logps

    # 隐式奖励
    chosen_rewards = beta * chosen_logratios
    rejected_rewards = beta * rejected_logratios

    # DPO 损失
    logits = chosen_rewards - rejected_rewards  # [batch]
    loss = -F.logsigmoid(logits).mean()

    return loss, chosen_rewards.mean(), rejected_rewards.mean()


def train_dpo():
    """DPO 训练循环"""
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

    vocab_size = 1000
    seq_len = 64
    batch_size = 16

    # 策略模型 (正在训练)
    policy_model = SimpleLM(vocab_size).to(device)
    # 参考模型 (冻结的 SFT 模型)
    reference_model = SimpleLM(vocab_size).to(device)
    reference_model.load_state_dict(policy_model.state_dict())  # 初始化相同
    reference_model.eval()
    for p in reference_model.parameters():
        p.requires_grad = False

    optimizer = torch.optim.AdamW(policy_model.parameters(), lr=5e-7)
    beta = 0.1

    print(f"策略模型参数量: {sum(p.numel() for p in policy_model.parameters())/1e6:.1f}M")
    print(f"DPO β = {beta}")
    print(f"{'='*70}")

    for step in range(300):
        # 模拟偏好数据: (prompt + chosen_response, prompt + rejected_response)
        # 实际中这些是已经 tokenize 好的序列
        chosen_ids = torch.randint(0, vocab_size, (batch_size, seq_len), device=device)
        rejected_ids = torch.randint(0, vocab_size, (batch_size, seq_len), device=device)
        chosen_labels = chosen_ids.clone()
        rejected_labels = rejected_ids.clone()

        # 计算策略模型的对数概率
        policy_chosen_logps = policy_model.get_log_probs(chosen_ids, chosen_labels)
        policy_rejected_logps = policy_model.get_log_probs(rejected_ids, rejected_labels)

        # 计算参考模型的对数概率 (不计算梯度)
        with torch.no_grad():
            ref_chosen_logps = reference_model.get_log_probs(chosen_ids, chosen_labels)
            ref_rejected_logps = reference_model.get_log_probs(rejected_ids, rejected_labels)

        # DPO 损失
        loss, chosen_reward, rejected_reward = dpo_loss(
            policy_chosen_logps, policy_rejected_logps,
            ref_chosen_logps, ref_rejected_logps,
            beta=beta,
        )

        optimizer.zero_grad()
        loss.backward()
        torch.nn.utils.clip_grad_norm_(policy_model.parameters(), 1.0)
        optimizer.step()

        if step % 50 == 0:
            # 准确率: 隐式奖励(chosen) > 隐式奖励(rejected)
            with torch.no_grad():
                p_c = policy_model.get_log_probs(chosen_ids, chosen_labels)
                p_r = policy_model.get_log_probs(rejected_ids, rejected_labels)
                r_c = reference_model.get_log_probs(chosen_ids, chosen_labels)
                r_r = reference_model.get_log_probs(rejected_ids, rejected_labels)
                acc = ((p_c - r_c) > (p_r - r_r)).float().mean()

            print(
                f"Step {step:3d} | Loss: {loss.item():.4f} | "
                f"Acc: {acc.item():.2%} | "
                f"R_w: {chosen_reward.item():.3f} | R_l: {rejected_reward.item():.3f}"
            )


# 实际工程: 使用 trl 库
DPO_TRAINING_CODE = """
from trl import DPOTrainer, DPOConfig
from transformers import AutoModelForCausalLM, AutoTokenizer

# 加载 SFT 模型作为起点
model = AutoModelForCausalLM.from_pretrained(
    "path/to/sft_model",
    torch_dtype=torch.bfloat16,
)
ref_model = AutoModelForCausalLM.from_pretrained(
    "path/to/sft_model",
    torch_dtype=torch.bfloat16,
)
tokenizer = AutoTokenizer.from_pretrained("path/to/sft_model")

# 偏好数据格式:
# {"prompt": "...", "chosen": "好回复", "rejected": "差回复"}

    output_dir="./dpo_output",
    per_device_train_batch_size=4,
    gradient_accumulation_steps=8,
    learning_rate=5e-7,          # DPO 的学习率比 SFT 小很多
    num_train_epochs=1,
    beta=0.1,                     # DPO 温度参数
    bf16=True,
    max_length=2048,
    max_prompt_length=1024,
    deepspeed="ds_config_zero3.json",
    gradient_checkpointing=True,

    model=model,
    ref_model=ref_model,
    args=training_args,
    train_dataset=preference_dataset,
    tokenizer=tokenizer,

trainer.save_model("./dpo_model")
"""

if __name__ == "__main__":
    train_dpo()
    print("\n\n" + "="*60)
    print("实际工程代码 (使用 trl 库):")
    print("="*60)

### 示例 4：PPO 训练核心逻辑

```python
"""
PPO (Proximal Policy Optimization) 在 RLHF 中的核心逻辑
简化版实现，展示四模型交互
"""
import torch
import torch.nn as nn
import torch.nn.functional as F
from typing import Dict, Tuple
from dataclasses import dataclass


@dataclass
class PPOConfig:
    """PPO 超参数"""
    clip_range: float = 0.2          # PPO clip 范围
    vf_coef: float = 0.1             # 价值函数损失系数
    kl_coef: float = 0.05            # KL 惩罚系数
    gamma: float = 1.0               # 折扣因子 (语言任务通常为 1)
    lam: float = 0.95                # GAE lambda
    n_ppo_epochs: int = 4            # 每批数据训练几轮 PPO
    mini_batch_size: int = 4         # PPO 小批次


class PPOTrainerSimplified:
    """
    简化版 PPO Trainer
    展示 RLHF 中四个模型的交互逻辑
    """

    def __init__(self, config: PPOConfig):
        self.config = config

    def compute_advantages(
        self,
        rewards: torch.Tensor,      # [batch, seq_len]
        values: torch.Tensor,       # [batch, seq_len]
    ) -> Tuple[torch.Tensor, torch.Tensor]:
        """
        GAE (Generalized Advantage Estimation)

        在 RLHF 中：
        - reward 只在最后一个 token 有值（RM 的打分）
        - 中间 token 的 reward = 0（或用 KL 惩罚）
        """
        advantages = torch.zeros_like(rewards)
        last_gae = 0

        for t in reversed(range(rewards.shape[1])):
            if t == rewards.shape[1] - 1:
                next_value = 0
            else:
                next_value = values[:, t + 1]

            delta = rewards[:, t] + self.config.gamma * next_value - values[:, t]
            last_gae = delta + self.config.gamma * self.config.lam * last_gae
            advantages[:, t] = last_gae

        returns = advantages + values
        return advantages, returns

    def ppo_loss(
        self,
        old_logprobs: torch.Tensor,   # π_old(a|s) 的对数概率
        new_logprobs: torch.Tensor,   # π_θ(a|s) 的对数概率
        advantages: torch.Tensor,
        values: torch.Tensor,         # V_θ(s) 当前价值估计
        returns: torch.Tensor,        # 目标回报
    ) -> Dict[str, torch.Tensor]:
        """
        PPO-Clip 损失计算

        L_clip = -min(r_t × A_t, clip(r_t, 1-ε, 1+ε) × A_t)
        """
        # 重要性采样比率
        ratio = torch.exp(new_logprobs - old_logprobs)

        # PPO-Clip
        surr1 = ratio * advantages
        surr2 = torch.clamp(
            ratio, 1 - self.config.clip_range, 1 + self.config.clip_range
        ) * advantages
        policy_loss = -torch.min(surr1, surr2).mean()

        # 价值函数损失
        value_loss = F.mse_loss(values, returns)

        # 总损失
        total_loss = policy_loss + self.config.vf_coef * value_loss

        return {
            "total_loss": total_loss,
            "policy_loss": policy_loss,
            "value_loss": value_loss,
            "approx_kl": ((ratio - 1) - (ratio.log())).mean(),
            "clip_fraction": ((ratio - 1).abs() > self.config.clip_range).float().mean(),
        }


def simulate_rlhf_step():
    """
    模拟一步完整的 RLHF 训练
    展示四个模型的交互
    """
    device = torch.device("cpu")  # 演示用 CPU
    batch_size = 4
    seq_len = 32
    vocab_size = 100

    print("RLHF PPO 训练步骤模拟")
    print("=" * 60)

    # ============ 四个模型 ============
    # 简化: 用随机张量模拟

    # Step 1: Actor 生成回复
    print("\n[1] Actor 生成回复...")
    prompts = torch.randint(0, vocab_size, (batch_size, 16))  # prompt tokens
    responses = torch.randint(0, vocab_size, (batch_size, seq_len))  # 生成的回复
    full_sequences = torch.cat([prompts, responses], dim=1)  # 拼接
    print(f"    Prompt: {prompts.shape}, Response: {responses.shape}")

    # Step 2: 用 Actor 计算生成时的 log prob（旧策略）
    print("\n[2] 记录 Actor 的旧策略 log prob...")
    old_logprobs = torch.randn(batch_size, seq_len) * 0.1 - 5  # 模拟 log prob
    print(f"    Old log probs shape: {old_logprobs.shape}")

    # Step 3: Reward Model 打分
    print("\n[3] Reward Model 对每个回复打分...")
    rm_scores = torch.randn(batch_size)  # RM 输出标量分数
    print(f"    RM scores: {rm_scores.tolist()}")

    # Step 4: Reference Model 计算 KL 惩罚
    print("\n[4] 计算与 Reference Model 的 KL 散度...")
    ref_logprobs = torch.randn(batch_size, seq_len) * 0.1 - 5
    kl_penalty = (old_logprobs - ref_logprobs).sum(dim=-1)  # 近似 KL
    kl_coef = 0.05
    print(f"    KL penalty: {kl_penalty.tolist()}")

    # Step 5: 构造 token 级 reward
    print("\n[5] 构造 token 级奖励...")
    rewards = torch.zeros(batch_size, seq_len)
    # 最后一个 token 的 reward = RM score - KL penalty
    rewards[:, -1] = rm_scores - kl_coef * kl_penalty
    # 中间 token 可以加 per-token KL 惩罚
    per_token_kl = old_logprobs - ref_logprobs
    rewards -= kl_coef * per_token_kl
    print(f"    Token rewards shape: {rewards.shape}")

    # Step 6: Critic 估计价值
    print("\n[6] Critic 估计每个位置的价值...")
    values = torch.randn(batch_size, seq_len) * 0.1
    print(f"    Values shape: {values.shape}")

    # Step 7: 计算 GAE 优势
    print("\n[7] 计算 GAE 优势和回报...")
    ppo_config = PPOConfig()
    trainer = PPOTrainerSimplified(ppo_config)
    advantages, returns = trainer.compute_advantages(rewards, values)
    advantages = (advantages - advantages.mean()) / (advantages.std() + 1e-8)
    print(f"    Advantages: mean={advantages.mean():.4f}, std={advantages.std():.4f}")

    # Step 8: PPO 更新
    print("\n[8] PPO 更新 (多轮)...")
    for ppo_epoch in range(ppo_config.n_ppo_epochs):
        new_logprobs = old_logprobs + torch.randn_like(old_logprobs) * 0.01
        new_values = values + torch.randn_like(values) * 0.01

        losses = trainer.ppo_loss(
            old_logprobs=old_logprobs,
            new_logprobs=new_logprobs,
            advantages=advantages,
            values=new_values,
            returns=returns,
        )

        print(f"    PPO Epoch {ppo_epoch}: "
              f"total={losses['total_loss']:.4f}, "
              f"policy={losses['policy_loss']:.4f}, "
              f"value={losses['value_loss']:.4f}, "
              f"kl={losses['approx_kl']:.4f}, "
              f"clip_frac={losses['clip_fraction']:.2%}")

    print(f"\n{'='*60}")
    print("PPO 训练步骤完成!")
    print(f"\n显存需求估算 (7B 模型):")
    print(f"  Actor (训练中):   ~56 GB (FP16 参数 + FP32 优化器)")
    print(f"  Critic (训练中):  ~56 GB")
    print(f"  Reward Model:     ~14 GB (FP16, 冻结)")
    print(f"  Reference Model:  ~14 GB (FP16, 冻结)")
    print(f"  总计:             ~140 GB → 至少 4× A100-40GB")


if __name__ == "__main__":
    simulate_rlhf_step()
```

## 工程视角

### 对齐训练的完整流水线

```
Step 0: 预训练模型 (Base Model)
        │
Step 1: SFT 数据准备
        ├─ 人工编写 (~1K-10K 条高质量)
        ├─ GPT-4 生成 + 人工审核 (~10K-100K 条)
        └─ 开源数据 (ShareGPT, OpenAssistant, UltraChat)
        │
Step 2: SFT 训练
        ├─ 学习率: 2e-5
        ├─ Epochs: 2-3
        ├─ 只对 assistant 部分算 loss
        └─ 输出: SFT Model
        │
Step 3: 偏好数据收集
        ├─ 用 SFT Model 生成多个回复
        ├─ 人类标注偏好排序
        └─ 或使用 AI 标注 (Constitutional AI)
        │
Step 4a: RLHF 路线            Step 4b: DPO 路线
        ├─ 训练 RM                    ├─ 直接用偏好数据
        ├─ PPO 训练                    ├─ SFT-like 训练
        └─ 需要 4 模型                 └─ 只需 2 模型
        │                              │
Step 5: 评估
        ├─ 自动评估: MT-Bench, AlpacaEval, MMLU
        ├─ 人工评估: 盲评 A/B 测试
        └─ 安全评估: 有害请求测试
```

### RLHF vs DPO 选型指南

```
选 DPO 的场景 (推荐大多数人):
  ✓ GPU 资源有限 (< 8×A100)
  ✓ 团队 RL 经验不足
  ✓ 已有高质量偏好数据
  ✓ 需要快速迭代
  ✓ 开源模型微调

选 RLHF/PPO 的场景:
  ✓ 资源充足 (≥ 16×A100)
  ✓ 需要在线探索 (模型生成 → 人类评估 → 更新)
  ✓ 追求极致性能
  ✓ 大规模商业模型 (GPT-4 级别)
  ✓ 团队有 RL 调参经验

DPO 变体选择:
  - DPO:     最基础，适合大多数场景
  - IPO:     对噪声标签更鲁棒
  - KTO:     不需要成对偏好数据，只需要 "好/坏" 标签
  - ORPO:    不需要参考模型，更省显存
  - SimPO:   用长度归一化的对数概率作为隐式奖励
```

### 常见的坑和解决方案

```
1. SFT 过拟合
   症状: 训练 loss 很低，但生成质量下降（死板、重复）
   原因: 数据量太少 or epoch 太多
   解决: 减少 epoch (1-3)，增加数据多样性，加 dropout

2. RM 奖励 hacking
   症状: RM 分数持续上升，但实际质量没提升甚至下降
   原因: 策略模型找到了 RM 的漏洞（如总是生成很长的回复）
   解决: 增大 KL 惩罚系数 β，改进 RM 数据多样性

3. DPO 分布偏移
   症状: 训练初期有效，后期效果停滞
   原因: 训练数据是 SFT 模型生成的，但策略已经偏移
   解决: 迭代式 DPO (用当前模型生成新数据，重新收集偏好)

4. KL 散度爆炸
   症状: 策略与参考模型差距越来越大
   解决: 降低学习率，增大 KL 系数，减少训练步数

5. 格式/长度偏差
   症状: 模型总是生成列表格式或极长回复
   原因: 标注者偏向于选择格式化/长回复
   解决: 长度惩罚、格式控制、多维度评分
```

## 小结

| 阶段 | 目标 | 数据 | 关键超参 | 典型规模 |
|------|------|------|---------|---------|
| SFT | 学会对话格式 | 指令-回复对 | lr=2e-5, epochs=2-3 | 10K-100K 条 |
| RM 训练 | 学习人类偏好 | 偏好排序 | lr=1e-5, epochs=1 | 50K-500K 对 |
| PPO (RLHF) | 优化偏好 | 在线生成 | β=0.01-0.1, clip=0.2 | 4 模型同时 |
| DPO | 直接偏好优化 | 离线偏好对 | lr=5e-7, β=0.1 | 2 模型 |

**核心结论**：
1. **数据质量 > 数据数量 > 模型大小**（InstructGPT 的发现）
2. DPO 是目前性价比最高的对齐方案
3. RLHF (PPO) 效果上限更高，但工程复杂度也高得多
4. 对齐训练的学习率要比预训练/SFT **小一个量级**
