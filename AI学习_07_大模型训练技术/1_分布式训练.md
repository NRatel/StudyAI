## 直觉与概述

### 为什么需要分布式训练？

一个直觉：GPT-3 有 175B 参数，每个参数用 FP16（2字节）存储就需要 350GB。而 2024 年最好的单张 GPU（H100 80GB）也只有 80GB 显存。**模型本身都放不进一张卡**，更别说训练时还需要存梯度、优化器状态。

粗略估算训练时的显存占用：

```
模型参数:         175B × 2B = 350 GB  (FP16)
梯度:             175B × 2B = 350 GB  (FP16)
Adam 优化器状态:   175B × 8B = 1400 GB (FP32 参数副本 + 一阶/二阶矩)
──────────────────────────────────
总计:             ≈ 2.1 TB
```

2.1TB 显存——即使用 8 张 H100（80GB × 8 = 640GB），朴素地复制模型到每张卡也装不下。这就是分布式训练要解决的核心问题。

### 三种基本策略

```
┌─────────────────────────────────────────────────────┐
│              分布式训练策略                            │
├─────────────┬─────────────────┬─────────────────────┤
│  数据并行    │   模型并行       │   ZeRO 优化         │
│  (DDP)      │  (张量/流水线)    │  (切分冗余状态)      │
│             │                 │                     │
│ 每卡完整模型 │ 模型切成多块     │ 优化器/梯度/参数     │
│ 数据不同     │ 每卡一块         │  分散到多卡          │
└─────────────┴─────────────────┴─────────────────────┘
```

- **数据并行**：模型小到放得下单卡时，复制模型到每张卡，每卡处理不同数据，最后同步梯度
- **模型并行**：模型太大放不下单卡时，把模型切开，分到不同卡上
- **ZeRO**：不切模型计算图，而是切"冗余的状态"（优化器状态、梯度、参数），需要时再通信取回

## 严谨定义与原理

### 1. 数据并行 (Data Parallelism / DDP)

**核心思想**：N 张卡各持有模型的完整副本，每张卡处理 `batch_size / N` 条数据，前向反向各自计算，然后 **AllReduce 同步梯度**，保证所有卡的参数更新一致。

**AllReduce 操作**：

```
GPU 0: grad_0 ──┐
GPU 1: grad_1 ──┼── AllReduce ──→ 每张卡都得到 mean(grad_0, grad_1, ..., grad_N)
GPU 2: grad_2 ──┤
GPU N: grad_N ──┘
```

AllReduce 通常用 Ring-AllReduce 实现：N 张卡排成环，每张卡只与左右邻居通信，通信量为 `2 × (N-1)/N × 模型大小`，约等于 `2 × 模型大小`，**与卡数无关**。

**DDP vs 旧版 DataParallel**：

| 特性 | DataParallel (DP) | DistributedDataParallel (DDP) |
|------|-------------------|-------------------------------|
| 进程模型 | 单进程多线程 | 多进程（每卡一个） |
| 通信 | 主卡收集梯度 | Ring-AllReduce |
| GIL 瓶颈 | 有（Python GIL） | 无 |
| 负载均衡 | 主卡瓶颈 | 均衡 |
| 推荐程度 | 不推荐 | **标准方案** |

**数学上**：DDP 等价于在全局 batch 上计算梯度。设全局 batch 为 $B$，分到 $N$ 张卡，每卡 batch 为 $b = B/N$：

$$\nabla_\theta L = \frac{1}{B} \sum_{i=1}^{B} \nabla_\theta \ell(x_i) = \frac{1}{N} \sum_{k=1}^{N} \left( \frac{1}{b} \sum_{j=1}^{b} \nabla_\theta \ell(x_j^{(k)}) \right)$$

每卡先计算局部平均梯度，AllReduce 再取全局平均——数学上完全等价。

### 2. 张量并行 (Tensor Parallelism)

**核心思想**：把单个层（如线性层、注意力层）内部的矩阵运算**按维度切开**，分配到不同 GPU 上并行计算。

**以线性层 $Y = XA$ 为例**：

将权重矩阵 $A$ 按列切分为 $[A_1, A_2]$，分到 2 张 GPU：

```
GPU 0: Y_0 = X × A_0     (A 的前半列)
GPU 1: Y_1 = X × A_1     (A 的后半列)
结果:   Y = [Y_0, Y_1]    (拼接)
```

**MLP 层的张量并行**（Megatron-LM 方案）：

```
MLP: Y = Dropout(GeLU(X × A) × B)

Step 1: 将 A 按列切分
  GPU 0: Z_0 = GeLU(X × A_0)
  GPU 1: Z_1 = GeLU(X × A_1)

Step 2: 将 B 按行切分
  GPU 0: Y_0 = Z_0 × B_0
  GPU 1: Y_1 = Z_1 × B_1

Step 3: AllReduce
  Y = Y_0 + Y_1
```

关键技巧：**GeLU 是非线性**，不能先加再激活，所以必须先列切 A → 各自 GeLU → 再行切 B → AllReduce。这让每个 MLP 层只需要 **一次 AllReduce**（加上前面的一次 broadcast，共 2 次集合通信）。

**注意力层的张量并行**：

```
多头注意力天然适合张量并行！

GPU 0: head_0, head_1, ..., head_{h/2-1}    (前一半注意力头)
GPU 1: head_{h/2}, ..., head_{h-1}           (后一半注意力头)

各自计算完 → 输出投影矩阵 W_O 按行切分 → AllReduce
```

**优点**：层内计算完全并行，没有"等上一层算完"的问题
**缺点**：通信在层内，要求 GPU 之间**极高带宽**（通常限制在同一节点的 NVLink 连接内）

### 3. 流水线并行 (Pipeline Parallelism)

**核心思想**：把模型按层切分，每组连续的层放到一张 GPU 上，数据像流水线一样依次流过。

```
GPU 0: 层 0~23   GPU 1: 层 24~47   GPU 2: 层 48~71   GPU 3: 层 72~95

数据流: Input → GPU0 → GPU1 → GPU2 → GPU3 → Loss
反向:   Loss → GPU3 → GPU2 → GPU1 → GPU0
```

**朴素流水线的问题——气泡 (Bubble)**：

```
时间 →
GPU 0: [F0][  等  ][  等  ][  等  ][B0]
GPU 1:     [F1][  等  ][  等  ][B1]
GPU 2:          [F2][  等  ][B2]
GPU 3:               [F3][B3]

F = Forward, B = Backward
大量时间在等待 → 利用率很低！
```

**微批次 (Micro-batch) 流水线**：把一个 mini-batch 切成多个 micro-batch，像流水线一样填满各个阶段。

**GPipe 方案**：先跑完所有 micro-batch 的前向，再跑反向。

```
GPU 0: [F0][F1][F2][F3][    ][B3][B2][B1][B0]
GPU 1:     [F0][F1][F2][F3][B3][B2][B1][B0]
GPU 2:         [F0][F1][F2][F3][B3][B2][B1][B0]
GPU 3:             [F0][F1][F2][F3][B3][B2][B1][B0]
```

**1F1B 方案**（PipeDream）：前向和反向交替执行，进一步减少气泡。

```
GPU 0: [F0][F1][F2][F3][B0][F4][B1][F5][B2]...
GPU 1:     [F0][F1][F2][B0][F3][B1][F4][B2]...
GPU 2:         [F0][F1][B0][F2][B1][F3][B2]...
GPU 3:             [F0][B0][F1][B1][F2][B2]...
```

**气泡比例**：设流水线阶段数为 $p$，micro-batch 数为 $m$，气泡时间占比约为：

$$\text{Bubble ratio} = \frac{p - 1}{m + p - 1}$$

当 $m \gg p$ 时气泡趋近于零。实践中通常 $m \geq 4p$。

### 4. ZeRO 优化 (Zero Redundancy Optimizer)

**核心洞察**：在数据并行中，每张卡都持有**完整的**优化器状态、梯度和参数——这是巨大的冗余。能否把这些状态也"切开"分到不同卡？

**三个阶段**：

| 阶段 | 切分内容 | 显存节省（相对DDP） | 通信量 |
|------|---------|-------------------|--------|
| ZeRO-1 | 优化器状态 | 4x | = DDP |
| ZeRO-2 | + 梯度 | 8x | = DDP |
| ZeRO-3 | + 模型参数 | 线性于 N | 1.5x DDP |

**ZeRO-1**：每张卡只存 $1/N$ 的优化器状态。

```
传统 DDP (N=4):
  每卡: 全部参数(FP16) + 全部梯度(FP16) + 全部优化器状态(FP32×3)
  每卡优化器显存: 175B × 12B = 2.1TB / 4 × 4 = 2.1TB (没省！每卡都完整)

ZeRO-1 (N=4):
  每卡: 全部参数(FP16) + 全部梯度(FP16) + 1/4 优化器状态(FP32×3)
  优化器显存: 175B × 12B / 4 = 525GB / 4 = 131GB/卡
```

**ZeRO-2**：再把梯度也切分。反向传播时，一个梯度算完就 Reduce-Scatter 到负责的卡，其他卡丢弃。

**ZeRO-3**：连模型参数也切分。需要某层参数时，All-Gather 临时拿到完整参数，用完就丢。

```
ZeRO-3 前向过程:
  for layer in model.layers:
      All-Gather(layer.params)      # 临时集齐完整参数
      output = layer.forward(input)  # 前向计算
      丢弃非本卡负责的参数            # 释放显存
```

**ZeRO-3 通信量分析**：
- 前向：每层一次 All-Gather → $\Psi$ 字节
- 反向：每层一次 All-Gather + 一次 Reduce-Scatter → $2\Psi$ 字节
- 总计：$3\Psi$（DDP 的 AllReduce 是 $2\Psi$），多了 50% 通信，但**显存降到 1/N**

### 5. 3D 并行

**核心思想**：将数据并行、张量并行、流水线并行组合起来，各取所长。

```
典型配置 (64 GPU 训练):
- 张量并行 (TP=4): 同一节点内 4 张 GPU，NVLink 高带宽互连
- 流水线并行 (PP=4): 4 个节点，每节点放模型的 1/4 层
- 数据并行 (DP=4): 4 组相同配置，各自处理不同数据

总 GPU: TP × PP × DP = 4 × 4 × 4 = 64
```

**为什么这样分配**：

```
通信量从大到小: 张量并行 > 数据并行 > 流水线并行
带宽从高到低: NVLink(节点内) > InfiniBand(节点间)

所以:
- 张量并行 → 放在节点内 (NVLink, ~600 GB/s)
- 流水线并行 → 放在节点间 (InfiniBand, ~200 Gb/s)
- 数据并行 → 跨所有副本 (Ring-AllReduce)
```

**Megatron-LM + DeepSpeed 的 3D 并行**是目前训练超大模型的标准方案（GPT-3、Llama 等都用了类似策略）。

## Python 代码示例

### 示例 1：PyTorch DDP 基础训练

```python
"""
PyTorch DistributedDataParallel 基础示例
展示数据并行的标准写法
"""
import os
import torch
import torch.nn as nn
import torch.distributed as dist
from torch.nn.parallel import DistributedDataParallel as DDP
from torch.utils.data import DataLoader, DistributedSampler, TensorDataset


def setup_distributed():
    """初始化分布式环境"""
    # torchrun 会自动设置这些环境变量
    dist.init_process_group(backend="nccl")  # NCCL 是 NVIDIA GPU 最优的后端
    local_rank = int(os.environ["LOCAL_RANK"])
    torch.cuda.set_device(local_rank)
    return local_rank


def cleanup():
    dist.destroy_process_group()


class SimpleTransformerBlock(nn.Module):
    """简化的 Transformer block，用于演示"""
    def __init__(self, d_model=512, nhead=8, dim_ff=2048):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, nhead, batch_first=True)
        self.ff = nn.Sequential(
            nn.Linear(d_model, dim_ff),
            nn.GELU(),
            nn.Linear(dim_ff, d_model),
        )
        self.ln1 = nn.LayerNorm(d_model)
        self.ln2 = nn.LayerNorm(d_model)

    def forward(self, x):
        # Pre-LN Transformer
        h = self.ln1(x)
        x = x + self.attn(h, h, h, need_weights=False)[0]
        x = x + self.ff(self.ln2(x))
        return x


def train_ddp():
    local_rank = setup_distributed()
    world_size = dist.get_world_size()
    rank = dist.get_rank()

    # 1. 创建模型并包装为 DDP
    model = nn.Sequential(
        nn.Embedding(10000, 512),
        SimpleTransformerBlock(512, 8, 2048),
        SimpleTransformerBlock(512, 8, 2048),
        nn.Linear(512, 10000),
    ).cuda(local_rank)

    model = DDP(model, device_ids=[local_rank])  # 核心：DDP 包装

    # 2. 使用 DistributedSampler 确保每张卡的数据不重复
    dataset = TensorDataset(
        torch.randint(0, 10000, (10000, 128)),  # 假数据
        torch.randint(0, 10000, (10000, 128)),
    )
    sampler = DistributedSampler(dataset, num_replicas=world_size, rank=rank)
    dataloader = DataLoader(dataset, batch_size=32, sampler=sampler)

    # 3. 标准训练循环
    optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4)
    loss_fn = nn.CrossEntropyLoss()

    for epoch in range(3):
        sampler.set_epoch(epoch)  # 关键：每个 epoch 打乱顺序
        for batch_idx, (input_ids, labels) in enumerate(dataloader):
            input_ids = input_ids.cuda(local_rank)
            labels = labels.cuda(local_rank)

            logits = model(input_ids)  # [B, seq_len, vocab]
            loss = loss_fn(logits.view(-1, 10000), labels.view(-1))

            optimizer.zero_grad()
            loss.backward()       # DDP 自动在 backward 中触发 AllReduce
            optimizer.step()

            if rank == 0 and batch_idx % 50 == 0:
                print(f"Epoch {epoch}, Step {batch_idx}, Loss: {loss.item():.4f}")

    cleanup()


# 启动命令 (4 GPU):
# torchrun --nproc_per_node=4 train_ddp.py
if __name__ == "__main__":
    train_ddp()
```

### 示例 2：模拟张量并行的矩阵切分

```python
"""
张量并行的核心思想：矩阵按列/行切分后分到不同 GPU
这里用 CPU 模拟展示原理
"""
import torch
import torch.nn.functional as F


def demo_column_parallel_linear():
    """
    列并行线性层: Y = X @ A
    将 A 按列切分到 2 个 "GPU"
    """
    batch_size, in_features, out_features = 4, 8, 6

    X = torch.randn(batch_size, in_features)
    A = torch.randn(in_features, out_features)

    # --- 标准计算 ---
    Y_standard = X @ A
    print(f"标准计算 Y shape: {Y_standard.shape}")  # [4, 6]

    # --- 列并行（模拟 2 GPU）---
    A_gpu0 = A[:, :3]  # 前 3 列 → GPU 0
    A_gpu1 = A[:, 3:]  # 后 3 列 → GPU 1

    Y_gpu0 = X @ A_gpu0  # GPU 0 独立计算
    Y_gpu1 = X @ A_gpu1  # GPU 1 独立计算

    Y_parallel = torch.cat([Y_gpu0, Y_gpu1], dim=-1)  # 拼接
    print(f"列并行计算 Y shape: {Y_parallel.shape}")  # [4, 6]
    print(f"结果一致: {torch.allclose(Y_standard, Y_parallel)}")  # True


def demo_row_parallel_linear():
    """
    行并行线性层: Y = X @ B
    将 B 按行切分，但需要对 X 也做对应切分
    """
    batch_size, in_features, out_features = 4, 6, 8

    X = torch.randn(batch_size, in_features)
    B = torch.randn(in_features, out_features)

    # --- 标准计算 ---
    Y_standard = X @ B

    # --- 行并行（模拟 2 GPU）---
    # X 也要按列切分（对应 B 的行切分）
    X_gpu0 = X[:, :3]   # X 的前 3 列 → GPU 0
    X_gpu1 = X[:, 3:]   # X 的后 3 列 → GPU 1
    B_gpu0 = B[:3, :]   # B 的前 3 行 → GPU 0
    B_gpu1 = B[3:, :]   # B 的后 3 行 → GPU 1

    Y_gpu0 = X_gpu0 @ B_gpu0  # GPU 0 的部分结果
    Y_gpu1 = X_gpu1 @ B_gpu1  # GPU 1 的部分结果

    Y_parallel = Y_gpu0 + Y_gpu1  # AllReduce (求和)
    print(f"\n行并行计算结果一致: {torch.allclose(Y_standard, Y_parallel)}")


def demo_megatron_mlp():
    """
    Megatron-LM 风格的 MLP 张量并行
    MLP: Y = Dropout(GeLU(X @ A) @ B)
    A 列切分，B 行切分 → 只需一次 AllReduce
    """
    batch_size, d_model, d_ff = 4, 8, 12
    tp_size = 2  # 张量并行度

    X = torch.randn(batch_size, d_model)
    A = torch.randn(d_model, d_ff)
    B = torch.randn(d_ff, d_model)

    # --- 标准计算 ---
    Y_standard = F.gelu(X @ A) @ B

    # --- Megatron 张量并行 ---
    # Step 1: A 按列切分
    A_parts = A.chunk(tp_size, dim=1)  # 每个 [d_model, d_ff/2]
    # Step 2: 各自 GeLU（非线性必须在切分后做）
    hidden_parts = [F.gelu(X @ A_parts[i]) for i in range(tp_size)]
    # Step 3: B 按行切分
    B_parts = B.chunk(tp_size, dim=0)  # 每个 [d_ff/2, d_model]
    # Step 4: 各自矩阵乘
    Y_parts = [hidden_parts[i] @ B_parts[i] for i in range(tp_size)]
    # Step 5: AllReduce（求和）
    Y_parallel = sum(Y_parts)

    print(f"\nMegatron MLP 张量并行结果一致: {torch.allclose(Y_standard, Y_parallel, atol=1e-5)}")


if __name__ == "__main__":
    demo_column_parallel_linear()
    demo_row_parallel_linear()
    demo_megatron_mlp()
```

### 示例 3：模拟 ZeRO 三阶段的显存分配

```python
"""
模拟 ZeRO 优化器的三个阶段对显存的影响
不涉及真实训练，只计算显存分配
"""


def calculate_memory(
    param_count_billions: float,
    num_gpus: int,
    zero_stage: int = 0,
    mixed_precision: bool = True,
):
    """
    计算不同 ZeRO 阶段的显存占用

    参数:
        param_count_billions: 参数量 (十亿)
        num_gpus: GPU 数量
        zero_stage: ZeRO 阶段 (0=DDP, 1, 2, 3)
        mixed_precision: 是否使用混合精度

    返回:
        每张 GPU 的显存占用 (GB)
    """
    P = param_count_billions * 1e9  # 参数数量
    N = num_gpus

    if mixed_precision:
        # 混合精度训练的显存组成:
        # - FP16 参数: 2P bytes
        # - FP16 梯度: 2P bytes
        # - FP32 优化器状态 (Adam): 4P (参数副本) + 4P (一阶矩) + 4P (二阶矩) = 12P bytes
        param_mem = 2 * P      # FP16 参数
        grad_mem = 2 * P       # FP16 梯度
        optim_mem = 12 * P     # FP32 优化器状态
    else:
        param_mem = 4 * P
        grad_mem = 4 * P
        optim_mem = 8 * P  # FP32 Adam: 参数副本 + 一阶 + 二阶 (参数本身已算)

    if zero_stage == 0:
        # 传统 DDP: 每卡完整副本
        per_gpu = param_mem + grad_mem + optim_mem
    elif zero_stage == 1:
        # ZeRO-1: 切分优化器状态
        per_gpu = param_mem + grad_mem + optim_mem / N
    elif zero_stage == 2:
        # ZeRO-2: 切分优化器状态 + 梯度
        per_gpu = param_mem + grad_mem / N + optim_mem / N
    elif zero_stage == 3:
        # ZeRO-3: 切分优化器状态 + 梯度 + 参数
        per_gpu = param_mem / N + grad_mem / N + optim_mem / N
    else:
        raise ValueError(f"Invalid ZeRO stage: {zero_stage}")

    per_gpu_gb = per_gpu / (1024 ** 3)
    return per_gpu_gb


def main():
    # 分析不同模型规模和 ZeRO 阶段的显存需求
    configs = [
        ("GPT-2 (1.5B)", 1.5),
        ("GPT-3 (175B)", 175),
        ("Llama-2 70B", 70),
    ]
    gpu_counts = [1, 8, 64, 256]

    for name, params in configs:
        print(f"\n{'='*70}")
        print(f"模型: {name} ({params}B 参数)")
        print(f"{'='*70}")
        print(f"{'GPU数':>6} | {'DDP':>10} | {'ZeRO-1':>10} | {'ZeRO-2':>10} | {'ZeRO-3':>10}")
        print("-" * 70)

        for n_gpu in gpu_counts:
            mems = []
            for stage in [0, 1, 2, 3]:
                mem = calculate_memory(params, n_gpu, stage)
                mems.append(mem)

            print(
                f"{n_gpu:>6} | "
                f"{mems[0]:>8.1f}GB | "
                f"{mems[1]:>8.1f}GB | "
                f"{mems[2]:>8.1f}GB | "
                f"{mems[3]:>8.1f}GB"
            )

    # 详细分析一个具体场景
    print(f"\n{'='*70}")
    print("详细分析: 175B 模型, 64 GPU, ZeRO-3")
    print(f"{'='*70}")
    P = 175e9
```

### 示例 4：流水线并行的调度模拟

```python
"""
模拟 GPipe 和 1F1B 两种流水线调度策略
可视化气泡（空闲时间）的差异
"""
from typing import List, Tuple


def simulate_gpipe(num_stages: int, num_microbatches: int) -> List[List[str]]:
    """
    模拟 GPipe 调度: 先全部前向，再全部反向

    返回: schedule[stage][time_slot] 的操作
    """
    total_time = 2 * num_microbatches + 2 * (num_stages - 1)
    schedule = [["  .  "] * total_time for _ in range(num_stages)]

    # 前向阶段
    for mb in range(num_microbatches):
        for stage in range(num_stages):
            t = mb + stage
            schedule[stage][t] = f" F{mb:02d} "

    # 反向阶段 (在所有前向完成后)
    forward_end = num_microbatches + num_stages - 1
    for mb in range(num_microbatches - 1, -1, -1):
        for stage in range(num_stages - 1, -1, -1):
            t = forward_end + (num_microbatches - 1 - mb) + (num_stages - 1 - stage)
            if t < total_time:
                schedule[stage][t] = f" B{mb:02d} "

    return schedule


def simulate_1f1b(num_stages: int, num_microbatches: int) -> List[List[str]]:
    """
    模拟 1F1B (One Forward One Backward) 调度
    进入稳态后，每个 stage 交替做一次前向一次反向
    """
    total_time = 2 * num_microbatches + 2 * (num_stages - 1)
    schedule = [["  .  "] * total_time for _ in range(num_stages)]

    for stage in range(num_stages):
        f_time = stage  # 第一个前向开始时间
        warmup = num_stages - stage - 1  # warmup 前向数

        # Warmup 阶段: 连续前向
        for i in range(min(warmup + 1, num_microbatches)):
            schedule[stage][f_time + i] = f" F{i:02d} "

        # 稳态阶段: 1F1B 交替
        next_f = warmup + 1
        next_b = 0
        t = f_time + warmup + 1

        while next_f < num_microbatches or next_b < num_microbatches:
            if next_b < num_microbatches and t < total_time:
                schedule[stage][t] = f" B{next_b:02d} "
                next_b += 1
                t += 1
            if next_f < num_microbatches and t < total_time:
                schedule[stage][t] = f" F{next_f:02d} "
                next_f += 1
                t += 1

    return schedule


def print_schedule(schedule: List[List[str]], title: str):
    """可视化打印调度"""
    print(f"\n{'='*80}")
    print(f" {title}")
    print(f"{'='*80}")

    for stage, row in enumerate(schedule):
        line = f"GPU {stage}: "
        line += "|".join(row)
        print(line)

    # 计算气泡率
    total_slots = len(schedule) * len(schedule[0])
    busy_slots = sum(
        1 for row in schedule for slot in row if slot.strip() != "."
    )
    bubble_ratio = 1 - busy_slots / total_slots
    print(f"\n气泡率: {bubble_ratio:.1%}")
    print(f"理论公式: (p-1)/(m+p-1) = {(len(schedule)-1)/(len(schedule[0])//2 + len(schedule)-1):.1%}")


def main():
    num_stages = 4
    num_microbatches = 8

    print("流水线并行调度可视化")
    print(f"流水线阶段数 (p) = {num_stages}")
    print(f"微批次数量 (m)   = {num_microbatches}")

    # GPipe
    gpipe_schedule = simulate_gpipe(num_stages, num_microbatches)
    print_schedule(gpipe_schedule, "GPipe 调度 (全部前向 → 全部反向)")

    # 1F1B
    schedule_1f1b = simulate_1f1b(num_stages, num_microbatches)
    print_schedule(schedule_1f1b, "1F1B 调度 (交替前向/反向)")

    # 气泡率对比
    print(f"\n{'='*80}")
    print("不同配置下的理论气泡率:")
    print(f"{'stages(p)':>10} | {'micro-batches(m)':>16} | {'气泡率':>10}")
    print("-" * 45)
    for p in [4, 8, 16]:
        for m in [4, 16, 64, 128]:
            bubble = (p - 1) / (m + p - 1)
            print(f"{p:>10} | {m:>16} | {bubble:>9.1%}")


if __name__ == "__main__":
    main()
```

## 工程视角

### 方案选择决策树

```
你有多少 GPU？模型多大？
│
├─ 模型放得下单卡（<10B 用 FP16 约 20GB）
│   └─ 用 DDP 就够了
│       如果想进一步省显存 → ZeRO-1 或 ZeRO-2
│
├─ 模型放不下单卡但放得下单机（8卡 × 80GB = 640GB）
│   ├─ 用 ZeRO-3（最简单，不改代码）
│   └─ 或者 张量并行（更高效但需要改模型代码）
│
└─ 模型放不下单机
    └─ 3D 并行:
        TP=节点内GPU数(通常4或8) × PP=节点数/DP副本数 × DP=剩余
```

### 常见的实际配置

```
7B 模型 (Llama-2 7B):
  - 8× A100-80GB: DDP 或 ZeRO-2, batch_size=4/GPU
  - 4× A100-40GB: ZeRO-3, batch_size=2/GPU

70B 模型 (Llama-2 70B):
  - 64× A100-80GB: TP=8, PP=2, DP=4
  - 或 ZeRO-3 on 64 GPUs（更简单但稍慢）

175B 模型 (GPT-3 规模):
  - 256× A100-80GB: TP=8, PP=8, DP=4
  - 必须 3D 并行，ZeRO-3 通信量太大
```

### 通信瓶颈与优化

```python
# 通信量估算 (175B 模型)
param_size_fp16 = 175e9 * 2  # 350 GB

# DDP AllReduce: 2 × 模型大小
ddp_comm = 2 * param_size_fp16 / 1e9  # 700 GB/step
# 如果用 200 Gbps (25 GB/s) 的 InfiniBand:
ddp_time = ddp_comm / 25  # 28 秒/step → 太慢!

# ZeRO-3: 3 × 模型大小（但 overlap 可以隐藏）
zero3_comm = 3 * param_size_fp16 / 1e9  # 1050 GB/step

# 张量并行 (节点内 NVLink 600 GB/s):
# 每层 2 次 AllReduce，每次只传 activation (远小于参数)
# 假设 d_model=12288, batch=32, seq=2048
activation_size = 32 * 2048 * 12288 * 2  # ~3 GB/layer
tp_comm_per_layer = 2 * activation_size / 1e9  # ~6 GB
tp_time_per_layer = tp_comm_per_layer / 600  # ~0.01 秒 → 很快!
```

**核心原则**：张量并行通信的是 **激活值**（与 batch 和 seq_len 相关），远小于参数量；数据并行/ZeRO 通信的是 **参数/梯度**（与模型大小相关）。所以张量并行放在高带宽链路上。

### 训练稳定性注意事项

1. **学习率 Warmup**：分布式训练有效 batch 变大，需要更长的 warmup
2. **梯度裁剪**：`max_grad_norm=1.0` 是标准配置
3. **Loss Spike**：大模型训练中偶尔出现 loss 突然飙升，常用策略：
   - 降低学习率重新开始
   - 跳过该 batch
   - 从最近的 checkpoint 恢复
4. **随机种子**：每张卡的数据采样种子不同，但模型初始化种子相同
5. **Checkpoint**：定期保存，分布式 checkpoint 需要处理好分片

### 关键指标：MFU (Model FLOPs Utilization)

衡量训练效率的核心指标：

$$\text{MFU} = \frac{\text{实际每秒浮点运算}}{\text{GPU 理论峰值}}$$

```
MFU 参考值:
- 单卡训练: 50-60%
- DDP (节点内): 45-55%
- 3D 并行 (多节点): 30-50%
- ZeRO-3 (多节点): 25-40%

A100 理论峰值: 312 TFLOPS (BF16)
H100 理论峰值: 989 TFLOPS (BF16) / ~1979 TFLOPS (FP8)
```

MFU 低于 30% 通常说明通信成为瓶颈，需要优化并行策略。

## 小结

| 策略 | 切什么 | 通信内容 | 适用场景 |
|------|--------|---------|---------|
| DDP | 数据 | 梯度 (AllReduce) | 模型能放下单卡 |
| 张量并行 | 层内矩阵 | 激活值 | 节点内高带宽 |
| 流水线并行 | 层间 | 激活值 (点对点) | 节点间 |
| ZeRO-1 | 优化器状态 | 梯度 (AllReduce) | 省显存但不改代码 |
| ZeRO-2 | + 梯度 | 梯度 (Reduce-Scatter) | 进一步省显存 |
| ZeRO-3 | + 参数 | 参数 (All-Gather) | 模型放不下单卡 |
| 3D并行 | 以上组合 | 混合 | 超大模型训练 |
