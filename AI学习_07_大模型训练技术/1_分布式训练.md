# 分布式训练

> **前置知识**：[GPT 系列深度解析](../AI学习_05_GPT系列深度解析/README.md)
>
> **本节目标**：理解为什么大模型训练需要很多 GPU 协作，以及常见并行方式在拆什么。

---

## 1. 大模型为什么不能简单单卡训练

大模型训练会同时消耗：

- 参数显存。
- 激活值显存。
- 梯度显存。
- 优化器状态显存。
- 大量矩阵计算。

模型足够大时，一张 GPU 放不下，也算不完。

## 2. 数据并行：多张卡看不同数据

数据并行最容易理解。

每张 GPU 放一份模型，处理不同小批数据。算完梯度后，大家同步一下，再一起更新参数。

像多个学生做不同练习题，最后汇总错题经验。

优点是简单。缺点是模型本身太大时，每张卡都放一份模型仍然放不下。

## 3. 模型并行：把模型本身拆开

模型并行会把模型切到不同 GPU 上。

比如一层太大，就把矩阵切开；层数太多，就把不同层放到不同设备。

这样可以训练更大的模型，但设备之间通信会变复杂。

## 4. 流水线并行：像工厂一样分工

流水线并行把模型按层分段。前几层在一组 GPU，后几层在另一组 GPU。

数据像工件一样一批批流过不同工位。

它能减轻单设备压力，但要处理流水线空闲、批次切分和同步问题。

## 5. 分布式训练的真正难点

难点不只是“多插几张卡”。

真正麻烦的是：

- 设备之间通信很贵。
- 显存要精细管理。
- 某些 GPU 等待其他 GPU 会浪费时间。
- 训练不稳定时排查困难。

所以大模型训练是一项系统工程，不只是模型结构问题。
