## 直觉与概述

### 为什么要用半精度？

一个直觉：FP32 用 4 字节存一个数，FP16 只用 2 字节——**显存直接减半，计算速度翻倍**。NVIDIA 的 Tensor Core 专门为半精度矩阵乘法优化，A100 的 FP16 算力（312 TFLOPS）是 FP32（19.5 TFLOPS）的 16 倍。

但直接把所有数据类型从 FP32 换成 FP16 会出问题：

```
FP32: 1位符号 + 8位指数 + 23位尾数  → 范围 ±3.4×10³⁸,  精度 ~7位有效数字
FP16: 1位符号 + 5位指数 + 10位尾数  → 范围 ±6.5×10⁴,   精度 ~3位有效数字
BF16: 1位符号 + 8位指数 + 7位尾数   → 范围 ±3.4×10³⁸,  精度 ~2位有效数字
```

FP16 的问题：
- **下溢**：梯度值小于 `5.96e-8`（FP16 最小正数）就变成 0
- **上溢**：值超过 65504 就变成 inf
- **精度损失**：参数更新量 `lr × grad` 如果太小，加到参数上可能被"吞掉"

所以不能"纯用 FP16"，要**混合精度**。

### 混合精度的核心想法

```
计算用 FP16 (快)        ←→        参数更新用 FP32 (准)

前向/反向:  FP16 参数 → FP16 激活 → FP16 梯度     (GPU Tensor Core 加速)
                                        ↓
参数更新:   FP32 主参数 ← FP32(梯度) × lr           (CPU 或 GPU 上的精确计算)
                  ↓
下一轮:     FP32 主参数 → 转回 FP16 → 下一次前向
```

**三个关键技巧**：
1. **FP32 主权重 (Master Weights)**：保留一份 FP32 参数用于更新
2. **Loss Scaling**：放大 loss 让梯度不下溢
3. **算术精度控制**：某些操作（如 Softmax、LayerNorm）用 FP32 计算

## 严谨定义与原理

### 1. 数值格式对比

| 格式 | 总位数 | 指数 | 尾数 | 范围 | 最小正数 | 适用场景 |
|------|-------|------|------|------|---------|---------|
| FP32 | 32 | 8 | 23 | ±3.4e38 | 1.2e-38 | 参数更新 |
| FP16 | 16 | 5 | 10 | ±6.5e4 | 6.0e-8 | 前向/反向计算 |
| BF16 | 16 | 8 | 7 | ±3.4e38 | 1.2e-38 | 前向/反向（更稳定） |
| TF32 | 19 | 8 | 10 | ±3.4e38 | 1.2e-38 | A100+ 默认 FP32 模式 |
| FP8 E4M3 | 8 | 4 | 3 | ±448 | 1.95e-3 | H100+ 前向 |
| FP8 E5M2 | 8 | 5 | 2 | ±57344 | 6.1e-8 | H100+ 反向 |

**BF16 vs FP16**：
- BF16 保留了 FP32 的指数范围（8位指数，与 FP32 相同），所以**不需要 Loss Scaling**，不易上溢/下溢
- FP16 精度更高（10位尾数 vs BF16 的 7位），但指数范围太小（5位指数，最大 65504），容易溢出
- **核心权衡**：BF16 范围更大不易溢出但精度略低；FP16 精度更高但范围小需要 Loss Scaling
- 现代 GPU（A100 及之后的 Ampere+ 架构）推荐使用 BF16，这也是 GPT-3 之后的主流选择

### 2. Loss Scaling 原理

问题：反向传播中，深层的梯度值可能非常小（如 1e-8），在 FP16 中会下溢为 0。

解决方案：前向结束后，把 loss 乘以一个大数（如 1024），这样反向传播的梯度都被等比放大，不会下溢。更新参数前再除以这个数。

```python
# 伪代码
scaled_loss = loss * scale_factor     # 放大 loss
scaled_loss.backward()                 # 梯度被等比放大
for param in model.parameters():
    param.grad /= scale_factor         # 还原真实梯度
optimizer.step()                        # 用真实梯度更新
```

**动态 Loss Scaling**：
- 初始 scale 设一个很大的值（如 2^16）
- 如果梯度没有 inf/nan，下次 scale × 2（尝试更大）
- 如果梯度出现 inf/nan，跳过本步更新，scale ÷ 2

### 3. DeepSpeed 框架

DeepSpeed 是微软开发的分布式训练框架，核心功能：

```
DeepSpeed
├── ZeRO 优化器 (Stage 1/2/3)
├── 混合精度训练
├── 梯度累积
├── 激活检查点 (Activation Checkpointing)
├── ZeRO-Offload (CPU/NVMe 卸载)
├── 稀疏注意力
├── 流水线并行
└── 通信优化
```

**DeepSpeed 的核心优势**：几乎不改代码，通过 JSON 配置文件控制一切。

**ZeRO-Offload / ZeRO-Infinity**：当 GPU 显存仍然不够时，把优化器状态甚至参数卸载到 CPU 内存或 NVMe SSD。

```
ZeRO-Offload (Stage 2 + CPU):
  GPU: FP16 参数 + FP16 梯度计算
  CPU: FP32 优化器状态 + 参数更新

ZeRO-Infinity (Stage 3 + CPU + NVMe):
  GPU: 当前层的 FP16 参数 (临时)
  CPU: 部分优化器状态
  NVMe: 全部优化器状态 + 参数分片
```

### 4. FSDP (Fully Sharded Data Parallel)

PyTorch 原生的 ZeRO-3 实现。核心思想与 ZeRO-3 相同：参数、梯度、优化器状态全部分片。

**FSDP vs DeepSpeed ZeRO-3**：

| 特性 | DeepSpeed ZeRO-3 | PyTorch FSDP |
|------|------------------|--------------|
| 维护方 | 微软 | Meta/PyTorch |
| 代码侵入 | 低（JSON 配置） | 中（需要包装模型） |
| 与 PyTorch 集成 | 插件 | 原生 |
| 混合精度 | 自己实现 | 用 PyTorch AMP |
| 生态 | HuggingFace 支持好 | 原生 PyTorch 生态 |
| 性能 | 略优（通信优化多） | 接近 |

### 5. 梯度累积 (Gradient Accumulation)

**问题**：显存不够放不下大 batch，但大 batch 对训练稳定性很重要。

**方案**：多次小 batch 前向反向累积梯度，攒够了再更新一次参数。

```
有效 batch_size = micro_batch_size × gradient_accumulation_steps × num_gpus

例如: 有效 batch = 4 × 8 × 64 = 2048
  - 每卡每步处理 4 条数据
  - 累积 8 步梯度
  - 64 张卡
```

数学上完全等价于用 batch_size=2048 训练。

### 6. 激活检查点 (Activation Checkpointing / Gradient Checkpointing)

**问题**：反向传播需要前向时的激活值。对于 96 层的 GPT-3，存所有激活需要巨大显存。

**方案**：只保存部分层的激活值，其他层的激活在反向传播时**重新计算**。

```
普通方式: 所有激活都存 → 显存 O(L)，计算 1x
激活检查点: 每隔 k 层存一次 → 显存 O(L/k)，计算 ~1.33x

典型配置: 每个 Transformer block 作为一个 checkpoint 单元
  - 前向: 只保存每个 block 的输入
  - 反向: 需要某 block 内部激活时，从该 block 输入重新前向计算
  - 额外计算开销: ~33% (每个 block 的前向算了两遍)
  - 显存节省: 激活值从 O(层数×隐藏维度) 降到 O(sqrt(层数)×隐藏维度)
```

## Python 代码示例

### 示例 1：PyTorch 原生混合精度训练

```python
"""
PyTorch AMP (Automatic Mixed Precision) 混合精度训练
展示 FP16 和 BF16 两种模式
"""
import torch
import torch.nn as nn
from torch.cuda.amp import autocast, GradScaler


class SimpleModel(nn.Module):
    def __init__(self, vocab_size=32000, d_model=1024, n_layers=6, n_heads=16):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, d_model)
        self.layers = nn.ModuleList([
            nn.TransformerEncoderLayer(
                d_model=d_model,
                nhead=n_heads,
                dim_feedforward=4 * d_model,
                batch_first=True,
                norm_first=True,  # Pre-LN
            )
            for _ in range(n_layers)
        ])
        self.ln_f = nn.LayerNorm(d_model)
        self.head = nn.Linear(d_model, vocab_size, bias=False)

    def forward(self, input_ids):
        x = self.embed(input_ids)
        for layer in self.layers:
            x = layer(x)
        x = self.ln_f(x)
        return self.head(x)


def train_with_amp(use_bf16: bool = False):
    """
    混合精度训练示例
    use_bf16=False: FP16 + Loss Scaling (需要 GradScaler)
    use_bf16=True:  BF16 (不需要 GradScaler)
    """
    device = torch.device("cuda")
    model = SimpleModel().to(device)
    optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4, weight_decay=0.01)
    loss_fn = nn.CrossEntropyLoss()

    # FP16 需要 GradScaler 做动态 Loss Scaling
    # BF16 不需要（范围足够大）
    dtype = torch.bfloat16 if use_bf16 else torch.float16
    scaler = GradScaler(enabled=not use_bf16)  # BF16 时禁用

    print(f"训练模式: {'BF16' if use_bf16 else 'FP16 + Loss Scaling'}")
    print(f"模型参数量: {sum(p.numel() for p in model.parameters()) / 1e6:.1f}M")

    for step in range(100):
        input_ids = torch.randint(0, 32000, (8, 512), device=device)
        labels = torch.randint(0, 32000, (8, 512), device=device)

        optimizer.zero_grad()

        # autocast 区域内自动将支持的运算转为半精度
        with autocast(device_type="cuda", dtype=dtype):
            logits = model(input_ids)
            loss = loss_fn(logits.view(-1, 32000), labels.view(-1))
            # 注意: loss 在 autocast 内仍为 FP32（CrossEntropyLoss 自动升精度）

        # GradScaler 处理 loss scaling + 梯度检查 + 参数更新
        scaler.scale(loss).backward()
        scaler.unscale_(optimizer)  # 在 clip 前 unscale
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
        scaler.step(optimizer)
        scaler.update()

        if step % 20 == 0:
            print(f"  Step {step:3d}, Loss: {loss.item():.4f}, "
                  f"Scale: {scaler.get_scale():.0f}")


def check_precision_impact():
    """展示不同精度下的数值差异"""
    # 小梯度在 FP16 中下溢
    grad_fp32 = torch.tensor(1e-8, dtype=torch.float32)
    grad_fp16 = grad_fp32.half()
    grad_bf16 = grad_fp32.bfloat16()

    print("\n精度对比:")
    print(f"  原始值 (FP32): {grad_fp32.item():.2e}")
    print(f"  FP16:          {grad_fp16.item():.2e}  {'(下溢!)' if grad_fp16.item() == 0 else ''}")
    print(f"  BF16:          {grad_bf16.item():.2e}  {'(下溢!)' if grad_bf16.item() == 0 else ''}")

    # Loss Scaling 的效果
    scale = 1024.0
    scaled_grad_fp16 = (grad_fp32 * scale).half()
    recovered = scaled_grad_fp16.float() / scale
    print(f"\n  Loss Scaling (scale={scale:.0f}):")
    print(f"  放大后 FP16:    {scaled_grad_fp16.item():.2e}")
    print(f"  还原后:         {recovered.item():.2e}")


if __name__ == "__main__":
    check_precision_impact()
    # 如果有 GPU:
    # train_with_amp(use_bf16=False)  # FP16 模式
    # train_with_amp(use_bf16=True)   # BF16 模式
```

### 示例 3：PyTorch FSDP 训练

```python
"""
PyTorch FSDP (Fully Sharded Data Parallel) 训练示例
FSDP 是 PyTorch 原生的 ZeRO-3 实现
"""
import os
import torch
import torch.nn as nn
import torch.distributed as dist
from functools import partial
from torch.distributed.fsdp import (
    FullyShardedDataParallel as FSDP,
    MixedPrecision,
    ShardingStrategy,
    CPUOffload,
)
from torch.distributed.fsdp.wrap import (
    transformer_auto_wrap_policy,
    size_based_auto_wrap_policy,
)


class TransformerBlock(nn.Module):
    """单个 Transformer Block — FSDP 的最小包装单元"""
    def __init__(self, d_model=2048, n_heads=16):
        super().__init__()
        self.ln1 = nn.LayerNorm(d_model)
        self.attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.ln2 = nn.LayerNorm(d_model)
        self.mlp = nn.Sequential(
            nn.Linear(d_model, 4 * d_model),
            nn.GELU(),
            nn.Linear(4 * d_model, d_model),
        )

    def forward(self, x):
        h = self.ln1(x)
        x = x + self.attn(h, h, h, need_weights=False)[0]
        x = x + self.mlp(self.ln2(x))
        return x


class LLM(nn.Module):
    def __init__(self, vocab_size=32000, d_model=2048, n_layers=24, n_heads=16):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, d_model)
        self.blocks = nn.ModuleList([
            TransformerBlock(d_model, n_heads) for _ in range(n_layers)
        ])
        self.ln_f = nn.LayerNorm(d_model)
        self.head = nn.Linear(d_model, vocab_size, bias=False)

    def forward(self, input_ids):
        x = self.embed(input_ids)
        for block in self.blocks:
            x = block(x)
        return self.head(self.ln_f(x))


def train_fsdp():
    dist.init_process_group("nccl")
    local_rank = int(os.environ["LOCAL_RANK"])
    torch.cuda.set_device(local_rank)

    # --- 混合精度策略 ---
    bf16_policy = MixedPrecision(
        param_dtype=torch.bfloat16,    # 参数存储为 BF16
        reduce_dtype=torch.bfloat16,   # 梯度规约用 BF16
        buffer_dtype=torch.bfloat16,   # Buffer 用 BF16
    )

    # --- 自动包装策略: 每个 TransformerBlock 作为一个 FSDP 单元 ---
    wrap_policy = partial(
        transformer_auto_wrap_policy,
        transformer_layer_cls={TransformerBlock},  # 指定哪种类被独立包装
    )

    # --- 创建模型并用 FSDP 包装 ---
    model = LLM()
    model = FSDP(
        model,
        auto_wrap_policy=wrap_policy,         # 自动包装策略
        mixed_precision=bf16_policy,          # 混合精度
        sharding_strategy=ShardingStrategy.FULL_SHARD,  # 等价于 ZeRO-3
        # ShardingStrategy.SHARD_GRAD_OP → 等价于 ZeRO-2
        # ShardingStrategy.NO_SHARD → 等价于 DDP
        cpu_offload=CPUOffload(offload_params=False),  # 可选 CPU 卸载
        device_id=local_rank,
    )

    optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4, weight_decay=0.01)
    loss_fn = nn.CrossEntropyLoss()

    # --- 训练循环 ---
    for step in range(100):
        input_ids = torch.randint(0, 32000, (4, 512), device=f"cuda:{local_rank}")
        labels = torch.randint(0, 32000, (4, 512), device=f"cuda:{local_rank}")

        logits = model(input_ids)
        loss = loss_fn(logits.view(-1, 32000), labels.view(-1))

        loss.backward()
        torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
        optimizer.step()
        optimizer.zero_grad()

        if local_rank == 0 and step % 20 == 0:
            print(f"Step {step}, Loss: {loss.item():.4f}")

    # --- 保存模型 (需要特殊处理) ---
    from torch.distributed.fsdp import FullStateDictConfig, StateDictType
    save_policy = FullStateDictConfig(offload_to_cpu=True, rank0_only=True)
    with FSDP.state_dict_type(model, StateDictType.FULL_STATE_DICT, save_policy):
        state_dict = model.state_dict()
        if local_rank == 0:
            torch.save(state_dict, "model_fsdp.pt")

    dist.destroy_process_group()


# 启动: torchrun --nproc_per_node=8 train_fsdp.py
if __name__ == "__main__":
    train_fsdp()
```

### 示例 4：梯度累积与激活检查点

```python
"""
梯度累积 + 激活检查点
在显存有限时实现大 batch 训练
"""
import torch
import torch.nn as nn
from torch.utils.checkpoint import checkpoint


class CheckpointedTransformerBlock(nn.Module):
    """支持激活检查点的 Transformer Block"""
    def __init__(self, d_model=1024, n_heads=16):
        super().__init__()
        self.ln1 = nn.LayerNorm(d_model)
        self.attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.ln2 = nn.LayerNorm(d_model)
        self.mlp = nn.Sequential(
            nn.Linear(d_model, 4 * d_model),
            nn.GELU(),
            nn.Linear(4 * d_model, d_model),
        )

    def forward(self, x):
        h = self.ln1(x)
        x = x + self.attn(h, h, h, need_weights=False)[0]
        x = x + self.mlp(self.ln2(x))
        return x


class GPTWithCheckpointing(nn.Module):
    def __init__(self, vocab_size=32000, d_model=1024, n_layers=24, n_heads=16,
                 use_checkpointing: bool = True):
        super().__init__()
        self.use_checkpointing = use_checkpointing
        self.embed = nn.Embedding(vocab_size, d_model)
        self.blocks = nn.ModuleList([
            CheckpointedTransformerBlock(d_model, n_heads)
            for _ in range(n_layers)
        ])
        self.ln_f = nn.LayerNorm(d_model)
        self.head = nn.Linear(d_model, vocab_size, bias=False)

    def forward(self, input_ids):
        x = self.embed(input_ids)
        for block in self.blocks:
            if self.use_checkpointing and self.training:
                # 激活检查点: 不保存中间激活值，反向时重新计算
                x = checkpoint(block, x, use_reentrant=False)
            else:
                x = block(x)
        return self.head(self.ln_f(x))


def train_with_gradient_accumulation(
    model: nn.Module,
    accumulation_steps: int = 8,
    micro_batch_size: int = 4,
    num_steps: int = 100,
):
    """
    梯度累积训练

    有效 batch = micro_batch_size × accumulation_steps = 4 × 8 = 32
    """
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    model = model.to(device)
    optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4)
    loss_fn = nn.CrossEntropyLoss()

    effective_batch = micro_batch_size * accumulation_steps
    print(f"微批次大小: {micro_batch_size}")
    print(f"累积步数: {accumulation_steps}")
    print(f"有效批次大小: {effective_batch}")
    print(f"激活检查点: {'开启' if model.use_checkpointing else '关闭'}")

    for step in range(num_steps):
        total_loss = 0.0

        for micro_step in range(accumulation_steps):
            input_ids = torch.randint(0, 32000, (micro_batch_size, 512), device=device)
            labels = torch.randint(0, 32000, (micro_batch_size, 512), device=device)

            logits = model(input_ids)
            # 除以累积步数，保证梯度平均值正确
            loss = loss_fn(logits.view(-1, 32000), labels.view(-1)) / accumulation_steps

            loss.backward()  # 梯度会在 .grad 上累积，不会清零
            total_loss += loss.item()

        # 累积完毕，执行一次参数更新
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
        optimizer.step()
        optimizer.zero_grad()  # 更新后才清零

        if step % 10 == 0:
            print(f"Step {step:3d}, Loss: {total_loss:.4f}")


def compare_memory_usage():
    """对比激活检查点开启/关闭的显存差异"""
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    if device.type != "cuda":
        print("需要 GPU 才能对比显存")
        return

    for use_ckpt in [False, True]:
        torch.cuda.reset_peak_memory_stats()
        model = GPTWithCheckpointing(
            d_model=1024, n_layers=24, use_checkpointing=use_ckpt
        ).to(device)

        input_ids = torch.randint(0, 32000, (8, 512), device=device)
        labels = torch.randint(0, 32000, (8, 512), device=device)

        logits = model(input_ids)
        loss = nn.functional.cross_entropy(logits.view(-1, 32000), labels.view(-1))
        loss.backward()

        peak_mem = torch.cuda.max_memory_allocated() / 1e9
        mode = "开启检查点" if use_ckpt else "关闭检查点"
        print(f"{mode}: 峰值显存 {peak_mem:.2f} GB")

        del model, input_ids, labels, logits, loss
        torch.cuda.empty_cache()


if __name__ == "__main__":
    model = GPTWithCheckpointing(use_checkpointing=True)
    param_count = sum(p.numel() for p in model.parameters()) / 1e6
    print(f"模型参数量: {param_count:.1f}M\n")

    # compare_memory_usage()  # 需要 GPU
    train_with_gradient_accumulation(model, accumulation_steps=8, num_steps=50)
```

### 训练监控要点

```
关键指标:
1. Loss 曲线 — 稳定下降，无突刺
2. 学习率 — Warmup → Peak → Decay
3. 梯度范数 — 稳定在 0.1~10 之间
4. GPU 利用率 — >90% 说明通信不是瓶颈
5. 吞吐量 — tokens/sec/GPU
6. MFU — >40% 为良好

Loss Spike 处理:
- 轻微 spike (2-3x): 通常自动恢复，继续训练
- 严重 spike (10x+): 从最近 checkpoint 回滚，降低 lr
- 持续发散: 检查数据质量、减小 batch size、增加 warmup

常用监控工具:
- Weights & Biases (wandb)
- TensorBoard
- DeepSpeed 自带 flops profiler
```

### FP16 vs BF16 选择指南

```
选 BF16:
  ✓ GPU 支持 (A100/H100/4090 等 Ampere+ 架构)
  ✓ 不需要 loss scaling (省心)
  ✓ 训练更稳定 (范围大)
  ✓ 大模型预训练首选

选 FP16:
  ✓ 老 GPU (V100)
  ✓ 精度要求高的任务 (尾数 10 位 vs BF16 的 7 位)
  ✓ 需要配合 Loss Scaling

实际情况:
  2023 年之后的训练几乎全部使用 BF16
  FP8 在 H100 上开始用于前向传播 (Transformer Engine)
```

## 小结

| 技术 | 核心作用 | 额外开销 | 推荐场景 |
|------|---------|---------|---------|
| BF16 混合精度 | 显存减半，速度翻倍 | 需要 FP32 主权重 | 所有训练 |
| FP16 + Loss Scaling | 同上 | Loss Scaling 复杂度 | V100 等老卡 |
| 梯度累积 | 用小 batch 模拟大 batch | 训练时间略增 | 显存不够时 |
| 激活检查点 | 显存降到 O(sqrt(L)) | ~33% 额外计算 | 深层大模型 |
| DeepSpeed | ZeRO + 混合精度 + 一键配置 | 学习配置 | 多卡训练 |
| FSDP | PyTorch 原生 ZeRO-3 | 需要包装模型 | PyTorch 原生党 |
