# 02 经典网络架构

这个模块不是为了让你停在 CNN、RNN 时代，而是帮你看懂 Transformer 出现之前，深度学习是怎样处理图像和序列的。

{{img:ch02_00_classic_architecture_bridge}}

## 先抓住这条线

不同架构的差别，来自它们对数据结构的不同假设：

- 图像有空间结构，所以 CNN 用滑动窗口看局部图案。
- 文本和时间序列有先后顺序，所以 RNN 按顺序一步步读。
- 长序列需要记忆，所以 LSTM/GRU 加了门控。
- 输入和输出长度可能不同，所以编码器-解码器先读完再生成。

## 和 Transformer 的关系

Transformer 并不是凭空出现的。它继承了“序列建模”和“编码器-解码器”的问题意识，只是用注意力机制换掉了 RNN 那种一步步读的方式。
