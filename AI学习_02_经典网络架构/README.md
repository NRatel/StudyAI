# AI学习：02 经典网络架构

> 读本模块时，先抓住 CNN、RNN、Seq2Seq 各自为什么出现、解决哪类数据问题。它们是通向 Transformer 的背景桥梁，不需要把所有历史架构都当作主线必修。

## 本模块导读

**学什么**：三大经典神经网络架构 —— CNN 处理空间数据（图像）、RNN/LSTM/GRU 处理序列数据（文本/时间序列）、编码器-解码器框架处理序列到序列的映射（翻译/摘要）。

**为什么学**：这些架构能帮你更顺地理解 Transformer 的来龙去脉。Transformer 的注意力机制正是为了解决 RNN 的长距离依赖问题而兴起；CNN 中的卷积思想在后续模型中反复出现；编码器-解码器框架也直接延续到了 Transformer 架构中。

{{img:ch02_00_classic_architecture_bridge}}

**读完能看懂的主线**：
- 理解 CNN 如何从图像中提取特征（卷积、池化、特征图）
- 理解 RNN 为什么能处理变长序列，以及为什么原始 RNN 不够用（梯度消失）
- 理解 LSTM/GRU 如何通过门控机制解决长距离依赖
- 理解 Seq2Seq 编码器-解码器框架，为学习 Transformer 做准备
- 为后续注意力机制、Transformer、多模态模型建立架构直觉

**建议阅读顺序**：CNN → RNN → LSTM/GRU → Seq2Seq，本模块最终目的是为 Transformer 做铺垫。

**路径建议**：本模块是背景桥梁 / 推荐补读。想快速抓住 GPT 主线，可以先走 `01 → 03 → 04 → 05`；如果读到 Transformer、扩散模型或多模态时发现 CNN/RNN/Seq2Seq 背景不够，再回到本模块补齐。

**前置知识**：[01 神经网络基础](../AI学习_01_神经网络基础/README.md)（前向传播、反向传播、优化器、正则化）

**预估阅读时间**：8~10 小时

## 读完能解释

1. CNN 为什么适合处理图像？卷积、池化、参数共享分别解决什么问题？
2. RNN 为什么适合处理序列？它为什么又会遇到长距离依赖和梯度消失？
3. LSTM/GRU 的“门”到底在控制什么信息？
4. Seq2Seq 的信息瓶颈是什么？为什么它自然引出了注意力机制？
