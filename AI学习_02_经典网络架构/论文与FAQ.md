# 论文与FAQ

> 本文件用于补充经典网络架构的历史脉络和常见误区。正文负责讲清概念；这里帮助你把 CNN、RNN、LSTM、Seq2Seq、Transformer 放到同一张地图上。

## 一、关键论文脉络

| 年份 | 论文/方法 | 为什么重要 |
|------|-----------|------------|
| 1997 | LSTM | 用门控记忆缓解 RNN 长距离依赖问题 |
| 1998 | LeNet-5 | 早期 CNN 代表，证明卷积适合图像识别 |
| 2012 | AlexNet | 让深度 CNN 在 ImageNet 上爆发，带动深度学习复兴 |
| 2014 | VGG | 用小卷积核堆深网络，结构规整，便于理解 |
| 2014 | GoogLeNet/Inception | 展示多尺度卷积组合的价值 |
| 2014 | Seq2Seq | 用编码器-解码器处理输入输出长度不同的序列任务 |
| 2014 | Bahdanau Attention | 解决 Seq2Seq 固定向量瓶颈，引出注意力机制 |
| 2015 | ResNet | 用残差连接让很深的网络更容易训练 |
| 2017 | Transformer | 用自注意力取代循环结构，成为后续大模型基础 |

学习建议：

- CNN 重点看：局部连接、权值共享、感受野。
- RNN/LSTM 重点看：隐藏状态、门控、长距离依赖。
- Seq2Seq 重点看：编码器、解码器、信息瓶颈。
- ResNet 重点看：残差连接为什么让深层网络更容易训练。

## 二、常见误区与 FAQ

### Q1：CNN 只能用于图像吗？

不是。CNN 的核心是“局部模式 + 权值共享”，这不只适用于图像。

| 数据 | CNN 关注什么 |
|------|--------------|
| 图像 | 局部边缘、纹理、形状 |
| 文本 | 局部词组模式 |
| 音频 | 局部时间片段 |
| 时间序列 | 局部变化趋势 |

只是图像天然有二维空间结构，所以 CNN 在视觉任务中最经典。

### Q2：为什么 CNN 比全连接网络适合图像？

因为图像有两个重要性质：

1. **局部性**：相邻像素关系最密切，边缘、纹理都来自局部区域。
2. **平移性**：同一个边缘检测器可以在图片任何位置使用。

卷积正好利用这两点：

- 局部连接减少不必要的全局连接。
- 权值共享让同一个卷积核在整张图上复用。

所以 CNN 不是“更花哨的全连接”，而是把图像结构先验写进了模型。

### Q3：RNN 为什么适合序列？

RNN 有一个隐藏状态 $h_t$，会随着时间步更新：

```text
当前输入 x_t + 上一步记忆 h_{t-1} → 当前记忆 h_t
```

这让它天然适合处理顺序数据，比如句子、语音、时间序列。

RNN 的问题是：信息要一步一步传递，距离太远时容易衰减，所以它不擅长记住很长的依赖关系。

### Q4：LSTM 和 GRU 哪个更好？

没有绝对答案。

| 模型 | 特点 |
|------|------|
| LSTM | 门更多，记忆结构更细，表达能力强 |
| GRU | 结构更简洁，参数更少，训练更轻 |

一般理解：

- 数据和任务复杂时，LSTM 可能更稳。
- 资源有限或想要简洁结构时，GRU 常是好选择。

本章重点不是背公式，而是理解：**门控机制让模型学会“该记什么、该忘什么”。**

### Q5：ResNet 为什么有效？

深层网络难训练，不只是因为表达能力不够，而是因为信息和梯度在很多层中传递会变困难。

ResNet 的残差连接让一层学习：

```text
输出 = 输入 + 需要补充的变化
```

也就是说，模型不必每层都从零重建表示，只需要学习“在原来基础上改一点”。这让深层网络更容易训练。

直觉类比：让学生修改一篇已有文章，比从白纸开始写整篇文章容易。

### Q6：RNN 被 Transformer 淘汰了吗？

在大语言模型主线中，Transformer 取代了 RNN 的核心位置；但这不等于 RNN 思想毫无价值。

RNN 的优点：

- 顺序建模直观。
- 对流式数据很自然。
- 某些小模型或低资源场景仍有意义。

Transformer 的优势：

- 并行计算强。
- 长距离依赖路径短。
- 更适合大规模预训练。

学习 RNN 仍然有价值，因为它能帮助你理解“序列记忆”这个问题，以及 Transformer 为什么要换一种方式解决它。

### Q7：Seq2Seq 和 Transformer 是什么关系？

Seq2Seq 是一种任务框架：输入一个序列，输出另一个序列。

```text
英文句子 → 中文句子
长文章 → 摘要
问题 → 答案
```

早期 Seq2Seq 常用 RNN/LSTM 做编码器和解码器。后来 Transformer 也可以做编码器-解码器，并且更擅长并行计算和长距离关系建模。

所以：

```text
Seq2Seq 是“输入到输出”的框架
Transformer 是实现这个框架的一种强大架构
```

### Q8：网络越深越好吗？

不一定。

更深的网络可能表达更复杂的函数，但也更难训练。没有残差连接、归一化、合适初始化时，深层网络可能比浅层更差。

现代深层网络能成功，靠的不只是“层数多”，还包括：

- 残差连接
- 归一化
- 合理初始化
- 足够数据
- 合适优化方法
