# 编码器-解码器框架（Seq2Seq）

> **前置知识**：[RNN 循环神经网络](2_RNN循环神经网络.md)、[LSTM 与 GRU](3_LSTM与GRU.md)

---

## 直觉与概述

### 为什么需要 Seq2Seq？

前面学的 RNN/LSTM 可以处理序列，但有一个隐含假设：**输入和输出等长**（多对多同步模式）。现实中很多任务不满足这个假设：

| 任务 | 输入长度 | 输出长度 | 等长？ |
|------|---------|---------|--------|
| 机器翻译 | "我爱你"（3 词） | "I love you"（3 词） | 偶然等长 |
| 文本摘要 | 500 词文章 | 50 词摘要 | 不等长 |
| 对话生成 | "你好"（2 字） | "你好，很高兴认识你！"（8 字） | 不等长 |
| 语音识别 | 16000 帧音频 | 5 个字 | 不等长 |

**Seq2Seq（Sequence-to-Sequence）** 框架的核心思想：

1. **编码器（Encoder）**：读入整个输入序列，把信息"压缩"成一个固定长度的向量
2. **解码器（Decoder）**：从这个向量出发，一步步"展开"生成输出序列

> **类比**：编码器 = 阅读理解（把一段话读完，形成理解），解码器 = 用自己的话复述（基于理解，逐字生成回答）。

### Seq2Seq 的历史地位

Seq2Seq 是连接 RNN 时代和 Transformer 时代的桥梁：
- 2014 年由 Sutskever 等人提出，奠定了序列到序列任务的基本范式
- 信息瓶颈问题催生了**注意力机制**（2014，Bahdanau）
- Transformer（2017）保留了编码器-解码器的整体架构，但用注意力完全替代了 RNN

## 严谨定义与原理

### 1. 基本 Seq2Seq 架构

```
输入序列: x₁, x₂, ..., xₘ

         ┌────┐   ┌────┐   ┌────┐   ┌────┐
  x₁ ──→ │ h₁ │──→│ h₂ │──→│ h₃ │──→│ hₘ │ ──→ c (上下文向量)
         └────┘   └────┘   └────┘   └────┘
         编码器 (Encoder LSTM)

                   c
                   │
         ┌────┐   ┌────┐   ┌────┐   ┌────┐
  <BOS>→ │ s₁ │──→│ s₂ │──→│ s₃ │──→│ s₄ │──→ <EOS>
         └─┬──┘   └─┬──┘   └─┬──┘   └─┬──┘
           ↓        ↓        ↓        ↓
           y₁       y₂       y₃       y₄
         解码器 (Decoder LSTM)
```

**编码器**：
$$h_t = \text{LSTM}(x_t, h_{t-1})$$

最终隐藏状态作为上下文向量：$\text{ctx} = h_m$（注意：这里的 ctx 不是 LSTM 的 cell state $c$）

**解码器**：
$$s_t = \text{LSTM}(y_{t-1}, s_{t-1}), \quad s_0 = c$$
$$P(y_t | y_{<t}, x) = \text{softmax}(W_o s_t + b_o)$$

解码器是**自回归（Autoregressive）** 的：每一步的输出 $y_t$ 作为下一步的输入。

### 2. Teacher Forcing

训练和推理时，解码器的输入来源不同：

| 阶段 | 解码器第 t 步的输入 | 优点 | 缺点 |
|------|-------------------|------|------|
| **训练（Teacher Forcing）** | 真实标签 $y_{t-1}^*$ | 收敛快、训练稳定 | 训练/推理输入分布不匹配 |
| **推理** | 自己上一步的输出 $\hat{y}_{t-1}$ | 真实使用场景 | 错误会累积（一步错步步错） |

这种不匹配叫做 **Exposure Bias**：训练时解码器总是看到"正确答案"，从未学过如何从"自己的错误"中恢复。

缓解方法（训练层面）：
- **Scheduled Sampling**：训练时以一定概率（随 epoch 增加）用自己的输出替代真实标签，让模型学会从自身错误中恢复

另外，**Beam Search** 是推理阶段的搜索策略（保留 top-k 候选序列），它缓解的是贪心解码的搜索误差，而非训练层面的 Exposure Bias，两者解决的是不同层面的问题。

> **局限**：基础 Seq2Seq 将整个输入序列压缩为一个固定长度的上下文向量，这个向量就是信息瓶颈——序列越长，信息丢失越严重。

### 3. 信息瓶颈问题

基本 Seq2Seq 的致命弱点：**所有输入信息被压缩进一个固定长度的向量 ctx**。

```
"The European Commission said on Thursday it disagreed with German
 opinion that the imports of Japanese luxury products threatened to
 disrupt the Community market."
                    ↓
              c = [0.23, -0.45, 0.78, ...]  ← 256维向量
                    ↓
          翻译需要从这个向量中恢复所有信息
```

问题：
- 输入序列越长，信息丢失越严重
- 实验证明：超过 20~30 词后，翻译质量急剧下降（Cho et al., 2014）
- 上下文向量 ctx 成了"瓶颈"——所有信息都要从这个窄口挤过去

### 4. 注意力机制（Bahdanau Attention, 2014）

注意力机制的核心思想：**解码器每一步不再只看一个固定的 c，而是"回头看"编码器的所有隐藏状态，动态决定关注哪些位置**。

#### 计算过程

**第一步：计算注意力分数**

解码器在第 $i$ 步时，对编码器每个位置 $j$ 计算一个分数：

$$e_{ij} = a(s_{i-1}, h_j)$$

其中 $a$ 是一个小型神经网络（对齐模型）：

$$e_{ij} = v^T \tanh(W_s s_{i-1} + W_h h_j)$$

**第二步：归一化为注意力权重**

$$\alpha_{ij} = \frac{\exp(e_{ij})}{\sum_{k=1}^{m} \exp(e_{ik})}$$

$\alpha_{ij}$ 表示"解码第 $i$ 个输出时，应该关注输入第 $j$ 个位置多少"。

**第三步：计算上下文向量**

$$c_i = \sum_{j=1}^{m} \alpha_{ij} h_j$$

注意：现在每个解码步 $i$ 都有自己的上下文向量 $c_i$，不再是固定的 $c$。

**第四步：融入解码器**

$$s_i = \text{LSTM}([y_{i-1}; c_i], s_{i-1})$$

或者在输出层融入：

$$\tilde{s}_i = \tanh(W_c[c_i; s_i])$$
$$P(y_i) = \text{softmax}(W_o \tilde{s}_i)$$

#### 注意力的直觉

以英法翻译为例：

```
编码器输入:   The  cat  sat  on  the  mat
注意力权重: [0.01 0.02 0.85 0.05 0.02 0.05]  ← 翻译 "assis" 时
                        ↑
                   关注 "sat"
```

注意力权重形成一个矩阵，可以可视化为**对齐矩阵**，显示输入输出之间的对应关系。

#### 为什么注意力能解决信息瓶颈

| 基本 Seq2Seq | + 注意力 |
|-------------|---------|
| 上下文 = 固定向量 ctx | 上下文 = 每步动态加权 $c_i$ |
| 长序列信息压缩丢失 | 解码器可直接访问任意输入位置 |
| 梯度需要穿过整个编码器 | 注意力提供了"梯度捷径" |

> **关键洞察**：注意力的本质是一种**可微的软寻址**——给定一个查询（解码器状态），在一组键值对（编码器隐藏状态）中做加权检索。这个思想直接演化为 Transformer 中的 Query-Key-Value 注意力。

> **局限**：加入注意力后信息瓶颈问题得到缓解，但编码器和解码器仍基于 RNN 的顺序计算——$h_t$ 必须等 $h_{t-1}$ 算完，无法在时间步维度上并行，训练速度受限于序列长度。这一局限直接促成了 Transformer 用自注意力完全替代 RNN 的设计。

## Python 代码示例

### 示例 1：基本 Seq2Seq（数字序列反转）

```python
import torch
import torch.nn as nn
import random

# === 任务：将数字序列反转 ===
# 输入: [3, 1, 4, 1, 5] → 输出: [5, 1, 4, 1, 3]
# 用 0 作为 <PAD>, 10 作为 <BOS>, 11 作为 <EOS>

VOCAB_SIZE = 12  # 0-9 + BOS(10) + EOS(11)
BOS, EOS = 10, 11
HIDDEN = 64
EMBED_DIM = 32


class Encoder(nn.Module):
    def __init__(self):
        super().__init__()
        self.embed = nn.Embedding(VOCAB_SIZE, EMBED_DIM)
        self.lstm = nn.LSTM(EMBED_DIM, HIDDEN, batch_first=True)

    def forward(self, x):
        # x: (batch, seq_len)
        embedded = self.embed(x)           # (batch, seq_len, embed_dim)
        outputs, (h, c) = self.lstm(embedded)
        return h, c                        # 最终隐藏状态作为上下文


class Decoder(nn.Module):
    def __init__(self):
        super().__init__()
        self.embed = nn.Embedding(VOCAB_SIZE, EMBED_DIM)
        self.lstm = nn.LSTM(EMBED_DIM, HIDDEN, batch_first=True)
        self.fc = nn.Linear(HIDDEN, VOCAB_SIZE)

    def forward(self, x, h, c):
        # x: (batch, 1) — 当前时间步的输入
        embedded = self.embed(x)           # (batch, 1, embed_dim)
        output, (h, c) = self.lstm(embedded, (h, c))
        pred = self.fc(output.squeeze(1))  # (batch, vocab_size)
        return pred, h, c


class Seq2Seq(nn.Module):
    def __init__(self):
        super().__init__()
        self.encoder = Encoder()
        self.decoder = Decoder()

    def forward(self, src, tgt, teacher_forcing_ratio=0.5):
        batch_size = src.size(0)
        tgt_len = tgt.size(1)

        # 编码
        h, c = self.encoder(src)

        # 解码（自回归 + Teacher Forcing）
        outputs = []
        dec_input = torch.full((batch_size, 1), BOS, dtype=torch.long)

        for t in range(tgt_len):
            pred, h, c = self.decoder(dec_input, h, c)
            outputs.append(pred)

            # Teacher Forcing：以一定概率用真实标签
            if random.random() < teacher_forcing_ratio:
                dec_input = tgt[:, t].unsqueeze(1)
            else:
                dec_input = pred.argmax(dim=1).unsqueeze(1)

        return torch.stack(outputs, dim=1)  # (batch, tgt_len, vocab_size)


# === 数据生成 ===
def make_batch(batch_size=32, seq_len=5):
    # 随机生成 1~9 的序列
    src = torch.randint(1, 10, (batch_size, seq_len))
    tgt = src.flip(dims=[1])  # 反转作为目标
    return src, tgt


# === 训练 ===
model = Seq2Seq()
optimizer = torch.optim.Adam(model.parameters(), lr=0.003)
criterion = nn.CrossEntropyLoss()

for epoch in range(200):
    src, tgt = make_batch(64, 5)
    output = model(src, tgt, teacher_forcing_ratio=0.5)

    # output: (batch, tgt_len, vocab_size), tgt: (batch, tgt_len)
    loss = criterion(output.reshape(-1, VOCAB_SIZE), tgt.reshape(-1))
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

    if (epoch + 1) % 50 == 0:
        pred = output.argmax(dim=2)
        acc = (pred == tgt).float().mean()
        print(f"Epoch {epoch+1}, Loss: {loss:.4f}, Acc: {acc:.4f}")

# === 推理测试 ===
model.eval()
with torch.no_grad():
    test_src = torch.tensor([[3, 1, 4, 1, 5]])
    h, c = model.encoder(test_src)
    dec_input = torch.tensor([[BOS]])
    result = []
    for _ in range(5):
        pred, h, c = model.decoder(dec_input, h, c)
        token = pred.argmax(dim=1)
        result.append(token.item())
        dec_input = token.unsqueeze(1)
    print(f"\n输入: {test_src.tolist()[0]}")
    print(f"期望: {test_src.tolist()[0][::-1]}")
    print(f"预测: {result}")
```

运行输出（示例）：
```
Epoch 50, Loss: 0.5832, Acc: 0.8250
Epoch 100, Loss: 0.1203, Acc: 0.9688
Epoch 150, Loss: 0.0324, Acc: 0.9969
Epoch 200, Loss: 0.0118, Acc: 1.0000

输入: [3, 1, 4, 1, 5]
期望: [5, 1, 4, 1, 3]
预测: [5, 1, 4, 1, 3]
```

### 示例 2：加入 Bahdanau 注意力

```python
import torch
import torch.nn as nn
import torch.nn.functional as F


class AttentionEncoder(nn.Module):
    def __init__(self):
        super().__init__()
        self.embed = nn.Embedding(VOCAB_SIZE, EMBED_DIM)
        self.lstm = nn.LSTM(EMBED_DIM, HIDDEN, batch_first=True)

    def forward(self, x):
        embedded = self.embed(x)
        outputs, (h, c) = self.lstm(embedded)
        return outputs, h, c  # 返回所有时间步的输出（注意力需要）


class BahdanauAttention(nn.Module):
    """Bahdanau (Additive) Attention"""
    def __init__(self, hidden_size):
        super().__init__()
        self.W_s = nn.Linear(hidden_size, hidden_size, bias=False)
        self.W_h = nn.Linear(hidden_size, hidden_size, bias=False)
        self.v = nn.Linear(hidden_size, 1, bias=False)

    def forward(self, decoder_state, encoder_outputs):
        # decoder_state: (batch, hidden)
        # encoder_outputs: (batch, src_len, hidden)
        src_len = encoder_outputs.size(1)

        # 扩展 decoder_state 以匹配 encoder_outputs
        s = decoder_state.unsqueeze(1).expand(-1, src_len, -1)  # (batch, src_len, hidden)

        # 注意力分数
        energy = self.v(torch.tanh(self.W_s(s) + self.W_h(encoder_outputs)))  # (batch, src_len, 1)
        energy = energy.squeeze(2)  # (batch, src_len)

        # 归一化
        alpha = F.softmax(energy, dim=1)  # (batch, src_len)

        # 加权求和
        context = torch.bmm(alpha.unsqueeze(1), encoder_outputs)  # (batch, 1, hidden)
        context = context.squeeze(1)  # (batch, hidden)

        return context, alpha


class AttentionDecoder(nn.Module):
    def __init__(self):
        super().__init__()
        self.embed = nn.Embedding(VOCAB_SIZE, EMBED_DIM)
        self.attention = BahdanauAttention(HIDDEN)
        self.lstm = nn.LSTM(EMBED_DIM + HIDDEN, HIDDEN, batch_first=True)
        self.fc = nn.Linear(HIDDEN * 2, VOCAB_SIZE)  # 拼接 context 和 output

    def forward(self, x, h, c, encoder_outputs):
        embedded = self.embed(x)  # (batch, 1, embed_dim)

        # 计算注意力
        context, alpha = self.attention(h.squeeze(0), encoder_outputs)

        # 拼接 embedding 和 context 作为 LSTM 输入
        lstm_input = torch.cat([embedded, context.unsqueeze(1)], dim=2)
        output, (h, c) = self.lstm(lstm_input, (h, c))

        # 拼接 output 和 context 做预测
        pred = self.fc(torch.cat([output.squeeze(1), context], dim=1))
        return pred, h, c, alpha


class AttentionSeq2Seq(nn.Module):
    def __init__(self):
        super().__init__()
        self.encoder = AttentionEncoder()
        self.decoder = AttentionDecoder()

    def forward(self, src, tgt, teacher_forcing_ratio=0.5):
        batch_size = src.size(0)
        tgt_len = tgt.size(1)

        encoder_outputs, h, c = self.encoder(src)

        outputs = []
        attentions = []
        dec_input = torch.full((batch_size, 1), BOS, dtype=torch.long)

        for t in range(tgt_len):
            pred, h, c, alpha = self.decoder(dec_input, h, c, encoder_outputs)
            outputs.append(pred)
            attentions.append(alpha)

            if random.random() < teacher_forcing_ratio:
                dec_input = tgt[:, t].unsqueeze(1)
            else:
                dec_input = pred.argmax(dim=1).unsqueeze(1)

        outputs = torch.stack(outputs, dim=1)
        attentions = torch.stack(attentions, dim=1)  # (batch, tgt_len, src_len)
        return outputs, attentions


# === 训练注意力模型 ===
attn_model = AttentionSeq2Seq()
optimizer = torch.optim.Adam(attn_model.parameters(), lr=0.003)

for epoch in range(200):
    src, tgt = make_batch(64, 8)  # 更长的序列
    output, attn_weights = attn_model(src, tgt, teacher_forcing_ratio=0.5)
    loss = criterion(output.reshape(-1, VOCAB_SIZE), tgt.reshape(-1))
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

    if (epoch + 1) % 50 == 0:
        pred = output.argmax(dim=2)
        acc = (pred == tgt).float().mean()
        print(f"Epoch {epoch+1}, Loss: {loss:.4f}, Acc: {acc:.4f}")
```

## 工程师视角

### Seq2Seq → Transformer 的演化

```
Seq2Seq (2014)
  │
  ├─ 问题: 信息瓶颈
  │
  ▼
Seq2Seq + Attention (2014, Bahdanau)
  │
  ├─ 问题: 仍依赖 RNN, 无法并行
  │
  ▼
Transformer (2017, Vaswani)
  │
  ├─ 保留: 编码器-解码器架构、注意力机制
  └─ 替换: 用 Self-Attention 替代 RNN, 完全可并行
```

Transformer 的注意力机制是 Bahdanau Attention 的泛化：
- Bahdanau：Query = 解码器状态，Key = Value = 编码器输出
- Transformer：Query、Key、Value 都有独立的线性变换，且引入了 Self-Attention（自己看自己）

### Beam Search 解码

贪心解码每步选概率最高的 token，但全局不一定最优。Beam Search 维护 $k$ 个候选序列：

```
Beam Size = 3

步骤1: 选 top-3 token → ["I", "The", "A"]
步骤2: 每个扩展 top-3 → 9 个候选 → 保留总分 top-3
        → ["I love", "I like", "The cat"]
步骤3: 继续扩展...
```

```python
# Beam Search 伪代码
def beam_search(model, src, beam_size=3, max_len=50):
    # 编码
    encoder_out, h, c = model.encoder(src)

    # 初始 beam: [(序列, 累积 log 概率, h, c)]
    beams = [([BOS], 0.0, h, c)]

    for _ in range(max_len):
        candidates = []
        for seq, score, h, c in beams:
            if seq[-1] == EOS:
                candidates.append((seq, score, h, c))
                continue
            # 扩展
            pred, h_new, c_new = model.decoder(seq[-1], h, c)
            log_probs = torch.log_softmax(pred, dim=-1)
            top_k = log_probs.topk(beam_size)
            for prob, idx in zip(top_k.values, top_k.indices):
                candidates.append((seq + [idx], score + prob, h_new, c_new))

        # 保留 top-k
        beams = sorted(candidates, key=lambda x: x[1], reverse=True)[:beam_size]

        # 全部结束则停止
        if all(b[0][-1] == EOS for b in beams):
            break

    return beams[0][0]  # 返回得分最高的序列
```

### 工程实践要点

| 要点 | 说明 |
|------|------|
| **Teacher Forcing 比例** | 训练初期用 1.0（纯 Teacher Forcing），后期逐渐降低到 0.3~0.5 |
| **长度惩罚** | Beam Search 倾向于生成短序列（log 概率越乘越小），需要除以长度做归一化 |
| **解码器输入** | 训练用 ground truth；推理用自己的输出 + Beam Search |
| **共享 Embedding** | 编码器和解码器的 Embedding 层可以共享（如果源语言和目标语言相同） |
| **双向编码器** | 编码器通常用双向 LSTM，解码器只能用单向（自回归约束） |

### 从 Attention 到 Self-Attention

本节介绍的注意力是**交叉注意力（Cross-Attention）**：解码器关注编码器。

下一模块将学习**自注意力（Self-Attention）**：序列关注自身的每个位置。这是 Transformer 的核心创新，也是理解 GPT、BERT 等模型的关键。

## 本章小结

| 概念 | 一句话总结 |
|------|-----------|
| Seq2Seq | 编码器压缩输入 → 固定向量 → 解码器展开输出 |
| Teacher Forcing | 训练时用真实标签加速收敛，但导致 Exposure Bias |
| 信息瓶颈 | 固定长度向量无法承载长序列的全部信息 |
| Bahdanau Attention | 解码每步动态关注编码器不同位置，解决信息瓶颈 |
| 注意力权重 | 可解释的对齐矩阵，显示输入输出的对应关系 |
| Beam Search | 维护 top-k 候选序列，比贪心解码质量更高 |
| 交叉注意力 → 自注意力 | 本节的交叉注意力是 Transformer 自注意力的直接前身 |
