# 编码器解码器框架

> **前置知识**：[RNN 循环神经网络](2_RNN循环神经网络.md)
>
> **本节目标**：理解为什么有些任务要先把输入读成表示，再一步步生成输出。

---

## 1. 有些任务不是一个输入配一个标签

图像分类是“图片 → 类别”。但翻译、摘要、问答这类任务更复杂：

```text
一段英文 → 一段中文
长文章 → 短摘要
问题 + 材料 → 答案
```

输入和输出长度不一定一样，结构也不一样。

编码器-解码器框架就是为这类任务准备的。

## 2. 编码器：先把输入读懂

编码器负责读输入，把它压缩成一组内部表示。

可以把它想成读书的人：先把原文读完，在脑子里形成理解。

这个内部表示不一定是人能直接读懂的句子，但它包含模型认为有用的信息。

## 3. 解码器：再把答案写出来

解码器负责根据编码器给出的表示，一步步生成输出。

比如翻译时：

```text
编码器读英文句子
解码器逐词生成中文句子
```

生成每个词时，解码器会参考已经生成的内容，以及编码器提供的输入信息。

## 4. 早期瓶颈：读完整篇只交一张纸条

早期 Seq2Seq 模型常把整段输入压成一个固定长度向量。

这像读完一整篇文章后，只给翻译员递一张很小的纸条。短句还行，长句就容易丢信息。

注意力机制后来解决了这个问题：解码器生成每个词时，可以回头看输入中最相关的位置。

## 5. 和 Transformer 的关系

Transformer 也可以使用编码器-解码器结构，只是里面不再靠 RNN 逐步传递状态，而是靠注意力直接建立关系。

所以编码器-解码器是一种任务框架；RNN、LSTM、Transformer 是实现这个框架的不同方式。
