# RNN 循环神经网络

## 序列不是一袋词
### 核心问题：序列不是一袋词

普通前馈网络默认输入长度固定，且各位置没有天然顺序。但语言、语音、时间序列都不一样：

- "我不喜欢这部电影" 和 "我喜欢这部电影不" 不是一回事。
- 股票、天气、传感器数据都依赖时间顺序。
- 一句话的后半句常常要靠前半句理解。

RNN（Recurrent Neural Network，循环神经网络）的核心想法是：**读序列时带着一个会更新的记忆状态**。

### 阅读类比

你读一句话时，不会每看到一个词就忘掉前面的词。你会带着一个不断更新的"当前理解"：

```
我 → 我很 → 我很喜欢 → 我很喜欢这部 → 我很喜欢这部电影
```

RNN 的隐藏状态 $h_t$ 就像这个"当前理解"。每读入一个新 token，RNN 同时看：

- 当前输入 $x_t$
- 上一步记忆 $h_{t-1}$

然后更新成新的记忆 $h_t$。

---

## RNN 的核心机制
### 1. 基本 RNN 公式

基本 RNN 的递推公式：

$$h_t = \tanh(W_{xh}x_t + W_{hh}h_{t-1} + b_h)$$

$$y_t = W_{hy}h_t + b_y$$

其中：

- $x_t$：第 $t$ 个时间步的输入
- $h_t$：第 $t$ 个时间步的隐藏状态
- $W_{xh}$：输入到隐藏状态的权重
- $W_{hh}$：上一隐藏状态到当前隐藏状态的权重
- $y_t$：当前时间步输出

最重要的是 $W_{hh}h_{t-1}$：它让过去的信息影响现在。

### 2. 展开图：循环其实是共享参数的深层网络

RNN 可以沿时间展开：

```
h0 → h1 → h2 → h3 → ...
     ↑    ↑    ↑
     x1   x2   x3
```

每个时间步看起来像一层网络，但这些"层"共享同一套参数 $W_{xh}$、$W_{hh}$、$W_{hy}$。

这带来两个结果：

- 序列多长都能处理，因为参数不会随长度增加。
- 时间步越长，反向传播路径越长，梯度问题越严重。

### 3. 隐藏状态到底记了什么

$h_t$ 不是原文的拷贝，而是模型压缩出的状态向量。

它可能包含：

- 当前句子的主题
- 最近出现的重要词
- 语法状态
- 时间序列的近期趋势

但它容量有限。所有过去信息都要被压进一个固定长度向量，这就是 RNN 处理长序列时的根本压力。

### 4. RNN 的常见输入输出模式

| 模式 | 例子 | 说明 |
|------|------|------|
| One-to-One | 普通分类 | 不是 RNN 重点 |
| One-to-Many | 图像生成描述 | 一个输入生成序列 |
| Many-to-One | 情感分类 | 读完整句后输出一个结果 |
| Many-to-Many（同步） | 词性标注 | 每个输入对应一个输出 |
| Many-to-Many（异步） | 翻译、摘要 | 输入输出长度可不同 |

Seq2Seq 架构会在后面专门讲 Many-to-Many 的异步情况。

### 5. BPTT：沿时间反向传播

RNN 的训练使用 **BPTT（Backpropagation Through Time）**。本质很简单：

1. 把 RNN 沿时间展开成一条长链。
2. 正向读完整个序列。
3. 从损失出发，沿时间反向传播梯度。

问题也正来自这里：如果序列很长，梯度要穿过很多时间步，连乘效应会变得严重。

### 6. 为什么原始 RNN 记不住长距离依赖

假设一句话有 100 个 token，最后一个词的理解依赖第 1 个词。梯度从最后传回第 1 个时间步时，要经过 100 次左右的连续变换。

如果每一步都让梯度变小一点，最后就几乎没信号了。这就是**梯度消失**。

如果每一步都让梯度变大一点，最后就会爆炸。这就是**梯度爆炸**。

梯度爆炸可以用梯度裁剪缓解；梯度消失更棘手，因为早期信息根本传不回来。LSTM 和 GRU 就是为了解决这个问题而设计的。
