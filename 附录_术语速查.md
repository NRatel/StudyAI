# 附录：术语速查

## 使用说明

这份附录不是背诵清单，而是读正文时的“概念字典”。遇到陌生词，先看一句话定义，再回到对应章节继续读。

每个术语只回答三个问题：

- **是什么**：一句话定义。
- **别误会**：最常见的理解偏差。
- **主要出现于**：建议回看的模块。

## 一、基础范式

| 术语 | 是什么 | 别误会 | 主要出现于 |
|------|--------|--------|------------|
| AI | 让机器表现出感知、理解、生成、决策等智能行为的技术总称 | AI 不是某一种模型，而是一大类目标和方法 | 全部 |
| 机器学习 | 让模型从数据中学习规律，而不是由人手写所有规则 | “学习”不是像人一样理解世界，而是优化参数 | 01 |
| 深度学习 | 使用多层神经网络学习复杂表示的机器学习方法 | 深度不是越深越好，关键是结构、数据和训练稳定性 | 01/02/03 |
| 模型 | 输入到输出的可计算函数，通常包含大量可训练参数 | 模型不是知识库；它更像压缩过的模式和能力 | 全部 |
| 参数 | 模型内部可学习的数值，如权重和偏置 | 参数多通常能力强，但也更贵、更难训练 | 01/05/07 |
| 训练 | 用数据和损失函数不断调整参数的过程 | 训练不是把数据原样存进去 | 01/07 |
| 推理 | 训练完成后，模型根据输入生成预测或回答的过程 | 推理不等于逻辑推理；部署语境里常指 inference | 05/08/11 |
| 泛化 | 模型在没见过的新样本上仍能表现良好的能力 | 训练集效果好不代表泛化好 | 01 |
| 过拟合 | 模型把训练数据细节甚至噪声记得太死，导致新数据表现差 | 过拟合不是模型“太聪明”，而是泛化失败 | 01 |
| 自监督学习 | 从数据自身构造训练信号，例如遮住词让模型猜 | 自监督不是没有监督，而是监督信号不是人工标注 | 04 |

## 二、神经网络与训练

| 术语 | 是什么 | 别误会 | 主要出现于 |
|------|--------|--------|------------|
| 神经元 | 对输入做加权求和，再通过激活函数输出的计算单元 | 它只是数学抽象，不等同于真实生物神经元 | 01 |
| 权重 | 决定某个输入信号有多重要的参数 | 权重不是人工指定的规则，而是在训练中学出来的 | 01 |
| 偏置 | 在线性变换中加入的可学习常数项 | 偏置不是“误差”，而是让决策边界可平移 | 01 |
| 激活函数 | 给线性变换加入非线性的函数，如 ReLU、GELU | 没有非线性，多层网络会退化成单层线性模型 | 01 |
| 损失函数 | 衡量模型预测和真实目标差距的函数 | 损失不是准确率；它是优化器要下降的目标 | 01 |
| 梯度 | 损失对参数的变化率，告诉参数该往哪个方向调 | 梯度大不一定好，过大可能训练不稳定 | 01 |
| 反向传播 | 用链式法则从损失反向计算各参数梯度的方法 | 反向传播负责算梯度，不负责更新参数 | 01 |
| 优化器 | 根据梯度更新参数的方法，如 SGD、Adam、AdamW | Adam 不是“自动变好”的魔法，只是更稳定的更新规则 | 01/07 |
| 正则化 | 限制模型过度记忆训练数据的技术，如权重衰减、Dropout | 正则化不是提升训练集分数，而是提升泛化 | 01 |
| 归一化 | 调整中间数值分布以稳定训练的方法，如 BatchNorm、LayerNorm | 它不是简单把数值缩小，而是改善训练动态 | 01/03 |

## 三、Transformer 与语言模型

| 术语 | 是什么 | 别误会 | 主要出现于 |
|------|--------|--------|------------|
| Token | 模型处理文本的基本单位，可以是词、子词或字节片段 | token 不等于汉字或英文单词 | 04/07 |
| Embedding | 把离散 token 映射成连续向量的表示 | embedding 不是词典释义，而是模型可计算的语义坐标 | 04/09/10 |
| Hidden State | 模型中间层对当前位置的向量表示 | 它不是人类可直接读懂的“想法” | 03/05 |
| Logit | Softmax 之前的原始分数 | logit 还不是概率，经过 Softmax 后才变成概率分布 | 01/05 |
| Softmax | 把一组分数转换成概率分布的函数 | Softmax 输出高不代表一定真实，只代表模型更偏向这个选项 | 01/03/05 |
| Attention | 按相关性从其他位置加权读取信息的机制 | Attention 不是“注意力可解释一切”，它只是加权信息读取 | 03 |
| Q/K/V | Query 表示“我要找什么”，Key 表示“我如何被匹配”，Value 表示“被找到后提供什么” | Q/K/V 通常来自同一输入的不同线性投影，不是三份原始数据 | 03 |
| Self-Attention | 同一序列内部 token 彼此查看和加权读取 | 它本身不知道顺序，需要位置编码补充顺序信息 | 03 |
| Multi-Head Attention | 多组注意力头并行从不同子空间看关系 | 多头不是简单重复计算，而是学习不同关系视角 | 03 |
| 位置编码 | 给 Transformer 注入 token 位置信息的方法 | 位置编码不是装饰；没有它模型难以理解顺序 | 03/13 |
| Transformer | 以 Attention、FFN、残差连接、LayerNorm 等组成的序列建模架构 | Transformer 不只有 GPT，BERT、T5、ViT 也属于这条家族 | 03/04/05 |
| Encoder | 双向理解输入的 Transformer 组件或架构路线 | Encoder 擅长理解，不天然适合从左到右生成 | 03/04 |
| Decoder | 带因果掩码、适合自回归生成的 Transformer 组件或架构路线 | Decoder-only 不是简化版，而是大语言模型的主流生成路线 | 03/04/05 |
| 自回归 | 按顺序一个 token 一个 token 生成，每一步依赖前文 | 自回归不等于只能短文本，它只是生成方式 | 04/05/08 |
| 预训练 | 在大规模通用数据上先学习基础能力 | 预训练模型通常还需要对齐或任务适配才好用 | 04/05 |
| 微调 | 在较小的特定数据上继续训练模型以适配任务或风格 | 微调不一定注入新事实，很多知识更新更适合 RAG | 04/09 |
| 对齐 | 让模型输出更符合人类偏好、安全边界和任务要求 | 对齐不是让模型“更懂知识”，主要是调整行为 | 05/07 |
| 上下文窗口 | 模型一次能读取的 token 范围 | 窗口大不代表一定能可靠使用所有内容 | 08/09/13 |
| KV Cache | 推理时缓存历史 token 的 Key/Value，避免重复计算 | KV Cache 省计算但占显存，长对话会越来越贵 | 08/13 |

## 四、生成式 AI 与应用架构

| 术语 | 是什么 | 别误会 | 主要出现于 |
|------|--------|--------|------------|
| 生成模型 | 学习数据分布并生成新样本的模型 | 生成不是复制训练样本，而是从学到的分布中采样 | 06 |
| VAE | 用编码器把数据压到潜变量，再用解码器重建数据的生成模型 | VAE 生成通常更平滑，但细节可能偏模糊 | 06 |
| GAN | 生成器和判别器对抗训练的生成模型 | GAN 强在清晰样本，但训练容易不稳定 | 06 |
| 扩散模型 | 先加噪破坏数据，再学习反向去噪生成数据的模型 | 扩散不是物理扩散模拟，而是一种生成建模方法 | 06 |
| CFG | Classifier-Free Guidance，用条件和无条件预测差值增强条件控制 | CFG 太强会提高匹配度但损害多样性和自然度 | 06 |
| RAG | 生成前先检索外部知识，再把相关内容交给模型回答 | RAG 不会自动保证正确，检索和引用质量仍然关键 | 09 |
| 向量数据库 | 存储和检索向量表示的数据库，常用于语义搜索 | 它不是普通数据库的替代品，适合相似度检索 | 09 |
| Chunk | 文档切分后的文本块，是 RAG 检索的基本单位 | chunk 越大不一定越好，会带来噪声和成本 | 09 |
| Reranker | 对初步检索结果重新排序的模型或步骤 | Reranker 不增加知识，只改善上下文排序质量 | 09 |
| 幻觉 | 模型生成看似可信但缺乏事实依据或错误的内容 | 幻觉不是“模型故意撒谎”，多来自概率生成和知识缺口 | 05/09 |
| Prompt | 给模型的输入指令、上下文和示例组合 | Prompt 不是咒语，核心是清楚约束任务和信息来源 | 08/11/12 |
| Tool Use | 模型调用外部函数、API、搜索、代码执行等工具 | 工具调用强不等于模型本身知道更多，而是能接入外部能力 | 12 |
| Agent | 能根据目标自主规划、调用工具并根据结果迭代的系统 | Agent 不等于多轮聊天；关键是行动和反馈闭环 | 12 |
| 记忆 | Agent 或对话系统保存和利用历史信息的机制 | 记忆不等于把所有聊天记录永久塞进上下文 | 12 |
| MCP | Model Context Protocol，用于标准化模型和外部工具/上下文的连接 | MCP 是接入协议，不是让模型变聪明的新模型 | 12 |

## 五、前沿与系统术语

| 术语 | 是什么 | 别误会 | 主要出现于 |
|------|--------|--------|------------|
| 多模态 | 模型同时处理文本、图像、音频、视频等多种信息形态 | 多模态不是简单把多个模型拼起来，关键是跨模态对齐和融合 | 10 |
| CLIP | 用图文对比学习把图像和文本对齐到同一语义空间的模型 | CLIP 不是图像生成模型，它主要做理解和对齐 | 10 |
| ViT | 把图像切成 patch 后交给 Transformer 处理的视觉模型 | ViT 不是 CNN 的小改版，而是图像 token 化后的 Transformer 路线 | 10 |
| CoT | Chain-of-Thought，让模型显式或隐式进行多步推理 | CoT 不是万能提示词，简单任务或事实缺失时帮助有限 | 11 |
| Test-time Compute | 推理时投入更多计算来换取更好答案 | 想更久有收益递减，不能凭空创造模型不知道的知识 | 11 |
| 推理模型 | 经过专门训练，在复杂推理任务上愿意花更多计算的模型 | 推理模型底层通常仍是 Transformer，不一定是新架构 | 11 |
| MoE | Mixture of Experts，每个 token 只激活部分专家的稀疏架构 | MoE 总参数大不等于每次推理都用全部参数 | 13 |
| 长上下文 | 模型能读取更长输入序列的能力 | 长上下文不等于长文本理解一定可靠，仍有检索和注意力分布问题 | 09/13 |
| RoPE | 用旋转方式注入相对位置信息的位置编码方法 | RoPE 扩展能拉长上下文，但不自动解决所有长文本理解问题 | 03/13 |
| 量化 | 用更低精度表示模型权重或激活以降低显存和计算成本 | 量化不是无损压缩，低比特可能影响质量 | 08 |
| 推测解码 | 用小模型草拟多个 token，再由大模型验证来加速生成 | 它不改变最终分布，核心是“草稿 + 验证” | 08 |
| FlashAttention | 优化注意力计算的高效实现方式 | 它不改变 Attention 数学定义，只改变计算组织方式 | 03/13 |

## 快速阅读原则

如果一个术语暂时看不懂，不要立刻停下来查到最深。先判断它属于哪一类：

| 类型 | 阅读策略 |
|------|----------|
| 主线概念 | 需要回到对应章节读清楚，如 Attention、预训练、RAG |
| 数学工具 | 查 [数学基础速览](附录_数学基础速览.md)，只补当前需要的部分 |
| 工程实现 | 先知道它解决什么问题，如 vLLM、FSDP、MCP，不必背 API |
| 前沿模型名 | 看趋势和代表意义，不必背发布时间和排名 |
