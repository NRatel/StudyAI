# 正则化（Regularization）

## 直觉与概述

### 核心问题：过拟合

你训练了一个神经网络，训练集上准确率 99%，拿到新数据上一测——65%。这就是**过拟合（Overfitting）**：模型把训练数据中的噪声、巧合、个例都当成了"规律"来记忆，而没有学到真正的泛化模式。

**类比**：想象一个学生准备考试。

- **过拟合的学生**：把历年真题的答案全背下来了。题目稍微改个数字就不会做——他背的是答案，不是方法。
- **泛化良好的学生**：理解了解题原理，即使遇到从没见过的题目，也能举一反三。

正则化就是一系列技术手段，让模型从"背答案"转向"学原理"。具体做法各异，但核心思想只有一个：**限制模型的复杂度，让它不要"过度拟合"训练数据中的噪声**。

### 偏差-方差权衡（Bias-Variance Tradeoff）

在理解正则化之前，需要建立一个关键直觉：**偏差-方差权衡**。

任何模型的预测误差可以分解为三部分：

$$\text{总误差} = \text{偏差}^2 + \text{方差} + \text{不可约噪声}$$

- **偏差（Bias）**：模型过于简单，无法捕捉数据的真实模式。比如用一条直线去拟合一条抛物线——不管怎么调参数，直线就是无法贴合曲线。这叫**欠拟合（Underfitting）**。
- **方差（Variance）**：模型过于复杂，对训练数据的微小变化极度敏感。换一批训练数据，模型的预测就完全不同。这叫**过拟合（Overfitting）**。
- **不可约噪声**：数据本身的随机性，任何模型都无法消除。

这三者之间存在矛盾：

| 模型复杂度 | 偏差 | 方差 | 训练误差 | 测试误差 |
|-----------|------|------|---------|---------|
| 过低（欠拟合） | 高 | 低 | 高 | 高 |
| 适中（刚好） | 中 | 中 | 中 | **最低** |
| 过高（过拟合） | 低 | 高 | 很低 | 高 |

**正则化的本质**：在模型已经足够复杂（低偏差）的前提下，通过各种手段降低方差，找到偏差和方差的最佳平衡点。

用 Python 工程师熟悉的语言来说：正则化就是给模型加"约束条件"——就像代码审查中要求"不准写超过 200 行的函数"，不是因为长函数一定不对，而是经验表明短函数通常质量更高。

### 正则化技术全景

本章将逐一讲解以下正则化技术：

| 技术 | 作用方式 | 核心直觉 |
|------|---------|---------|
| L2 正则化（权重衰减） | 惩罚大权重 | 让模型参数保持小值，等于限制模型复杂度 |
| L1 正则化 | 惩罚权重的绝对值 | 让部分权重变成 0，实现特征选择 |
| Dropout | 随机关闭神经元 | 强迫网络不依赖单个神经元，等于模型集成 |
| Batch Normalization | 标准化每层激活值 | 稳定训练、加速收敛，附带正则化效果 |
| Layer Normalization | 沿特征维度标准化 | 适用于变长序列，Transformer 的标配 |
| 早停（Early Stopping） | 监控验证集，适时停止 | 隐式限制模型复杂度 |
| 数据增强 | 增加训练数据多样性 | 让模型见更多"变体"，减少对特定样本的记忆 |

## 严谨定义与原理

### 一、L1/L2 正则化（权重衰减）

#### L2 正则化

**思想**：在原始损失函数上加一个惩罚项，惩罚权重的平方和。权重越大，惩罚越重。

$$\mathcal{L}_{\text{reg}} = \mathcal{L}_{\text{original}} + \frac{\lambda}{2} \sum_{i} w_i^2 = \mathcal{L}_{\text{original}} + \frac{\lambda}{2} \|\mathbf{w}\|_2^2$$

其中：
- $\mathcal{L}_{\text{original}}$ 是原始损失（如交叉熵）
- $\lambda$ 是正则化强度（超参数），控制惩罚力度
- $\|\mathbf{w}\|_2^2 = \sum_i w_i^2$ 是权重的 L2 范数的平方
- $\frac{1}{2}$ 是为了求导后消掉系数，纯粹为了数学简洁

**对梯度的影响**：加上正则化项后，权重更新规则变为：

$$\frac{\partial \mathcal{L}_{\text{reg}}}{\partial w_i} = \frac{\partial \mathcal{L}_{\text{original}}}{\partial w_i} + \lambda w_i$$

$$w_i \leftarrow w_i - \eta \left( \frac{\partial \mathcal{L}_{\text{original}}}{\partial w_i} + \lambda w_i \right)$$

$$w_i \leftarrow (1 - \eta\lambda) w_i - \eta \frac{\partial \mathcal{L}_{\text{original}}}{\partial w_i}$$

注意 $(1 - \eta\lambda)$ 这个因子：每次更新时，权重先乘以一个略小于 1 的数，相当于每一步都把权重"衰减"一点点。这就是 **权重衰减（Weight Decay）** 名称的由来。

**为什么小权重 = 更简单的模型？**

直觉上：
- 大权重意味着模型对某些特征的反应非常剧烈——输入稍有变化，输出就剧烈波动，这正是过拟合的特征
- 小权重意味着模型对输入的响应比较平滑——决策边界更"圆润"，不会出现奇怪的锯齿

数学上：考虑一个简单函数 $f(x) = w_1 x_1 + w_2 x_2$。如果 $w_1 = 100$，那么 $x_1$ 的微小噪声会被放大 100 倍。如果限制 $w_1$ 不超过 1，模型就不得不寻找更稳健的模式。

#### L1 正则化

$$\mathcal{L}_{\text{reg}} = \mathcal{L}_{\text{original}} + \lambda \sum_{i} |w_i| = \mathcal{L}_{\text{original}} + \lambda \|\mathbf{w}\|_1$$

**L1 vs L2 的关键区别——稀疏性**：

L1 正则化倾向于产生**稀疏解**（许多权重恰好为 0），而 L2 倾向于让所有权重都变小但不为零。

为什么？看梯度：

- L2 的梯度：$\frac{\partial}{\partial w_i} \frac{\lambda}{2} w_i^2 = \lambda w_i$。当 $w_i$ 已经很小时，梯度也很小，推力不足以把它推到 0
- L1 的梯度：$\frac{\partial}{\partial w_i} \lambda |w_i| = \lambda \cdot \text{sign}(w_i)$。不论 $w_i$ 多小，梯度的绝对值始终是 $\lambda$，有恒定的力量把它推向 0

**几何直觉**（著名的菱形 vs 圆形图）：

想象二维情况下的等高线图。原始损失的等高线是一组椭圆，正则化项的等高线是：
- L2：一组同心圆（$w_1^2 + w_2^2 = c$）
- L1：一组同心菱形（$|w_1| + |w_2| = c$）

最优解在两组等高线的交点。菱形的角在坐标轴上，椭圆更容易与角相切——这意味着 L1 的最优解更可能出现在坐标轴上（即某些 $w_i = 0$）。

| 对比 | L1 正则化 | L2 正则化 |
|------|----------|----------|
| 惩罚项 | $\lambda \|\mathbf{w}\|_1$ | $\frac{\lambda}{2} \|\mathbf{w}\|_2^2$ |
| 梯度 | $\lambda \cdot \text{sign}(w)$（恒定） | $\lambda w$（与权重成正比） |
| 权重趋势 | 稀疏（很多为 0） | 均匀缩小（都接近 0 但不为 0） |
| 适用场景 | 特征选择、模型压缩 | 防过拟合的通用手段 |
| 深度学习中 | 较少使用 | 非常常用（以权重衰减形式） |

#### 权重衰减 vs L2 正则化：在 Adam 中不等价

在 SGD 中，L2 正则化和权重衰减在数学上完全等价。但在 **Adam** 等自适应学习率优化器中，两者**不等价**。

**原因**：Adam 会根据梯度的历史信息调整每个参数的学习率。L2 正则化的梯度（$\lambda w_i$）也会被 Adam 的自适应机制处理——那些历史梯度大的参数，正则化的效果反而被削弱了。

这就是 **AdamW**（Decoupled Weight Decay）被提出的原因。AdamW 把权重衰减从梯度计算中分离出来，直接在参数更新步骤中执行衰减：

$$\theta_{t+1} = (1 - \eta \lambda) \theta_t - \eta \cdot \frac{\hat{m}_t}{\sqrt{\hat{v}_t} + \epsilon}$$

而标准 Adam + L2 正则化是：

$$g_t' = g_t + \lambda \theta_t \quad \text{(将 L2 梯度混入原始梯度)}$$
$$\theta_{t+1} = \theta_t - \eta \cdot \frac{\hat{m}_t'}{\sqrt{\hat{v}_t'} + \epsilon}$$

注意：在第二种写法中，$\lambda \theta_t$ 这个正则化梯度经过了 Adam 的二阶矩缩放，导致实际衰减效果和超参数之间的关系变得难以预测。

**工程结论**：现代深度学习几乎全部使用 **AdamW**，而不是 Adam + L2 正则化。在 PyTorch 中，`torch.optim.AdamW` 默认 `weight_decay=0.01`。

### 二、Dropout

#### 原理

Dropout（Srivastava et al., 2014）是深度学习中最经典的正则化技术之一。

**训练阶段**：在每次前向传播时，以概率 $p$ 随机将部分神经元的输出置为 0（"关掉"它们）。每次前向传播都会随机选择不同的神经元关掉。

$$\mathbf{r} \sim \text{Bernoulli}(1-p)$$
$$\tilde{\mathbf{h}} = \mathbf{r} \odot \mathbf{h}$$

其中：
- $\mathbf{h}$ 是某一层的激活值
- $\mathbf{r}$ 是一个掩码向量，每个元素独立地以概率 $(1-p)$ 为 1，概率 $p$ 为 0
- $\odot$ 是逐元素乘法
- $p$ 是 dropout 概率（通常 0.1~0.5）

**直觉：为什么有效？**

1. **强迫冗余表示**：任何一个神经元都可能被关掉，所以网络不能依赖任何单个神经元。它必须学习冗余的、分布式的表征——即使丢掉一部分信息，剩下的部分也能做出合理判断。

2. **近似模型集成**：每次前向传播，被关掉的神经元不同，相当于在指数级数量的"子网络"中随机采样一个来训练。一个有 $n$ 个神经元的网络，理论上有 $2^n$ 种不同的子网络。最终的模型相当于所有这些子网络的集成（ensemble）——而集成学习是减少方差的经典方法。

3. **贝叶斯视角**（进阶）：Dropout 可以被解释为近似贝叶斯推断。贝叶斯方法不学习一个固定的权重值，而是学习权重的概率分布。Dropout 训练出的模型在推理时进行的"平均"操作，近似于对权重后验分布的蒙特卡洛采样。Gal & Ghahramani (2016) 在论文 *"Dropout as a Bayesian Approximation"* 中严格证明了这一点。

#### 测试时的缩放问题

训练时关掉了一部分神经元，测试时全部打开——但这会导致一个问题：

训练时，某一层的期望输出为 $(1-p) \cdot h$（因为有 $p$ 比例的输出被置零）。测试时，所有神经元都在，输出为 $h$。期望值不匹配。

**解决方案一：测试时缩放**

测试时将输出乘以 $(1-p)$：

$$h_{\text{test}} = (1-p) \cdot h$$

**解决方案二：Inverted Dropout（反转 Dropout）**——实践中更常用

训练时将未被关掉的神经元输出除以 $(1-p)$：

$$\tilde{h}_{\text{train}} = \frac{\mathbf{r} \odot \mathbf{h}}{1-p}$$

这样训练时的期望输出就是 $h$，测试时不需要做任何缩放。PyTorch 的 `nn.Dropout` 默认使用 Inverted Dropout。

#### Dropout 的常见配置

| 场景 | 典型 Dropout 率 $p$ |
|------|-------------------|
| 全连接层 | 0.5 |
| 卷积层 | 0.1~0.3（或不用，因为参数共享本身就是正则化） |
| Transformer 注意力/FFN | 0.1 |
| 小数据集 | 偏大（0.3~0.5） |
| 大数据集 | 偏小（0.0~0.1）或不用 |

### 三、Batch Normalization（批归一化）

#### 原理

Batch Normalization（Ioffe & Szegedy, 2015）对每个 mini-batch 内的激活值进行标准化处理。

**步骤**：

给定一个 mini-batch $\mathcal{B} = \{x_1, x_2, \ldots, x_m\}$（这里 $x_i$ 是某一层某个特征维度上的值）：

1. **计算 batch 均值**：

$$\mu_\mathcal{B} = \frac{1}{m} \sum_{i=1}^{m} x_i$$

2. **计算 batch 方差**：

$$\sigma_\mathcal{B}^2 = \frac{1}{m} \sum_{i=1}^{m} (x_i - \mu_\mathcal{B})^2$$

3. **标准化**：

$$\hat{x}_i = \frac{x_i - \mu_\mathcal{B}}{\sqrt{\sigma_\mathcal{B}^2 + \epsilon}}$$

其中 $\epsilon$ 是一个很小的数（如 $10^{-5}$），防止除以零。

4. **缩放和偏移**（可学习参数）：

$$y_i = \gamma \hat{x}_i + \beta$$

$\gamma$ 和 $\beta$ 是可学习的参数，允许网络在需要时"恢复"标准化之前的表达能力。如果网络发现标准化对某个特征不利，它可以学到 $\gamma = \sigma_\mathcal{B}$, $\beta = \mu_\mathcal{B}$，等效于没有做 BN。

#### BN 解决了什么问题？

**原始解释——内部协变量偏移（Internal Covariate Shift）**：

Ioffe & Szegedy 在原论文中认为：训练过程中，前一层参数的更新会改变后一层输入的分布，导致每一层都在不断适应"漂移"的输入分布，减慢了训练。BN 通过标准化消除这种漂移。

**争议**：

Santurkar et al. (2018) 在 *"How Does Batch Normalization Help Optimization?"* 中提出了不同看法：BN 的好处主要不是消除内部协变量偏移，而是让损失函数的**景观更平滑**（使损失函数的梯度变化更稳定），从而允许使用更大的学习率。

**实际好处**（无论理论解释如何，以下好处在实践中被广泛验证）：

1. **允许更大的学习率**：没有 BN 时，大学习率容易导致梯度爆炸；有了 BN，激活值被控制在合理范围内
2. **加速收敛**：通常能减少 50%~70% 的训练迭代次数
3. **减少对初始化的敏感度**：即使初始化不太好，BN 也能把激活值"拉回"合理范围
4. **轻微的正则化效果**：因为每个 mini-batch 的均值和方差是噪声估计（取决于 batch 中的样本），这为训练引入了噪声，类似于 Dropout 的正则化效果

#### BN 的推理阶段

训练时使用 mini-batch 的统计量。推理时不能依赖 batch（可能一次只推理一个样本），所以使用训练过程中累积的**全局统计量**——运行均值和运行方差（通过指数移动平均维护）：

$$\mu_{\text{running}} \leftarrow (1-\alpha) \cdot \mu_{\text{running}} + \alpha \cdot \mu_\mathcal{B}$$
$$\sigma^2_{\text{running}} \leftarrow (1-\alpha) \cdot \sigma^2_{\text{running}} + \alpha \cdot \sigma^2_\mathcal{B}$$

其中 $\alpha$ 是动量参数（PyTorch 默认 0.1）。

推理时使用：

$$\hat{x} = \frac{x - \mu_{\text{running}}}{\sqrt{\sigma^2_{\text{running}} + \epsilon}}$$

**关键工程点**：这就是为什么 PyTorch 中训练和推理时必须切换 `model.train()` 和 `model.eval()`——BN 层在两种模式下的行为不同。

### 四、Layer Normalization（层归一化）

#### 原理

Layer Normalization（Ba et al., 2016）与 BN 的核心区别在于**归一化的维度不同**。

- **Batch Norm**：沿 **batch 维度** 归一化——对同一个特征，计算整个 batch 中该特征的均值和方差
- **Layer Norm**：沿 **特征维度** 归一化——对同一个样本，计算该样本所有特征的均值和方差

数学上，假设一个样本的某一层有 $H$ 个特征值 $\{h_1, h_2, \ldots, h_H\}$：

$$\mu = \frac{1}{H}\sum_{i=1}^{H} h_i \qquad \sigma^2 = \frac{1}{H}\sum_{i=1}^{H}(h_i - \mu)^2$$

$$\hat{h}_i = \frac{h_i - \mu}{\sqrt{\sigma^2 + \epsilon}} \qquad y_i = \gamma_i \hat{h}_i + \beta_i$$

#### BN vs LN 对比

| 对比 | Batch Norm | Layer Norm |
|------|-----------|------------|
| 归一化维度 | batch 维度 | 特征维度 |
| 训练/推理一致性 | 不一致（需要运行统计量） | 一致（不依赖 batch） |
| batch size 敏感性 | 小 batch 不稳定 | 不受 batch size 影响 |
| 序列任务 | 不适合变长序列 | 适合变长序列 |
| 主要用于 | CNN（图像） | Transformer（NLP、多模态） |

#### 为什么 Transformer 用 LN 而不是 BN？

1. **序列长度不固定**：NLP 任务中，每个 batch 内的序列长度不同。BN 需要对同一位置的特征跨 batch 计算统计量，但不同序列的同一位置含义不同（第 5 个 token 在不同句子中意义完全不同）

2. **batch 独立性**：LN 对每个样本独立计算，不依赖 batch 中的其他样本。这意味着：
   - 单样本推理时行为一致
   - 不需要维护运行统计量
   - 在分布式训练中更简单（不需要跨 GPU 同步统计量）

3. **小 batch 稳定性**：大模型训练时由于显存限制，batch size 可能很小。BN 在小 batch（如 batch_size=1~4）下统计量估计不准，性能严重下降

### 五、早停（Early Stopping）

#### 原理

早停是最简单也最实用的正则化技术之一。

**做法**：
1. 将数据划分为训练集和验证集
2. 训练过程中，每隔一定 epoch，在验证集上计算 loss
3. 当验证集 loss 开始持续上升（训练集 loss 仍在下降）时，停止训练
4. 使用验证集 loss 最低时的模型参数

**为什么有效？**

训练的早期，模型在学习数据中的真实模式——训练 loss 和验证 loss 同时下降。训练的后期，模型开始记忆训练数据中的噪声——训练 loss 继续下降，但验证 loss 开始上升。早停就是在"学习信号"和"记忆噪声"的交界点停下来。

**与 L2 正则化的关系**：

Bishop (1995) 证明了，在线性模型中，早停和 L2 正则化是等价的。训练迭代次数 $T$ 与正则化系数 $\lambda$ 的关系大约为 $T \approx \frac{1}{\eta\lambda}$（$\eta$ 是学习率）。直觉上：参数从零开始，如果早早停下来，参数还没来得及变大——这和 L2 正则化限制参数大小的效果类似。

#### Patience（耐心值）

验证 loss 可能出现短暂波动（上升一两个 epoch 后又下降）。为了避免过早停止，通常设置一个 **patience** 参数——只有验证 loss 连续 patience 个 epoch 都没有改善时，才真正停止。

### 六、数据增强（Data Augmentation）

数据增强不是对模型本身动手脚，而是通过增加训练数据的多样性来间接防止过拟合。

**核心思想**：对已有训练样本施加合理的变换，生成"新"的训练样本，但不改变其标签。

**常见的增强手段**：

| 领域 | 增强方法 |
|------|---------|
| 图像 | 随机裁剪、水平翻转、旋转、色彩抖动、Cutout、Mixup、CutMix |
| 文本 | 同义词替换、回译（翻成另一种语言再翻回来）、随机删词 |
| 音频 | 时间拉伸、添加噪声、变调 |

**为什么有效？** 让模型见过更多"变体"后，它就不得不学习那些在所有变体中都保持不变的特征——这些才是真正有区分度的特征，而非特定样本的噪声。

```python
# ================================================================
# 从零实现 L2 正则化
# 任务：二分类（XOR 变体），展示 L2 正则化如何约束权重
# ================================================================

# 生成一个简单的非线性数据集
def make_data(n=200):
    """生成围绕原点的非线性可分数据"""
    X = np.random.randn(n, 2)
    # 标签：x1*x2 > 0 为类别1，否则为类别0（XOR 模式）
    y = ((X[:, 0] * X[:, 1]) > 0).astype(np.float64).reshape(-1, 1)
    return X, y

X_train, y_train = make_data(200)
X_val, y_val = make_data(50)

# Sigmoid 和它的导数
def sigmoid(z):
    return 1.0 / (1.0 + np.exp(-np.clip(z, -500, 500)))

# 简单的两层网络：2 -> 16 -> 1
class SimpleNet:
    def __init__(self, hidden_size=16):
        # He 初始化
        self.W1 = np.random.randn(2, hidden_size) * np.sqrt(2.0 / 2)
        self.b1 = np.zeros((1, hidden_size))
        self.W2 = np.random.randn(hidden_size, 1) * np.sqrt(2.0 / hidden_size)
        self.b2 = np.zeros((1, 1))

    def forward(self, X):
        self.z1 = X @ self.W1 + self.b1
        self.a1 = np.maximum(0, self.z1)  # ReLU
        self.z2 = self.a1 @ self.W2 + self.b2
        self.a2 = sigmoid(self.z2)
        return self.a2

    def compute_loss(self, X, y, lam=0.0):
        """
        二分类交叉熵 + L2 正则化

        lam: L2 正则化系数 (lambda)
             lam=0 表示不使用正则化
        """
        m = X.shape[0]
        pred = self.forward(X)
        # 交叉熵损失
        eps = 1e-8
        ce_loss = -np.mean(y * np.log(pred + eps) + (1 - y) * np.log(1 - pred + eps))

        # L2 正则化项：(lambda/2) * (||W1||^2 + ||W2||^2)
        l2_penalty = (lam / 2) * (np.sum(self.W1 ** 2) + np.sum(self.W2 ** 2))

        return ce_loss + l2_penalty

    def backward(self, X, y, lam=0.0):
        """反向传播，返回梯度"""
        m = X.shape[0]
        pred = self.a2

        # 输出层梯度
        dz2 = (pred - y) / m
        dW2 = self.a1.T @ dz2 + lam * self.W2  # ← L2 正则化梯度
        db2 = np.sum(dz2, axis=0, keepdims=True)

        # 隐藏层梯度
        da1 = dz2 @ self.W2.T
        dz1 = da1 * (self.z1 > 0).astype(float)  # ReLU 导数
        dW1 = X.T @ dz1 + lam * self.W1           # ← L2 正则化梯度
        db1 = np.sum(dz1, axis=0, keepdims=True)

        return dW1, db1, dW2, db2

    def update(self, grads, lr=0.01):
        dW1, db1, dW2, db2 = grads
        self.W1 -= lr * dW1
        self.b1 -= lr * db1
        self.W2 -= lr * dW2
        self.b2 -= lr * db2


# 对比：无正则化 vs 有 L2 正则化
print("=" * 60)
print("对比实验：L2 正则化的效果")
print("=" * 60)

for lam, label in [(0.0, "无正则化"), (0.1, "L2 正则化 λ=0.1")]:
    net = SimpleNet(hidden_size=16)
    lr = 0.1

    for epoch in range(1001):
        loss = net.compute_loss(X_train, y_train, lam=lam)
        grads = net.backward(X_train, y_train, lam=lam)
        net.update(grads, lr=lr)

        if epoch % 250 == 0:
            val_loss = net.compute_loss(X_val, y_val, lam=0.0)  # 验证集不加正则项
            print(f"[{label}] Epoch {epoch:4d} | "
                  f"训练 Loss: {loss:.4f} | 验证 Loss: {val_loss:.4f}")

    # 查看权重大小
    w_norm = np.sqrt(np.sum(net.W1**2) + np.sum(net.W2**2))
    print(f"[{label}] 权重 L2 范数: {w_norm:.4f}")
    print()
```

### 2. 用 NumPy 从零实现 Dropout

```python
import numpy as np

np.random.seed(42)

# ================================================================
# 从零实现 Inverted Dropout
# ================================================================

def dropout_forward(h, p=0.5, training=True):
    """
    Inverted Dropout 前向传播

    参数:
        h: 激活值，shape (batch_size, features)
        p: dropout 概率（置零的概率）
        training: 是否处于训练模式

    返回:
        h_out: dropout 后的激活值
        mask: dropout 掩码（反向传播需要）
    """
    if not training or p == 0:
        return h, None

    # 生成掩码：每个神经元以概率 (1-p) 保留
    mask = (np.random.rand(*h.shape) > p).astype(np.float64)

    # Inverted Dropout：保留的神经元除以 (1-p)
    # 这样训练时期望值 = h，推理时不需要额外缩放
    h_out = (h * mask) / (1 - p)

    return h_out, mask


def dropout_backward(dout, mask, p=0.5):
    """
    Inverted Dropout 反向传播
    梯度只传给未被 mask 掉的神经元
    """
    if mask is None:
        return dout
    return (dout * mask) / (1 - p)


# ---- 演示 ----
batch_size, features = 4, 8
h = np.random.randn(batch_size, features)

print("原始激活值:")
print(h.round(3))
print(f"均值: {h.mean():.4f}")
print()

# 训练模式：一些神经元被置零，其余被放大
h_dropped, mask = dropout_forward(h, p=0.5, training=True)
print("Dropout 后 (p=0.5, 训练模式):")
print(h_dropped.round(3))
print(f"掩码 (1=保留, 0=丢弃):")
print(mask.astype(int))
print(f"均值: {h_dropped.mean():.4f} (与原始均值接近，因为 inverted scaling)")
print()

h_test, _ = dropout_forward(h, p=0.5, training=False)
print("推理模式：输出与原始完全相同")
print(f"差异: {np.abs(h_test - h).max():.10f}")
```

```python
class TransformerBlock(nn.Module):
    """
    简化的 Transformer 块，展示 LayerNorm 的典型用法
    使用 Pre-LN 结构（GPT-2/3 之后的主流做法）
    """
    def __init__(self, d_model=64, n_heads=4, dropout=0.1):
        super().__init__()
        # 注意力子层
        self.ln1 = nn.LayerNorm(d_model)        # ← Pre-LN
        self.attn = nn.MultiheadAttention(d_model, n_heads, dropout=dropout, batch_first=True)
        self.drop1 = nn.Dropout(dropout)

        # FFN 子层
        self.ln2 = nn.LayerNorm(d_model)        # ← Pre-LN
        self.ffn = nn.Sequential(
            nn.Linear(d_model, 4 * d_model),
            nn.GELU(),
            nn.Linear(4 * d_model, d_model),
            nn.Dropout(dropout)
        )

    def forward(self, x):
        # Pre-LN: 先归一化再做注意力，残差连接绕过归一化
        normed = self.ln1(x)
        attn_out, _ = self.attn(normed, normed, normed)
        x = x + self.drop1(attn_out)            # 残差连接

        normed = self.ln2(x)
        ffn_out = self.ffn(normed)
        x = x + ffn_out                         # 残差连接
        return x
```

### 5. 早停的完整实现

```python
import torch
import torch.nn as nn
import torch.optim as optim
import copy

# ================================================================
# 早停（Early Stopping）的完整实现
# ================================================================

class EarlyStopping:
    """
    早停工具类

    用法:
        early_stop = EarlyStopping(patience=10, min_delta=1e-4)
        for epoch in range(max_epochs):
            train(...)
            val_loss = validate(...)
            if early_stop(val_loss, model):
                print("早停触发")
                model = early_stop.best_model  # 恢复最佳模型
                break
    """

    def __init__(self, patience=10, min_delta=1e-4, verbose=True):
        """
        参数:
            patience: 验证 loss 不再改善后等待多少个 epoch
            min_delta: 认为 loss 有改善的最小下降量
            verbose: 是否打印信息
        """
        self.patience = patience
        self.min_delta = min_delta
        self.verbose = verbose
        self.counter = 0
        self.best_loss = float('inf')
        self.best_model = None
        self.best_epoch = 0

    def __call__(self, val_loss, model):
        """
        返回 True 表示应该停止训练
        """
        if val_loss < self.best_loss - self.min_delta:
            # 验证 loss 有显著改善
            self.best_loss = val_loss
            self.best_model = copy.deepcopy(model.state_dict())
            self.best_epoch = 0  # 这里借用于记录信息
            self.counter = 0
            if self.verbose:
                print(f"  ✓ 验证 Loss 改善至 {val_loss:.4f}，保存模型")
        else:
            # 没有改善
            self.counter += 1
            if self.verbose and self.counter % 5 == 0:
                print(f"  ✗ 已 {self.counter}/{self.patience} 个 epoch 无改善")

        return self.counter >= self.patience
```

### 6. 综合实验：多种正则化技术的组合

```python
import torch
import torch.nn as nn
import torch.optim as optim

torch.manual_seed(42)

# ================================================================
# 综合对比：不同正则化组合的效果
# ================================================================

def make_data_torch(n, noise=0.3):

    w_true = torch.zeros(20)
    w_true[:5] = torch.tensor([2.0, -1.5, 1.0, -0.5, 0.8])

class FlexibleModel(nn.Module):
    """支持多种正则化配置的模型"""

    def __init__(self, use_dropout=False, use_batchnorm=False, use_layernorm=False, p=0.5):
        super().__init__()

        layers = []

        # 第一层
        layers.append(nn.Linear(20, 128))
        if use_batchnorm:
            layers.append(nn.BatchNorm1d(128))
        if use_layernorm:
            layers.append(nn.LayerNorm(128))
        layers.append(nn.ReLU())
        if use_dropout:

        # 第二层

        if use_batchnorm:
            layers.append(nn.BatchNorm1d(64))
        if use_layernorm:
            layers.append(nn.LayerNorm(64))
        layers.append(nn.ReLU())
        if use_dropout:
            layers.append(nn.Dropout(p))

        # 输出层
        layers.append(nn.Linear(64, 1))
        layers.append(nn.Sigmoid())

        self.net = nn.Sequential(*layers)

    def forward(self, x):
        return self.net(x)


def train_model(model, name, weight_decay=0.0, epochs=300):
    """训练模型并返回结果"""
    criterion = nn.BCELoss()
    # 使用 AdamW，weight_decay 参数直接控制权重衰减
    optimizer = optim.AdamW(model.parameters(), lr=0.001, weight_decay=weight_decay)

    best_val_loss = float('inf')
    final_train_loss = 0
    final_val_loss = 0

    for epoch in range(epochs):
        model.train()
        pred = model(X_train)
        loss = criterion(pred, y_train)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

        model.eval()
        with torch.no_grad():
            val_pred = model(X_val)
            val_loss = criterion(val_pred, y_val).item()

        final_train_loss = loss.item()
        final_val_loss = val_loss
        best_val_loss = min(best_val_loss, val_loss)

    model.eval()
    with torch.no_grad():
        train_acc = ((model(X_train) > 0.5).float() == y_train).float().mean().item()
        val_acc = ((model(X_val) > 0.5).float() == y_val).float().mean().item()

    gap = final_val_loss - final_train_loss
    print(f"{name:40s} | 训练Loss={final_train_loss:.4f} | 验证Loss={final_val_loss:.4f} | "
          f"Gap={gap:+.4f} | 训练Acc={train_acc:.3f} | 验证Acc={val_acc:.3f}")


print("=" * 110)
print("综合对比：不同正则化组合")
print(f"训练集: {len(X_train)} 样本 | 验证集: {len(X_val)} 样本")
print("=" * 110)

configs = [
    ("无正则化 (baseline)",
     FlexibleModel(), 0.0),
    ("仅 Dropout (p=0.5)",
     FlexibleModel(use_dropout=True, p=0.5), 0.0),
    ("仅权重衰减 (AdamW, wd=0.01)",
     FlexibleModel(), 0.01),
    ("仅 BatchNorm",
     FlexibleModel(use_batchnorm=True), 0.0),
    ("仅 LayerNorm",
     FlexibleModel(use_layernorm=True), 0.0),
    ("Dropout + 权重衰减",
     FlexibleModel(use_dropout=True, p=0.3), 0.01),
    ("LayerNorm + Dropout + 权重衰减 (现代标配)",
     FlexibleModel(use_layernorm=True, use_dropout=True, p=0.3), 0.01),
]

for name, model, wd in configs:
    train_model(model, name, weight_decay=wd)
```

## 工程师视角

### 1. 现代深度学习的正则化标配

在 2024-2026 年的主流实践中，最常用的正则化组合是：

```
AdamW（内置权重衰减） + Dropout(0.1) + LayerNorm
```

具体来说：

| 组件 | 现代实践 | 说明 |
|------|---------|------|
| 优化器 | **AdamW**（而不是 Adam） | weight_decay=0.01~0.1 |
| 归一化 | **LayerNorm**（而不是 BatchNorm） | Transformer 架构的标配 |
| Dropout | **0.0~0.1**（比以前小） | 大数据集 + 大模型时 dropout 率很低 |
| 早停 | 几乎总是使用 | 监控验证 loss，保存最佳 checkpoint |
| 数据增强 | 视觉任务必备 | NLP 中较少使用传统增强 |
| L1 正则化 | 深度学习中很少用 | 更多用于传统 ML 的特征选择 |

### 2. `model.train()` vs `model.eval()` ：一个经典 bug

```python
# !!!! 这是新手最常犯的错误之一 !!!!

# 训练时
model.train()       # Dropout 开启，BatchNorm 用 batch 统计量
loss = criterion(model(X_train), y_train)
loss.backward()
optimizer.step()

# 推理/验证时
model.eval()        # Dropout 关闭，BatchNorm 用全局统计量
with torch.no_grad():
    pred = model(X_test)

# 忘记 model.eval() 的后果:
# 1. Dropout 仍在随机丢弃神经元 → 每次推理结果不同！
# 2. BatchNorm 用当前 batch 的统计量 → 单样本推理结果不稳定
# 3. 验证/测试指标偏低且波动大，你可能误以为模型没训好
```

**经验法则**：在任何 `with torch.no_grad()` 之前，先检查是否调用了 `model.eval()`。训练循环开始前确保 `model.train()`。

### 3. BatchNorm 在小 batch 时不稳定

```python
# 为什么 Transformer 用 LayerNorm 的工程原因：

# 场景1：大模型训练，GPU 显存紧张
# 实际 batch_size 可能只有 1~4
# BatchNorm 需要从 batch 中估计均值和方差
# batch_size=2 时，你用 2 个样本去估计整个数据集的统计量 → 非常不准

# 场景2：序列生成（自回归）
# 推理时一次只处理一个样本 (batch_size=1)
# BatchNorm 要么用训练时积累的统计量（可能不匹配），
# 要么在 batch_size=1 上计算（方差=0，完全崩溃）

# 场景3：梯度累积
# 为了模拟大 batch，常用梯度累积
# 但 BatchNorm 看到的仍是小 mini-batch → 统计量不准

# LayerNorm 完全不依赖 batch，所以没有以上任何问题
```

### 4. 过拟合的诊断清单

当你发现训练 loss 持续下降但验证 loss 开始上升时，按以下顺序排查和处理：

```
第一步：确认是过拟合，而不是 bug
├── 检查 model.eval() 是否正确调用
├── 检查验证集有没有数据泄露（与训练集重叠）
└── 检查验证集的 DataLoader 有没有错误的增强

第二步：增加数据（最有效的方法）
├── 收集更多数据
├── 数据增强
└── 检查数据质量（标签噪声）

第三步：增加正则化
├── 增大 weight_decay（0.01 → 0.1）
├── 增大 dropout（0.1 → 0.3）
├── 使用早停
└── 减小模型大小（最后手段）

第四步：调整学习率
├── 使用学习率调度（如 cosine annealing）
└── 降低初始学习率
```

### 5. 常见坑与最佳实践

**Dropout 的位置**：Dropout 应该放在激活函数之后、下一个线性层之前。不要放在输出层。

```python
# 正确
nn.Linear(128, 64),
nn.ReLU(),
nn.Dropout(0.3),   # ← 放在 ReLU 之后
nn.Linear(64, 32),

# 不要在输出层加 Dropout
nn.Linear(64, num_classes)   # ← 输出层，不加 Dropout
```

**BatchNorm 与 Dropout 的冲突**：研究表明（Li et al., 2019），同时使用 BN 和 Dropout 可能互相干扰。BN 期望稳定的统计量，而 Dropout 引入的随机性会扰乱这种稳定性。在实践中：
- CNN 中：用 BN，少用或不用 Dropout
- Transformer 中：用 LN + Dropout，它们配合良好

**weight_decay 不要施加在所有参数上**：通常偏置（bias）和 LayerNorm 的参数不需要权重衰减。

```python
# PyTorch 中排除 bias 和 LN 参数的权重衰减
param_groups = [
    {
        "params": [p for n, p in model.named_parameters()
                   if "bias" not in n and "norm" not in n and "ln" not in n],
        "weight_decay": 0.01
    },
    {
        "params": [p for n, p in model.named_parameters()
                   if "bias" in n or "norm" in n or "ln" in n],
        "weight_decay": 0.0       # ← bias 和归一化层不加权重衰减
    }
]
optimizer = optim.AdamW(param_groups, lr=1e-4)
```

**Dropout 在大型预训练模型中的趋势**：GPT-3、LLaMA 等大模型的 dropout 率通常设为 0.0~0.1。当数据足够多、模型足够大时，过拟合不再是主要问题，反而是欠拟合（模型还没学够）更常见。但在微调（fine-tuning）阶段，由于数据变少了，Dropout 又变得重要起来。

## 附录：关键公式速查

| 技术 | 核心公式 |
|------|---------|
| L2 正则化 | $\mathcal{L} = \mathcal{L}_0 + \frac{\lambda}{2}\|\mathbf{w}\|_2^2$ |
| L1 正则化 | $\mathcal{L} = \mathcal{L}_0 + \lambda\|\mathbf{w}\|_1$ |
| Dropout（训练时） | $\tilde{h} = \frac{\mathbf{r} \odot h}{1-p}, \quad \mathbf{r} \sim \text{Bernoulli}(1-p)$ |
| Batch Norm | $\hat{x} = \frac{x - \mu_\mathcal{B}}{\sqrt{\sigma_\mathcal{B}^2 + \epsilon}}, \quad y = \gamma\hat{x} + \beta$ |
| Layer Norm | $\hat{h}_i = \frac{h_i - \mu}{\sqrt{\sigma^2 + \epsilon}}, \quad y_i = \gamma_i\hat{h}_i + \beta_i$（均值和方差沿特征维度计算） |
| AdamW 更新 | $\theta_{t+1} = (1 - \eta\lambda)\theta_t - \eta \cdot \frac{\hat{m}_t}{\sqrt{\hat{v}_t} + \epsilon}$ |
