## 1. 直觉与概述

### 1.1 损失函数是什么

类比：你参加一场考试，试卷上有标准答案，而你写了自己的答案。**损失函数就是"扣分规则"**——它是一个函数，接受"你的答案"和"标准答案"，输出一个非负数字，数字越大表示差距越大。

在神经网络的语境下：

```
损失 = L(模型预测值, 真实标签)
```

- 模型预测值（prediction / output）：神经网络前向传播的输出
- 真实标签（ground truth / label）：训练数据中的正确答案
- 损失值（loss）：一个标量数字，越小越好

### 1.2 为什么需要损失函数

神经网络的训练本质是一个**优化问题**：调整网络中的数百万个参数，使得模型的预测尽可能接近真实答案。优化器（如 SGD、Adam）的工作方式是：

1. 算出损失值 $L$
2. 对每个参数求 $\frac{\partial L}{\partial \theta}$（损失关于参数的梯度）
3. 沿梯度的反方向更新参数：$\theta \leftarrow \theta - \eta \cdot \frac{\partial L}{\partial \theta}$

没有损失函数，优化器就没有"导航信号"——它不知道当前模型好不好，也不知道参数该往哪个方向调整。损失函数把"模型预测好不好"这个模糊的概念量化为一个可微分的数字，使得基于梯度的优化成为可能。

### 1.3 损失函数的基本要求

| 要求 | 原因 |
|------|------|
| 输出非负标量 | 便于比较和优化 |
| 可微分（至少几乎处处可微） | 反向传播需要计算梯度 |
| 预测越准，损失越小 | 确保最小化损失 = 提高预测质量 |
| 梯度具有有效的方向性 | 远离正确答案时，梯度应该足够大，能把参数"推"回来 |

最后一条尤为关键，它直接决定了"为什么分类问题不用 MSE 而用交叉熵"——后面会详细展开。

## 2. 均方误差 (MSE)

### 2.1 定义与公式

**均方误差（Mean Squared Error, MSE）**是最直观的损失函数：预测值与真实值之差的平方，再对所有样本取平均。

对单个样本：

$$L_{\text{MSE}} = (y - \hat{y})^2$$

对 $N$ 个样本的 mini-batch：

$$L_{\text{MSE}} = \frac{1}{N} \sum_{i=1}^{N} (y_i - \hat{y}_i)^2$$

其中 $y_i$ 是第 $i$ 个样本的真实值，$\hat{y}_i$ 是模型预测值。

> **注意**：有些文献和框架（如某些数学推导）使用 $\frac{1}{2}(y - \hat{y})^2$，多乘了一个 $\frac{1}{2}$ 纯粹是为了求导时消掉系数 2，使梯度公式更简洁。PyTorch 默认不带这个 $\frac{1}{2}$。

### 2.2 几何直觉

$$(y - \hat{y})^2 = \text{预测值与真实值之间"距离"的平方}$$

- 距离越大，损失越大——而且是**平方级别**地增大。预测偏差 2 倍，损失增大 4 倍。
- 这意味着 MSE 对**大误差特别敏感**（对异常值 outlier 不够鲁棒），但对小误差比较宽容。
- 几何上，MSE 构成一个**抛物线型**的损失曲面——在最低点附近平滑，梯度逐渐趋近于零，有利于精确收敛。

### 2.3 MSE 的梯度

对单个样本的 MSE 损失，对预测值 $\hat{y}$ 求导：

$$\frac{\partial L_{\text{MSE}}}{\partial \hat{y}} = -2(y - \hat{y}) = 2(\hat{y} - y)$$

关键观察：**梯度与误差成正比**。预测偏差越大，梯度越大，参数更新的步幅越大。这在回归问题中是理想的行为。

### 2.4 适用场景

MSE 适合**回归问题**——预测一个连续的数值：

- 房价预测
- 温度预测
- 年龄估计
- 股票价格（虽然实际上更复杂）

**不适合**分类问题——原因在第 4 节详细分析。

## 3. 交叉熵损失 (Cross Entropy Loss)

交叉熵是分类问题的标准损失函数。理解它需要一点信息论的背景，但回报巨大——它在分类任务中的表现远胜 MSE。

### 3.1 信息论视角速览

> 详细推导见 [附录 2.5 信息论基础](../附录_数学基础速览.md#25-信息论基础熵kl-散度交叉熵)，此处给出与损失函数直接相关的核心概念。

**信息量**：一个事件发生后带来的"惊讶程度"。概率越低的事件发生时，信息量越大。

$$I(x) = -\log p(x)$$

直觉：如果有人告诉你"明天太阳会升起"（概率接近 1），你不惊讶，信息量接近 0。如果有人说"明天会下陨石雨"（概率极低），你非常惊讶，信息量很大。

**熵**：一个概率分布的"平均不确定性"——所有可能事件的信息量的期望。

$$H(P) = -\sum_i p_i \log p_i$$

**交叉熵**：用分布 $Q$ 的编码方案去编码服从分布 $P$ 的数据，平均需要的信息量。

$$H(P, Q) = -\sum_i p_i \log q_i$$

**核心关系**：

$$H(P, Q) = H(P) + D_{KL}(P \| Q)$$

其中 $D_{KL}(P \| Q) \geq 0$ 是 KL 散度，衡量 $Q$ 偏离 $P$ 的程度。当 $P$ 是固定的真实分布时，$H(P)$ 是常数，因此：

$$\text{最小化交叉熵} \Leftrightarrow \text{最小化 KL 散度} \Leftrightarrow \text{让模型分布尽量逼近真实分布}$$

### 3.2 二分类交叉熵 (Binary Cross Entropy, BCE)

#### 场景

只有两个类别（如垃圾邮件 vs 正常邮件），模型通过 Sigmoid 输出一个概率 $p \in (0, 1)$，表示"属于正类"的概率。

#### 公式

$$L_{\text{BCE}} = -\left[y \cdot \log(p) + (1 - y) \cdot \log(1 - p)\right]$$

其中：
- $y \in \{0, 1\}$：真实标签
- $p = \sigma(z)$：模型输出经过 Sigmoid 后的概率

#### 为什么是这个形式

逐项分析：

| 真实标签 | 损失简化为 | 直觉 |
|----------|-----------|------|
| $y = 1$（正类） | $-\log(p)$ | 模型给正类的概率 $p$ 越高越好；$p \to 1$ 时损失趋近 0，$p \to 0$ 时损失趋近 $+\infty$ |
| $y = 0$（负类） | $-\log(1 - p)$ | 模型给负类的概率 $(1-p)$ 越高越好；$p \to 0$ 时损失趋近 0，$p \to 1$ 时损失趋近 $+\infty$ |

$-\log(\cdot)$ 的形状是这个损失函数的灵魂：

- 当模型很自信且正确时（$p$ 接近正确答案），$-\log(p) \approx 0$，损失几乎为零
- 当模型很自信但错误时（$p$ 远离正确答案），$-\log(p) \to +\infty$，损失急剧增大

这种"答错了严厉惩罚，答对了几乎不惩罚"的特性，远比 MSE 更适合分类任务。

#### 对 mini-batch 的平均

$$L_{\text{BCE}} = -\frac{1}{N} \sum_{i=1}^{N} \left[y_i \cdot \log(p_i) + (1 - y_i) \cdot \log(1 - p_i)\right]$$

### 3.3 多分类交叉熵 (Categorical Cross Entropy)

#### 场景

$K$ 个类别（如手写数字识别 0~9，$K=10$），模型对每个类别输出一个概率。

#### Softmax 回顾

模型的原始输出是 $K$ 维的向量 $\mathbf{z} = [z_1, z_2, \dots, z_K]$（称为 **logits**），还不是概率。通过 Softmax 函数将其转化为概率分布：

$$p_k = \text{Softmax}(z_k) = \frac{e^{z_k}}{\sum_{j=1}^{K} e^{z_j}}, \quad k = 1, 2, \dots, K$$

Softmax 保证：
- $p_k > 0$（指数函数输出恒正）
- $\sum_{k=1}^{K} p_k = 1$（归一化）

#### 公式

对单个样本，真实标签是 one-hot 向量 $\mathbf{y} = [y_1, y_2, \dots, y_K]$（只有一个位置为 1，其余为 0）：

$$L_{\text{CE}} = -\sum_{k=1}^{K} y_k \cdot \log(p_k)$$

由于 $\mathbf{y}$ 是 one-hot 的，只有正确类别 $c$ 对应的 $y_c = 1$，其余都是 0，所以：

$$L_{\text{CE}} = -\log(p_c)$$

**一句话总结**：多分类交叉熵损失 = 模型给正确类别分配的概率的负对数。

对 mini-batch：

$$L_{\text{CE}} = -\frac{1}{N} \sum_{i=1}^{N} \log(p_{i, c_i})$$

### 3.4 交叉熵 = 负对数似然（MLE 视角）

> 详细推导见 [附录 2.3 最大似然估计](../附录_数学基础速览.md#23-最大似然估计mle)。

从最大似然估计（MLE）的角度看，训练神经网络就是找到使训练数据概率最大的参数 $\theta$：

$$\hat{\theta} = \arg\max_\theta \prod_{i=1}^{N} p(y_i | x_i; \theta)$$

取对数（乘法变加法）：

$$\hat{\theta} = \arg\max_\theta \sum_{i=1}^{N} \log p(y_i | x_i; \theta)$$

等价于最小化**负对数似然（NLL）**：

$$\hat{\theta} = \arg\min_\theta \left( -\sum_{i=1}^{N} \log p(y_i | x_i; \theta) \right)$$

对比交叉熵损失 $L_{\text{CE}} = -\frac{1}{N}\sum_{i=1}^{N} \log(p_{i, c_i})$，除了一个 $\frac{1}{N}$ 的常数因子（不影响优化方向），**交叉熵损失就是负对数似然**。

这个等价关系非常重要：
- 它说明交叉熵不是一个拍脑袋想出来的启发式公式，而是有坚实的统计学理论支撑
- 最小化交叉熵 = 最大似然估计 = 让模型学到训练数据的真实分布

## 4. 为什么分类不用 MSE 而用交叉熵

这是一个面试高频问题，也是理解损失函数选择的关键。

### 4.1 问题的根源：Sigmoid + MSE 的梯度平坦问题

考虑二分类场景：模型输出 logit $z$，经过 Sigmoid 得到概率 $p = \sigma(z) = \frac{1}{1+e^{-z}}$，损失为 MSE。

链式法则求梯度：

$$\frac{\partial L_{\text{MSE}}}{\partial z} = \frac{\partial L_{\text{MSE}}}{\partial p} \cdot \frac{\partial p}{\partial z} = 2(p - y) \cdot \sigma'(z)$$

其中 $\sigma'(z) = \sigma(z)(1 - \sigma(z)) = p(1-p)$。

所以：

$$\frac{\partial L_{\text{MSE}}}{\partial z} = 2(p - y) \cdot p(1 - p)$$

**问题出在 $p(1-p)$ 这个因子上**：

| 模型输出 $p$ | $p(1-p)$ | 状况 |
|-------------|-----------|------|
| $p \approx 0.5$ | $\approx 0.25$ | 梯度正常 |
| $p \approx 0.01$（对 $y=1$ 大错特错） | $\approx 0.01$ | 梯度很小！ |
| $p \approx 0.99$（对 $y=0$ 大错特错） | $\approx 0.01$ | 梯度很小！ |

当模型给出的预测大错特错（比如真实标签是 1，模型非常自信地输出接近 0）时，Sigmoid 的导数 $p(1-p)$ 趋近于 0，导致**梯度消失**。模型明知自己错了，却几乎无法更新参数来纠正——像是站在一个极其平坦的高原上，迈不开步。

### 4.2 交叉熵解决了这个问题

同样的场景，改用交叉熵损失：

$$L_{\text{BCE}} = -[y \log(p) + (1-y) \log(1-p)]$$

$$\frac{\partial L_{\text{BCE}}}{\partial z} = \frac{\partial L_{\text{BCE}}}{\partial p} \cdot \frac{\partial p}{\partial z}$$

关键：交叉熵对 $p$ 的导数中的 $\frac{1}{p}$ 或 $\frac{1}{1-p}$ 项与 Sigmoid 导数中的 $p(1-p)$ **完美抵消**，最终：

$$\frac{\partial L_{\text{BCE}}}{\partial z} = p - y$$

这个梯度的优美之处：

| 模型输出 $p$ | 真实标签 $y$ | 梯度 $p - y$ | 直觉 |
|-------------|-------------|-------------|------|
| 0.99 | 1 | $-0.01$ | 预测很准，梯度很小（几乎不需要调整）|
| 0.01 | 1 | $-0.99$ | 预测大错，梯度很大（强力纠正）|
| 0.50 | 1 | $-0.50$ | 不确定，中等梯度（正常学习）|

**大错时梯度大，接近正确时梯度小**——这正是我们想要的理想行为。

### 4.3 总结对比

| 特性 | MSE + Sigmoid | 交叉熵 + Sigmoid |
|------|-------------|---------------|
| 梯度公式 | $2(p-y) \cdot p(1-p)$ | $p - y$ |
| 大错时梯度 | 很小（Sigmoid 饱和导致） | 很大（正确的行为） |
| 收敛速度 | 慢（初始阶段梯度平坦） | 快（梯度与误差成正比） |
| 理论基础 | 纯几何距离 | MLE / 信息论 |
| 适用场景 | 回归问题 | 分类问题 |

## 5. 其他常见损失函数

### 5.1 Hinge Loss（铰链损失）

来自支持向量机（SVM），用于二分类：

$$L_{\text{hinge}} = \max(0, 1 - y \cdot f(x))$$

其中 $y \in \{-1, +1\}$，$f(x)$ 是模型的原始输出（不需要 Sigmoid）。

直觉：只要模型的预测在正确一侧且有足够的"间隔"（margin > 1），损失就为 0。Hinge Loss 不关心"有多正确"，只关心"是否足够正确"。

适用场景：SVM、部分特征学习任务中仍会遇到。

### 5.2 Huber Loss（鲁棒回归损失）

MSE 对离群值（outlier）过于敏感，L1 Loss（$|y - \hat{y}|$）在 0 点不可微。Huber Loss 是两者的结合：

$$L_{\text{Huber}} = \begin{cases} \frac{1}{2}(y - \hat{y})^2, & |y - \hat{y}| \leq \delta \\ \delta \cdot (|y - \hat{y}| - \frac{1}{2}\delta), & |y - \hat{y}| > \delta \end{cases}$$

- 误差小于阈值 $\delta$ 时，退化为 MSE（平滑、精确）
- 误差大于 $\delta$ 时，退化为 L1（线性增长，不会被离群值主导）

适用场景：数据中有异常值的回归问题，强化学习中也常用。

### 5.3 Focal Loss

处理类别极度不均衡的问题（如目标检测中，背景样本远多于目标样本）：

$$L_{\text{focal}} = -\alpha_t (1 - p_t)^\gamma \log(p_t)$$

在交叉熵基础上乘了 $(1 - p_t)^\gamma$ 因子。对于"容易"分类正确的样本（$p_t$ 接近 1），这个因子接近 0，等于降低了它们的权重，让模型聚焦于"困难"样本。

适用场景：目标检测（RetinaNet）、严重类别不均衡的分类任务。

### 5.4 损失函数选择速查表

| 任务类型 | 推荐损失函数 | PyTorch 类 |
|----------|------------|-----------|
| 回归（连续值） | MSE | `nn.MSELoss()` |
| 回归（有离群值） | Huber Loss | `nn.HuberLoss()` |
| 二分类 | Binary Cross Entropy | `nn.BCEWithLogitsLoss()` |
| 多分类 | Cross Entropy | `nn.CrossEntropyLoss()` |
| 多标签分类 | Binary Cross Entropy（逐标签） | `nn.BCEWithLogitsLoss()` |
| 类别不均衡 | Focal Loss | 需自行实现或用 torchvision |

## 6. Python 代码示例

### 6.1 用 numpy 实现 MSE 和交叉熵

```python
import numpy as np

# ============================================================
# 1. MSE 损失函数
# ============================================================

def mse_loss(y_true, y_pred):
    """均方误差损失

    Args:
        y_true: 真实值, shape (N,) 或 (N, D)
        y_pred: 预测值, 与 y_true 同 shape
    Returns:
        标量损失值
    """
    return np.mean((y_true - y_pred) ** 2)


def mse_gradient(y_true, y_pred):
    """MSE 对 y_pred 的梯度"""
    N = y_true.shape[0]
    return 2 * (y_pred - y_true) / N


# 验证
y_true_reg = np.array([1.0, 2.0, 3.0, 4.0])
y_pred_reg = np.array([1.1, 2.3, 2.7, 4.2])
print("=== MSE 损失 ===")
print(f"y_true: {y_true_reg}")
print(f"y_pred: {y_pred_reg}")
print(f"MSE Loss: {mse_loss(y_true_reg, y_pred_reg):.6f}")
print(f"MSE Grad: {mse_gradient(y_true_reg, y_pred_reg)}")


# ============================================================
# 2. 二分类交叉熵损失
# ============================================================

def binary_cross_entropy(y_true, p_pred, eps=1e-12):
    """二分类交叉熵损失

    Args:
        y_true: 真实标签 (0 或 1), shape (N,)
        p_pred: 预测概率 (Sigmoid 输出), shape (N,), 值域 (0, 1)
        eps: 防止 log(0) 的极小值
    Returns:
        标量损失值
    """
    # 裁剪概率，防止 log(0)
    p_pred = np.clip(p_pred, eps, 1 - eps)
    loss = -(y_true * np.log(p_pred) + (1 - y_true) * np.log(1 - p_pred))
    return np.mean(loss)


def sigmoid(z):
    """Sigmoid 函数（数值稳定版）"""
    return np.where(z >= 0,
                    1 / (1 + np.exp(-z)),
                    np.exp(z) / (1 + np.exp(z)))


# 验证
y_true_bin = np.array([1, 0, 1, 1, 0])
logits_bin = np.array([2.0, -1.5, 0.5, 3.0, -2.0])
p_pred_bin = sigmoid(logits_bin)
print("\n=== 二分类交叉熵 ===")
print(f"y_true:  {y_true_bin}")
print(f"logits:  {logits_bin}")
print(f"sigmoid: {p_pred_bin.round(4)}")
print(f"BCE Loss: {binary_cross_entropy(y_true_bin, p_pred_bin):.6f}")


# ============================================================
# 3. 多分类交叉熵损失
# ============================================================

def softmax(logits):
    """Softmax 函数（数值稳定版：减去最大值防止 exp 溢出）

    Args:
        logits: shape (N, K)，N 个样本，K 个类别
    Returns:
        概率分布, shape (N, K)
    """
    # 每个样本减去自己的最大 logit（数值稳定技巧）
    shifted = logits - np.max(logits, axis=-1, keepdims=True)
    exp_shifted = np.exp(shifted)
    return exp_shifted / np.sum(exp_shifted, axis=-1, keepdims=True)


def cross_entropy_loss(logits, y_true, eps=1e-12):
    """多分类交叉熵损失（从 logits 出发）

    Args:
        logits: shape (N, K)，模型的原始输出
        y_true: shape (N,)，每个样本的真实类别索引 (0 ~ K-1)
    Returns:
        标量损失值
    """
    N = logits.shape[0]
    probs = softmax(logits)
    # 取出每个样本在真实类别上的概率
    correct_probs = probs[np.arange(N), y_true]
    # 负对数似然
    loss = -np.log(np.clip(correct_probs, eps, 1.0))
    return np.mean(loss)


# 验证：4 个样本，3 个类别
logits_multi = np.array([
    [2.0, 1.0, 0.1],   # 样本 0
```

### 6.2 MSE vs 交叉熵：梯度大小对比

这是理解"为什么分类不用 MSE"的核心实验。

```python
import numpy as np

def sigmoid(z):
    return np.where(z >= 0,
                    1 / (1 + np.exp(-z)),
                    np.exp(z) / (1 + np.exp(z)))

# ============================================================
# 实验：对同一个二分类问题，比较 MSE 和 BCE 的梯度
# ============================================================
# 场景：真实标签 y=1，模型从"大错特错"到"非常正确"

y = 1.0
logits = np.array([-5.0, -3.0, -1.0, 0.0, 1.0, 3.0, 5.0])
probs = sigmoid(logits)

print("logit  |  p=σ(z)  |  MSE 梯度 (dL/dz)  |  BCE 梯度 (dL/dz)  |  比率")
print("-" * 80)

for z, p in zip(logits, probs):
    # MSE 梯度: 2(p - y) * p * (1 - p)
    grad_mse = 2 * (p - y) * p * (1 - p)

    # BCE 梯度: p - y
    grad_bce = p - y

    ratio = abs(grad_bce / grad_mse) if abs(grad_mse) > 1e-15 else float('inf')

    print(f"z={z:+5.1f}  |  p={p:.4f}  |  MSE: {grad_mse:+.6f}   "
          f"|  BCE: {grad_bce:+.6f}   |  {ratio:.1f}x")

print()
print("关键观察：")
print("  - z=-5 时（大错特错），MSE 梯度只有 -0.013，而 BCE 梯度是 -0.993")
print("  - BCE 的梯度是 MSE 的约 73 倍！")
print("  - MSE 在'大错特错'时梯度反而很小，导致学习缓慢")
print("  - BCE 的梯度始终和误差成正比（p-y），行为更合理")
```

### 6.3 PyTorch 中的损失函数

```python
import torch
import torch.nn as nn

# ============================================================
# 1. MSELoss
# ============================================================
mse_criterion = nn.MSELoss()

y_true = torch.tensor([1.0, 2.0, 3.0, 4.0])
y_pred = torch.tensor([1.1, 2.3, 2.7, 4.2])

loss_mse = mse_criterion(y_pred, y_true)
print(f"=== PyTorch MSELoss ===")
print(f"MSE Loss: {loss_mse.item():.6f}")  # 0.057500


# ============================================================
# 2. BCEWithLogitsLoss（推荐方式：接受 logits，内部做 Sigmoid）
# ============================================================
bce_criterion = nn.BCEWithLogitsLoss()

y_true_bin = torch.tensor([1.0, 0.0, 1.0, 1.0, 0.0])
logits_bin = torch.tensor([2.0, -1.5, 0.5, 3.0, -2.0])

loss_bce = bce_criterion(logits_bin, y_true_bin)
print(f"\n=== PyTorch BCEWithLogitsLoss ===")
print(f"BCE Loss: {loss_bce.item():.6f}")  # 0.210570

# 注意：BCEWithLogitsLoss = Sigmoid + BCELoss，且数值更稳定
# 不要自己先过 Sigmoid 再用 BCELoss，会损失数值精度！


# ============================================================
# 3. CrossEntropyLoss（接受 logits，内部做 log_softmax + NLLLoss）
# ============================================================
ce_criterion = nn.CrossEntropyLoss()

# 注意：logits shape 是 (N, K)，标签 shape 是 (N,) 且为 long 类型
logits_multi = torch.tensor([
    [2.0, 1.0, 0.1],
    [0.5, 2.5, 0.3],
    [0.1, 0.2, 3.0],
    [1.0, 1.0, 1.0],
])
y_true_multi = torch.tensor([0, 1, 2, 0])  # 注意是 int/long，不是 one-hot

loss_ce = ce_criterion(logits_multi, y_true_multi)
print(f"\n=== PyTorch CrossEntropyLoss ===")
print(f"CE Loss: {loss_ce.item():.6f}")  # 0.453925

# ============================================================
# 重要细节：CrossEntropyLoss 内含 Softmax！
# ============================================================
# 以下两种写法等价：
# 方式 A（推荐）：直接传 logits
loss_a = nn.CrossEntropyLoss()(logits_multi, y_true_multi)

# 方式 B（手动分解）：log_softmax + NLLLoss
log_probs = nn.functional.log_softmax(logits_multi, dim=1)
loss_b = nn.NLLLoss()(log_probs, y_true_multi)

print(f"\n=== 验证 CrossEntropyLoss = log_softmax + NLLLoss ===")
print(f"方式 A (CrossEntropyLoss):       {loss_a.item():.6f}")
print(f"方式 B (log_softmax + NLLLoss):  {loss_b.item():.6f}")
print(f"两者相等: {torch.allclose(loss_a, loss_b)}")

# 不推荐：先过 softmax 再传给 CrossEntropyLoss
# CrossEntropyLoss 期望输入是 raw logits，先做 softmax 会导致双重 softmax，
# 数值行为和梯度都会偏离预期，虽然不会报错但会影响训练效果
probs_wrong = torch.softmax(logits_multi, dim=1)
loss_wrong = nn.CrossEntropyLoss()(probs_wrong, y_true_multi)  # 不推荐！
print(f"\n不推荐用法（双重 softmax）: {loss_wrong.item():.6f}  <-- 数值偏离预期")
print(f"正确用法:                 {loss_ce.item():.6f}")


# ============================================================
# 4. 完整的训练循环示例
# ============================================================
print("\n=== 完整训练循环 ===")
torch.manual_seed(42)

# 构造一个简单数据集：3 类分类，100 个样本，4 维特征
N, D, K = 100, 4, 3
X = torch.randn(N, D)
y = torch.randint(0, K, (N,))

# 简单的两层网络
model = nn.Sequential(
    nn.Linear(D, 16),
    nn.ReLU(),
    nn.Linear(16, K),  # 输出层不需要 Softmax！CrossEntropyLoss 会处理
)

criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.01)

for epoch in range(5):
    logits = model(X)           # (100, 3) — 原始 logits，无 softmax
    loss = criterion(logits, y) # 内部: log_softmax + NLLLoss

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

    # 计算准确率
    preds = logits.argmax(dim=1)
    acc = (preds == y).float().mean()
    print(f"  Epoch {epoch+1}: loss={loss.item():.4f}, acc={acc.item():.2%}")
```

## 7. 工程师视角

### 7.1 PyTorch CrossEntropyLoss 内部做了什么

PyTorch 的 `nn.CrossEntropyLoss` 等价于 `log_softmax + NLLLoss` 的组合，其内部实现流程是：

```
输入 logits (N, K)
    ↓
log_softmax: 计算 log(softmax(logits))
    ↓
NLLLoss: 取出每个样本正确类别的 log 概率，取负，再求平均
    ↓
输出标量 loss
```

**黄金规则：模型的最后一层不要加 Softmax。** 直接输出 raw logits，让 `CrossEntropyLoss` 处理。这不仅是约定，更是因为数值稳定性（见下节）。

### 7.2 数值稳定性：为什么用 log_softmax 而不是 log(softmax(x))

先看"朴素"的计算方式：

```python
# 朴素方式（数值不稳定）
probs = softmax(logits)     # 步骤 1: 指数运算可能溢出
log_probs = log(probs)      # 步骤 2: 如果某个概率接近 0，log(0) = -inf
```

问题 1：**指数溢出**。如果 logit 值很大（如 1000），$e^{1000}$ 会产生 `inf`。虽然减去最大值可以缓解，但分两步计算增加了中间精度损失。

问题 2：**log(0) 未定义**。Softmax 输出的某些概率可能因浮点精度限制变成 0（特别是 float16 训练时），此时 $\log(0) = -\infty$。

`log_softmax` 的做法是将两步合并为一步，利用数学恒等式直接计算：

$$\log(\text{softmax}(z_k)) = \log\frac{e^{z_k}}{\sum_j e^{z_j}} = z_k - \log\sum_j e^{z_j}$$

具体实现（PyTorch 内部逻辑）：

```python
def log_softmax_stable(logits):
    """数值稳定的 log_softmax

    关键技巧：先减去最大值，避免 exp 溢出
    """
    # 减去最大值（不影响 softmax 结果，因为分子分母同时除以 e^max）
    max_val = logits.max(dim=-1, keepdim=True).values
    shifted = logits - max_val

    # log_softmax = shifted - log(sum(exp(shifted)))
    log_sum_exp = torch.log(torch.exp(shifted).sum(dim=-1, keepdim=True))
    return shifted - log_sum_exp
```

对比：

```python
import torch

logits = torch.tensor([[100.0, 200.0, 300.0]])  # 极端大值

# 朴素方式：会产生 nan
probs = torch.softmax(logits, dim=1)
log_probs_naive = torch.log(probs)
print(f"朴素方式: {log_probs_naive}")  # tensor([[-200., -100., 0.]]) -- 这个例子恰好没问题

# 但在 float16 下就会出问题
logits_fp16 = logits.half()
probs_fp16 = torch.softmax(logits_fp16, dim=1)
log_probs_fp16 = torch.log(probs_fp16)
print(f"朴素方式 (float16): {log_probs_fp16}")  # 可能出现 -inf

# log_softmax 方式：始终稳定
log_probs_stable = torch.log_softmax(logits_fp16, dim=1)
print(f"log_softmax (float16): {log_probs_stable}")  # 稳定输出
```

**工程总结**：

| 做法 | 推荐度 | 原因 |
|------|--------|------|
| `nn.CrossEntropyLoss(logits, targets)` | 强烈推荐 | 一站式处理，数值最稳定 |
| `log_softmax` + `NLLLoss` | 可以 | 需要中间结果时用，数值稳定 |
| `softmax` + `log` + 手动求和 | 不推荐 | 数值不稳定，尤其在 float16 |
| 模型最后加 `Softmax` + `CrossEntropyLoss` | 不推荐 | 双重 softmax，数值和梯度行为偏离预期 |

### 7.3 标签平滑 (Label Smoothing)

#### 问题

标准交叉熵使用 one-hot 标签：正确类别概率为 1，其他为 0。这驱使模型输出越来越极端的 logits（让正确类别的概率尽量接近 1），导致：

1. **过度自信**：模型对自己的预测过于确定，泛化性变差
2. **logits 爆炸**：为了让 softmax 输出趋近 1，logits 不断增大

#### 解决方案

将 one-hot 标签"软化"——把一小部分概率均匀分给其他类别：

$$y_k^{\text{smooth}} = \begin{cases} 1 - \varepsilon + \frac{\varepsilon}{K}, & k = c \text{（正确类别）} \\ \frac{\varepsilon}{K}, & k \neq c \text{（其他类别）} \end{cases}$$

其中 $\varepsilon$ 是平滑系数（常用 0.1），$K$ 是类别数。

例如，3 分类问题，真实标签为类别 0，$\varepsilon = 0.1$：
- 原始标签：$[1.0, \; 0.0, \; 0.0]$
- 平滑标签：$[0.933, \; 0.033, \; 0.033]$

#### PyTorch 实现

```python
import torch
import torch.nn as nn

# PyTorch 内置支持（推荐）
criterion = nn.CrossEntropyLoss(label_smoothing=0.1)

logits = torch.randn(8, 10)       # 8 个样本，10 个类别
targets = torch.randint(0, 10, (8,))

loss = criterion(logits, targets)
print(f"带标签平滑的损失: {loss.item():.4f}")

# 手动实现（帮助理解原理）
def cross_entropy_with_label_smoothing(logits, targets, epsilon=0.1):
    """
    Args:
        logits: (N, K) 原始 logits
        targets: (N,) 类别索引
        epsilon: 平滑系数
    """
    K = logits.size(1)
    log_probs = torch.log_softmax(logits, dim=1)  # (N, K)

    # one-hot 部分的损失（标准交叉熵）
    nll = -log_probs.gather(dim=1, index=targets.unsqueeze(1)).squeeze(1)  # (N,)

    # 均匀分布部分的损失（等于所有 log_probs 的平均的负值）
    smooth_loss = -log_probs.mean(dim=1)  # (N,)

    # 加权组合
    loss = (1 - epsilon) * nll + epsilon * smooth_loss
    return loss.mean()

loss_manual = cross_entropy_with_label_smoothing(logits, targets, epsilon=0.1)
print(f"手动实现的损失:   {loss_manual.item():.4f}")
# 两者应该非常接近（可能有微小浮点误差）
```

#### 何时使用标签平滑

| 场景 | 是否使用 | 原因 |
|------|---------|------|
| 大规模分类（ImageNet、语言模型） | 通常使用 | 防止过拟合，提升泛化 |
| 小数据集 + 少类别 | 视情况 | 数据少时可能帮助防过拟合 |
| 知识蒸馏 | 通常不用 | 教师模型的 soft label 本身就是"平滑的" |
| 标签本身有噪声 | 推荐使用 | 减弱对噪声标签的过拟合 |

### 核心知识点回顾

```
损失函数
├── 作用：量化"预测 vs 真实"的差距，为优化器提供方向
├── MSE（均方误差）
│   ├── 公式：(1/N) * Σ(y - ŷ)²
│   ├── 梯度与误差成正比
│   └── 适用：回归问题
├── 交叉熵
│   ├── 二分类 BCE：-[y·log(p) + (1-y)·log(1-p)]
│   ├── 多分类 CE：-log(p_正确类)
│   ├── 等价于负对数似然（MLE 视角）
│   ├── 等价于最小化 KL 散度（信息论视角）
│   └── 适用：分类问题
├── 为什么分类用 CE 不用 MSE
│   ├── Sigmoid + MSE → 梯度含 p(1-p) 因子 → 大错时梯度消失
│   └── Sigmoid + CE  → 梯度 = p - y → 大错时梯度大（正确行为）
└── 工程要点
    ├── CrossEntropyLoss 内含 Softmax，模型最后一层不要加 Softmax
    ├── log_softmax 比 log(softmax(x)) 数值更稳定
    └── 标签平滑防止过拟合和过度自信
```
