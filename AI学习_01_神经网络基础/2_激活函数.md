# 激活函数

> **前置知识**：了解神经网络的基本结构（前向传播 $y = Wx + b$），理解矩阵乘法和导数的含义。如需回顾，参考 [1_感知机与多层网络](1_感知机与多层网络.md) 和 [附录：数学基础速览](../附录_数学基础速览.md) 3.1 节（导数与偏导数）。

## 一、直觉与概述

### 1.1 为什么需要激活函数？

先看一个关键事实：**没有激活函数的多层网络，等价于单层网络**。

假设一个 2 层线性网络：

$$y = W_2(W_1 x + b_1) + b_2 = W_2 W_1 x + W_2 b_1 + b_2 = W' x + b'$$

其中 $W' = W_2 W_1$，$b' = W_2 b_1 + b_2$。无论你叠多少层线性变换，结果仍然是一个线性变换。这意味着：

- 2 层 = 1 层
- 100 层 = 1 层
- 深度完全没有意义

线性函数只能学到线性边界（高维空间中的超平面），而现实世界的问题几乎都是非线性的：图片中猫和狗的分界不是一条直线，语言的语义空间也远非线性可描述。

**激活函数的作用：在每一层的线性变换之后引入非线性，打破线性叠加的局限，让多层网络真正拥有"深度"的意义。**

### 1.2 类比理解

把激活函数想象成一个**信号滤波器**或**阀门**：

- **线性变换**（$Wx + b$）：像一个混音台的推子，对输入信号做加权混合
- **激活函数**：像一个阀门，决定信号如何通过——完全通过、部分通过、截断、还是变换形状

不同的激活函数就是不同类型的阀门：
- Sigmoid 像一个 S 型阀门，把任意大的信号压缩到 0~1 之间
- ReLU 像一个单向阀，正信号完全通过，负信号完全截断
- GELU 像一个"软"单向阀，根据信号值的概率决定通过多少

### 1.3 激活函数在网络中的位置

一个典型的神经网络层的计算流程：

```
输入 x  →  线性变换 z = Wx + b  →  激活函数 a = f(z)  →  输出 a（作为下一层的输入）
```

用 Python 伪代码表示：

```python
# 单层前向传播
z = W @ x + b       # 线性变换：加权求和
a = activation(z)   # 非线性激活：注入非线性
```

整个网络就是这个过程的多次重复，每层可以用不同的激活函数。

## 二、逐个详解：每种激活函数的原理

### 2.1 Sigmoid

**数学公式**：

$$\sigma(x) = \frac{1}{1 + e^{-x}}$$

**输出范围**：$(0, 1)$

**函数图像描述**：经典的 S 形曲线。在 $x=0$ 处输出 0.5，$x$ 趋向正无穷时输出趋向 1，$x$ 趋向负无穷时输出趋向 0。曲线在 $x=0$ 附近变化最快，两端趋于平坦（饱和）。

**导数公式**：

$$\sigma'(x) = \sigma(x)(1 - \sigma(x))$$

这个导数有一个优雅的性质：可以直接用 Sigmoid 的输出值来计算，不需要重新算原始输入。导数最大值在 $x=0$ 处取到，为 $0.25$。

**优点**：
- 输出有明确的概率含义（0 到 1 之间），天然适合二分类的输出层
- 处处可导，数学性质好

**缺点**：
- **梯度消失**（最致命的问题）：导数最大值只有 0.25。在反向传播中，每经过一层 Sigmoid，梯度至少乘以 0.25。5 层网络，梯度衰减到 $0.25^5 \approx 0.001$，底层几乎学不动
- **输出非零中心**：输出恒正（0 到 1），导致后续层的梯度更新总是同向的，优化路径呈锯齿形，收敛变慢
- **指数运算开销**：$e^{-x}$ 比简单的比较运算慢得多

**现代用途**：几乎不用于隐藏层，仅用于二分类输出层（输出概率）和门控机制（如 LSTM 的遗忘门）。

---

### 2.2 Tanh

**数学公式**：

$$\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}} = 2\sigma(2x) - 1$$

**输出范围**：$(-1, 1)$

**函数图像描述**：形状与 Sigmoid 相似的 S 型曲线，但上下对称于原点。$x=0$ 处输出 0，正负方向分别趋向 $\pm 1$。

**导数公式**：

$$\tanh'(x) = 1 - \tanh^2(x)$$

导数最大值在 $x=0$ 处取到，为 $1.0$（比 Sigmoid 的 0.25 大 4 倍）。

**与 Sigmoid 的关系**：Tanh 本质上是 Sigmoid 的缩放平移版本：$\tanh(x) = 2\sigma(2x) - 1$。

**优点**：
- **零中心输出**：输出均值为 0，解决了 Sigmoid 输出恒正导致的锯齿优化问题
- 导数最大值为 1（Sigmoid 只有 0.25），梯度消失问题相对较轻

**缺点**：
- **仍有梯度消失**：当 $|x|$ 较大时，导数趋近于 0，深层网络仍面临梯度消失
- 同样有指数运算开销

**现代用途**：比 Sigmoid 用得稍多，常见于 RNN/LSTM 的隐藏状态计算。在需要 $(-1, 1)$ 范围输出时使用。

---

### 2.3 ReLU（Rectified Linear Unit）

**数学公式**：

$$\text{ReLU}(x) = \max(0, x) = \begin{cases} x & \text{if } x > 0 \\ 0 & \text{if } x \leq 0 \end{cases}$$

**输出范围**：$[0, +\infty)$

**函数图像描述**：极其简单的折线——$x \leq 0$ 时输出为 0（紧贴 x 轴），$x > 0$ 时输出等于输入（45 度斜线）。在 $x=0$ 处有一个"拐点"（不可导点）。

**导数公式**：

$$\text{ReLU}'(x) = \begin{cases} 1 & \text{if } x > 0 \\ 0 & \text{if } x < 0 \end{cases}$$

$x=0$ 处严格来说不可导，但实践中约定为 0（或 1，不影响训练）。

**为什么 ReLU 是一个革命性突破**：

1. **彻底解决正半轴的梯度消失**：$x > 0$ 时导数恒为 1，无论网络多深，梯度都不会因为激活函数而衰减
2. **计算极其高效**：只需一次比较运算（`max(0, x)`），没有指数、除法，比 Sigmoid 快一个数量级
3. **稀疏激活**：大约 50% 的神经元输出为 0，带来天然的稀疏性，类似生物神经元的工作方式

**优点**：
- 正半轴梯度恒为 1，有效缓解梯度消失
- 计算速度极快
- 稀疏激活，有一定的正则化效果

**缺点**：
- **死神经元（Dead Neurons）问题**：如果某个神经元的输入始终为负（比如因为一次较大的梯度更新导致权重偏移），那么它的输出永远为 0，梯度也永远为 0，这个神经元就"死"了，再也不会被更新。严重时可能有 10%~40% 的神经元死亡
- **输出非零中心**：与 Sigmoid 有相同的问题
- $x=0$ 处不可导（实际影响极小）

**现代用途**：仍然是最广泛使用的隐藏层激活函数，尤其在 CNN 和中小型网络中。简单、高效、效果好。

---

### 2.4 Leaky ReLU 与 PReLU

**数学公式**：

$$\text{LeakyReLU}(x) = \begin{cases} x & \text{if } x > 0 \\ \alpha x & \text{if } x \leq 0 \end{cases} = \max(\alpha x, x)$$

其中 $\alpha$ 是一个很小的正数，通常取 $0.01$。

**PReLU（Parametric ReLU）**：$\alpha$ 不是固定值，而是一个可学习的参数，由网络通过反向传播自动调整。

**输出范围**：$(-\infty, +\infty)$

**函数图像描述**：和 ReLU 几乎一样，但 $x < 0$ 时不是完全为 0，而是一条非常平缓的斜线（斜率为 $\alpha$，默认 0.01）。

**导数公式**：

$$\text{LeakyReLU}'(x) = \begin{cases} 1 & \text{if } x > 0 \\ \alpha & \text{if } x < 0 \end{cases}$$

**优点**：
- 完全解决了死神经元问题：$x < 0$ 时梯度不为 0，神经元始终有机会被更新
- 保留了 ReLU 的计算效率优势

**缺点**：
- $\alpha$ 值需要选择（虽然 0.01 是广泛接受的默认值）
- 实际效果相比 ReLU 的提升并不总是显著

**现代用途**：当模型中出现大量死神经元时，用 Leaky ReLU 替换 ReLU 是第一选择。GAN 的判别器中较常用（训练不稳定场景下死神经元更容易出现）。

---

### 2.5 GELU（Gaussian Error Linear Unit）

**数学公式**：

$$\text{GELU}(x) = x \cdot \Phi(x)$$

其中 $\Phi(x)$ 是标准正态分布的累积分布函数（CDF）：

$$\Phi(x) = P(X \leq x) = \frac{1}{2}\left[1 + \text{erf}\left(\frac{x}{\sqrt{2}}\right)\right]$$

**实用近似公式**（PyTorch 中的 `tanh` 近似，误差 < 0.004）：

$$\text{GELU}(x) \approx 0.5 x \left[1 + \tanh\left(\sqrt{\frac{2}{\pi}}\left(x + 0.044715 x^3\right)\right)\right]$$

**输出范围**：大约 $[-0.17, +\infty)$。负半轴有一个极小值约 $-0.17$（在 $x \approx -0.75$ 处），之后趋向 0。

**函数图像描述**：整体形状类似 ReLU，但在 $x=0$ 附近是光滑的（没有尖角）。$x$ 很大时接近恒等函数 $y=x$；$x$ 很负时接近 0（但不完全为 0，有一个小的负值凸起然后趋向 0）。

**导数公式**：

$$\text{GELU}'(x) = \Phi(x) + x \cdot \phi(x)$$

其中 $\phi(x) = \frac{1}{\sqrt{2\pi}}e^{-x^2/2}$ 是标准正态的概率密度函数。

**直觉理解——为什么要乘以 $\Phi(x)$**：

GELU 的核心思想是"**根据输入值的大小，以概率方式决定是否保留**"：
- 输入 $x$ 很大（正）→ $\Phi(x) \approx 1$ → 输出 $\approx x$（几乎完全保留）
- 输入 $x \approx 0$ → $\Phi(x) \approx 0.5$ → 输出 $\approx 0.5x$（保留一半）
- 输入 $x$ 很负 → $\Phi(x) \approx 0$ → 输出 $\approx 0$（几乎完全抑制）

这是一种"**软门控**"——与 ReLU 的硬截断（非 0 即全通过）不同，GELU 提供了一个平滑的过渡。可以理解为一种"**确定性的 Dropout**"：不是随机丢弃，而是根据输入值本身的统计意义来决定保留比例。

**为什么比 ReLU 好（在 Transformer 场景下）**：

1. **处处光滑可导**：没有 ReLU 在 $x=0$ 处的不可导点，梯度流动更顺畅
2. **允许小的负值通过**：不会完全杀死负信号，避免信息丢失
3. **非单调性**：在 $x \approx -0.75$ 附近有一个小的负值输出区域，这为网络提供了更丰富的表达能力
4. **与 Transformer 的配合**：Transformer 的 FFN 层使用 GELU 比 ReLU 在多项 NLP 基准上表现更好（BERT 论文中验证）

**现代用途**：Transformer 系列模型（BERT、GPT-2/3/4、Claude 等）的标准激活函数。是当前 NLP 领域的事实标准。

---

### 2.6 Swish / SiLU（Sigmoid Linear Unit）

**数学公式**：

$$\text{Swish}(x) = x \cdot \sigma(x) = \frac{x}{1 + e^{-x}}$$

其中 $\sigma(x)$ 是 Sigmoid 函数。

> Swish 由 Google Brain 在 2017 年通过自动搜索发现，SiLU 是同一函数的另一个名字（Sigmoid Linear Unit）。

**输出范围**：大约 $[-0.278, +\infty)$。

**函数图像描述**：与 GELU 非常相似——光滑、在正半轴近似线性、在负半轴有一个小的负值凸起然后趋向 0。两者的差异在曲线细节上，肉眼几乎难以区分。

**导数公式**：

$$\text{Swish}'(x) = \sigma(x) + x \cdot \sigma(x)(1 - \sigma(x)) = \sigma(x)(1 + x(1 - \sigma(x)))$$

**与 GELU 的关系**：

- GELU：$x \cdot \Phi(x)$，其中 $\Phi$ 是正态分布 CDF
- Swish：$x \cdot \sigma(x)$，其中 $\sigma$ 是 Sigmoid（逻辑分布 CDF）

两者结构完全相同（$x$ 乘以一个 S 型函数），只是用了不同的 S 型函数。在实际使用中效果非常接近，差异通常在误差范围内。

**现代用途**：在一些视觉模型（如 EfficientNet）和 LLaMA 系列模型中使用。LLaMA 使用 SiLU 作为其 FFN 层的激活函数（配合 GLU 门控结构）。

---

### 2.7 Softmax（特殊的激活函数）

虽然 Softmax 不是常规意义上的逐元素激活函数（它的输入和输出是整个向量），但它在神经网络中扮演重要角色，有必要了解。

**数学公式**：对于向量 $\mathbf{z} = (z_1, z_2, \dots, z_K)$：

$$\text{Softmax}(z_i) = \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}}$$

**性质**：
- 输出的每个元素都在 $(0, 1)$ 之间
- 所有输出之和为 1
- 因此可以解释为概率分布

**现代用途**：多分类问题的输出层，Transformer 注意力机制中计算注意力权重。

---

### 2.8 如何选择激活函数：决策流程

```
你要选择哪一层的激活函数？
│
├─→ 输出层
│   │
│   ├─→ 二分类 → Sigmoid（输出概率 0~1）
│   ├─→ 多分类 → Softmax（输出概率分布）
│   ├─→ 回归（任意实数）→ 无激活函数（线性输出）
│   └─→ 回归（有界输出）→ Sigmoid 或 Tanh（根据范围选择）
│
└─→ 隐藏层
    │
    ├─→ Transformer / NLP 模型 → GELU（BERT/GPT 的标配）
    │                            或 SiLU（LLaMA 的选择）
    │
    ├─→ CNN / 一般前馈网络 → ReLU（默认首选，简单高效）
    │   │
    │   └─→ 出现死神经元？→ 换 Leaky ReLU 或 PReLU
    │
    └─→ RNN / LSTM → Tanh（隐藏状态）+ Sigmoid（门控）
```

**一句话总结**：不确定用什么就用 ReLU；做 Transformer 就用 GELU；输出层根据任务选。

> **补充提醒**：输出层激活函数和隐藏层激活函数的分工不同——隐藏层激活函数负责引入非线性以增强表达能力，输出层激活函数负责将结果映射到任务所需的数值范围（概率、分布等），两者的选择逻辑完全独立。

## 三、Python 代码示例

### 3.1 用 NumPy 实现所有激活函数及其导数

```python
import numpy as np

# ============================================================
# 激活函数实现
# ============================================================

def sigmoid(x):
    """Sigmoid: 将任意实数压缩到 (0, 1)"""
    return 1 / (1 + np.exp(-np.clip(x, -500, 500)))  # clip 防止溢出

def sigmoid_derivative(x):
    """Sigmoid 导数: σ(x)(1 - σ(x))"""
    s = sigmoid(x)
    return s * (1 - s)

def tanh(x):
    """Tanh: 将任意实数压缩到 (-1, 1)"""
    return np.tanh(x)

def tanh_derivative(x):
    """Tanh 导数: 1 - tanh²(x)"""
    return 1 - np.tanh(x) ** 2

def relu(x):
    """ReLU: 正数不变，负数截断为 0"""
    return np.maximum(0, x)

def relu_derivative(x):
    """ReLU 导数: x > 0 时为 1，否则为 0"""
    return (x > 0).astype(float)

def leaky_relu(x, alpha=0.01):
    """Leaky ReLU: 正数不变，负数乘以小系数 α"""
    return np.where(x > 0, x, alpha * x)

def leaky_relu_derivative(x, alpha=0.01):
    """Leaky ReLU 导数: x > 0 时为 1，否则为 α"""
    return np.where(x > 0, 1.0, alpha)

def gelu(x):
    """GELU: x * Φ(x)，使用 tanh 近似"""
    return 0.5 * x * (1 + np.tanh(np.sqrt(2 / np.pi) * (x + 0.044715 * x**3)))

def gelu_derivative(x):
    """GELU 导数的数值近似（解析公式较长，用中心差分）"""
    h = 1e-7
    return (gelu(x + h) - gelu(x - h)) / (2 * h)

def swish(x):
    """Swish/SiLU: x * sigmoid(x)"""
    return x * sigmoid(x)

def swish_derivative(x):
    """Swish 导数: σ(x) + x * σ(x) * (1 - σ(x))"""
    s = sigmoid(x)
    return s + x * s * (1 - s)

# ============================================================
# 验证：用数值导数检查解析导数是否正确
# ============================================================

def numerical_derivative(f, x, h=1e-7):
    """中心差分法计算数值导数"""
    return (f(x + h) - f(x - h)) / (2 * h)

x_test = np.array([-2.0, -1.0, -0.5, 0.0, 0.5, 1.0, 2.0])

print("=== 导数验证（解析 vs 数值）===")
for name, func, deriv in [
    ("Sigmoid", sigmoid, sigmoid_derivative),
    ("Tanh", tanh, tanh_derivative),
    ("ReLU", relu, relu_derivative),
    ("LeakyReLU", leaky_relu, leaky_relu_derivative),
    ("Swish", swish, swish_derivative),
]:
    analytical = deriv(x_test)
    numerical = numerical_derivative(func, x_test)
    max_err = np.max(np.abs(analytical - numerical))
    print(f"  {name:12s}: 最大误差 = {max_err:.2e}  {'✓ 通过' if max_err < 1e-5 else '✗ 有误'}")
```

### 3.3 实验：Sigmoid vs ReLU 训练速度对比

下面的代码从零开始构建一个简单的全连接网络，分别用 Sigmoid 和 ReLU 作为激活函数训练，直观展示 ReLU 的训练优势。

```python
import numpy as np
import matplotlib.pyplot as plt

# ============================================================
# 从零构建一个 3 层全连接网络（不依赖任何框架）
# 任务：学习 XOR 的扩展版（2D 非线性分类）
# ============================================================

# ---- 生成数据：同心圆分类问题 ----
np.random.seed(42)
N = 500  # 每类样本数

# 内圈（标签 0）
r_inner = np.random.uniform(0, 1, N)
theta_inner = np.random.uniform(0, 2 * np.pi, N)
X_inner = np.column_stack([r_inner * np.cos(theta_inner), r_inner * np.sin(theta_inner)])

# 外圈（标签 1）
r_outer = np.random.uniform(1.5, 2.5, N)
theta_outer = np.random.uniform(0, 2 * np.pi, N)
X_outer = np.column_stack([r_outer * np.cos(theta_outer), r_outer * np.sin(theta_outer)])

X = np.vstack([X_inner, X_outer])  # (1000, 2)
y = np.array([0] * N + [1] * N)    # (1000,)

# 打乱数据
shuffle_idx = np.random.permutation(len(y))
X, y = X[shuffle_idx], y[shuffle_idx]

# 标准化
X = (X - X.mean(axis=0)) / X.std(axis=0)

# ---- 激活函数 ----
def sigmoid(x):
    return 1 / (1 + np.exp(-np.clip(x, -500, 500)))

def sigmoid_grad(x):
    s = sigmoid(x)
    return s * (1 - s)

def relu(x):
    return np.maximum(0, x)

def relu_grad(x):
    return (x > 0).astype(float)

# ---- 网络结构：2 -> 32 -> 32 -> 1 ----
def init_params(layer_dims):
    """Xavier 初始化"""
    params = {}
    for l in range(1, len(layer_dims)):
        scale = np.sqrt(2.0 / layer_dims[l - 1])  # He initialization (好搭配 ReLU)
        params[f"W{l}"] = np.random.randn(layer_dims[l], layer_dims[l - 1]) * scale
        params[f"b{l}"] = np.zeros((layer_dims[l], 1))
    return params

def forward(X, params, activation_fn):
    """前向传播"""
    cache = {"A0": X.T}  # (features, batch)
    L = len(params) // 2  # 层数

    for l in range(1, L):
        Z = params[f"W{l}"] @ cache[f"A{l-1}"] + params[f"b{l}"]
        A = activation_fn(Z)
        cache[f"Z{l}"] = Z
        cache[f"A{l}"] = A

    # 输出层始终用 Sigmoid（二分类）
    ZL = params[f"W{L}"] @ cache[f"A{L-1}"] + params[f"b{L}"]
    AL = sigmoid(ZL)
    cache[f"Z{L}"] = ZL
    cache[f"A{L}"] = AL

    return AL, cache

def compute_loss(AL, Y):
    """二元交叉熵损失"""
    m = Y.shape[1]
    eps = 1e-8
    loss = -np.mean(Y * np.log(AL + eps) + (1 - Y) * np.log(1 - AL + eps))
    return loss

def backward(params, cache, Y, activation_grad_fn):
    """反向传播"""
    grads = {}
    L = len(params) // 2
    m = Y.shape[1]

    # 输出层梯度
    AL = cache[f"A{L}"]
    dZ = AL - Y  # Sigmoid + 交叉熵的简化梯度

    for l in range(L, 0, -1):
        A_prev = cache[f"A{l-1}"]
        grads[f"dW{l}"] = (1 / m) * dZ @ A_prev.T
        grads[f"db{l}"] = (1 / m) * np.sum(dZ, axis=1, keepdims=True)

        if l > 1:
            dA = params[f"W{l}"].T @ dZ
            dZ = dA * activation_grad_fn(cache[f"Z{l-1}"])

    return grads

def train(X, y, activation_fn, activation_grad_fn, lr=0.05, epochs=300):
    """训练循环"""
    Y = y.reshape(1, -1)
    layer_dims = [2, 32, 32, 1]  # 3 层网络
    params = init_params(layer_dims)
    losses = []

    for epoch in range(epochs):
        # 前向传播
        AL, cache = forward(X, params, activation_fn)
        loss = compute_loss(AL, Y)
        losses.append(loss)

        # 反向传播
        grads = backward(params, cache, Y, activation_grad_fn)

        # 参数更新
        for l in range(1, len(layer_dims)):
            params[f"W{l}"] -= lr * grads[f"dW{l}"]
            params[f"b{l}"] -= lr * grads[f"db{l}"]

        if epoch % 50 == 0:
            preds = (AL > 0.5).astype(int)
            acc = np.mean(preds == Y) * 100
            print(f"  Epoch {epoch:3d} | Loss: {loss:.4f} | Accuracy: {acc:.1f}%")

    # 最终准确率
    AL_final, _ = forward(X, params, activation_fn)
    preds = (AL_final > 0.5).astype(int)
    final_acc = np.mean(preds == Y.reshape(1, -1)) * 100
    return losses, final_acc

# ---- 训练对比 ----
print("=" * 50)
print("Sigmoid 激活函数训练：")
print("=" * 50)
losses_sigmoid, acc_sigmoid = train(X, y, sigmoid, sigmoid_grad, lr=0.5, epochs=300)

print()
print("=" * 50)
print("ReLU 激活函数训练：")
print("=" * 50)
losses_relu, acc_relu = train(X, y, relu, relu_grad, lr=0.05, epochs=300)

# ---- 画训练曲线对比 ----
plt.figure(figsize=(10, 6))
plt.plot(losses_sigmoid, label=f"Sigmoid (final acc: {acc_sigmoid:.1f}%)", color="#e74c3c", linewidth=2)
plt.plot(losses_relu, label=f"ReLU (final acc: {acc_relu:.1f}%)", color="#2ecc71", linewidth=2)
plt.xlabel("Epoch", fontsize=12)
plt.ylabel("Loss (Binary Cross-Entropy)", fontsize=12)
plt.title("Training Loss: Sigmoid vs ReLU", fontsize=14, fontweight="bold")
plt.legend(fontsize=12)
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.savefig("sigmoid_vs_relu_training.png", dpi=150, bbox_inches="tight")
plt.show()

print(f"\n最终结果：")
print(f"  Sigmoid — 准确率: {acc_sigmoid:.1f}%")
print(f"  ReLU    — 准确率: {acc_relu:.1f}%")
print(f"\n关键观察：ReLU 通常收敛更快，尤其在更深的网络中优势更明显。")
print(f"Sigmoid 的学习率需要设得更大（本例 0.5 vs ReLU 的 0.05），")
print(f"因为 Sigmoid 导数最大值只有 0.25，梯度天然偏小。")
```

### 4.1 现代实践总结

| 场景 | 推荐激活函数 | 原因 |
|------|-------------|------|
| 隐藏层（Transformer/NLP） | GELU | BERT/GPT 验证有效，光滑可导 |
| 隐藏层（LLaMA 系列） | SiLU (Swish) | 配合 GLU 门控结构效果优异 |
| 隐藏层（CNN/一般网络） | ReLU | 计算快，效果稳定，工程友好 |
| 隐藏层（GAN 判别器） | Leaky ReLU | 避免训练不稳定时的死神经元 |
| 输出层（二分类） | Sigmoid | 输出概率 $\in (0, 1)$ |
| 输出层（多分类） | Softmax | 输出概率分布 |
| 输出层（回归） | 无（线性） | 不限制输出范围 |
| RNN/LSTM 门控 | Sigmoid | 门控值需要在 0~1 之间 |
| RNN/LSTM 隐藏状态 | Tanh | 输出范围 $(-1, 1)$，零中心 |

### 4.2 PyTorch 中的使用方式

```python
import torch
import torch.nn as nn

# ---- 方式一：作为独立层使用（推荐，在 nn.Sequential 中清晰直观）----
model = nn.Sequential(
    nn.Linear(768, 3072),
    nn.GELU(),           # GELU 激活
    nn.Linear(3072, 768),
)

# ---- 方式二：函数式调用（在自定义 forward 中更灵活）----
import torch.nn.functional as F

class TransformerFFN(nn.Module):
    def __init__(self, d_model=768, d_ff=3072):
        super().__init__()
        self.linear1 = nn.Linear(d_model, d_ff)
        self.linear2 = nn.Linear(d_ff, d_model)

    def forward(self, x):
        x = self.linear1(x)
        x = F.gelu(x)          # 函数式调用
        x = self.linear2(x)
        return x

# ---- 常用激活函数的 PyTorch 对照表 ----
# Sigmoid:      nn.Sigmoid()      或 F.sigmoid() 或 torch.sigmoid()
# Tanh:         nn.Tanh()         或 F.tanh()    或 torch.tanh()

# Softmax:      nn.Softmax(dim=-1) 或 F.softmax(x, dim=-1)

# ---- 快速验证各激活函数输出 ----

]:
```

### 4.3 为什么 Transformer 用 GELU 而不是 ReLU

这是一个高频面试题，也是工程选型中的关键决策。原因有以下几个层面：

**1. 实验验证：GELU 在 NLP 任务上持续优于 ReLU**

BERT 论文（2018）和 GPT-2 论文（2019）的消融实验都发现，在相同模型规模和训练配置下，GELU 在下游 NLP 任务上比 ReLU 有稳定的提升（通常 0.5~1.5 个百分点）。

**2. 光滑性对 Transformer 更重要**

Transformer 的训练依赖精确的梯度信息（Adam 优化器需要可靠的一阶和二阶矩估计）。ReLU 在 $x=0$ 处的不可导点以及硬截断会在梯度中引入额外的噪声和不连续性。GELU 处处光滑，梯度流更加稳定。

对于 CNN 来说，ReLU 的稀疏性是一个优势（图像中大量信息是局部的，稀疏响应有助于特征选择）。但对于 NLP 来说，token 的每一个维度都可能携带语义信息，ReLU 将约 50% 的信号硬截断为 0 可能会丢失有用信息。

**3. 概率门控与 Dropout 的精神一致**

GELU 可以理解为"按照输入值在标准正态分布中的排名来决定通过比例"。这与 Transformer 广泛使用的 Dropout（按概率随机屏蔽）在理念上一致，GELU 是 Dropout 的确定性、输入依赖版本。

**4. 非单调性提供额外的表达能力**

GELU 在 $x \approx -0.75$ 附近有一个小的负输出区域，这意味着它是非单调的。虽然幅度很小，但这为网络提供了比单调函数（ReLU、Sigmoid、Tanh）更丰富的函数逼近能力。

**一句话总结**：Transformer 对梯度质量的要求比 CNN 更高，GELU 的光滑性和软门控特性更匹配这种需求。

### 4.4 LLaMA 的选择：SiLU + GLU

LLaMA（Meta, 2023）及其后续模型使用了一种更精巧的结构——**SwiGLU（Swish-Gated Linear Unit）**：

$$\text{SwiGLU}(x) = \text{SiLU}(x W_1) \otimes (x W_3)$$

其中 $\otimes$ 是逐元素乘法。这把 SiLU 激活函数和门控线性单元（GLU）结合在一起：
- $x W_1$ 经过 SiLU 激活后作为"门控信号"
- $x W_3$ 是待过滤的信息
- 两者逐元素相乘实现信息过滤

这种门控结构让激活函数不仅决定"通过多少"，还让网络学会"用什么信号来控制门"，表达能力更强。代价是多了一个投影矩阵 $W_3$。

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class SwiGLU_FFN(nn.Module):
    """LLaMA 风格的 FFN：使用 SwiGLU 激活"""
    def __init__(self, d_model=768, d_ff=2048):
        super().__init__()
        self.w1 = nn.Linear(d_model, d_ff, bias=False)  # 门控投影
        self.w2 = nn.Linear(d_ff, d_model, bias=False)   # 输出投影
        self.w3 = nn.Linear(d_model, d_ff, bias=False)   # 信息投影

    def forward(self, x):
        # SwiGLU: SiLU(x @ W1) * (x @ W3)
        gate = F.silu(self.w1(x))    # 门控信号
        value = self.w3(x)           # 待过滤信息
        hidden = gate * value        # 逐元素相乘
        return self.w2(hidden)       # 投影回原始维度

# 对比经典 FFN（BERT/GPT-2 风格）
class Classic_FFN(nn.Module):
    """经典 Transformer FFN：使用 GELU 激活"""
    def __init__(self, d_model=768, d_ff=3072):
        super().__init__()
        self.linear1 = nn.Linear(d_model, d_ff)
        self.linear2 = nn.Linear(d_ff, d_model)

    def forward(self, x):
        return self.linear2(F.gelu(self.linear1(x)))

# 参数量对比
classic = Classic_FFN(768, 3072)
swiglu = SwiGLU_FFN(768, 2048)  # d_ff 更小以保持总参数量相近

classic_params = sum(p.numel() for p in classic.parameters())
swiglu_params = sum(p.numel() for p in swiglu.parameters())
print(f"Classic FFN (GELU) 参数量: {classic_params:,}")
print(f"SwiGLU FFN (SiLU)  参数量: {swiglu_params:,}")
```

## 五、总结与速查表

### 激活函数对比一览

| 激活函数 | 公式 | 输出范围 | 导数范围 | 核心优势 | 核心劣势 | 典型用途 |
|---------|------|---------|---------|---------|---------|---------|
| Sigmoid | $\frac{1}{1+e^{-x}}$ | $(0, 1)$ | $(0, 0.25]$ | 概率输出 | 梯度消失 | 输出层（二分类） |
| Tanh | $\tanh(x)$ | $(-1, 1)$ | $(0, 1]$ | 零中心 | 梯度消失 | RNN/LSTM |
| ReLU | $\max(0, x)$ | $[0, +\infty)$ | $\{0, 1\}$ | 快速、简单 | 死神经元 | CNN、一般网络 |
| Leaky ReLU | $\max(\alpha x, x)$ | $(-\infty, +\infty)$ | $\{\alpha, 1\}$ | 无死神经元 | 需选 $\alpha$ | GAN 判别器 |
| GELU | $x \cdot \Phi(x)$ | $\approx[-0.17, +\infty)$ | 连续光滑 | 光滑、软门控 | 计算稍慢 | Transformer/NLP |
| Swish/SiLU | $x \cdot \sigma(x)$ | $\approx[-0.28, +\infty)$ | 连续光滑 | 光滑、自门控 | 计算稍慢 | LLaMA、EfficientNet |

### 核心要点回顾

1. **没有激活函数，深度网络等于浅层网络**。非线性是深度学习的基石。
2. **Sigmoid/Tanh 有梯度消失问题**，已不再用于隐藏层。
3. **ReLU 是现代深度学习的默认选择**，简单高效，但有死神经元风险。
4. **GELU 是 Transformer 的标配**，光滑性和软门控特性更适合注意力机制。
5. **输出层的选择由任务决定**，不是性能问题而是数学建模问题。
6. 激活函数的选择通常不是模型性能的瓶颈——网络结构、数据质量、训练策略的影响更大。**当你不确定时，用 ReLU（CNN）或 GELU（Transformer）不会错。**
