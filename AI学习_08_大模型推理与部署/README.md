# AI学习：08 大模型推理与部署

## 本模块导读

**学什么**：大语言模型从训练完成到实际可用，中间横亘着一道关键门槛——推理效率。本模块系统学习推理阶段的核心瓶颈（自回归解码为什么慢）、关键优化技术（KV Cache、PagedAttention、连续批处理、推测解码）、模型压缩（量化）、以及主流推理框架和部署方案。

**为什么学**：训练一个好模型只是故事的上半场。真正让模型在生产环境中以可接受的延迟和成本服务用户，需要推理优化和高效部署。这也是当前工业界大模型落地最核心的工程能力。

**学完能做什么**：
- 理解自回归推理的本质瓶颈（内存带宽受限，而非计算受限）
- 掌握 KV Cache 的原理和 PagedAttention 的优化思路
- 理解 INT8/INT4 量化的原理和各主流量化方案（GPTQ/AWQ/GGUF）
- 能选择合适的推理框架（vLLM / TGI / TensorRT-LLM / llama.cpp）
- 理解推测解码如何在不损失质量的前提下加速推理
- 理解 Prompt Engineering 为什么有效

**前置知识**：[05 GPT系列深度解析](../AI学习_05_GPT系列深度解析/README.md)

**预估学习时间**：6~8 小时

**重要程度**：★★★★

```
1_KVCache与推理优化 ──→ 2_量化技术 ──→ 3_推理框架与部署
     (核心瓶颈)          (模型压缩)       (实战框架)
```

第 1 章是本模块的基石：只有理解了自回归推理为什么慢、KV Cache 为什么关键，后面的量化和框架选型才有意义。

## 核心问题预览

在开始学习前，带着这些问题：

1. 一个 7B 模型在 GPU 上推理时，瓶颈到底是"算力不够"还是"内存带宽不够"？
2. KV Cache 为什么能大幅加速推理？缓存的具体是什么？
3. 把模型从 FP16 量化到 INT4，参数少了 4 倍，为什么精度损失可以很小？
4. vLLM 比朴素 HuggingFace 推理快 10~24 倍，优化了什么？
5. 推测解码如何做到"一次验证多个 token"而不损失生成质量？
