# KV Cache 与推理优化

> **前置知识**：[Transformer 架构](../AI学习_03_注意力机制与Transformer/3_Transformer架构.md)
>
> **本节目标**：理解大模型为什么逐 token 生成很慢，以及 KV Cache 如何避免重复计算。

---

## 1. 生成是一点点往外吐

GPT 这类模型通常每次只生成一个 token。

生成第 100 个 token 时，模型需要参考前面 99 个 token。生成第 101 个时，又要参考前面 100 个。

如果每次都把全部历史重新算一遍，就会非常浪费。

{{img:ch08_01_kv_cache_library}}

## 2. KV Cache：把已经算过的东西存起来

在注意力机制里，每个 token 会产生 Key 和 Value。

过去 token 的 K/V 一旦算好，在后续生成中可以反复使用。

KV Cache 的思路就是：

> 前面 token 的 K/V 别重复算，存起来，下一个 token 直接拿来用。

这能显著减少重复计算。

## 3. 它加速的是后续生成

KV Cache 对长回复特别重要。

第一次处理 Prompt 时，模型仍然要读完整输入。后面每生成一个新 token，只需要算新 token 的部分，再和缓存里的历史 K/V 做注意力。

所以它主要优化的是解码阶段。

## 4. 代价：更吃显存

缓存不是免费的。

上下文越长、并发请求越多，KV Cache 占用的显存越大。

这也是为什么长上下文服务很贵：不仅计算多，缓存也大。

## 5. 其他推理优化在做什么

除了 KV Cache，还有很多优化：

- 批处理：把多个请求合在一起算。
- PagedAttention：更高效管理 KV Cache。
- 推测解码：用小模型先猜，大模型验证。
- FlashAttention：更高效计算注意力。

它们目标类似：减少浪费，提高速度和吞吐。
