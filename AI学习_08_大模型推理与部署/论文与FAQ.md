# 论文与 FAQ

这个模块建议把系统论文当作“怎么省计算、怎么省显存、怎么提高吞吐”的案例看。

## 推荐了解的主题

| 主题 | 可以关注什么 |
|------|--------------|
| KV Cache | 为什么自回归生成可以复用历史 |
| FlashAttention | 注意力计算如何更省显存 |
| PagedAttention / vLLM | KV Cache 如何像分页内存一样管理 |
| 量化方法 | 低比特表示如何平衡质量和成本 |
| 推测解码 | 小模型猜、大模型验如何加速 |

## FAQ

### KV Cache 会让模型更聪明吗？

不会。它主要提升推理效率，不改变模型能力。

### 量化一定会降低效果吗？

可能会，但好的量化方法能把损失控制得很小。越低比特，风险越高。

### 推理框架是不是只影响速度？

主要影响速度、吞吐、显存和成本。间接也会影响用户体验和可用性。

### 长上下文为什么贵？

因为注意力计算和 KV Cache 都会随上下文增长而增加，尤其在高并发时成本明显。
