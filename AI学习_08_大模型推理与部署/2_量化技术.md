## 直觉与概述

### 量化是什么？

想象你有一幅高清照片（每个像素用 24 bit 表示颜色），你把它转换成 GIF 格式（每个像素只用 8 bit）。文件大小变成原来的三分之一，画质有一点点损失，但大多数时候你几乎看不出区别。

**模型量化**做的事情本质上一样：把模型权重从高精度（FP16，每个数 16 bit）压缩到低精度（INT8 每个数 8 bit，INT4 每个数 4 bit），从而：

```
FP16 → INT8:  模型大小减半，推理速度提升 ~2x
FP16 → INT4:  模型大小减到 1/4，推理速度提升 ~3x
```

### 为什么量化对大模型如此重要？

回忆上一章的结论：**大模型推理的瓶颈是内存带宽**。每生成一个 token，都需要把整个模型权重从显存搬到计算单元。

```
LLaMA-2 7B:
  FP16 权重: 14 GB  → 加载一次需 14GB / 2TB/s = 7ms
  INT4 权重: 3.5 GB → 加载一次需 3.5GB / 2TB/s = 1.75ms
  → 理论加速 4x（实际约 2~3x，因为有反量化开销）
```

量化不只是"省显存"，更重要的是**直接提高推理速度**——因为需要搬运的数据更少了。

### 关键直觉：大模型权重的冗余性

为什么量化损失可以很小？因为大模型的权重分布有两个特点：

1. **大部分权重集中在零附近**：权重的分布近似正态/拉普拉斯分布，大量值接近 0
2. **少量异常值（outlier）**：极少数权重的绝对值远大于其他值（Dettmers 2022 发现约 0.1% 的权重是 outlier）

```
典型权重分布:
                        ▓
                       ▓▓▓
                      ▓▓▓▓▓
                     ▓▓▓▓▓▓▓
                   ▓▓▓▓▓▓▓▓▓▓
              ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓
  ──────────────────────────────────────
  -0.5    -0.25     0      0.25     0.5
                  ↑ 大部分值在这个范围

  但偶尔出现 outlier: -3.2, +4.1（绝对值是均值的10倍+）
```

处理好这些 outlier 是量化方法的核心挑战。

## 严谨定义与原理

### 1. 量化的基本数学

#### 均匀量化（Uniform Quantization）

将浮点数映射到整数的标准方法：

**量化（浮点 → 整数）**：

$$x_q = \text{round}\left(\frac{x}{s}\right) + z$$

**反量化（整数 → 浮点）**：

$$\hat{x} = s \cdot (x_q - z)$$

其中：
- $s$（scale，缩放因子）：决定量化的"精度"
- $z$（zero-point，零点偏移）：决定量化的"偏移"
- $x_q$：量化后的整数值

#### 对称量化 vs 非对称量化

| 类型 | 零点 $z$ | 适用场景 | 特点 |
|------|----------|---------|------|
| 对称量化 | $z = 0$ | 权重（分布对称） | 计算更简单 |
| 非对称量化 | $z \neq 0$ | 激活值（可能不对称，如 ReLU 后） | 表示范围更灵活 |

**对称量化**（以 INT8 为例，范围 [-127, 127]）：

$$s = \frac{\max(|x|)}{127}, \quad x_q = \text{round}\left(\frac{x}{s}\right)$$

**量化误差**：

$$\epsilon = x - \hat{x} = x - s \cdot \text{round}\left(\frac{x}{s}\right)$$

误差上界为 $|s|/2$，即半个量化步长。

#### 分组量化（Group Quantization）

不对整个张量使用一个 scale，而是把张量分成多个组，每组独立计算 scale：

```
整张量量化（per-tensor）: 整个权重矩阵共用一个 scale
  → 如果有一个 outlier = 100，其他都在 [-1, 1]
  → scale 被 outlier 拉大，其他值的精度被严重牺牲

分组量化（per-group，如 group_size=128）:
  → 每 128 个权重独立计算 scale
  → outlier 只影响它所在的组，不波及其他
  → 精度显著提升，代价是多存一些 scale 值
```

GPTQ 和 AWQ 都默认使用 group_size=128 的分组量化。

### 2. PTQ vs QAT — 两种量化路线

| | PTQ (Post-Training Quantization) | QAT (Quantization-Aware Training) |
|---|---|---|
| 全称 | 训练后量化 | 量化感知训练 |
| 时机 | 模型训练完成后直接量化 | 训练过程中模拟量化误差 |
| 计算成本 | 低（几小时，只需少量校准数据） | 高（需要重新训练或微调） |
| 精度 | INT8 几乎无损，INT4 有少量损失 | 通常优于 PTQ |
| 适用场景 | 大模型（训练成本太高） | 追求极致精度的场景 |
| 代表方法 | GPTQ, AWQ, LLM.int8() | QLoRA (某种程度上) |

**对于大语言模型，PTQ 是主流**：因为重新训练一个 70B 模型的成本实在太高，而 PTQ 已经能达到很好的效果。

### 3. LLM.int8() — 混合精度量化的先驱

Dettmers et al. (2022) 发现了一个关键现象：**Transformer 中存在约 0.1% 的"涌现特征"（emergent features），它们的激活值比其他特征大 100 倍**。

如果直接对所有特征做 INT8 量化，这些 outlier 会严重破坏量化精度。

**LLM.int8() 的解决方案——混合精度分解**：

```
矩阵乘法 Y = XW 分解为:

1. 识别 outlier 维度: 激活值中绝对值 > 6.0 的维度（约占 0.1%）

2. 分开计算:
   - 正常维度 (99.9%): INT8 × INT8 → INT8 矩阵乘法（高效）
   - Outlier 维度 (0.1%): FP16 × FP16 → FP16 矩阵乘法（保精度）

3. 合并结果: Y = Y_int8 + Y_fp16
```

```
         正常维度 (INT8)              Outlier 维度 (FP16)
  X_normal × W_normal = Y1    +   X_outlier × W_outlier = Y2
  ┌─────────────────────┐         ┌──────────┐
  │ 0.1  -0.3   0.5 ... │ INT8    │ 45.2 ... │ FP16
  │ -0.2  0.4  -0.1 ... │ ×      │ -38.1 .. │ ×
  │ ...                  │ INT8    │ ...      │ FP16
  └─────────────────────┘         └──────────┘
     99.9% 的维度                   0.1% 的维度
```

效果：**在 175B 模型上实现零精度损失的 INT8 推理**，显存减半。

### 4. GPTQ — 逐层最优量化

Frantar et al. (2022) 提出 GPTQ，这是目前最广泛使用的 LLM 量化方法之一。

**核心思想**：逐列量化权重矩阵，每量化一列后，调整剩余列来补偿量化误差。

**目标**：最小化量化前后输出的差异

$$\arg\min_{\hat{W}} ||WX - \hat{W}X||_2^2$$

其中 $W$ 是原始权重，$\hat{W}$ 是量化后的权重，$X$ 是校准数据的激活值。

**算法流程**（简化版）：

```
输入: 权重矩阵 W (shape: [out, in]), 校准数据的 Hessian H = 2XX^T

for col = 0, 1, ..., in-1:
    1. 量化当前列: w_hat[col] = quantize(W[col])
    2. 计算量化误差: delta = W[col] - w_hat[col]
    3. 用误差补偿后续列: W[col+1:] -= delta × H[col, col+1:] / H[col, col]
       （关键: 把当前列的误差"分摊"到还没量化的列上）
```

**GPTQ 的优势**：
- 速度快：7B 模型量化只需 ~4 分钟（GPU）
- 精度高：利用 Hessian 信息做最优补偿
- INT4 量化后精度损失很小（perplexity 增加 < 0.5）

### 5. AWQ — 保护重要权重

Lin et al. (2023) 观察到：**不是所有权重都同等重要——少数"显著"通道的量化误差对模型输出影响特别大**。

**核心发现**：

```
权重的重要性 ∝ 对应激活值的大小

如果某个输入通道的激活值始终很大（在校准数据上）:
  → 该通道的权重即使有小的量化误差，也会被放大
  → 应该"保护"这些权重
```

**AWQ 的做法——Per-channel Scaling**：

```
原始矩阵乘法: Y = XW

AWQ 的变换:
  Y = X · W = (X · diag(s)^{-1}) · (diag(s) · W)
            = X' · W'

其中 s 是 per-channel 的缩放因子:
  - 对重要通道: s 较大 → W' 中该通道权重被放大 → 量化误差相对更小
  - 对不重要通道: s 较小 → W' 中该通道权重正常量化
```

直觉：给重要权重乘一个大数使其更"圆整"（easier to quantize），同时给对应的输入除以同一个数以保持数学等价。

**AWQ vs GPTQ 对比**：

| 特性 | GPTQ | AWQ |
|------|------|-----|
| 核心思路 | 量化后补偿误差 | 量化前保护重要权重 |
| 需要校准数据 | 是（~128 条） | 是（~128 条） |
| 量化速度 | 7B ~4分钟 | 7B ~10分钟 |
| INT4 精度 | 优秀 | 略优于 GPTQ（尤其在更小 group） |
| 推理速度 | 依赖后端 | 通常略快（权重结构更友好） |
| 生态 | 非常成熟 | 成熟且在快速增长 |

### 6. GGUF — CPU 友好的量化格式

GGUF（GPT-Generated Unified Format）是 llama.cpp 生态使用的量化格式，专为 **CPU 推理** 设计。

**GGUF 量化类型命名规则**：

```
Q4_K_M  →  Q: Quantized
            4: 4 bit
            K: K-quant (改进的量化方案)
            M: Medium (中等精度/大小平衡)

常见类型:
  Q2_K    - 2 bit, 极致压缩，精度损失较大
  Q4_0    - 4 bit, 基础量化
  Q4_K_M  - 4 bit, K-quant Medium, 推荐日常使用
  Q5_K_M  - 5 bit, K-quant Medium, 精度更好
  Q6_K    - 6 bit, 接近 FP16 精度
  Q8_0    - 8 bit, 几乎无损
```

**GGUF 的 K-Quant 方案**——"不同层用不同量化位数"：

```
注意力层的 Q/K/V 投影: 精度敏感 → 用更高位数 (如 6 bit)
FFN 的 down_proj:       精度不太敏感 → 用更低位数 (如 4 bit)
Embedding 层:           对量化最敏感 → 保持 FP16 或用 8 bit

→ 整体平均 ~4.5 bit，但精度远好于均匀 4 bit 量化
```

### 7. 各量化方法对比总览

| 方法 | 位数 | 速度 | 精度保持 | 硬件 | 典型使用场景 |
|------|------|------|---------|------|-------------|
| LLM.int8() | 8+16混合 | 中 | 极好 | GPU | 首次尝试量化 |
| GPTQ | 4/3/2 | 快 | 好 | GPU | 服务器部署 |
| AWQ | 4 | 快 | 好+ | GPU | 服务器部署（精度优先） |
| GGUF (Q4_K_M) | 4~6混合 | 中 | 好 | CPU/GPU | 本地部署、边缘设备 |
| bitsandbytes NF4 | 4 | 快 | 好 | GPU | QLoRA 训练 |

## Python 代码示例

### 示例 1：手动实现对称量化与反量化

```python
import torch
import numpy as np

def symmetric_quantize(tensor, n_bits=8):
    """
    对称均匀量化: 将 FP32 张量量化为 n_bits 整数。

    参数:
        tensor: 原始浮点张量
        n_bits: 量化位数 (8=INT8, 4=INT4)
    返回:
        quantized: 量化后的整数张量
        scale: 缩放因子
    """
    # 对称量化: 范围 [-2^(n-1)+1, 2^(n-1)-1]
    qmax = 2 ** (n_bits - 1) - 1  # INT8: 127, INT4: 7
    qmin = -qmax                   # INT8: -127, INT4: -7

    # 计算 scale: 使最大绝对值映射到 qmax
    abs_max = tensor.abs().max()
    scale = abs_max / qmax

    # 量化: round(x / scale), 并裁剪到合法范围
    quantized = torch.clamp(torch.round(tensor / scale), qmin, qmax).to(torch.int8)

    return quantized, scale

def dequantize(quantized, scale):
    """反量化: 整数 → 浮点"""
    return quantized.float() * scale

# ========== 演示: 不同位数的量化效果 ==========
torch.manual_seed(42)
# 模拟一个权重矩阵（正态分布 + 少量 outlier）
weight = torch.randn(256, 256) * 0.1
# 人为添加 outlier（模拟 Transformer 中的涌现特征）
weight[0, 0] = 3.5
weight[100, 50] = -2.8

print(f"原始权重: shape={weight.shape}, 范围=[{weight.min():.3f}, {weight.max():.3f}]")
print(f"原始大小: {weight.numel() * 4 / 1024:.1f} KB (FP32)\n")

for n_bits in [8, 4, 2]:
    q, scale = symmetric_quantize(weight, n_bits)
    w_hat = dequantize(q, scale)

    # 量化误差
    mse = ((weight - w_hat) ** 2).mean().item()
    max_err = (weight - w_hat).abs().max().item()
    size_kb = weight.numel() * n_bits / 8 / 1024

    print(f"INT{n_bits}: MSE={mse:.6f}, MaxErr={max_err:.4f}, "
          f"大小={size_kb:.1f} KB, 压缩比={4*8/n_bits:.0f}x")

# 输出示例:
# INT8: MSE=0.000001, MaxErr=0.0138, 大小=64.0 KB, 压缩比=4x   ← 几乎无损
# INT4: MSE=0.000094, MaxErr=0.2500, 大小=32.0 KB, 压缩比=8x   ← 很小的损失
# INT2: MSE=0.004700, MaxErr=1.1667, 大小=16.0 KB, 压缩比=16x  ← 明显损失
```

### 示例 2：分组量化 vs 全张量量化

```python
import torch

def per_tensor_quantize(tensor, n_bits=4):
    """全张量量化: 整个张量共用一个 scale"""
    qmax = 2 ** (n_bits - 1) - 1
    scale = tensor.abs().max() / qmax
    quantized = torch.clamp(torch.round(tensor / scale), -qmax, qmax)
    return quantized * scale, scale

def per_group_quantize(tensor, n_bits=4, group_size=128):
    """
    分组量化: 每 group_size 个元素独立计算 scale。
    这是 GPTQ/AWQ 使用的核心策略。
    """
    orig_shape = tensor.shape
    # 展平并按 group_size 分组
    flat = tensor.reshape(-1)
    n = flat.numel()
    # 补齐到 group_size 的倍数
    pad = (group_size - n % group_size) % group_size
    if pad > 0:
        flat = torch.cat([flat, torch.zeros(pad)])
    groups = flat.reshape(-1, group_size)

    qmax = 2 ** (n_bits - 1) - 1
    # 每组独立计算 scale
    scales = groups.abs().amax(dim=1, keepdim=True) / qmax  # [n_groups, 1]
    quantized = torch.clamp(torch.round(groups / (scales + 1e-10)), -qmax, qmax)
    dequantized = quantized * scales

    # 恢复原始形状
    result = dequantized.reshape(-1)[:n].reshape(orig_shape)
    return result, scales.squeeze()

# ========== 对比实验 ==========
torch.manual_seed(42)
# 模拟有 outlier 的权重
weight = torch.randn(1024, 1024) * 0.02
# 在某些位置注入 outlier
for i in range(10):
    r, c = torch.randint(0, 1024, (2,))
    weight[r, c] = torch.randn(1) * 5.0  # 250 倍于平均值的 outlier

print("INT4 量化对比:")
print(f"原始权重范围: [{weight.min():.2f}, {weight.max():.2f}]")
print(f"原始权重 std:  {weight.std():.4f}\n")

# 全张量量化
w_tensor, s = per_tensor_quantize(weight, n_bits=4)
mse_tensor = ((weight - w_tensor) ** 2).mean().item()

# 分组量化
for gs in [128, 64, 32]:
    w_group, scales = per_group_quantize(weight, n_bits=4, group_size=gs)
    mse_group = ((weight - w_group) ** 2).mean().item()
    overhead = scales.numel() * 2 / (weight.numel() * 0.5) * 100  # scale 的额外开销
    print(f"Group Size={gs:>4d}: MSE={mse_group:.8f}, "
          f"误差比全张量低 {mse_tensor/mse_group:.1f}x, "
          f"scale 额外开销 {overhead:.1f}%")

print(f"Per-Tensor:     MSE={mse_tensor:.8f} (baseline)")

# 典型结果:
# Group=128:  MSE 约为 per-tensor 的 1/50,  额外开销 ~3%
# Group=64:   MSE 约为 per-tensor 的 1/90,  额外开销 ~6%
# Group=32:   MSE 约为 per-tensor 的 1/150, 额外开销 ~12%
# 结论: group_size=128 是精度和开销的最佳平衡点
```

### 示例 3：GPTQ 核心思想——误差补偿的直觉演示

```python
import torch

def naive_quantize_matrix(W, n_bits=4):
    """朴素量化: 直接对每列独立量化，不做误差补偿"""
    qmax = 2 ** (n_bits - 1) - 1
    W_hat = torch.zeros_like(W)
    for col in range(W.shape[1]):
        s = W[:, col].abs().max() / qmax
        q = torch.clamp(torch.round(W[:, col] / (s + 1e-10)), -qmax, qmax)
        W_hat[:, col] = q * s
    return W_hat

def gptq_like_quantize(W, H, n_bits=4):
    """
    GPTQ 风格量化: 逐列量化 + 误差补偿。

    核心: 每量化一列，把误差"推"给后续列，
    使得 ||WX - W_hat X||^2 最小化（而非 ||W - W_hat||^2）。

    参数:
        W: 权重矩阵 [out_features, in_features]
        H: Hessian 矩阵 = X @ X.T / n_samples [in, in]
           反映每个输入维度的"重要性"
    """
    W_hat = W.clone()
    qmax = 2 ** (n_bits - 1) - 1
    n_cols = W.shape[1]
    H_diag = H.diag()

    for col in range(n_cols):
        w_col = W_hat[:, col]

        # 1. 量化当前列
        s = w_col.abs().max() / qmax
        q = torch.clamp(torch.round(w_col / (s + 1e-10)), -qmax, qmax)
        w_quantized = q * s

        # 2. 计算量化误差
        delta = w_col - w_quantized
        W_hat[:, col] = w_quantized

        # 3. 误差补偿: 把误差分摊到后续列
        if col < n_cols - 1 and H_diag[col] > 1e-6:
            # 按 Hessian 信息分配误差
            compensation = delta.unsqueeze(1) * H[col, col+1:].unsqueeze(0) / H_diag[col]
            W_hat[:, col+1:] += compensation

    return W_hat

# ========== 对比实验 ==========
torch.manual_seed(42)
out_dim, in_dim = 256, 256
W = torch.randn(out_dim, in_dim) * 0.1

# 模拟校准数据生成 Hessian
n_samples = 128
X = torch.randn(in_dim, n_samples) * 0.5
H = (X @ X.T) / n_samples + 1e-4 * torch.eye(in_dim)  # 加正则化

# 朴素量化
W_naive = naive_quantize_matrix(W, n_bits=4)
# GPTQ 风格量化
W_gptq = gptq_like_quantize(W, H, n_bits=4)

# 评估: 计算量化前后输出的差异 ||WX - W_hat X||^2
X_test = torch.randn(in_dim, 1000)
Y_orig = W @ X_test

err_naive = ((Y_orig - W_naive @ X_test) ** 2).mean().item()
err_gptq  = ((Y_orig - W_gptq @ X_test) ** 2).mean().item()

print("INT4 量化效果对比:")
print(f"  朴素量化 输出 MSE: {err_naive:.6f}")
print(f"  GPTQ量化 输出 MSE: {err_gptq:.6f}")
print(f"  GPTQ 误差降低: {err_naive / err_gptq:.1f}x")
print()
print("关键: GPTQ 最小化的是 '输出误差' 而非 '权重误差'")
print("      它利用校准数据的 Hessian 来决定误差补偿的方向和大小")
```

### 示例 4：AWQ 核心思想——重要通道保护演示

```python
import torch

def compute_channel_importance(X_calibration):
    """
    计算每个输入通道的重要性（基于激活值的平均幅度）。
    AWQ 的核心观察: 激活值大的通道 → 对应权重更重要。
    """
    # X_calibration: [n_samples, in_features]
    return X_calibration.abs().mean(dim=0)  # [in_features]

def awq_like_quantize(W, importance, n_bits=4, alpha=0.5):
    """
    AWQ 风格量化: 对重要通道施加 per-channel scaling 后再量化。

    Y = XW = (X * diag(s)^{-1}) * (diag(s) * W)
    s 的选择使得重要通道的权重更容易量化。
    """
    in_features = W.shape[1]
    qmax = 2 ** (n_bits - 1) - 1

    # 计算 per-channel scale: 重要通道 scale 更大
    # s = importance^alpha, alpha 是超参数 (0~1)
    s = importance.pow(alpha).clamp(min=1e-5)

    # 缩放权重: W' = W * diag(s)
    W_scaled = W * s.unsqueeze(0)

    # 对缩放后的权重做量化
    W_hat_scaled = torch.zeros_like(W_scaled)
    for col in range(in_features):
        scale = W_scaled[:, col].abs().max() / qmax
        q = torch.clamp(torch.round(W_scaled[:, col] / (scale + 1e-10)), -qmax, qmax)
        W_hat_scaled[:, col] = q * scale

    # 反缩放: W_hat = W_hat_scaled / diag(s)
    W_hat = W_hat_scaled / s.unsqueeze(0)

    return W_hat

# ========== 对比 ==========
torch.manual_seed(42)
out_dim, in_dim = 256, 256

# 创建权重矩阵
W = torch.randn(out_dim, in_dim) * 0.1

# 校准数据: 某些通道的激活值明显更大（模拟 outlier 特征）
X_cal = torch.randn(128, in_dim) * 0.5
# 人为让某些通道的激活值很大
important_channels = [10, 50, 100, 200]
for ch in important_channels:
    X_cal[:, ch] *= 20.0  # 这些通道的激活值是其他通道的 20 倍

importance = compute_channel_importance(X_cal)

# 朴素量化
qmax = 7  # INT4
W_naive = torch.zeros_like(W)
for col in range(in_dim):
    s = W[:, col].abs().max() / qmax
    q = torch.clamp(torch.round(W[:, col] / (s + 1e-10)), -qmax, qmax)
    W_naive[:, col] = q * s

# AWQ 量化
W_awq = awq_like_quantize(W, importance, n_bits=4)

# 评估输出误差
X_test = torch.randn(1000, in_dim) * 0.5
for ch in important_channels:
    X_test[:, ch] *= 20.0

Y_orig = X_test @ W.T
err_naive = ((Y_orig - X_test @ W_naive.T) ** 2).mean().item()
err_awq   = ((Y_orig - X_test @ W_awq.T) ** 2).mean().item()

print("INT4 量化效果对比（有 outlier 通道）:")
print(f"  朴素量化 输出 MSE: {err_naive:.6f}")
print(f"  AWQ 量化 输出 MSE: {err_awq:.6f}")
print(f"  AWQ 误差降低: {err_naive / err_awq:.1f}x")
print(f"\n重要通道 indices: {important_channels}")
print(f"重要通道 importance: {importance[important_channels].tolist()}")
print(f"其他通道平均 importance: {importance.mean():.3f}")
```

## 工程师视角

### 量化方案选型指南

```
你要量化一个大模型？
  │
  ├─ 在 GPU 上部署?
  │    ├─ 追求吞吐量 → GPTQ + vLLM (最成熟的组合)
  │    ├─ 追求精度   → AWQ + vLLM (精度略优)
  │    └─ 首次尝试   → bitsandbytes INT8 (最简单, pip install 即用)
  │
  └─ 在 CPU / 笔记本 / 手机?
       └─ GGUF + llama.cpp (Q4_K_M 最推荐)
```

### 量化精度损失的真实影响

社区大量测试表明：

| 量化 | Perplexity 变化 | 实际体感 |
|------|-----------------|---------|
| FP16 → INT8 | < 0.1 | 几乎无法区分 |
| FP16 → INT4 (GPTQ/AWQ) | 0.1~0.5 | 大部分任务无影响，复杂推理可能微降 |
| FP16 → INT4 (GGUF Q4_K_M) | 0.2~0.8 | 日常对话几乎无影响 |
| FP16 → INT3 | 1.0~3.0 | 明显可感知的质量下降 |
| FP16 → INT2 | > 5.0 | 基本不可用 |

**实践建议**：INT4 (4-bit) 是当前精度和效率的最佳平衡点。INT3 以下只适合对质量不敏感的场景。

### 量化的隐藏坑

**坑 1: 量化对不同任务的影响不同**
```
量化对以下任务影响较小:
  - 日常对话、翻译、摘要
  - 情感分析、简单分类

量化对以下任务影响较大:
  - 数学推理（尤其是多步推理）
  - 代码生成（语法错误增加）
  - 知识密集型 QA（罕见知识可能丢失）
```

**坑 2: 量化位数不等于"等效模型大小"**
```
错误直觉: "7B INT4 ≈ 1.75B FP16 的效果"
正确理解: 7B INT4 ≈ 7B FP16 的 95%+ 效果
  → 参数量不变，只是每个参数的精度降低
  → 模型的知识和能力大部分保留
```

**坑 3: KV Cache 量化是额外的优化维度**
```
上面讨论的都是"权重量化"。KV Cache 也可以量化:
  - KV Cache INT8 量化: 减少一半 KV Cache 显存
  - 精度影响比权重量化大（因为 KV 在注意力计算中直接参与）
  - vLLM 支持 FP8 KV Cache，是平衡点
```

## 本章小结

| 概念 | 一句话总结 |
|------|-----------|
| 量化 | 将模型从 FP16 压缩到 INT8/INT4，减少内存并加速推理 |
| 对称量化 | scale = max(\|x\|) / qmax，零点为 0，适合权重 |
| 分组量化 | 每 group_size 个元素独立计算 scale，精度远优于全张量量化 |
| PTQ vs QAT | PTQ 训练后直接量化（主流），QAT 训练中模拟量化（精度更好但成本高） |
| LLM.int8() | 混合精度：0.1% 的 outlier 用 FP16，其余用 INT8 |
| GPTQ | 逐列量化 + Hessian 误差补偿，最小化输出误差 |
| AWQ | per-channel scaling 保护重要通道，精度略优于 GPTQ |
| GGUF | llama.cpp 格式，不同层用不同位数，CPU 友好 |
| INT4 | 当前精度与效率的最佳平衡点 |
