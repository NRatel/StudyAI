# AI 及大模型原理 — 整体知识体系思维导图

这张图只负责帮你定位：每一章在整条 AI 主线里解决什么问题。第一次读时不必把所有分支都记住，先顺着主干走。

```text
AI 及大模型原理
│
├─ 01 神经网络基础
│  └─ 模型怎样从“答错”中调参数
│     输入 → 输出 → 损失 → 反向传播 → 参数更新
│
├─ 02 经典网络架构
│  └─ Transformer 之前，人们怎样处理图像和序列
│     CNN 看局部图案，RNN/LSTM 处理顺序，Seq2Seq 先读再写
│
├─ 03 注意力机制与 Transformer
│  └─ 词和词之间怎样直接建立关系
│     Attention 决定重点看谁，Transformer 把它堆成可扩展结构
│
├─ 04 预训练语言模型演进
│  └─ 文字怎样变成模型能理解的向量
│     One-hot → Embedding → 上下文表示 → BERT / GPT
│
├─ 05 GPT 系列深度解析
│  └─ 为什么“预测下一个 token”能长出通用能力
│     自回归生成 → Scaling Laws → 涌现 → 指令微调 → 对齐
│
├─ 06 生成模型与扩散模型
│  └─ AI 为什么能生成新图片和新内容
│     学数据分布，扩散模型通过“加噪再去噪”生成
│
├─ 07 大模型训练技术
│  └─ 大模型为什么训练贵、难、需要系统工程
│     分布式训练、混合精度、数据工程、对齐训练
│
├─ 08 大模型推理与部署
│  └─ 模型上线后为什么还要优化速度和成本
│     KV Cache 复用历史，量化压缩模型，推理框架提高吞吐
│
├─ 09 RAG 与应用架构
│  └─ 模型回答前怎样先查资料
│     分块 → Embedding → 检索 → 重排 → 带证据生成
│
├─ 10 多模态模型
│  └─ 模型怎样同时处理文字、图片、语音和视频
│     不同模态先变成向量或 token，再接入语言模型
│
├─ 11 推理与思维链
│  └─ 复杂问题为什么要拆步骤、多检查
│     分步推理、多路径尝试、验证器、测试时计算
│
├─ 12 Agent 与工具使用
│  └─ 模型怎样从“回答问题”走向“完成任务”
│     观察 → 计划 → 调工具 → 看反馈 → 更新状态 → 继续
│
└─ 13 长文本与高效架构
   └─ 上下文更长、模型更大时怎样控制成本
      长上下文、MoE、高效注意力、SSM / Mamba 等路线
```

## 三条主线

| 主线 | 包含章节 | 解决的问题 |
|------|----------|------------|
| 模型怎么学 | 01 → 03 → 04 → 05 | 从神经网络训练到 GPT 生成能力 |
| 模型怎么扩展 | 06 → 10 → 11 → 13 | 从生成、多模态、推理到长文本和高效架构 |
| 模型怎么落地 | 07 → 08 → 09 → 12 | 从训练、推理、外部知识到 Agent 工具使用 |
