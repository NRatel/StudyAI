SFT / RLHF / DPO 速查对比表

在进入细节之前，先建立全局印象。下表对比本章涉及的三种核心对齐方法：

维度SFT（监督微调）RLHF（PPO）DPO（直接偏好优化）
一句话用高质量问答对教模型"好回答长什么样"用人类偏好排序 + 强化学习教模型"哪个更好"用偏好数据直接优化，跳过奖励模型
输入数据(指令, 优质回答) 对(指令, 回答A > 回答B) 排序对同 RLHF（偏好对）
模型数量1 个4 个（策略 + 参考 + 奖励 + 价值）2 个（策略 + 参考）
训练稳定性稳定较差（PPO 超参敏感）稳定
实现复杂度低高（需 RL 框架）低（标准监督学习）
优化粒度token 级别回答级别回答级别
适用场景必做的第一步资源充足、追求极致多数场景的首选

三者不是互相替代关系，而是典型的三阶段流水线：预训练 -> SFT -> RLHF 或 DPO。以下各节将逐步展开。

一、直觉与概述

1.1 预训练模型的"人格缺陷"

经过海量文本预训练后，GPT 拥有了惊人的语言能力。但它的训练目标是"预测下一个 token"，而不是"成为有用的助手"，这导致三类典型失败：

失败类型例子根因
不遵循指令问"用三句话总结"，它写了三页训练目标是续写，不是服从
编造事实一本正经地胡说八道训练目标是"像真的"，不是"是真的"
有害输出给出危险建议、歧视性内容训练数据里有大量有害文本

预训练模型 = 一个博览群书的天才，但没有职业素养。 什么都知道一点，但可能胡说八道、说出不当言论，因为他追求的是"听起来像"而不是"是对的"。

1.2 对齐的目标：HHH 框架

"对齐"（Alignment）指让 AI 的行为与人类意图和价值观对齐。Anthropic 提出的 HHH 框架：

  • Helpful（有用）：理解用户意图，给出高质量回答
  • Honest（诚实）：不编造事实，对不确定的事情坦诚
  • Harmless（无害）：拒绝有害请求，不输出歧视、暴力内容

1.3 对齐的三阶段流程

从 InstructGPT（OpenAI, 2022）确立的经典流程：

阶段 1：预训练 → 学习语言能力（能力强，但不听话）
       ↓
阶段 2：SFT（监督微调）→ 学习遵循指令（听话了，但不够好）
       ↓
阶段 3：RLHF / DPO → 学习人类偏好（有用、诚实、无害）

关键洞察：每阶段解决不同问题——预训练解决"能力"，SFT 解决"格式和服从"，RLHF/DPO 解决"质量和安全"。跳过任何一步效果都会大打折扣。

二、SFT：监督微调

2.1 核心思想

SFT 极其朴素：收集高质量的（问题, 回答）对，用语言建模目标微调预训练模型。

预训练SFT
数据来源互联网爬取人工编写的对话数据
数据规模数万亿 token数万~数十万条
数据质量参差不齐严格质量控制
训练目标预测下一个 token（全文）预测下一个 token（仅回答部分）

2.2 数据格式与关键细节

SFT 数据采用对话格式（如 ChatML）。关键细节：计算损失时只对 assistant 回答部分计算交叉熵，system 和 user 部分被 mask 掉——我们只想让模型学"如何回答"。

<|im_start|>system
You are a helpful assistant.<|im_end|>
<|im_start|>user
解释什么是梯度消失问题。<|im_end|>
<|im_start|>assistant
梯度消失问题是指...（详细解释）<|im_end|>

2.3 数据质量 >> 数据数量

  • InstructGPT：约 13,000 条高质量数据就能显著改善模型行为
  • LIMA（Meta, 2023）：仅 1,000 条精心筛选数据，效果接近 GPT-4
  • 工程经验：一条专家精心编写的回答，抵得上 100 条随意标注的数据

2.4 SFT 的局限性

  1. 天花板是标注者能力：模型只能学到标注者写出的回答质量
  2. 没有学到"比较"能力：SFT 教"好回答长什么样"，没教"这个比那个好在哪"

类比：SFT 像给学生看范文，RLHF 像让老师批改作文并打分——后者能让学生理解"好"和"更好"的区别。

三、RLHF：基于人类反馈的强化学习

3.1 为什么需要强化学习？

"好回答"很难用数学公式定义。 交叉熵能告诉模型"正确的下一个 token"，但无法告诉它"这个回答比那个更有帮助"。RLHF 的核心想法：让人类告诉我们什么是好的，训练模型学习这种偏好，再用它作为奖励信号优化语言模型。

3.2 完整流程（三步骤，四个模型）

步骤 1：收集人类偏好数据
  对同一个 prompt，SFT 模型生成多个回答 → 人类排序（A > B > C）
       ↓
步骤 2：训练奖励模型（Reward Model）
  输入 (prompt, response) → 输出标量分数，让人类偏好的回答得分更高
       ↓
步骤 3：PPO 优化策略模型
  策略模型生成回答 → 奖励模型打分 → PPO 更新策略 → KL 惩罚防止偏离太远

3.3 奖励模型训练

奖励模型 $r_\phi$ 在 SFT 模型基础上换一个线性头输出标量分数。对每个偏好对 $(y_w, y_l)$，使用 Bradley-Terry 模型训练：

$$\mathcal{L}_{\text{RM}} = -\mathbb{E}_{(x, y_w, y_l)}\left[\log \sigma(r_\phi(x, y_w) - r_\phi(x, y_l))\right]$$

直觉：让奖励模型给"好回答"打更高分，给"差回答"打更低分，分数差足够大。

3.4 PPO 优化

优化策略模型 $\pi_\theta$，使其生成高奖励回答，同时不偏离参考模型 $\pi_{\text{ref}}$ 太远：

$$\max_{\pi_\theta} \mathbb{E}_{x,\, y \sim \pi_\theta}\left[r_\phi(x, y)\right] - \beta \cdot D_{\text{KL}}\left[\pi_\theta \| \pi_{\text{ref}}\right]$$
  • 第一项：最大化奖励（"让回答更好"）
  • 第二项：KL 惩罚（"别跑太远"），防止 reward hacking
  • $\beta$：平衡系数

为什么需要 KL 惩罚？ 没有约束时，策略模型会"黑掉"奖励模型——找到高分但无意义的回答模式。KL 惩罚是安全绳。

3.5 RLHF 的四个模型

PPO 训练阶段内存中需同时维护：

模型角色是否更新
策略模型 $\pi_\theta$生成回答是
参考模型 $\pi_{\text{ref}}$SFT 冻结副本，计算 KL否
奖励模型 $r_\phi$给回答打分否
价值模型 $V_\psi$估算状态价值（PPO 需要）是

7B 模型的 RLHF 训练约需 $7B \times 4 = 28B$ 参数的显存（加优化器状态更多）。这是 RLHF 实际训练中最大的挑战之一。

3.6 为什么 RLHF 有效？

阶段优化目标优化粒度
预训练$\max P(\text{next token})$token 级别
SFT$\max P(\text{good answer} \mid \text{instruction})$token 级别
RLHF$\max r_\phi(\text{answer}) - \beta \cdot D_{\text{KL}}$回答级别

SFT 教每个字怎么写，RLHF 教整篇文章怎么写才得高分。

四、DPO：直接偏好优化

4.1 RLHF 的痛点

  1. 需要单独训练奖励模型
  2. PPO 训练不稳定，超参数敏感
  3. 四个模型同时在显存，硬件需求极高
  4. 三阶段流水线，任何一步出问题都影响最终效果

2023 年，Stanford 的 Rafailov 等人提出 DPO，给出了优雅的解答。

4.2 核心洞察

RLHF 优化目标的最优解有解析形式。将其代入 Bradley-Terry 模型后，可以推导出奖励函数完全由策略模型和参考模型的对数概率比表示：

$$r(x, y) = \beta \log \frac{\pi^*(y|x)}{\pi_{\text{ref}}(y|x)} + \beta \log Z(x)$$

关键发现：不需要单独训练奖励模型——策略模型本身隐含地定义了奖励函数。

4.3 DPO 损失函数

将奖励表达式代入 Bradley-Terry 模型，配分函数 $Z(x)$ 被消掉，得到：

$$\mathcal{L}_{\text{DPO}} = -\mathbb{E}_{(x, y_w, y_l)}\left[\log \sigma\left(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right)\right]$$
符号含义
$y_w, y_l$偏好回答（win）和不偏好回答（lose）
$\pi_\theta / \pi_{\text{ref}}$策略模型 / 参考模型（冻结的 SFT 模型）
$\beta$温度参数，控制偏离程度，典型值 0.1~0.5

直觉：增大策略模型对"好回答"的概率，减小对"差回答"的概率（均相对于参考模型）。

4.4 DPO vs RLHF 对比

维度RLHF (PPO)DPO
训练阶段SFT → RM → PPOSFT → 直接优化
模型数4 个2 个（策略 + 参考）
显存需求极高（4x）中等（2x）
超参数敏感度高低（主要 $\beta$ 和学习率）
训练稳定性较差好（标准监督学习）
实现复杂度高（需 RL 框架）低（几十行 PyTorch）
在线数据支持不支持（离线方法）
效果强（尤其大规模场景）多数场景足够好

4.5 DPO 的局限

  1. 离线方法：无法在训练中生成新回答获取反馈，限制探索能力
  2. 分布偏移：训练中策略偏离了生成偏好数据时的分布
  3. 隐式奖励假设：人类偏好可能不满足 Bradley-Terry 模型假设

后续 IPO、Online DPO 等尝试解决这些问题。

五、Constitutional AI：用 AI 辅助对齐

Anthropic（2022）提出的方法：用一组明确原则（"宪法"）指导 AI 自我批评和修正，减少对人类标注的依赖。

两阶段流程：

  1. 监督阶段：模型生成回答 → 根据宪法原则自我批评 → 修正回答 → 用修正后的数据做 SFT
  2. RL 阶段（RLAIF）：AI 根据宪法原则选择更好的回答（替代人类标注者） → 训练 RM → PPO 优化
维度RLHFRLAIF (CAI)
偏好来源人类标注者AI + 宪法原则
规模化能力受限于标注者数量近乎无限
成本高（人力）低（计算）
一致性标注者间可能不一致AI 判断更一致

六、RLHF 之后的趋势

6.1 KTO（Kahneman-Tversky Optimization, 2024）

不需要偏好对，只需每个回答的好/差标签（二元反馈）。用户"点赞/点踩"就是天然数据源。参考前景理论，"损失"权重大于"收益"。

6.2 ORPO（Odds Ratio Preference Optimization, 2024）

把 SFT 和偏好优化合并为一步：$\mathcal{L}_{\text{ORPO}} = \mathcal{L}_{\text{SFT}} + \lambda \cdot \mathcal{L}_{\text{OR}}$。不需要参考模型。

6.3 在线 DPO 与迭代 DPO

解决 DPO 的离线局限：训练中用当前策略生成新回答并获取 AI 偏好标注，或多轮迭代。

6.4 GRPO（Group Relative Policy Optimization, 2025）

DeepSeek 提出，用于 DeepSeek-R1 训练。去掉价值模型，用组内相对排名作为基准线。比 PPO 少一个模型，更稳定。

6.5 趋势总结

RLHF (PPO)  → 4个模型，复杂但效果好
DPO         → 2个模型，简单稳定
KTO         → 不需偏好对，只需好/差标签
ORPO        → SFT + 对齐一步完成
Online DPO / GRPO → 在线生成 + 简化架构

总体方向：更简单、更少模型、更易获取的反馈、更稳定的训练。

八、关键要点总结

#要点说明
1对齐解决的不是能力问题预训练给能力，对齐让模型听话且安全
2SFT 是必要的第一步学会对话格式和基本指令遵循
3RLHF 优化的是人类偏好从 token 级别跃升到回答级别的优化
4DPO 是 RLHF 的优雅简化数学等价（理想条件下），工程简单得多
5数据质量 >> 方法选择再好的算法也救不了垃圾数据
6对齐有代价（对齐税）过度对齐会损害能力和创造力
7趋势越来越简单RLHF → DPO → KTO → ORPO
8对齐不是一次性的持续收集反馈、更新数据、重新训练
