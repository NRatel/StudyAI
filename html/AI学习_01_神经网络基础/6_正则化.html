正则化（Regularization）

从过拟合说起

核心问题：过拟合

你训练了一个神经网络，训练集准确率 99%，新数据准确率 65%。这就是过拟合（Overfitting）：模型把训练数据里的噪声、巧合、个例都当成了规律。

类比考试：

  • 过拟合的学生：背下历年真题答案，题目改个数字就不会。
  • 泛化好的学生：理解方法，换题也能做。

正则化就是让模型少背答案、多学规律的一组方法。它们的共同目标是：

限制模型复杂度，减少对训练集细节的过度依赖，提高泛化能力。

偏差-方差直觉

模型错误可以粗略理解为三部分：

$$\text{总误差} = \text{偏差}^2 + \text{方差} + \text{不可约噪声}$$
  • 偏差高：模型太简单，学不到真实规律，叫欠拟合。
  • 方差高：模型太敏感，把训练集细节也记住了，叫过拟合。
  • 不可约噪声：数据本身的随机性，再好的模型也消不掉。

正则化主要是在模型已经足够复杂时，压低方差，让模型更稳。

常见正则化方法

1. L2 正则化与权重衰减

L2 正则化在原始损失上加一个惩罚项：

$$\mathcal{L}_{\text{reg}} = \mathcal{L}_{\text{original}} + \frac{\lambda}{2}\|\mathbf{w}\|_2^2$$

权重越大，惩罚越重。直觉是：

  • 大权重会放大输入中的微小变化，模型更容易对噪声敏感。
  • 小权重让模型变化更平滑，不容易学出奇怪的锯齿边界。

在 SGD 中，L2 正则化等价于权重衰减（Weight Decay）：每次更新时把权重轻轻缩小一点。

AdamW 中的 weight_decay 也来自这个思想，只是它把权重衰减和 Adam 的自适应更新分开处理，效果更稳定。

2. L1 正则化

L1 正则化惩罚权重绝对值：

$$\mathcal{L}_{\text{reg}} = \mathcal{L}_{\text{original}} + \lambda\|\mathbf{w}\|_1$$

它的特点是容易让很多权重变成 0，产生稀疏性。

对比L1L2
惩罚对象权重绝对值权重平方
参数效果很多权重变 0权重整体变小
常见用途特征选择、稀疏模型深度学习中更常用的防过拟合手段

对入门阶段来说，记住一句话就够：L1 更像做筛选，L2 更像让模型别太激进。

3. Dropout：训练时随机关掉一部分神经元

Dropout 的做法很直接：训练时随机把一部分神经元输出置为 0。

$$\tilde{\mathbf{h}} = \frac{\mathbf{r}\odot\mathbf{h}}{1-p},\quad \mathbf{r}\sim\text{Bernoulli}(1-p) $$

其中 $p$ 是 dropout 概率。

为什么有效？

  1. 减少依赖单个神经元：任何神经元都可能被关掉，网络必须学会分散表示。
  2. 近似模型集成：每次训练都像在训练一个略有不同的子网络，最终效果类似很多子网络的平均。
  3. 增加训练噪声：适度噪声能让模型不那么死记训练样本。

推理时不会随机关神经元。主流实现使用 Inverted Dropout：训练时把保留下来的激活除以 $1-p$，保证训练和推理时的期望尺度一致。

4. Batch Normalization：按 batch 标准化

Batch Normalization（BN）会把一个 mini-batch 中某个特征的激活值标准化：

$$\hat{x} = \frac{x-\mu_\mathcal{B}}{\sqrt{\sigma_\mathcal{B}^2+\epsilon}}$$

再用可学习参数恢复表达能力：

$$y = \gamma\hat{x}+\beta$$

BN 的好处：

  • 让中间激活值保持在比较稳定的范围。
  • 允许使用更大的学习率。
  • 降低模型对初始化的敏感度。
  • 因为 batch 统计量带有噪声，也有轻微正则化效果。

需要注意：BN 在训练和推理时行为不同。训练时用当前 batch 的均值和方差；推理时用训练过程中累计的运行均值和运行方差。

5. Layer Normalization：按特征标准化

Layer Normalization（LN）和 BN 的核心区别是归一化维度不同：

对比Batch NormLayer Norm
归一化维度同一特征跨 batch同一样本内部跨特征
是否依赖 batch size依赖不依赖
训练/推理行为不完全一致一致
常见场景CNNTransformer

为什么 Transformer 更常用 LN？

  • 序列长度变化大，不适合跨 batch 对同一位置做统计。
  • 单样本推理时行为稳定。
  • 不需要维护运行均值和运行方差。

后面学习 Transformer 时，LayerNorm 会反复出现。

6. 早停（Early Stopping）

早停是最简单的正则化方法：

  1. 划分训练集和验证集。
  2. 训练过程中观察验证集 loss。
  3. 如果训练集 loss 继续下降，但验证集 loss 开始变差，就停止训练。
  4. 保留验证集表现最好的模型。

直觉：训练早期，模型在学真实规律；训练太久后，模型开始记训练集噪声。早停就是在"学规律"和"背噪声"的分界点附近刹车。

7. 数据增强（Data Augmentation）

数据增强不是改模型，而是让训练数据变得更多样。

领域常见增强
图像随机裁剪、翻转、旋转、颜色扰动、Mixup、CutMix
文本同义替换、回译、随机删词
音频加噪声、变速、变调

核心思想：如果猫的照片翻转、裁剪、稍微变暗后仍然是猫，模型就应该学会"猫的本质特征"，而不是记住某张图片的像素细节。

常见误区

误区 1：正则化越强越好。

不是。正则化太弱会过拟合，太强会欠拟合。它是一个平衡问题。

误区 2：Dropout 是现代大模型的唯一防过拟合手段。

不是。大模型还依赖数据规模、权重衰减、归一化、训练策略等。Dropout 在不同架构中的重要性也不同。

误区 3：BatchNorm 和 LayerNorm 只是为了防过拟合。

不准确。它们也强烈影响训练稳定性和收敛速度。对 Transformer 来说，LayerNorm 更是架构稳定性的关键部件。

本节小结

技术核心直觉常见位置
L2 / 权重衰减让权重别太大，模型别太激进几乎所有深度模型
L1让部分权重变 0，产生稀疏性特征选择、压缩
Dropout随机关神经元，减少依赖全连接层、部分 Transformer
BatchNorm按 batch 稳定激活分布CNN
LayerNorm按样本特征维度稳定激活分布Transformer
早停验证集变差时及时停止小数据训练
数据增强制造合理变体，减少死记样本图像、语音、文本
