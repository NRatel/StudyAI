LSTM 与 GRU

前置知识：已理解 RNN 循环神经网络 中的隐藏状态、时间展开和梯度消失问题。
本节目标：理解 LSTM 和 GRU 如何用"门控"机制选择性记住、忘记和更新信息。

RNN 为什么需要门控

原始 RNN 的问题

普通 RNN 只有一个隐藏状态 $h_t$。每读一个新输入，它就把旧状态和新输入混在一起，得到新状态。

这有点像你一边听课一边只拿一张很小的便签做笔记：新信息不断写上去，旧信息很容易被覆盖。句子一长，开头的信息就淡了。

LSTM 和 GRU 的想法是：不要让记忆每一步都被粗暴刷新，而是加几个"门"，让模型自己决定：

  • 什么信息该忘？
  • 什么信息该写入？
  • 什么信息该输出给下一步？

门是什么

门（Gate）本质上是一个取值在 0 到 1 之间的向量，通常由 Sigmoid 产生。

可以把它理解成一排可调旋钮：

  • 接近 0：几乎关上，不让信息通过。
  • 接近 1：几乎打开，让信息通过。
  • 介于中间：部分通过。

LSTM 与 GRU 的核心机制

1. LSTM：给记忆一条专用通道

LSTM（Long Short-Term Memory）除了隐藏状态 $h_t$，还维护一个细胞状态 $c_t$。

可以把 $c_t$ 想象成一条贯穿时间的传送带。信息可以在上面较稳定地流动，而不是每一步都被完全重写。

LSTM 有三个核心门：

门作用问的问题
遗忘门 $f_t$控制旧记忆保留多少过去哪些信息该忘掉？
输入门 $i_t$控制新信息写入多少现在哪些信息该写进记忆？
输出门 $o_t$控制记忆暴露多少当前该拿出哪些信息使用？

简化理解：

旧记忆 c_{t-1}
  ├─ 遗忘门：保留有用旧信息
  ├─ 输入门：写入有用新信息
  ↓
新记忆 c_t
  └─ 输出门：生成隐藏状态 h_t

2. LSTM 的核心公式

核心公式如下，重点看每一行的含义，不需要背：

$$f_t = \sigma(W_f[h_{t-1}, x_t] + b_f)$$
$$i_t = \sigma(W_i[h_{t-1}, x_t] + b_i)$$
$$\tilde{c}_t = \tanh(W_c[h_{t-1}, x_t] + b_c)$$
$$c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t$$
$$o_t = \sigma(W_o[h_{t-1}, x_t] + b_o)$$
$$h_t = o_t \odot \tanh(c_t)$$

最关键的是这一行：

$$c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t$$

它用加法更新长期记忆，而不是每一步都把旧状态整体塞进一个非线性函数里重算。加法路径让梯度更容易跨时间传播，这就是 LSTM 缓解梯度消失的核心原因。

3. GRU：更简洁的门控 RNN

GRU（Gated Recurrent Unit）可以看作 LSTM 的轻量版。它没有单独的细胞状态，而是直接在隐藏状态上做门控。

GRU 有两个主要门：

门作用
更新门 $z_t$控制保留多少旧状态、写入多少新状态
重置门 $r_t$控制计算候选状态时看多少旧状态

核心公式：

$$z_t = \sigma(W_z[h_{t-1}, x_t])$$
$$r_t = \sigma(W_r[h_{t-1}, x_t])$$
$$\tilde{h}_t = \tanh(W_h[r_t \odot h_{t-1}, x_t])$$
$$h_t = (1-z_t)\odot h_{t-1} + z_t\odot \tilde{h}_t$$

其中最后一行最好理解：

  • $z_t$ 小：多保留旧状态。
  • $z_t$ 大：多采用新候选状态。

4. LSTM vs GRU

对比LSTMGRU
状态隐藏状态 + 细胞状态只有隐藏状态
门数量3 个门2 个门
表达能力更强，结构更细更简洁
参数量更多更少
常见选择长依赖复杂任务数据较少或希望模型轻量时

实际使用中，两者没有绝对胜负。LSTM 更经典、更细致；GRU 更简单、更快。理解它们的共同点比记差异更重要：都用门控机制解决原始 RNN 的记忆不稳定问题。

5. 为什么门控能缓解梯度消失

普通 RNN 每一步都把旧状态和新输入混合后过一次非线性函数，梯度要不断穿过这些非线性变换。

LSTM/GRU 引入了更直接的状态保留路径：

  • LSTM 用细胞状态 $c_t$ 承载长期记忆。
  • GRU 用更新门在旧状态和新状态之间插值。

当模型学到"这段信息很重要"时，门可以接近打开状态，让信息和梯度跨多个时间步保留下来。

常见误区

误区 1：LSTM 能无限记住所有历史。

不能。它只是比普通 RNN 更容易保留长距离信息，仍然受隐藏维度、数据和训练影响。

误区 2：GRU 是 LSTM 的低配版，所以一定更差。

不一定。GRU 参数更少，在一些任务上效果接近甚至优于 LSTM。模型越复杂不总是越好。

误区 3：学了 Transformer 就不用理解 LSTM/GRU。

仍然值得理解。门控、状态更新、长距离依赖这些思想后来在很多架构中反复出现。

本节小结

概念一句话总结
门控用 0~1 的开关控制信息流
LSTM用细胞状态 + 三个门管理长期记忆
细胞状态LSTM 中相对稳定的信息传送带
GRU更简洁的门控 RNN
更新门控制旧状态和新状态的混合比例
长距离依赖门控让重要信息更容易跨时间保留
