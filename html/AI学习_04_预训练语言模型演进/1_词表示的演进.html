词表示的演进

前置知识：Transformer 架构
本节目标：理解语言模型的第一个基本问题：文字如何变成模型能计算的向量。沿着 One-Hot → Word2Vec/GloVe → ELMo 的路线，看清“词表示”如何从无语义索引，走向上下文相关表示。

1. 词怎么变成模型能算的东西

神经网络只认识数字，不认识“猫”“狗”“银行”这些词。词表示要做的事，就是给每个词办一张“数学身份证”：

$$f: \text{词或 token} \to \mathbb{R}^d$$

但这张身份证不能只是编号。理想情况下，“猫”和“狗”应该离得近，“猫”和“拖拉机”应该离得远；“bank”在“存钱”附近和在“河岸”附近，也不该长得一模一样。

演进路线可以这样看：

One-Hot
  像座位号：能区分词，但不知道词义
     ↓
Word2Vec / GloVe
  像朋友圈画像：经常一起出现的词，语义更接近
     ↓
ELMo
  像根据场合换名片：同一个词在不同句子里有不同表示
     ↓
BERT / GPT
  用 Transformer 生成更强的上下文表示

关键跳跃有两次：

  • One-Hot → 静态词向量：从“每个词只是编号”变成“词和词之间有语义距离”。
  • 静态词向量 → 上下文词向量：从“一词一个固定向量”变成“同一个词在不同句子里有不同向量”。

2. 词表示方法的演进

2.1 One-Hot：只有身份，没有语义

给定词汇表大小 $|V|$，每个词用一个 $|V|$ 维向量表示，只有对应位置为 1，其余位置为 0：

$$\mathbf{e}_{\text{cat}} = [0, 0, 1, 0, \ldots, 0]^T$$

One-Hot 的优点是简单、明确；缺点也很致命：

问题说明
维度很高词表可能有几万到几十万个词
向量稀疏绝大多数位置都是 0
没有语义“猫”和“狗”的向量正交，模型看不出它们更相近
难以泛化学到“猫很可爱”，不能自然迁移到“狗很可爱”

一句话：One-Hot 是索引，不是理解。

2.2 分布式假设：看一个词的邻居

Word2Vec 和 GloVe 背后的共同思想是分布式假设：

一个词的含义，可以由它经常出现的上下文来刻画。

如果两个词经常出现在相似语境里，它们大概率语义相近。比如“医生”和“护士”都常和“医院、病人、手术”一起出现，所以它们在向量空间中应该更接近。

这就是词向量的核心直觉：让词在语义空间里站队。

2.3 Word2Vec：用预测任务学词义

Word2Vec 用一个很朴素的自监督任务学习词向量：给模型一段文本，让它根据词和上下文之间的关系做预测。

常见有两种形式：

方法预测方向直觉
CBOW用上下文预测中心词看左右邻居，猜中间是谁
Skip-gram用中心词预测上下文给一个词，猜它身边常出现什么词

例子：

句子：the cat sat on the mat

CBOW：      the, cat, on, the → sat
Skip-gram： sat → the, cat, on, the

训练结束后，模型内部的词向量会带上语义结构。经典例子是：

$$\mathbf{v}_{king} - \mathbf{v}_{man} + \mathbf{v}_{woman} \approx \mathbf{v}_{queen}$$

这个例子不必神化。它说明的是：词向量空间中不仅能表示“相似”，还可能捕捉到某些稳定的语义方向，例如性别、时态、国家-首都关系。

2.4 负采样：别每次都考全班

原始的词预测如果每次都在整个词表里做 softmax，代价很高。负采样的想法很像考试抽查：

  • 真正一起出现的词对是正样本。
  • 随机抽来的无关词是负样本。
  • 模型只需要学会“拉近正样本，推远负样本”。

这让训练效率大幅提高，也让 Word2Vec 能在很大的语料上训练。

2.5 GloVe：把全局共现也算进去

Word2Vec 更像边读文本边学习局部窗口关系；GloVe 则先统计词与词在整个语料中的共现情况，再学习词向量。

它关心的问题是：两个词共同出现得多不多？和哪些词一起出现？这些全局统计能揭示稳定语义关系。

维度Word2VecGloVe
核心信息局部上下文窗口全局共现统计
学习方式通过预测任务学习通过共现矩阵学习
结果静态词向量静态词向量

对初学者来说，不必纠结两者谁“本质更高级”。它们共同完成了一件重要的事：让词从离散符号变成有语义距离的稠密向量。

2.6 静态词向量的根本局限：一词多义

Word2Vec 和 GloVe 的共同限制是：每个词只有一个固定向量。

I went to the bank to deposit money.  → bank = 银行
I sat on the bank of the river.       → bank = 河岸

同一个 “bank” 在两个句子里意思完全不同，但静态词向量给它同一张身份证。这就像一个人白天是医生、晚上是乐队鼓手，系统却只允许他填一个身份。

2.7 ELMo：同一个词，按上下文换表示

ELMo 的核心贡献是上下文相关词向量：词的表示不再只由词本身决定，而由整个句子共同决定。

ELMo 使用双向 LSTM 语言模型：

  • 前向 LSTM 从左到右读句子。
  • 后向 LSTM 从右到左读句子。
  • 最终把不同层的表示组合起来，得到每个词在当前句子中的向量。

因此，“bank”在“deposit money”和“river”两个上下文中会得到不同表示。

需要注意：ELMo 的前向和后向语言模型是分别训练的；BERT 后来用 Transformer 和 MLM 目标，更直接地建模双向上下文。

2.8 词、子词与 Token

现代语言模型通常不是直接按“词”建模，而是按 token 建模。token 可以是一个词，也可以是词的一部分。

为什么要切成子词？因为自然语言里新词太多。如果只按完整词建词表，遇到没见过的词就会 OOV（out-of-vocabulary，词表外词）。子词分词把生词拆成更小片段，能显著缓解这个问题。

后续 BERT、GPT 里的 Embedding，本质上就是：先把文本切成 token，再把 token 映射成向量。

本节小结

概念一句话总结
One-Hot只表示词的身份，不表示语义
分布式假设一个词的意义由它常出现的上下文决定
Word2Vec通过预测中心词或上下文学习静态词向量
负采样用少量负样本替代全词表计算，提高训练效率
GloVe利用全局词共现统计学习静态词向量
静态词向量局限一词一个向量，无法处理一词多义
ELMo根据上下文为同一个词生成不同表示
Token现代语言模型的基本输入单位，常常是子词而非完整词
