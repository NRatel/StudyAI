BERT 与 GPT：两大路线

前置知识：自监督预训练范式、Transformer 架构
本节目标：理解 BERT 和 GPT 的核心差异：一个偏“双向理解”，一个偏“自回归生成”。重点不在记模型参数，而在理解技术路线为什么不同。

1. 两条预训练路线

2018 年，BERT 和 GPT 几乎同时把 NLP 推入“预训练模型”时代。它们都相信一件事：先在海量文本上预训练，再迁移到具体任务。

但它们对“语言能力应该怎么学”给出了不同答案：

BERT：先读完整段落，再回答问题
  Transformer Encoder + MLM
  强项：理解、分类、抽取、匹配

GPT：根据前文不断续写
  Transformer Decoder + CLM
  强项：生成、对话、推理、代码

可以把 BERT 想成“阅读理解高手”，把 GPT 想成“接话高手”。阅读理解高手适合判断、抽取、分类；接话高手一旦足够强，就能把很多任务都转成“把答案接着写出来”。

2. BERT 与 GPT 的核心差异

2.1 BERT 是什么

BERT（Bidirectional Encoder Representations from Transformers） 使用 Transformer Encoder 堆叠，通过 MLM 进行双向预训练。

核心特征：

  • Encoder-only：只使用 Transformer 编码器。
  • 双向注意力：每个位置可以看见左右所有位置。
  • MLM 预训练：遮住部分 token，让模型预测原词。
  • 下游微调：针对分类、NER、问答等任务加一个任务头。

简化结构：

Token + Segment + Position Embedding
          ↓
Transformer Encoder × N
          ↓
每个 token 的上下文表示
          ↓
分类 / 标注 / 抽取式问答任务头

BERT 的输入里常见两个特殊符号：

  • [CLS]：放在序列开头，常用于代表整句或整段文本。
  • [SEP]：分隔两个句子或文本片段。

2.2 BERT 擅长什么

BERT 的优势来自双向上下文。它适合那些需要“看完整输入再判断”的任务：

任务为什么适合 BERT
文本分类需要整体理解句子或段落
命名实体识别判断每个词在上下文中的角色
语义匹配比较两段文本是否相关或等价
抽取式问答在给定文章中定位答案范围

BERT 不自然适合长文本续写，因为它的训练目标不是“一个 token 一个 token 地生成下去”。

2.3 GPT 是什么

GPT（Generative Pre-trained Transformer） 使用 Transformer Decoder 堆叠，通过 CLM 进行自回归预训练。

核心特征：

  • Decoder-only：使用带因果掩码的 Transformer 解码器。
  • 单向注意力：每个位置只能看自己和之前的 token。
  • CLM 预训练：根据前文预测下一个 token。
  • 生成式使用方式：把任务写成 prompt，让模型生成答案。

简化结构：

Token + Position Embedding
          ↓
Masked Transformer Decoder × N
          ↓
下一个 token 的概率分布
          ↓
逐步生成文本

GPT 的核心动作很简单：给定前文，预测下一个 token。但当模型足够大、数据足够丰富时，这个动作可以承载翻译、问答、摘要、代码、推理等大量任务。

2.4 GPT 擅长什么

GPT 的优势来自生成范式。它适合那些可以表达为“输入 → 输出文本”的任务：

任务GPT 的处理方式
对话根据上下文生成回复
翻译根据源语言文本生成目标语言文本
摘要根据长文本生成短摘要
开放问答根据问题生成答案
代码生成根据需求描述生成代码

GPT 的限制是：它天然只能从左到右看上下文。在某些短文本理解、检索排序、结构化抽取任务上，小型 BERT 类模型仍可能更高效。

2.5 核心差异总表

维度BERTGPT
架构路线Encoder-onlyDecoder-only
注意力方向双向单向，因果掩码
预训练目标MLM，完形填空CLM，预测下一个 token
输入使用看完整输入后输出表示根据前文逐步生成
典型任务分类、NER、匹配、抽取式问答对话、续写、翻译、摘要、代码
迁移方式加任务头微调Prompt、微调、指令微调
核心优势理解类任务高效任务形式统一、生成能力强

2.6 为什么 Decoder-only 成为通用大模型主流

需要先加一个限定：主流不等于唯一正确。BERT 类模型并没有过时，只是 GPT 路线更适合做通用生成式 AI。

Decoder-only 成为主流，主要因为：

  1. 任务统一为生成

分类可以生成“正面/负面”，翻译可以生成译文，问答可以生成答案。不同任务不一定要改模型结构。

  1. 训练目标和使用方式一致

训练时预测下一个 token，使用时也是预测下一个 token。模型学到的能力可以直接用于生成。

  1. 大规模模型出现 In-Context Learning

GPT-3 展示了 few-shot 能力：在 prompt 里给几个例子，模型就能临时对齐任务格式，不一定需要重新训练参数。

  1. 和人机交互形式自然契合

用户输入自然语言，模型输出自然语言。聊天、写作、问答、编程辅助都可以落到同一套生成接口上。

2.7 但 BERT 仍然重要

BERT 的价值不应该被“GPT 更火”掩盖。它仍然是理解 Encoder-only 模型、文本表示、向量检索和判别式 NLP 任务的重要基础。

在这些场景里，BERT 类模型仍常有优势：

  • 需要快速判断相关性或类别。
  • 输出空间固定，例如标签集合很小。
  • 不需要生成长文本。
  • 成本、延迟、可控性比开放生成更重要。

一句话：GPT 更像通用写作者，BERT 更像高效阅卷老师。

本节小结

概念一句话总结
BERTTransformer Encoder + MLM，双向理解能力强
GPTTransformer Decoder + CLM，自回归生成能力强
双向注意力每个位置能看左右上下文，适合理解
因果注意力每个位置只能看前文，适合生成
任务头微调BERT 常通过分类头、标注头适配任务
Prompt / ICLGPT 可通过自然语言提示和示例适配任务
Decoder-only 主流因为生成范式能统一大量任务，并在大规模下表现出强通用性
BERT 的价值在理解、检索、分类等任务中仍然高效实用
