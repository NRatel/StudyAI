一、关键论文

论文列表

1. Wei et al., 2022 — Chain-of-Thought Prompting

标题：Chain-of-Thought Prompting Elicits Reasoning in Large Language Models

简评：开创性论文。发现在 few-shot prompt 中加入推理步骤（思维链），就能大幅提升大语言模型在数学、常识推理等任务上的表现。核心贡献不在于复杂的技术，而在于一个简单而深刻的观察：模型已经具备推理能力，只需要正确的提示方式来"激活"它。实验表明 CoT 是一种涌现能力，只在约 100B+ 参数的模型上才显著生效。

核心贡献：确立了 Chain-of-Thought Prompting 范式，开启了"通过提示增强推理"的研究方向。直接启发了后续所有 CoT 变体（Zero-shot CoT、Self-Consistency、ToT 等）以及最终的推理模型（o1、R1）。

推荐阅读方式：重点读 Section 2（方法）、Section 3（实验结果）、Figure 1（直观对比）。

2. Kojima et al., 2022 — Zero-shot CoT

标题：Large Language Models are Zero-Shot Reasoners

简评：Wei 2022 的 CoT 需要人工编写推理示例（few-shot），Kojima 的发现令人惊讶——只需在问题后加一句 "Let's think step by step"，模型就会自动生成推理过程。这意味着 LLM 的推理能力比想象中更容易激活，不需要精心设计的示例。论文还系统测试了不同触发短语的效果差异。

核心贡献：将 CoT 从 few-shot 推广到 zero-shot，极大降低了使用门槛。证明了推理能力是 LLM 的"内在属性"而非"示例教会的技巧"。

推荐阅读方式：重点读 Table 1（不同触发短语的效果对比），以及与 few-shot CoT 的对比实验。

3. Wang et al., 2023 — Self-Consistency

标题：Self-Consistency Improves Chain of Thought Reasoning in Language Models

简评：CoT 的一个弱点是推理链可能出错（一步错步步错）。Self-Consistency 提出了一个朴素但有效的解决方案：对同一问题多次采样（用较高温度），然后对最终答案做多数投票。核心洞察是"正确的推理路径比错误的更常见"——不同的错误方式产生不同的错误答案（票数分散），而正确答案的票数集中。

核心贡献：将集成学习（ensemble）的思想引入 LLM 推理，无需额外训练即可获得显著提升。直接启发了 Test-time Compute Scaling 的研究方向——推理阶段花更多计算能换更好结果。

推荐阅读方式：重点读 Section 3（方法，极简洁），Table 1-3（与 CoT 的效果对比），以及 Section 5 的消融实验（采样次数、温度等的影响）。

4. Yao et al., 2023 — Tree of Thoughts

标题：Tree of Thoughts: Deliberate Problem Solving with Large Language Models

简评：将推理结构从线性链扩展为树形搜索。核心创新是引入了经典 AI 搜索算法（BFS/DFS）的思想：在推理过程的每一步生成多个候选思路，用 LLM 自身评估每条路径的前景，然后剪枝保留最有希望的路径继续探索。在需要"回溯"的任务（如 24 点游戏）上效果显著：从 CoT 的 7.3% 提升到 74%。

核心贡献：将搜索算法与 LLM 推理相结合，证明了在特定任务上"搜索 + 评估"远优于单链推理。从思想上预示了推理模型（o1 等）的核心原理——推理时的计算量（搜索宽度/深度）可以显著提升推理质量。

推荐阅读方式：重点读 Figure 1（ToT 与 CoT 的结构对比）、Section 3（方法框架）、Section 4.1（24 点游戏案例分析）。

5. OpenAI, 2024 — Learning to Reason with LLMs (o1)

标题：Learning to Reason with LLMs（OpenAI 技术博客 / 系统卡片）

简评：OpenAI o1 的发布标志着从"提示推理"到"训练推理"的范式转变。o1 通过大规模强化学习训练模型在回答前自主进行长链推理（internal chain-of-thought）。关键突破是发现推理性能随 test-time compute 的增加而持续提升——模型"想得越久"，答案质量越高。o1 在 AIME 2024 数学竞赛中达到 83.3%（接近金牌选手水平），在 GPQA 博士级科学问题上超过人类博士生平均水平。

核心贡献：实验验证了 Test-time Compute Scaling 的有效性；确立了"推理模型"作为独立类别；引发了开源社区的推理模型竞赛（DeepSeek-R1、QwQ 等）。

推荐阅读方式：OpenAI 未发布完整论文，建议阅读官方博客 Learning to Reason with LLMs + o1 system card。重点关注 AIME/GPQA 的 scaling 曲线和与 GPT-4o 的对比。

6. DeepSeek, 2025 — DeepSeek-R1

标题：DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via Reinforcement Learning

简评：开源推理模型的标志性工作。最重要的发现是 DeepSeek-R1-Zero：直接从基础模型出发，不使用任何 SFT 数据，仅通过 GRPO 强化学习训练，模型就自发学会了长链推理、自我验证、回溯修正等能力。论文详细公开了 GRPO 算法、训练流程、以及将推理能力蒸馏到小模型（1.5B~70B）的方法。R1 在数学和编程基准上接近 o1 水平。

核心贡献：证明纯 RL 可以从零训练出推理能力（无需人工推理示例）；开源了完整的模型权重和训练细节；GRPO 算法为推理 RL 训练提供了更简洁高效的方案；蒸馏版本证明推理能力可以有效转移到小模型。

推荐阅读方式：重点读 Section 3（R1-Zero 的 RL 训练）——这是最有启发性的部分，尤其是模型自发涌现推理行为的分析。然后读 Section 4（蒸馏方法）和 Section 5（完整的 R1 训练流程）。

7. Snell et al., 2024 — Scaling LLM Test-Time Compute

标题：Scaling LLM Test-Time Compute Optimally can be More Effective than Scaling Model Parameters

简评：为 Test-time Compute Scaling 提供了理论和实验基础。核心发现是：在推理阶段增加计算量（更多采样、更长推理链、验证器等）的效果，可以等价甚至超过增加训练阶段的计算量（更大模型）。论文系统比较了不同的 TTC 策略（多数投票、Best-of-N + 验证器、顺序修正等），发现最优策略取决于问题难度和模型基础能力。

核心贡献：为"小模型 + 更多推理时间 ≈ 大模型 + 快速推理"提供了实验验证；提出了 TTC 的最优分配框架；为推理模型的设计和实际应用提供了理论指导。

推荐阅读方式：重点读 Section 4（不同 TTC 策略的对比）和 Figure 3-4（TTC vs Train-time Compute 的 Scaling 曲线）。

二、常见误区与FAQ

Q1：CoT 提示和推理模型是一回事吗？

不是，它们是两种不同的方法，虽然灵感相同。

CoT 提示（2022-2023）:
  方法: 通过提示让模型"写出推理步骤"
  模型: 不变，用通用 LLM（如 GPT-4）
  控制: 外部（由用户的提示决定）
  推理过程: 用户可见（就是模型输出的文本）
  效果: 在大模型上有效，但提升有上限

推理模型（2024-2026）:
  方法: 通过 RL 训练模型自主进行长链推理
  模型: 专门训练的推理模型（o1/o3/R1）
  控制: 内部（模型自主决定"想多久"）
  推理过程: 部分可见（开源模型可看到 <think> 标签）
  效果: 更强，且推理能力随计算量持续提升

类比：CoT 提示相当于"提醒学生把步骤写出来"，推理模型相当于"训练学生养成深度思考的习惯"。前者是临时性的外部干预，后者是能力的内化。

Q2：推理模型是不是所有场景都比普通模型好？

不是。推理模型有明确的优势场景和劣势场景。

推理模型更好的场景（值得等待和付费）：

  • 竞赛数学 / 复杂数学推导
  • 代码生成和调试（尤其是逻辑复杂的算法题）
  • 科学推理 / 数据分析
  • 需要多步逻辑推理的复杂问题

普通模型可能更好的场景（推理模型反而是浪费）：

  • 简单问答（"法国首都是什么？"）→ 推理模型不必要，还更贵更慢
  • 创意写作 → 推理链可能过度"理性化"，限制创造力
  • 翻译 / 摘要 → 不需要逻辑推理，快速模型即可
  • 延迟敏感的实时交互 → 推理模型的"思考时间"无法接受
  • 大批量低难度任务 → 成本太高

一个实用的判断标准：如果人类解决这个问题需要"仔细想一想"，那推理模型可能有帮助。如果人类可以"脱口而出"，那普通模型足够。

Q3：Self-Consistency 和推理模型的 TTC Scaling 有什么关系？

Self-Consistency 是 TTC Scaling 的"手工版"。

两者的核心思想相同：在推理阶段花更多计算来提升结果质量。区别在于实现方式：

维度Self-Consistency推理模型 TTC
额外计算方式多次独立采样，外部投票模型内部产生更长的思维链
控制方式开发者设定采样次数模型自主决定（或通过 reasoning_effort 参数）
信息利用各路径独立，不共享中间结果模型可以在思维链中自我纠错
效率较低（很多冗余计算）较高（每一步都基于之前的推理）
适用模型任何 LLM专门训练的推理模型

进化关系：

Self-Consistency (2022): 多次独立采样 + 投票 → 朴素但有效
     ↓ 启发
TTC Scaling 理论 (2024): 系统研究推理时计算的最优分配
     ↓ 指导
推理模型 (o1/R1): 将 TTC 内化到模型中，自动高效地使用推理计算

Q4：DeepSeek-R1-Zero 为什么能从零学会推理？这不反直觉吗？

这确实是 R1 论文最令人惊讶的发现。

直觉上的困惑：没有任何人工编写的推理示例，模型怎么知道要"一步步想"？

论文给出的解释：

  1. 基础模型已有潜在的推理能力：DeepSeek-V3 在预训练中见过大量推理文本（数学教科书、代码注释、论坛讨论等），已经"知道"推理的基本模式。但这种能力是潜在的，不会在普通对话中主动展现。
  2. RL 的奖励信号激活了这种潜在能力：当 RL 奖励"正确答案"时，模型发现"如果我先推理再回答，得到正确答案的概率更高"。于是推理行为被正向强化。
  3. 涌现过程：论文描述了一个"Aha moment"——在训练早期，模型的推理链很短、很粗糙。但随着训练推进，模型逐渐学会了更长、更精细的推理，甚至学会了自我纠错（"Wait, let me reconsider..."）。

类比：就像一个学过数学但从不主动演算的学生。GRPO 相当于给了一个激励机制——"你算对了有奖励"。学生自然会开始尝试列算式、检查步骤，因为这样拿奖的概率更高。推理行为是"被奖励引导出来的"，而不是"被教会的"。

Q5：CoT 为什么只在大模型上有效？小模型能不能用？

经验规律：CoT 在约 100B+ 参数的模型上效果显著，小模型效果有限甚至为负。

原因分析：

  1. 预训练数据量不足：小模型见过的推理文本更少，缺乏足够的"推理模式"可供激活。
  2. 链式错误放大：CoT 要求模型在每一步都正确。大模型每一步的正确率更高（比如 95%），N 步的正确率 = 0.95^N 仍然可接受。小模型每步正确率可能只有 70%，0.70^N 快速趋近于零。
  3. 错误推理 > 无推理：小模型可能生成看起来合理但实际错误的推理链，然后"忠实地"按照错误推理给出错误答案。这比直接猜（可能碰对）更糟。

小模型怎么办？

选项 1: 蒸馏（推荐）
  用大的推理模型生成推理数据，然后微调小模型
  例: DeepSeek-R1 → 蒸馏到 R1-Distill-7B
  效果: 7B 模型也能获得不错的推理能力

选项 2: 微调
  收集数学/推理数据集（如 MetaMath、GSM8K），
  用 SFT 训练小模型生成推理步骤
  效果: 一般，但比零样本 CoT 提示好

选项 3: 放弃 CoT
  对于非推理密集型任务，小模型直接回答可能更好

Q6：如何评估推理模型的真正推理能力 vs 记忆能力？

这是一个活跃的研究问题（截至 2026.4），业界尚无定论。

核心担忧：模型在 AIME/GSM8K 等基准上的高分，到底是"真正学会了推理"，还是"记住了训练数据中类似的题目"？

支持"真正推理"的证据：

  • 新题目的泛化能力：o1/R1 在全新的竞赛题目（从未出现在训练数据中）上也能表现出色
  • 推理过程的连贯性：模型产生的推理链通常逻辑连贯，不像是"拼凑记忆"
  • 错误修正能力：模型能在推理过程中发现并修正自己的错误，这难以用"记忆"解释

支持"记忆"担忧的证据：

  • 数据污染：GSM8K 等流行基准的题目很可能已被大量网页引用，进入了训练数据
  • 分布偏移下的脆弱性：当题目格式或措辞偏离常见模式时，推理能力有时会显著下降
  • 对抗测试：精心设计的"反直觉"题目（故意违反常见模式）可以让推理模型犯错

实用建议：

  • 不要过度依赖公开基准分数
  • 在自己的实际任务上评估，而非仅看 AIME/GSM8K
  • 测试"举一反三"的能力：模型能否在变体问题上也表现稳健
  • 使用 Self-Consistency：如果多次采样的答案不一致，说明模型并非"确定地知道答案"

Q7：开源推理模型（R1/QwQ）和闭源推理模型（o3）差距有多大？

截至 2026.4，差距已大幅缩小，但在最难的任务上仍有差距。

维度闭源（o3, 2025.4）开源最强（R1/Qwen3, 2025.1-4）
AIME 2024~90%+~80%（R1: 79.8%）
GPQA Diamond~80%+~70%+
编程竞赛~99th 百分位~97th 百分位
工具使用原生支持部分支持（快速改进中）
推理效率优化过的思维链思维链可能较冗长
成本高自部署可控
透明度思维链不可见思维链完全可见

选择建议：

  • 追求极致推理性能 → o3
  • 需要透明和可审计 → R1/Qwen3（可以看到完整思维过程）
  • 成本敏感 / 数据隐私 → 本地部署 R1-Distill 或 Qwen3
  • 大批量处理 → 本地部署小型蒸馏模型（R1-7B/14B）
