长文本技术

前置知识：注意力机制、位置编码（特别是 RoPE 旋转位置编码）
内容时效：截至 2026 年 4 月

长上下文为什么难

核心问题

GPT-2 的上下文窗口是 1024 个 token。GPT-4 Turbo 扩展到 128K。Gemini 1.5 Pro 支持 1M+。从 1K 到 1M，增长了 1000 倍。这不仅仅是"把数字调大"——长文本面临两个根本性挑战。

关键认知前置：长上下文支持 ≠ 长上下文有效利用。 模型宣称支持 128K 输入，不代表它能有效利用 128K 中每个位置的信息。研究发现（Liu et al., 2023, "Lost in the Middle"），多数模型对序列开头和末尾的信息利用率高，对中间位置的信息容易"遗忘"。也就是说，即使你把关键信息放在 128K 上下文的中间区域，模型也可能无法检索到它。2025-2026 年的模型（如 Claude 3.5、GPT-4 Turbo、Llama 3.1）已通过专门的训练策略大幅缓解此问题，但在多针测试（Multi-Needle NIAH）中仍有挑战。因此，"支持 128K"是必要条件，"有效利用 128K"才是真正的能力。

长文本面临两个根本性挑战：

  1. 计算与内存的二次方瓶颈：Self-Attention 的 $QK^T$ 矩阵大小为 $n \times n$，计算量和内存占用都是 $O(n^2)$。序列长度翻倍，计算量翻四倍。
  2. 位置编码的外推难题：模型在 4K 长度上训练，推理时面对 32K 甚至 128K 的序列，位置编码能否正常工作？

为什么 O(n^2) 是瓶颈

Self-Attention 的核心是计算 $QK^T$，其中 $Q, K \in \mathbb{R}^{n \times d}$，结果是 $n \times n$ 的注意力矩阵。让我们看看具体数字：

序列长度 n    注意力矩阵元素数    FP16 内存占用（单层单头）
    1K         1,000,000               2 MB
    4K        16,000,000              32 MB
   32K     1,024,000,000               2 GB
  128K    16,384,000,000              32 GB        ← 已超过单卡显存
    1M   1,000,000,000,000          2,000 GB       ← 完全不可行

实际情况更严重：多头注意力有 $h$ 个头，模型有 $L$ 层，训练还需要保存梯度。即使 FlashAttention 可以将内存从 $O(n^2)$ 降到 $O(n)$（不具化注意力矩阵），计算量仍然是 $O(n^2 \cdot d)$，绕不过去。

三条技术路线总览

路线核心思路代表方案本节内容
位置编码外推不改模型结构，让 RoPE 支持更长位置PI / NTK-Aware / YaRN第一部分
序列并行把长序列切块，分布到多 GPU 并行计算全注意力Ring Attention第二部分
压缩记忆将历史信息压缩到固定大小的记忆向量Infini-Attention第三部分

长文本建模的主要路线

一、位置编码外推——RoPE 扩展

回顾：RoPE 的核心思想

RoPE（详见 位置编码）对位置 $m$ 处的向量，将每对维度旋转角度 $m \theta_i$：

$$\theta_i = \text{base}^{-2i/d}, \quad \text{其中 } \text{base} = 10000$$

关键性质：位置 $m$ 和 $n$ 的 query-key 内积只依赖相对距离 $m - n$，这就是 RoPE 编码相对位置的方式。

外推失败的原因

训练时模型只见过 $m \in [0, L_{\text{train}})$ 的位置。当推理长度超过训练长度时：

  • 高频维度（$\theta_i$ 较大，$i$ 较小）的旋转角度 $m \theta_i$ 远超训练时见过的范围
  • 注意力分数的分布变得异常——模型对这些"从未见过的旋转角度"完全无法正确处理
  • 实验表现：超过训练长度 1.5~2 倍后，困惑度（perplexity）急剧上升

方法 1：位置插值 PI（Position Interpolation, Meta 2023）

核心思想极其简单：不让位置 ID 超出训练范围，把更长序列的位置线性压缩回训练范围。

如果训练长度 $L = 4096$，要扩展到 $L' = 32768$（8 倍），则将位置 $m$ 替换为：

$$m' = m \cdot \frac{L}{L'} = m / 8$$

位置 0 ~ 32767 被映射到 0 ~ 4095.875，全部落在训练时见过的范围内。

原始位置: 0, 1, 2, 3, ..., 32767
PI 映射:  0, 0.125, 0.25, 0.375, ..., 4095.875
                                        ↑
                                   不超过训练范围 4096

优点：实现极简（一行代码修改位置 ID），效果好，只需 ~1000 步微调即可恢复性能。

缺点：所有频率维度被均匀压缩。高频维度原本编码相邻位置的细微差别，压缩后分辨率下降——位置 0 和位置 1 的区分度变成了原来的 1/8。

方法 2：NTK-Aware 缩放（Bloc97 2023, 社区发现）

核心思想：高频维度编码局部位置，不需要太多缩放；低频维度编码远距离位置，需要大幅拉伸。应该不均匀缩放。

实现方式：修改 RoPE 的 base 常数。将 $\text{base} = 10000$ 替换为：

$$\text{base}' = \text{base} \cdot \alpha^{d/(d-2)}, \quad \alpha = L'/L$$

这个修改的效果：

  • 高频维度（$i$ 小）：$\theta_i$ 本身就大，base 的变化对它影响小，缩放因子接近 1
  • 低频维度（$i$ 大）：$\theta_i$ 本身就小，base 的变化对它影响大，缩放因子接近 $\alpha$

优点：无需任何微调即可获得不错的外推效果（"零样本外推"）。 缺点：对极长序列（>8 倍训练长度）精度仍有损失。

方法 3：YaRN（Yet another RoPE extensioN, 2023）

核心思想：综合 PI 和 NTK-Aware 的优势，并加入注意力温度修正。

YaRN 将频率维度按波长与训练长度的关系分为三组：

频率段判断标准处理方式
高频（波长 < 训练长度）训练时已见过完整周期不缩放，保持原样
低频（波长 >> 训练长度）训练时远未见过完整周期完全使用 PI 线性插值
中频（过渡区域）介于两者之间PI 和 NTK 缩放的线性混合

同时对注意力 logits 乘以温度因子 $\sqrt{1/t}$ 来补偿插值导致的注意力分布变化。

效果：截至 2025 年，YaRN 是最广泛使用的 RoPE 扩展方法之一。在 Llama、Qwen、Mistral 等模型上都有成功应用。

2025-2026 进展补充

  • LongRoPE (Microsoft, 2024)：为每个维度搜索最优的独立缩放因子，比 YaRN 更灵活
  • ABF (Adjusted Base Frequency, 2024-2025)：在预训练阶段直接使用更大的 base（如 500000 而非 10000），从源头减少外推需求。Llama 3.1/3.3、Qwen-2.5 均采用此策略
  • Dynamic NTK (2024-2025)：推理时根据实际输入长度动态选择缩放因子，短序列不缩放、长序列自适应缩放

二、Ring Attention（序列并行）

问题：单 GPU 放不下的长序列

即使位置编码能正确外推，128K 序列的 $QK^T$ 计算量仍然巨大。需要把长序列分布到多个 GPU 上。

核心思想（Liu et al., UC Berkeley, 2023）

将长度 $N$ 的序列均匀切成 $P$ 块，$P$ 个 GPU 各持有一块。通过"环形传递"KV 块，让每个 GPU 的 Q 块都与所有 KV 块完成注意力计算：

Step 0: 每个 GPU 持有本地 Q/K/V 块
  GPU 0: [Q₀, K₀, V₀]    GPU 1: [Q₁, K₁, V₁]
  GPU 2: [Q₂, K₂, V₂]    GPU 3: [Q₃, K₃, V₃]

Step 1: 每个 GPU 计算本地注意力 Attention(Qᵢ, Kᵢ, Vᵢ)
        同时，将 KV 块沿环传递：GPU 0→1, 1→2, 2→3, 3→0

Step 2: GPU 0 收到 K₃,V₃，计算 Attention(Q₀, K₃, V₃)
        同时继续传递 KV 块

Step 3: 类似地继续...

经过 P 步后: 每个 GPU 完成了 Qᵢ 与全部 K,V 的注意力计算

关键工程技巧：

  1. 通信与计算重叠：在传递第 $j$ 块 KV 的同时，计算第 $j-1$ 块的注意力。只要计算时间 >= 通信时间，通信开销就被完全隐藏。
  2. Online Softmax：由于 softmax 需要全局归一化，但每步只看到部分 KV，需要使用在线 softmax 技巧（维护运行中的最大值和指数和），逐步更新结果。
  3. 精确性：数学上与标准全注意力完全等价，不是近似。

效果：$P$ 个 GPU 可以处理 $P$ 倍长度的序列，每个 GPU 的内存需求降到 $O(N^2/P^2)$（注意力矩阵部分）或更优。

三、Infini-Attention（压缩记忆）

问题：序列真的无限长怎么办？

Ring Attention 需要 GPU 数量与序列长度成正比。对于持续对话、流式处理等场景，序列长度没有上界。能否用固定大小的记忆处理无限长序列？

核心思想（Munkhdalai et al., Google, 2024）

在标准注意力中嵌入一个压缩记忆模块。输入被切成等长的段（segment），每段内做标准注意力，段间通过压缩记忆传递信息：

输入序列: [seg₁] [seg₂] [seg₃] ...

处理 seg_t 时:
  ① 段内标准注意力 → A_local         （局部精确）
  ② 用 Q_t 查询压缩记忆 M → A_memory  （全局近似）
  ③ 门控融合: output = β·A_memory + (1-β)·A_local
  ④ 用 K_t, V_t 更新压缩记忆 M        （增量式更新）

压缩记忆 $M$ 是一个 $d_{\text{key}} \times d_{\text{value}}$ 的矩阵（大小固定，与序列长度无关）：

  • 检索：$A_{\text{memory}} = \frac{Q_t M_{t-1}}{Q_t z_{t-1}}$，其中 $z$ 是归一化项
  • 更新：$M_t = M_{t-1} + K_t^T V_t$（线性注意力风格的增量更新）
  • 门控：可学习参数 $\beta$ 控制"用局部注意力还是全局记忆"

本质：Infini-Attention 在段内保留了完整注意力的精度，段间用线性注意力的形式做近似压缩——是精度与效率的折中。

特性值
段内复杂度$O(s^2)$，$s$ 为段长度
段间记忆大小$O(d^2)$，固定不变
总复杂度$O(n \cdot s)$，$n$ 为总长度
信息保真度段内精确，段间有损压缩

本章小结

概念一句话总结
O(n^2) 瓶颈Self-Attention 的 $QK^T$ 大小与序列长度平方成正比，128K 时已不可承受
RoPE 外推失败训练时未见过的旋转角度导致注意力分数异常、困惑度飙升
PI（位置插值）线性压缩位置索引到训练范围内，简单有效，需轻量微调
NTK-Aware修改 RoPE base 实现不均匀缩放，高频少动低频多拉，免训练
YaRN分频段混合 PI + NTK + 温度修正，综合效果最佳
Ring Attention将序列分块到多 GPU，环形传递 KV，数学上与全注意力精确等价
Infini-Attention段内全注意力 + 段间固定大小压缩记忆，理论上支持无限长序列
长上下文 vs RAG互补关系：RAG 负责检索，长上下文负责推理

时效性说明

本文内容截至 2026 年 4 月。以下领域仍在快速迭代：

  • RoPE 扩展方法：YaRN/Dynamic NTK 是当前主流，新的频率调整策略持续涌现
  • 上下文窗口记录：Gemini 1.5 Pro 支持 1M，Google 已演示 10M token 实验
  • Ring Attention 变体：Striped Attention、Blockwise Parallel Transformer 等持续改进通信效率
