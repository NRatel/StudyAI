08 大模型推理与部署

读本模块时，先抓住模型上线后为什么慢、贵、占显存。KV Cache、量化、批处理、推测解码和推理框架，本质上都在围绕延迟、吞吐、显存和成本做取舍。

本模块导读

学什么：大语言模型从训练完成到实际可用，中间横亘着一道关键门槛——推理效率。本模块系统学习推理阶段的核心瓶颈（自回归解码为什么慢）、关键优化技术（KV Cache、PagedAttention、连续批处理、推测解码）、模型压缩（量化）、以及主流推理框架和部署方案。

本模块与模块 07 的边界：模块 07 讲的是如何训练出一个模型（分布式训练、混合精度、数据工程、对齐训练），本模块 08 讲的是如何把训练好的模型高效地部署和使用（推理优化、量化压缩、推理框架选型）。简单说：07 = 怎么造车，08 = 怎么把车开上路并开得又快又省油。

为什么学：训练一个好模型只是故事的上半场。真正让模型以可接受的延迟和成本服务用户，需要理解推理优化和高效部署背后的取舍。这也是看懂大模型应用落地的关键背景。

读完能看懂的主线：

  • 理解自回归推理的本质瓶颈（内存带宽受限，而非计算受限）
  • 理解 KV Cache 的原理和 PagedAttention 的优化思路
  • 理解 INT8/INT4 量化的原理和各主流量化方案（GPTQ/AWQ/GGUF）
  • 能看懂不同推理框架分别偏向高吞吐、低延迟、生态集成还是本地运行
  • 理解推测解码如何在不损失质量的前提下加速推理
  • 理解 Prompt Engineering 为什么有效
