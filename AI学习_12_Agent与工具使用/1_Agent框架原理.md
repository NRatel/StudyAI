# Agent 框架原理

> **前置知识**：[推理与思维链](../AI学习_11_推理与思维链/README.md)
>
> **本节目标**：理解 Agent 为什么不只是聊天模型，而是一套持续行动的循环。

---

## 1. Agent 和聊天机器人的区别

普通聊天模型主要回答问题。

Agent 还要完成任务。它需要：

- 理解目标。
- 查看当前环境。
- 制定下一步。
- 调用工具。
- 根据工具结果继续调整。

{{img:ch12_01_agent_loop_tools}}

## 2. Agent 的基本循环

一个典型 Agent 循环是：

```text
观察 → 思考 / 计划 → 行动 → 得到反馈 → 更新状态 → 再行动
```

比如让它整理一个项目，它可能先看文件列表，再读 README，再运行测试，再修改文件，最后汇总结果。

## 3. 工具让模型离开“纯文本世界”

模型自己不擅长可靠计算、实时信息和实际执行。

工具调用可以让它使用：

- 搜索。
- 数据库。
- 代码执行。
- 文件系统。
- 浏览器。
- 日历和业务系统。

模型负责决定用什么工具、什么时候用、如何解释结果。

## 4. 计划很重要，但计划会变

Agent 不能只在开头制定一次计划就不动。

真实任务里，工具结果可能和预期不同，文件可能缺失，测试可能失败。

所以 Agent 需要根据反馈调整计划。

## 5. Agent 的风险

Agent 能行动，也意味着风险更高。

需要关注：

- 工具权限。
- 错误操作。
- 循环失控。
- 把不可靠结果当事实。
- 任务目标理解偏差。

所以 Agent 系统通常需要权限边界、日志、确认机制和评估。
