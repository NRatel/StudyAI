# 记忆与状态管理

> **前置知识**：[Agent 框架原理](1_Agent框架原理.md)
>
> **本节目标**：理解 Agent 为什么需要状态和记忆，以及短期上下文和长期记忆有什么区别。

---

## 1. 只靠当前对话不够

复杂任务不是一句问答能完成的。

Agent 需要知道：

- 目标是什么。
- 已经做了什么。
- 哪些工具结果可用。
- 哪些决定已经确认。
- 哪些信息以后还要用。

这些都属于状态管理。

## 2. 短期记忆：当前任务上下文

短期记忆通常放在上下文窗口里。

它记录当前会话和任务中的关键信息，比如刚刚读过的文件、刚运行的命令、用户最新要求。

问题是上下文窗口有限，内容太多会挤掉旧信息。

## 3. 长期记忆：跨任务保留信息

长期记忆用于保存以后还可能用到的信息。

比如：

- 用户偏好。
- 项目约定。
- 常用工具。
- 过去任务结论。

长期记忆通常不会全部塞进上下文，而是在需要时检索出来。

## 4. 状态不是越多越好

记忆太多也会带来问题：

- 检索到无关旧信息。
- 旧信息过期。
- 隐私风险增加。
- 模型被错误记忆带偏。

所以记忆系统要能写入、检索、更新和遗忘。

## 5. 好的状态管理像工作记录

一个可靠 Agent 应该能清楚知道：

- 当前任务到哪一步。
- 哪些假设已确认。
- 哪些结果来自工具。
- 哪些事情还没做。

这比让模型“凭感觉记住”可靠得多。
