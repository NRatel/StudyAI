# 多 Agent 与 MCP

> **前置知识**：[Agent 框架原理](1_Agent框架原理.md)
>
> **本节目标**：理解多 Agent 协作解决什么问题，以及 MCP 为什么和工具接入有关。

---

## 1. 多 Agent：把复杂任务分角色

多 Agent 系统会让多个模型或角色协作。

比如：

- 一个负责规划。
- 一个负责检索。
- 一个负责写代码。
- 一个负责审查。
- 一个负责汇总。

这像一个小团队，而不是一个人包办所有事。

## 2. 多 Agent 不是越多越好

多个 Agent 会带来额外成本：

- 通信变复杂。
- 容易重复工作。
- 可能互相误导。
- 调度和终止更难。

如果任务简单，一个 Agent 加工具就够了。多 Agent 更适合复杂、可拆分、需要交叉检查的任务。

## 3. MCP：让模型更标准地接工具

MCP 可以理解成一种让模型和外部工具、数据源交互的标准接口。

它关心的是：

- 工具有哪些能力。
- 输入输出是什么格式。
- 模型如何发现和调用工具。
- 数据源如何暴露给模型。

有了标准接口，工具接入不必每次从零定制。

## 4. 工具标准化为什么重要

Agent 真正有用时，往往要接很多外部系统。

如果每个系统都用不同方式接入，维护会很乱。

标准化接口能让：

- 工具更容易复用。
- 权限更容易管理。
- 调试更清晰。
- 系统更容易扩展。

## 5. 最重要的是边界

Agent 能调用工具，不代表可以随便调用。

需要明确：

- 哪些工具允许用。
- 哪些操作需要确认。
- 哪些数据不能访问。
- 错误操作如何回滚。

多 Agent 和 MCP 的目标不是制造复杂系统，而是让复杂任务更可控。
