# AI 及大模型原理 — 学习目录索引

这份索引的作用不是催你把所有章节一次性读完，而是帮你选路线。AI 原理很大，合理的读法是：先抓主干，再按兴趣和问题回补。

## 三条学习路径

### 路径 A：核心+应用路径

目标：先理解大语言模型到底怎么工作，再补上推理部署、RAG 和推理模型这些最常见的应用问题。

```
01 神经网络基础
  → 03 注意力机制与 Transformer
  → 04 预训练语言模型演进
  → 05 GPT 系列深度解析
  → 08 大模型推理与部署
  → 09 RAG 与应用架构
  → 11 推理与思维链
```

适合：主要关心 ChatGPT、Claude、DeepSeek、RAG、推理模型这些现代大模型现象的学习者。

如果只想先抓核心原理，可以读到 05 暂停：01 → 03 → 04 → 05。

提示：如果读 03 时对“编码器-解码器”“序列建模”没概念，再回补 02；如果读 08 时想知道模型如何训练出来，再补 07。

### 路径 B：完整原理路径

目标：把神经网络、语言模型、生成模型和多模态模型串成完整知识图。

```
01 神经网络基础
  → 02 经典网络架构
  → 03 注意力机制与 Transformer
  → 04 预训练语言模型演进
  → 05 GPT 系列深度解析
  → 06 生成模型与扩散模型
  → 10 多模态模型
```

适合：希望系统理解“AI 模型谱系”的学习者，尤其是想同时理解文本、图像、视频生成路线的人。

### 路径 C：工程与前沿选读

目标：理解模型如何训练、部署、调用工具，以及当前架构如何继续演进。

```
07 大模型训练技术
08 大模型推理与部署
12 Agent 与工具使用
13 长文本与高效架构
```

适合：已经读过 GPT 主线，想继续理解工程系统或跟进前沿架构的人。13 属于高级选读，不建议放在 GPT 主线之前硬啃。

## 怎么判断自己该继续还是回补

读每个模块时，只要能回答模块 README 里的“学完本模块，你应该能回答”问题，就可以继续往后走。不必追求把每个公式、每个工程细节都一次性吃透。

如果连续两节都出现“术语认识但关系不清”的感觉，先回到最近的前置模块补地基。AI 学习最怕的不是慢，而是带着一串半懂的词继续往前冲。
