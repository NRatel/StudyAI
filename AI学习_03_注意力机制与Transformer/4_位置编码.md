## 1. 直觉与概述

### 1.1 问题：Self-Attention 是"位置盲"的

回顾 Self-Attention 的核心计算：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

这个计算有一个关键性质：**置换不变性**（permutation invariance）。把 "I love AI" 打乱成 "AI love I"，Self-Attention 对每个词的输出（经过适当重排后）完全一样——它只关心词的内容，不关心词出现在哪个位置。

但语言显然是有顺序的："狗咬人" 和 "人咬狗" 意思完全不同。如果模型无法区分词序，就无法理解语言。

### 1.2 解决方案：给每个位置注入"我在哪"的信息

**类比**：教室里的学生都戴着面具（只有内容信息，没有位置信息）。解决办法：给每个座位贴编号，让学生把编号别在胸前。这样讨论时，不仅知道对方说了什么，还知道对方坐在哪里。

位置编码（Positional Encoding）就是这个"座位编号"。注入方式有两种：

1. **加法注入**：$\text{input} = \text{token\_embedding} + \text{position\_encoding}$
2. **旋转注入**：在计算 Attention 时旋转 Q/K 向量（RoPE 的做法）

### 1.3 三种主流方案一览

| 方案 | 代表模型 | 核心思想 | 注入方式 |
|------|---------|----------|---------|
| 正弦位置编码（Sinusoidal） | 原始 Transformer (2017) | 用不同频率的正弦/余弦函数编码位置 | 加到嵌入上 |
| 可学习位置编码（Learned） | BERT, GPT-2 | 直接学一个位置嵌入表 | 加到嵌入上 |
| 旋转位置编码（RoPE） | LLaMA, GPT-NeoX, Qwen | 用旋转矩阵把位置编码为角度 | 旋转 Q/K 向量 |

## 2. 严谨定义与原理

### 2.1 正弦位置编码（Sinusoidal Positional Encoding）

#### 2.1.1 公式

对于位置 $pos$ 和维度 $i$（$i = 0, 1, \dots, d/2 - 1$）：

$$PE(pos, 2i) = \sin\left(\frac{pos}{10000^{2i/d}}\right), \quad PE(pos, 2i+1) = \cos\left(\frac{pos}{10000^{2i/d}}\right)$$

其中 $d$ 是模型维度。结果是一个 $(max\_len, d)$ 的矩阵，每一行是一个位置的编码向量。

| 符号 | 含义 |
|------|------|
| $pos$ | 词在序列中的位置，从 0 开始 |
| $i$ | 维度索引（每两个维度共享一个频率） |
| $10000^{2i/d}$ | 周期，随维度增大而增大 |

#### 2.1.2 直觉：为什么用正弦和余弦？

**直觉 1：不同频率捕捉不同尺度的位置关系**

$10000^{2i/d}$ 随 $i$ 增大而指数级增长：低维度周期短、变化快（捕捉相邻词的差异），高维度周期长、变化慢（捕捉远距离的相对位置）。

类比：钟表的秒针（高频）、分针（中频）、时针（低频），组合起来能唯一表示任何时间点。

```
维度 0-1 (高频):  ▁▂▃▄▅▆▇█▇▆▅▄▃▂▁▂▃▄▅▆▇█▇▆▅▄▃▂▁  (变化快)
维度 2-3 (中频):  ▁▁▂▃▃▄▅▅▆▇▇█▇▇▆▅▅▄▃▃▂▁▁▂▃▃▄▅▅▆  (变化中等)
维度 d-2,d-1(低频): ▁▁▁▁▁▂▂▂▂▂▃▃▃▃▃▄▄▄▄▄▅▅▅▅▅▆▆▆▆▆  (变化慢)

组合所有维度 → 每个位置有唯一的"指纹"
```

**直觉 2：相对位置可以用线性变换表示**

这是正弦编码最精妙的性质。对于任意固定偏移 $k$，存在一个只依赖 $k$ 的线性变换矩阵 $M_k$，使得 $PE(pos + k) = M_k \cdot PE(pos)$。

利用三角函数的和角公式，对每一对 sin/cos 维度（频率 $\omega$）：

$$\begin{bmatrix} \sin((pos+k)\omega) \\ \cos((pos+k)\omega) \end{bmatrix} = \begin{bmatrix} \cos(k\omega) & \sin(k\omega) \\ -\sin(k\omega) & \cos(k\omega) \end{bmatrix} \begin{bmatrix} \sin(pos \cdot \omega) \\ \cos(pos \cdot \omega) \end{bmatrix}$$

这意味着模型可以通过学习线性变换来"关注相对位置为 $k$ 的词"——"the cat sat" 中 "cat" 和 "sat" 的语法关系只取决于它们的距离，不取决于绝对位置。

#### 2.1.3 使用方式

正弦编码是**加法注入**，且是**固定的**（不参与训练）：

$$\text{input}_i = \text{Embedding}(token_i) + PE(i)$$

### 2.2 可学习位置编码（Learned Positional Embedding）

BERT 和 GPT-2 采用更直接的方式：`PositionEmbedding` 是一个 $(max\_len, d_{model})$ 的可训练参数矩阵。

```python
self.position_embedding = nn.Embedding(max_len, d_model)  # 比如 (512, 768)
positions = torch.arange(seq_len)
x = token_emb + self.position_embedding(positions)
```

| 对比维度 | 正弦编码 | 可学习编码 |
|---------|---------|-----------|
| 参数 | 无（固定公式） | $max\_len \times d_{model}$ |
| 理论性质 | 相对位置可线性表示 | 无理论保证 |
| 外推能力 | 公式可泛化，但实际效果退化 | **无法外推**——超过 $max\_len$ 直接报错 |
| 实际效果 | 两者几乎一样 | 两者几乎一样 |

可学习编码的致命问题：`max_len` 是死的。训练时 `max_len=512`，推理遇到 513 个 token 就崩溃。这推动了 RoPE 等方案的诞生。

### 2.3 RoPE 旋转位置编码（Rotary Position Embedding）

RoPE（Su et al. 2021）已成为现代大语言模型的标配（LLaMA、Qwen、Mistral 等几乎全部使用）。

#### 2.3.1 核心思想

不修改嵌入，而是在计算 Attention 时**旋转** Q 和 K 向量，使得点积 $\tilde{q}_m^T \tilde{k}_n$ 只依赖于内容和**相对位置** $m - n$。

#### 2.3.2 公式推导

**二维情况**：把 $q = [q_0, q_1]$ 看作复平面上的点，乘以单位复数 $e^{im\theta}$ 得到旋转：

$$\begin{bmatrix} \tilde{q}_0 \\ \tilde{q}_1 \end{bmatrix} = \begin{bmatrix} \cos(m\theta) & -\sin(m\theta) \\ \sin(m\theta) & \cos(m\theta) \end{bmatrix} \begin{bmatrix} q_0 \\ q_1 \end{bmatrix}$$

旋转后的点积：$\tilde{q}_m^T \tilde{k}_n = q^T R((n-m)\theta) k$，只依赖 $m - n$。

**推广到高维**：把 $d$ 维向量看成 $d/2$ 个二维子空间，每个子空间用不同频率独立旋转：

$$\theta_i = 10000^{-2i/d}, \quad i = 0, 1, \dots, d/2 - 1$$

实际实现用逐元素操作（不构造大矩阵）：

$$\tilde{q}_{2i} = q_{2i} \cos(m\theta_i) - q_{2i+1} \sin(m\theta_i), \quad \tilde{q}_{2i+1} = q_{2i} \sin(m\theta_i) + q_{2i+1} \cos(m\theta_i)$$

#### 2.3.3 RoPE 的关键优势

1. **天然的相对位置编码**：在数学上保证点积只依赖相对位置
2. **不增加额外参数**：旋转角度由位置和固定频率决定
3. **更好的长度外推**：编码相对位置，天然支持长文本扩展
4. **实现高效**：只有逐元素的乘法和加法

### 2.4 三种位置编码的对比总结

| 对比维度 | 正弦编码 | 可学习编码 | RoPE |
|---------|---------|-----------|------|
| 类型 | 绝对位置 | 绝对位置 | 相对位置 |
| 注入方式 | 加到嵌入上 | 加到嵌入上 | 旋转 Q/K 向量 |
| 额外参数 | 0 | $max\_len \times d$ | 0 |
| 外推能力 | 一般 | 差 | 较好 |
| 当前使用状况 | 较少 | 部分模型 | **大模型标配** |

> **一句话理清三者关系**：绝对位置编码（正弦、可学习）给每个位置分配一个固定的编码向量，模型需要自己从中学会理解距离；相对位置编码只关心两个位置之间的距离，更符合语言中"关系取决于间距而非坐标"的直觉；RoPE 用旋转矩阵巧妙地实现了相对位置编码——在数学上保证 $Q$ 和 $K$ 的点积只依赖于二者的位置差 $m - n$，无需额外参数，且天然支持长度外推。

## 3. Python 代码示例

### 3.1 numpy 实现正弦位置编码并可视化

```python
import numpy as np

# ============================================================
# 正弦位置编码：numpy 实现 + 性质验证
# ============================================================

def sinusoidal_position_encoding(max_len, d_model):
    """计算正弦位置编码矩阵，形状 (max_len, d_model)"""
    assert d_model % 2 == 0, "d_model 必须是偶数"
    PE = np.zeros((max_len, d_model))
    pos = np.arange(max_len)[:, np.newaxis]     # (max_len, 1)
    i = np.arange(0, d_model, 2)                # [0, 2, 4, ..., d-2]
    # exp-log 技巧避免大数: 10000^(2i/d) = exp(2i/d * ln(10000))
    div_term = np.exp(i * (-np.log(10000.0) / d_model))
    angles = pos * div_term                      # (max_len, d/2)
    PE[:, 0::2] = np.sin(angles)
    PE[:, 1::2] = np.cos(angles)
    return PE

# --- 生成与基本观察 ---
PE = sinusoidal_position_encoding(max_len=128, d_model=64)
print(f"形状: {PE.shape}")
print(f"位置 0（前 8 维）: {PE[0, :8].round(4)}")  # sin(0)=0, cos(0)=1 交替
print(f"位置 1（前 8 维）: {PE[1, :8].round(4)}")

# --- 验证线性变换性质 ---
print("\n=== 相对位置的线性变换 ===")
k = 3
omega = 1.0 / (10000.0 ** (0.0 / 64))  # 第一对维度的频率
R_k = np.array([[np.cos(k*omega), np.sin(k*omega)],
                [-np.sin(k*omega), np.cos(k*omega)]])

for pos in [0, 5, 10, 20]:
    actual = PE[pos + k, :2]
    predicted = R_k @ PE[pos, :2]
    error = np.linalg.norm(actual - predicted)
    print(f"  pos={pos:2d}, k={k}: 误差={error:.1e}")  # 应为 ~1e-16

# --- 验证距离与相似度的关系 ---
print("\n=== 余弦相似度 vs 距离 ===")
def cosine_sim(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

for offset in [1, 2, 5, 10, 25, 50]:
    sim = cosine_sim(PE[50], PE[50 + offset])
    bar = "#" * int(max(0, sim) * 30)
    print(f"  距离 {offset:2d}: {sim:+.4f}  {bar}")

# --- 热力图可视化 ---
try:
    import matplotlib.pyplot as plt
    fig, axes = plt.subplots(1, 2, figsize=(12, 5))

    axes[0].imshow(PE[:64, :], aspect='auto', cmap='RdBu_r', vmin=-1, vmax=1)
    axes[0].set_xlabel('Dimension'); axes[0].set_ylabel('Position')
    axes[0].set_title('PE Heatmap')

    for dim_idx in [0, 4, 16, 32, 62]:
        axes[1].plot(PE[:, dim_idx], label=f'dim {dim_idx}', alpha=0.8)
    axes[1].set_xlabel('Position'); axes[1].set_title('Different Frequencies')
    axes[1].legend(fontsize=8); axes[1].grid(True, alpha=0.3)

    plt.tight_layout()
    plt.savefig('sinusoidal_pe.png', dpi=150, bbox_inches='tight')
    plt.show()
except ImportError:
    print("\n(matplotlib 未安装, 跳过可视化)")
```

### 3.2 PyTorch 实现 RoPE

```python
import torch
import math

# ============================================================
# RoPE 实现（参考 LLaMA 的写法）
# ============================================================

def precompute_freqs_cis(dim: int, max_len: int, theta: float = 10000.0):
    """预计算旋转频率（复数形式 e^{i*m*theta_j}）"""
    freqs = 1.0 / (theta ** (torch.arange(0, dim, 2).float() / dim))  # (dim//2,)
    t = torch.arange(max_len, dtype=torch.float32)
    angles = torch.outer(t, freqs)                          # (max_len, dim//2)
    return torch.polar(torch.ones_like(angles), angles)     # 复数张量

def apply_rotary_emb(xq, xk, freqs_cis):
    """
    对 query 和 key 应用 RoPE 旋转。
    xq/xk: (batch, seq_len, n_heads, head_dim)
    freqs_cis: (seq_len, head_dim // 2)
    """
    # 实数 → 复数（把相邻两维看作一个复数的实部和虚部）
    xq_c = torch.view_as_complex(xq.float().reshape(*xq.shape[:-1], -1, 2))
    xk_c = torch.view_as_complex(xk.float().reshape(*xk.shape[:-1], -1, 2))

    # 广播: (1, seq_len, 1, head_dim//2)
    freqs_cis = freqs_cis.unsqueeze(0).unsqueeze(2)

    # 核心: 复数乘法 = 旋转!
    xq_out = torch.view_as_real(xq_c * freqs_cis).flatten(-2)
    xk_out = torch.view_as_real(xk_c * freqs_cis).flatten(-2)
    return xq_out.type_as(xq), xk_out.type_as(xk)

# --- 验证 ---
head_dim = 16
freqs_cis = precompute_freqs_cis(dim=head_dim, max_len=128)

torch.manual_seed(42)
xq = torch.randn(2, 10, 4, head_dim)
xk = torch.randn(2, 10, 4, head_dim)
xq_rot, xk_rot = apply_rotary_emb(xq, xk, freqs_cis[:10])

# 性质 1: 旋转不改变向量的模
print("=== 旋转不改变模 ===")
for i in range(3):
    before = xq[0, i, 0].norm().item()
    after = xq_rot[0, i, 0].norm().item()
    print(f"  位置 {i}: |q|={before:.4f} → {after:.4f}, 差={abs(before-after):.1e}")

# 性质 2: 点积只依赖相对位置
print("\n=== 点积编码相对位置 ===")
q_same = torch.randn(1, 1, 1, head_dim).expand(1, 20, 1, head_dim).clone()
k_same = torch.randn(1, 1, 1, head_dim).expand(1, 20, 1, head_dim).clone()
q_rot, k_rot = apply_rotary_emb(q_same, k_same, freqs_cis[:20])

print("  固定相对距离 = 1:")
for m in [0, 5, 10, 15]:
    dot = (q_rot[0, m, 0] * k_rot[0, m+1, 0]).sum().item()
    print(f"    q_{m}^T k_{m+1} = {dot:.4f}")

print("  固定相对距离 = 3:")
for m in [0, 5, 10]:
    dot = (q_rot[0, m, 0] * k_rot[0, m+3, 0]).sum().item()
    print(f"    q_{m}^T k_{m+3} = {dot:.4f}")

print("\n  相同距离 → 相同点积，证实 RoPE 编码的是相对位置。")
```

**预期输出**：

```
=== 旋转不改变模 ===
  位置 0: |q|=3.8547 → 3.8547, 差=0.0e+00

=== 点积编码相对位置 ===
  固定相对距离 = 1:
    q_0^T k_1 = 2.3145
    q_5^T k_6 = 2.3145
    q_10^T k_11 = 2.3145
    q_15^T k_16 = 2.3145
```

### 3.3 三种位置编码的 PyTorch 对比

```python
import torch
import torch.nn as nn
import math

# ============================================================
# 三种位置编码的对比
# ============================================================

class SinusoidalPE(nn.Module):
    """正弦位置编码（原始 Transformer）"""
    def __init__(self, d_model, max_len=5000):
        super().__init__()
        pe = torch.zeros(max_len, d_model)
        pos = torch.arange(max_len, dtype=torch.float).unsqueeze(1)
        div = torch.exp(torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model))
        pe[:, 0::2] = torch.sin(pos * div)
        pe[:, 1::2] = torch.cos(pos * div)
        self.register_buffer('pe', pe.unsqueeze(0))  # (1, max_len, d)

    def forward(self, x):
        return x + self.pe[:, :x.size(1), :]

class LearnedPE(nn.Module):
    """可学习位置编码（BERT / GPT-2 风格）"""
    def __init__(self, d_model, max_len=512):
        super().__init__()
        self.pos_emb = nn.Embedding(max_len, d_model)
        self.max_len = max_len

    def forward(self, x):
        seq_len = x.size(1)
        assert seq_len <= self.max_len, f"长度 {seq_len} > max_len {self.max_len}，无法外推！"
        return x + self.pos_emb(torch.arange(seq_len, device=x.device))

# --- 对比参数量 ---
d_model = 64
sin_pe = SinusoidalPE(d_model, max_len=128)
learned_pe = LearnedPE(d_model, max_len=128)

print("--- 参数量 ---")
for name, m in [("正弦编码", sin_pe), ("可学习编码", learned_pe)]:
    trainable = sum(p.numel() for p in m.parameters() if p.requires_grad)
    buffers = sum(b.numel() for b in m.buffers())
    print(f"  {name}: 可训练={trainable:,}, buffer={buffers:,}")

# --- 验证位置编码打破置换不变性 ---
x = torch.randn(1, 5, d_model)
perm = torch.tensor([3, 1, 4, 0, 2])

out_normal = sin_pe(x)
out_shuffled = sin_pe(x[:, perm, :])

diff = (out_normal[0, 0] - out_shuffled[0, 3]).abs().max().item()
print(f"\n--- 置换不变性测试 ---")
print(f"  Token 在位置 0 vs 位置 3 的输出差异: {diff:.6f}")
print(f"  {'不同 → 位置编码生效!' if diff > 1e-5 else '相同 → 位置编码未生效'}")

# --- 可学习编码的外推限制 ---
small_pe = LearnedPE(d_model=64, max_len=10)
try:
    small_pe(torch.randn(1, 15, 64))
    print("\n  长度 15 (max_len=10): 正常")
except AssertionError as e:
    print(f"\n  长度 15 (max_len=10): 报错 — {e}")
```

**预期输出**：

```
--- 参数量 ---
  正弦编码: 可训练=0, buffer=8,192
  可学习编码: 可训练=8,192, buffer=0

--- 置换不变性测试 ---
  Token 在位置 0 vs 位置 3 的输出差异: 1.068242
  不同 → 位置编码生效!

  长度 15 (max_len=10): 报错 — 长度 15 > max_len 10，无法外推！
```

## 4. 工程师视角

### 4.1 绝对位置 vs 相对位置编码

**绝对位置编码**（正弦、可学习）：给每个位置分配固定向量，加到嵌入上。模型需要自己学会理解"距离"。

**相对位置编码**（RoPE、Transformer-XL 等）：直接编码两个位置之间的距离。更符合语言直觉——"cat" 和 "sat" 的语法关系不取决于绝对位置。

现代大模型的实践已给出答案：**RoPE 全面胜出**。原因：语言关系基于相对位置、天然支持长度外推、实现效率高。

### 4.2 长度外推（Length Extrapolation）

**问题**：训练时只见过长度 $L_{train}$ 的序列，推理时能否处理 $L_{test} > L_{train}$？

| 编码方案 | 外推行为 |
|---------|---------|
| 可学习编码 | 完全不能。超过 max_len 直接报错 |
| 正弦编码 | 公式可计算任意位置，但 Attention 模式崩溃 |
| RoPE | 较好，但仍需配合扩展技术 |

**RoPE 的长度扩展技术**（详见模块 13）：

| 技术 | 核心思想 |
|------|---------|
| **Position Interpolation (PI)** | 将推理位置索引线性压缩到训练范围。训练 2k → 推理 4k：位置 0\~4095 映射到 0\~2047 |
| **NTK-Aware Scaling** | 调整频率底数（10000 → 更大值），扩展低频周期同时保持高频分辨率 |
| **YaRN** | 结合 PI 和 NTK，不同频率维度用不同插值策略 |
| **Dynamic NTK** | 根据实际长度动态调整底数 |

所有这些技术的前提：**使用 RoPE 作为基础**。这也是现代大模型几乎全部采用 RoPE 的原因。

### 4.3 ALiBi 简介

ALiBi（Attention with Linear Biases，Press et al. 2022）直接在 Attention score 上加线性偏置：

$$\text{Attention} = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}} + B\right) V, \quad B_{ij} = -m \cdot |i - j|$$

```
偏置矩阵 B (m=0.5):     直觉: 距离越远, 注意力被惩罚越多
  [ 0.0  -0.5  -1.0]     → 自然引入"近处的词更重要"的偏置
  [-0.5   0.0  -0.5]
  [-1.0  -0.5   0.0]
```

ALiBi 实现极简且外推能力强，被 BLOOM 等模型采用。但 RoPE + 扩展技术在长文本性能上已超过 ALiBi，目前 RoPE 是事实标准。

### 4.4 位置编码是长文本技术的基础

```
长文本能力的技术栈:

位置编码（基础层）← 本节
  └── RoPE + 长度扩展（PI, NTK, YaRN）
       └── 高效注意力（FlashAttention, 滑动窗口, 稀疏注意力）
            └── 长文本微调（渐进式扩展, 数据配比）
```

位置编码处于最底层——如果编码方案不支持长度扩展，上面的技术都无从施展。详见模块 13。

### 4.5 实际模型中的位置编码选择

| 模型 | 位置编码 | 训练长度 |
|------|---------|---------|
| Transformer (2017) | 正弦 | — |
| BERT (2018) | 可学习 | 512 |
| GPT-2 (2019) | 可学习 | 1024 |

趋势明确：**2023 年之后的主流大模型全部使用 RoPE**。

### 4.6 常见问题

**Q1: 为什么位置编码是"加"到 embedding 上，而不是"拼接"？**

拼接会增加维度（如 768 → 768 + PE 维度），导致后续所有层计算量增大。加法不改变维度，且 Attention 的线性投影足以学会分离位置和内容信息。

**Q2: RoPE 为什么只旋转 Q 和 K，不旋转 V？**

位置信息需要影响的是"哪些词互相关注"（通过 $QK^T$ 决定），而不是"关注到的内容是什么"（由 V 决定）。V 保持原样，确保聚合的内容不被位置信息污染。

**Q3: 正弦编码中底数 10000 有什么特别？**

它决定最低频维度的周期：$2\pi \times 10000 \approx 62832$ 个位置。足以覆盖常见序列长度，同时高频端有足够分辨率。RoPE 的长度扩展技术（NTK-Aware Scaling）正是通过调整这个底数来实现的。

## 本节小结

| 概念 | 一句话总结 |
|------|-----------|
| 置换不变性问题 | Self-Attention 不关心词序，需要额外注入位置信息 |
| 正弦位置编码 | 用不同频率的 sin/cos 编码位置，有优美的线性变换性质 |
| 可学习位置编码 | 直接学一个嵌入表，简单但无法外推 |
| RoPE | 旋转 Q/K 向量，天然编码相对位置，现代大模型标配 |
| 绝对 vs 相对编码 | 绝对编码给位置贴标签，相对编码关注位置间距离 |
| 长度外推 | 训练短推理长，RoPE + PI/NTK/YaRN 是主流方案 |
| ALiBi | Attention score 加线性偏置，简单但已被 RoPE 超越 |

**位置编码在知识体系中的位置**：

```
Self-Attention（内容交互） + 位置编码（顺序信息）  ← 本节
  = 完整的 Transformer 注意力机制
    → BERT, GPT-2（绝对位置编码时代）
      → LLaMA, Mistral, Qwen（RoPE 时代）
        → 长文本技术（模块 13）
```
