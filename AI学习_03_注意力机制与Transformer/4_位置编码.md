# 位置编码

> **前置知识**：[Transformer 架构](3_Transformer架构.md)
>
> **本节目标**：理解注意力机制为什么需要额外的位置信息，以及常见位置编码在解决什么问题。

---

## 1. 注意力本身不懂顺序

Self-Attention 会比较 token 之间的关系，但它本身不天然知道谁在前、谁在后。

如果不加位置信息，下面两句话里的词集合很像，但意思完全不同：

```text
狗咬人
人咬狗
```

语言的顺序非常重要，所以 Transformer 必须额外告诉模型位置。

## 2. 位置编码：给 token 加上座位号

位置编码可以理解成给每个 token 加一个“座位号”。

模型看到的不只是“这个词是什么”，还包括“这个词在第几个位置”。

最终输入大致是：

```text
词的表示 + 位置的表示
```

这样模型才能区分同样的词出现在不同位置时的含义。

## 3. 绝对位置和相对位置

绝对位置关心“这是第几个 token”。

相对位置更关心“两个 token 相隔多远”。语言里很多关系更依赖相对距离，比如修饰词和被修饰词通常不会离太远。

现代大模型常更重视相对位置信息，因为它更适合泛化到不同长度的文本。

## 4. RoPE：把位置放进旋转里

RoPE 是很多大模型使用的位置编码方式。

你可以把它理解成：token 的表示会根据位置做一种可控旋转。两个 token 之间的相对距离，会体现在它们表示的关系里。

不用一开始深究旋转公式。先记住它解决的是：

- 让模型知道顺序和距离。
- 更自然地表达相对位置。
- 方便扩展到较长上下文。

## 5. 长上下文为什么还会难

有了位置编码，不等于模型天然能处理无限长文本。

长文本还会遇到：

- 注意力计算成本上升。
- 训练时没见过那么长的序列。
- 远距离信息容易被大量内容稀释。

所以长上下文模型既需要位置编码，也需要训练策略、注意力优化和记忆机制配合。
