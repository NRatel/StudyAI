# Self-Attention 与 Multi-Head

> **前置知识**：[注意力机制](1_注意力机制.md)
>
> **本节目标**：理解一句话里的 token 如何互相看，以及多头注意力为什么能从多个角度理解关系。

---

## 1. Self-Attention：让词和词互相看

Self-Attention 的核心问题是：

> 这句话里的每个 token，应该从其他 token 那里拿多少信息？

比如“苹果发布了新手机，它很受欢迎”。模型看到“它”时，需要回头看“新手机”，而不是“苹果”这个水果含义。

## 2. Q、K、V：三个角色

Q、K、V 听起来抽象，其实可以这样理解：

- **Query**：我现在想找什么。
- **Key**：我这里有什么线索，能不能被你匹配上。
- **Value**：如果你觉得我有用，就从我这里拿走的信息。

每个 token 都会生成自己的 Q、K、V。

然后一个 token 用自己的 Q 去和其他 token 的 K 打分，分数越高，说明越相关。最后根据分数从 V 中汇总信息。

## 3. 为什么要打分再加权

句子里的词不是孤立的。

“银行”在“河边的银行”和“银行转账”里含义不同。Self-Attention 允许它根据周围词动态调整表示。

这就是上下文表示：同一个词，在不同上下文里会变成不同的向量。

## 4. Multi-Head：不只从一个角度看

一个注意力头可能擅长看主谓关系，另一个可能擅长看指代关系，还有一个可能关注局部短语。

Multi-Head Attention 就是让模型同时用多个注意力头观察句子。

它不是简单“多算几遍”，而是给模型多个视角：

- 有的头看语法关系。
- 有的头看远距离指代。
- 有的头看局部搭配。

最后这些视角会合并，形成更丰富的表示。

## 5. 代价：看得全面也更贵

Self-Attention 的优点是每个 token 可以直接看其他 token。

代价是：序列越长，token 两两匹配的计算越多。长上下文为什么贵，根源之一就在这里。
