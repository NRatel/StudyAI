# 论文与FAQ

> 本文件用于补充注意力机制与 Transformer 的论文脉络和常见误区。正文优先讲清 Q/K/V、自注意力、多头、位置编码和 Transformer 架构；这里负责把常见疑问收束起来。

---

## 一、关键论文脉络

| 年份 | 论文/方法 | 为什么重要 |
|------|-----------|------------|
| 2014 | Bahdanau Attention | 让 Seq2Seq 解码时能动态关注输入的不同位置 |
| 2015 | Luong Attention | 简化并系统比较注意力打分方式，点积注意力影响很大 |
| 2017 | Transformer | 完全基于注意力的架构，替代 RNN 成为主线 |
| 2018 | Relative Position | 强调相对位置比绝对编号更符合语言直觉 |
| 2020 | Pre-LN 分析 | 解释 LayerNorm 位置为何影响深层 Transformer 稳定性 |
| 2021 | RoPE | 用旋转方式把位置信息注入 Q/K，现代 LLM 常用 |
| 2022 | FlashAttention | 不改注意力定义，优化长序列注意力计算 |

阅读建议：

1. 必读主线是 Bahdanau Attention → Transformer。
2. 位置编码只需先理解“为什么要有位置”和“绝对/相对的区别”。
3. FlashAttention、Pre-LN 这类内容作为进阶补充，不影响初次理解 Transformer。

## 二、常见误区与 FAQ

### Q1：Transformer 没有 RNN，怎么知道顺序？

自注意力本身确实不天然知道顺序。它看的是 token 之间的内容关系，而不是第几个 token。

所以 Transformer 必须加入**位置编码**：

```text
词向量：这个 token 是什么
位置编码：这个 token 在哪里
二者相加：既知道内容，也知道位置
```

RNN 通过一步步读取天然带顺序；Transformer 通过显式位置编码获得顺序信息。

### Q2：为什么要有 Q、K、V？

Q、K、V 是把“查找信息”拆成三个角色：

| 角色 | 直觉 |
|------|------|
| Q（Query） | 我现在想找什么 |
| K（Key） | 我有什么特征可供匹配 |
| V（Value） | 如果被关注，要提供什么内容 |

如果三者都直接用同一个输入向量，模型很难区分“匹配依据”和“取回内容”。Q/K/V 线性变换让模型能在不同空间里分别学习这些角色。

### Q3：为什么要多头注意力？

单头注意力像一个人只从一个角度看句子。多头注意力让模型同时从多个角度看：

- 一个头可能关注主谓关系。
- 一个头可能关注相邻词。
- 一个头可能关注长距离指代。

多头不是简单“多算几遍”，而是让不同子空间学习不同关系，再把结果合并。

### Q4：为什么注意力要除以 $\sqrt{d_k}$？

点积维度越高，数值越容易变大。数值太大送进 Softmax，会让概率分布变得过于尖锐，梯度变弱。

除以 $\sqrt{d_k}$ 的作用是把分数拉回更合适的尺度：

```text
不缩放：Softmax 太自信，学习困难
缩放后：分布更平滑，梯度更稳定
```

不用记复杂推导，只要记住：这是为了让注意力分数的尺度稳定。

### Q5：Self-Attention 和 Cross-Attention 有什么区别？

| 类型 | Q 来自哪里 | K/V 来自哪里 | 直觉 |
|------|------------|--------------|------|
| Self-Attention | 当前序列 | 当前序列 | 句子内部互相看 |
| Cross-Attention | 解码器当前状态 | 编码器输出 | 生成时回头看输入 |

BERT/GPT 的核心主要是 Self-Attention；传统 Encoder-Decoder 翻译模型里会用到 Cross-Attention。

### Q6：Pre-LN 和 Post-LN 有必要现在掌握吗？

只需要知道结论即可。

- **Post-LN**：原始 Transformer 中常见，先做子层，再做 LayerNorm。
- **Pre-LN**：现代深层 Transformer 中常见，先做 LayerNorm，再做子层。

Pre-LN 通常让深层模型更稳定，所以现代大模型更常采用它。具体梯度分析属于进阶内容，不影响本模块主线。

### Q7：Transformer 的参数主要在哪里？

参数主要来自三类地方：

1. Embedding 层。
2. Attention 中的 Q/K/V/O 投影矩阵。
3. FFN 前馈网络。

一个常见误解是“序列越长，参数越多”。不对。序列长度主要影响计算量和显存，不直接增加模型参数。

### Q8：位置编码为什么不直接用 1、2、3？

直接用整数位置有两个问题：

- 数值范围会随长度变大，容易干扰词向量尺度。
- 它很难表达“相邻”“相隔 3 个位置”这类相对关系。

正弦位置编码、相对位置编码、RoPE 都是在解决同一个问题：让模型既知道 token 的位置，又能更自然地理解位置之间的关系。

### Q9：FlashAttention 是不是新的注意力机制？

不是。它不改变注意力的数学定义。

FlashAttention 主要改变计算组织方式，让长序列注意力更快、更省内存。初学时可以把它放在“高效计算”类别里，不必把它当成新的原理章节。
