# 注意力机制

## 风趣易懂的直觉讲解
### 从"压缩整句话"到"按需回头看"

在传统 Seq2Seq 中，编码器要把整句输入压缩成一个固定向量。句子越长，信息越容易挤丢。

注意力机制的想法很简单：

> 解码器每生成一步，都可以回头看输入序列的所有位置，并决定此刻最该关注哪里。

翻译 "The cat sat on the mat" 时，如果正在生成 "坐"，模型应该重点看 "sat"，而不是平均看所有词。

### 注意力的本质：可微的软查找

把注意力想成查资料：

- **Query（查询）**：我现在想找什么？
- **Key（索引）**：每条资料如何被匹配？
- **Value（内容）**：匹配上后真正取走什么？

注意力不是只选一个位置，而是给所有位置分配权重，再做加权平均。这叫**软寻址**：既能学习，又能反向传播。

## 准确概念定义与核心原理
### 1. 通用注意力流程

给定一个 Query $q$，一组 Key $\{k_i\}$ 和 Value $\{v_i\}$，注意力分三步：

1. **算相似度**：$q$ 和每个 $k_i$ 打分。
2. **转成权重**：用 softmax 把分数变成和为 1 的权重。
3. **加权求和**：用权重汇总所有 $v_i$。

公式：

$$\alpha_i = \text{softmax}(\text{score}(q, k_i))$$

$$\text{Attention}(q, K, V) = \sum_i \alpha_i v_i$$

直觉：越相关的位置权重越大，对最终输出贡献越大。

### 2. Q、K、V 为什么要分开

在自然语言里，同一个 token 可以承担不同角色。

例如在句子 "动物没有过马路，因为它太累了" 中，"它"需要找前文中最相关的指代对象。此时：

- Query：当前位置在问"我指的是谁？"
- Key：前文每个词提供"我是什么线索"
- Value：真正被取回的信息

Q/K/V 分开后，模型可以分别学习：

- 如何提出问题
- 如何被别人匹配
- 匹配后贡献什么信息

### 3. 缩放点积注意力

Transformer 使用的是**缩放点积注意力**：

$$\text{Attention}(Q,K,V) =
\text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

逐项理解：

| 部分 | 含义 |
|------|------|
| $QK^T$ | 每个 Query 和每个 Key 的相似度 |
| $\sqrt{d_k}$ | 防止点积数值过大，让 softmax 不至于过早饱和 |
| softmax | 把相似度变成注意力权重 |
| 乘以 $V$ | 按权重汇总真正内容 |

为什么要除以 $\sqrt{d_k}$？

如果向量维度很高，点积值会天然变大。softmax 遇到很大的数会变得极端：最大位置接近 1，其他位置接近 0，梯度会变差。缩放因子让分数保持在更稳定的范围。

### 4. 注意力和 RNN 的关键区别

| 对比 | RNN | 注意力 |
|------|-----|--------|
| 信息传递 | 逐步传递隐藏状态 | 任意位置可直接互相查看 |
| 长距离依赖 | 路径长，容易梯度消失 | 路径短，更容易建立联系 |
| 并行性 | 时间步依赖前一步 | 训练时各位置可并行计算 |
| 代价 | 对长序列较线性 | 标准注意力随序列长度平方增长 |

注意力不是没有代价。标准注意力要计算所有 token 两两关系，所以长序列会变贵。这也是后续长文本模型要重点优化的问题。

## 常见误区

**误区 1：注意力就是解释模型为什么这么判断。**

不完全是。注意力权重有一定可解释性，但不能简单等同于模型的全部推理原因。

**误区 2：Value 不重要，关键只看 Query 和 Key。**

Q/K 决定"看哪里"，V 决定"拿什么"。没有 V，注意力只是在打分，没有真正的信息汇总。

**误区 3：注意力一定比 RNN 更适合所有序列。**

不一定。注意力擅长并行和长距离交互，但标准注意力的序列长度平方成本很高。

## 本节小结

| 概念 | 一句话总结 |
|------|------------|
| Attention | 根据相关性对信息加权汇总 |
| Query | 当前要找什么 |
| Key | 每个位置如何被匹配 |
| Value | 匹配后真正取出的内容 |
| Softmax | 把相似度变成权重分布 |
| 缩放因子 | 防止高维点积让 softmax 饱和 |
