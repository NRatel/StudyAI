## 1. 直觉与概述

### 1.1 从 Bahdanau Attention 到通用注意力

上一模块中，我们学到 Bahdanau Attention 的核心操作：

```
解码器当前状态 s_{i-1}  ───→  与编码器所有隐藏状态 h_1, h_2, ..., h_m 比较
                                ↓
                          得到注意力权重 α_{i,1}, α_{i,2}, ..., α_{i,m}
                                ↓
                          加权求和得到上下文向量 c_i
```

现在退一步，把这个过程抽象成更一般的形式：

| Bahdanau 中的角色 | 通用注意力中的名字 | 做什么 |
|---|---|---|
| 解码器状态 $s_{i-1}$ | **Query（查询）** | "我想找什么？" |
| 编码器隐藏状态 $h_j$ | **Key（键）** | "我有什么可以被匹配的标签？" |
| 编码器隐藏状态 $h_j$ | **Value（值）** | "匹配成功后取出的内容" |

在 Bahdanau 中，Key 和 Value 恰好是同一个东西（都是编码器隐藏状态）。但在更一般的框架里，**Key 和 Value 可以不同**——Key 负责"被查找"，Value 负责"被提取"。

### 1.2 注意力的本质：可微的软寻址

最精确的一句话定义：**注意力是一种可微分的软寻址机制**。

类比：在图书馆用你的问题（Query）匹配每本书的标题（Key），按相关度给所有书打分，然后把所有书的内容（Value）按分数加权平均——不是只取最相关的一本，而是所有书都取，只是权重不同。

| 硬寻址（hash table） | 软寻址（注意力） |
|--------|----------------|
| 精确匹配一个 key | 与所有 key 计算相似度 |
| 返回一个 value | 返回所有 value 的加权平均 |
| 不可微（无法反向传播） | 可微（softmax 是连续函数） |

### 1.3 全景图

```
Bahdanau Attention（已学）
  │
  ├─ 本质：加性 score 函数 + 软寻址
  ▼
通用注意力框架：Attention(Q, K, V) ← 本节
  │
  ├─ 三种 score 函数：加性、点积、缩放点积
  ├─ 缩放点积注意力 → Transformer 的基础
  ▼
Self-Attention → Multi-Head Attention → Transformer（后续节）
```

## 2. 严谨定义与原理

### 2.1 通用注意力公式

给定一组查询 $Q$、键 $K$、值 $V$，注意力的通用计算过程为：

$$\text{Attention}(Q, K, V) = \text{softmax}\big(\text{score}(Q, K)\big) \cdot V$$

三步拆解：

1. **计算分数（score）**：衡量每个 Query 与每个 Key 的匹配程度
2. **归一化（softmax）**：将分数转为概率分布（非负、求和为 1），得到注意力权重
3. **加权求和**：用注意力权重对 Value 做加权平均，得到输出

核心差异在于 score 函数的选择。

### 2.2 三种 score 函数

设 query 向量 $\mathbf{q} \in \mathbb{R}^{d_q}$，key 向量 $\mathbf{k} \in \mathbb{R}^{d_k}$：

#### (1) 加性注意力（Additive / Bahdanau, 2014）

$$\text{score}(\mathbf{q}, \mathbf{k}) = \mathbf{v}^T \tanh(W_q \mathbf{q} + W_k \mathbf{k})$$

- $W_q \in \mathbb{R}^{d_a \times d_q}$，$W_k \in \mathbb{R}^{d_a \times d_k}$，$\mathbf{v} \in \mathbb{R}^{d_a}$ 都是可学习参数
- 优点：$\mathbf{q}$ 和 $\mathbf{k}$ 的维度可以不同
- 缺点：有额外的参数矩阵和非线性运算，计算较慢

#### (2) 点积注意力（Dot-Product / Luong, 2015）

$$\text{score}(\mathbf{q}, \mathbf{k}) = \mathbf{q}^T \mathbf{k}$$

- 无额外参数，要求 $d_q = d_k$，计算极其高效（纯矩阵乘法）

#### (3) 缩放点积注意力（Scaled Dot-Product / Vaswani, 2017）

$$\text{score}(\mathbf{q}, \mathbf{k}) = \frac{\mathbf{q}^T \mathbf{k}}{\sqrt{d_k}}$$

多了一个 $\frac{1}{\sqrt{d_k}}$ 的缩放因子。这就是 **Transformer 使用的 score 函数**。

### 2.3 为什么要除以 $\sqrt{d_k}$？

假设 $\mathbf{q}$ 和 $\mathbf{k}$ 的每个分量都是独立同分布的随机变量，均值为 0，方差为 1。则点积：

$$\mathbf{q}^T \mathbf{k} = \sum_{i=1}^{d_k} q_i k_i$$

每个 $q_i k_i$ 的期望为 0，方差为 1（因为 $\text{Var}(q_i k_i) = E[q_i^2]E[k_i^2] = 1$）。由于各分量独立：

$$\text{Var}(\mathbf{q}^T \mathbf{k}) = d_k$$

**点积的方差随维度线性增长**。当 $d_k = 64$（Transformer 常用值）时，点积值可达 $\pm 8$ 量级。此时 softmax 的行为：

```
输入量级适中 [-2, 1, 0, 3]  → softmax → [0.01, 0.04, 0.01, 0.94]  ← 梯度正常
输入量级过大 [-20, 10, 0, 30] → softmax → [0.00, 0.00, 0.00, 1.00]  ← 梯度消失
```

除以 $\sqrt{d_k}$ 后：$\text{Var}\left(\frac{\mathbf{q}^T \mathbf{k}}{\sqrt{d_k}}\right) = \frac{d_k}{d_k} = 1$，方差稳定在 1，softmax 输入保持合理范围。

> **工程直觉**：$\sqrt{d_k}$ 缩放是一种数值稳定性技巧，不改变注意力的语义，只保证训练能正常进行。类似思想在 Batch Norm、Layer Norm 中反复出现。

### 2.4 缩放点积注意力的完整公式

给定 $Q \in \mathbb{R}^{n \times d_k}$，$K \in \mathbb{R}^{m \times d_k}$，$V \in \mathbb{R}^{m \times d_v}$：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

逐步拆解矩阵维度：

| 步骤 | 计算 | 维度 | 含义 |
|------|------|------|------|
| 1 | $QK^T$ | $(n \times d_k)(d_k \times m) = n \times m$ | 每个 query 与每个 key 的点积分数 |
| 2 | $/\sqrt{d_k}$ | $n \times m$ | 缩放 |
| 3 | $\text{softmax}(\cdot)$ | $n \times m$ | 注意力权重（每行求和为 1） |
| 4 | $\cdot V$ | $(n \times m)(m \times d_v) = n \times d_v$ | 加权求和后的输出 |

其中 $n$ = query 序列长度，$m$ = key/value 序列长度。softmax 沿最后一个维度（key 维度 $m$）做归一化。

### 2.5 Q/K/V 的来源：线性投影

实际使用中，Q/K/V 通过**线性变换**从输入投影得到：

$$Q = X_q W^Q, \quad K = X_k W^K, \quad V = X_k W^V$$

其中 $W^Q \in \mathbb{R}^{d_{\text{model}} \times d_k}$，$W^K \in \mathbb{R}^{d_{\text{model}} \times d_k}$，$W^V \in \mathbb{R}^{d_{\text{model}} \times d_v}$ 是可学习的投影矩阵。

为什么需要投影？

1. **分工明确**：同一个向量在"被匹配"和"被提取"时承担不同角色，投影让模型学到不同的表示
2. **维度控制**：可以把高维输入映射到较低维的 Q/K/V 空间，减少计算量
3. **增加表达能力**：引入可学习参数，让模型决定"什么信息用于匹配、什么信息用于输出"

> **类比**：$W^K$ 学习提取"书的标题"（用于被查找），$W^V$ 学习提取"书的正文"（用于被读取），$W^Q$ 学习如何把问题转化为与标题可比较的形式。

### 2.6 三种 score 函数对比

| 特性 | 加性（Bahdanau） | 点积（Luong） | 缩放点积（Transformer） |
|------|----------------|-------------|----------------------|
| 公式 | $\mathbf{v}^T \tanh(W_q \mathbf{q} + W_k \mathbf{k})$ | $\mathbf{q}^T \mathbf{k}$ | $\frac{\mathbf{q}^T \mathbf{k}}{\sqrt{d_k}}$ |
| 额外参数 | $W_q$, $W_k$, $\mathbf{v}$ | 无 | 无 |
| 计算效率 | 低（tanh + 额外矩阵乘法） | 高 | 高 |
| 数值稳定性 | tanh 自带压缩 | 高维时 softmax 饱和 | 缩放解决饱和 |

实践中缩放点积注意力是绝对主流：**无额外参数、纯矩阵乘法可高度并行、数值稳定性有保证**。

## 3. Python 代码示例

### 3.1 用 numpy 实现缩放点积注意力

```python
import numpy as np

# ============================================================
# 缩放点积注意力的 numpy 实现
# Attention(Q, K, V) = softmax(Q @ K^T / sqrt(d_k)) @ V
# ============================================================

def softmax(x, axis=-1):
    """数值稳定的 softmax：减去最大值防止 exp 溢出"""
    x_max = np.max(x, axis=axis, keepdims=True)
    exp_x = np.exp(x - x_max)
    return exp_x / np.sum(exp_x, axis=axis, keepdims=True)

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    参数:
        Q: (n, d_k) 或 (batch, n, d_k)
        K: (m, d_k) 或 (batch, m, d_k)
        V: (m, d_v) 或 (batch, m, d_v)
        mask: (n, m)，True 的位置会被屏蔽
    返回: output, weights
    """
    d_k = Q.shape[-1]

    # 第一步: Q @ K^T → (n, m) 分数矩阵
    scores = Q @ K.swapaxes(-2, -1)

    # 第二步: 缩放
    scores = scores / np.sqrt(d_k)

    # 可选: 掩码（被屏蔽位置设为 -inf，softmax 后趋近 0）
    if mask is not None:
        scores = np.where(mask, -1e9, scores)

    # 第三步: softmax → 注意力权重
    weights = softmax(scores, axis=-1)

    # 第四步: 加权求和 → (n, d_v)
    output = weights @ V
    return output, weights


# === 测试 ===
np.random.seed(42)
seq_len, d_k = 4, 8

X = np.random.randn(seq_len, d_k)
Q = K = V = X  # Self-Attention 场景: Q/K/V 来自同一序列

output, weights = scaled_dot_product_attention(Q, K, V)

print("=== 缩放点积注意力（numpy）===\n")
print(f"输入 shape: {X.shape}, 输出 shape: {output.shape}")
print(f"\n注意力权重矩阵（每行求和为 1）:")
np.set_printoptions(precision=4, suppress=True)
print(weights)
print(f"每行之和: {weights.sum(axis=-1)}")
```

**预期输出**：

```
=== 缩放点积注意力（numpy）===

输入 shape: (4, 8), 输出 shape: (4, 8)

注意力权重矩阵（每行求和为 1）:
[[0.4180 0.1498 0.2340 0.1982]
 [0.1969 0.4549 0.1507 0.1975]
 [0.2434 0.1564 0.3755 0.2246]
 [0.2129 0.1995 0.2255 0.3620]]
每行之和: [1. 1. 1. 1.]
```

**观察**：对角线值较大（Q=K 时 $\mathbf{q}_i^T \mathbf{k}_i = \|\mathbf{q}_i\|^2$，自身相似度最高），每行是一个概率分布。

### 3.4 PyTorch 实现与官方 API 对比

```python
import torch
import torch.nn.functional as F
import numpy as np

def manual_scaled_dot_product_attention(Q, K, V, mask=None):
    """手写版本，与 numpy 版逻辑一致"""
    d_k = Q.size(-1)
    scores = torch.matmul(Q, K.transpose(-2, -1)) / (d_k ** 0.5)
    if mask is not None:
        scores = scores.masked_fill(mask == True, float('-inf'))
    weights = torch.softmax(scores, dim=-1)
    return torch.matmul(weights, V), weights

# === 无掩码对比 ===
torch.manual_seed(42)
batch, seq_len, d_k, d_v = 2, 6, 64, 64
Q = torch.randn(batch, seq_len, d_k)
K = torch.randn(batch, seq_len, d_k)
V = torch.randn(batch, seq_len, d_v)

out_manual, _ = manual_scaled_dot_product_attention(Q, K, V)
out_official = F.scaled_dot_product_attention(Q, K, V)  # PyTorch >= 2.0

diff = (out_manual - out_official).abs().max().item()
print(f"=== 手写 vs F.scaled_dot_product_attention ===")
print(f"最大绝对误差: {diff:.2e}  一致: {'是' if diff < 1e-5 else '否'}")

# === 因果掩码（causal mask）===
# 位置 i 只能看到 i 及之前的位置（解码器用）
causal_mask = torch.triu(torch.ones(seq_len, seq_len, dtype=torch.bool), diagonal=1)
print(f"\n因果掩码 (True=屏蔽):\n{causal_mask.int()}")

out_masked, weights_masked = manual_scaled_dot_product_attention(Q, K, V, mask=causal_mask)
out_masked_official = F.scaled_dot_product_attention(Q, K, V, is_causal=True)

diff_m = (out_masked - out_masked_official).abs().max().item()
print(f"\n带因果掩码 - 最大误差: {diff_m:.2e}  一致: {'是' if diff_m < 1e-5 else '否'}")
print(f"\n掩码后权重（第一个样本，下三角=可见区域）:")
np.set_printoptions(precision=3, suppress=True)
print(weights_masked[0].detach().numpy())
```

### 3.5 完整注意力层：带线性投影

```python
import torch
import torch.nn as nn

class SingleHeadAttention(nn.Module):
    """单头缩放点积注意力（带 Q/K/V 线性投影 + 输出投影）"""

    def __init__(self, d_model, d_k, d_v):
        super().__init__()
        self.W_Q = nn.Linear(d_model, d_k, bias=False)
        self.W_K = nn.Linear(d_model, d_k, bias=False)
        self.W_V = nn.Linear(d_model, d_v, bias=False)
        self.W_O = nn.Linear(d_v, d_model, bias=False)
        self.d_k = d_k

    def forward(self, X_q, X_kv, mask=None):
        """
        X_q:  (batch, n, d_model) — query 侧输入
        X_kv: (batch, m, d_model) — key/value 侧输入
              Self-Attention 时 X_q == X_kv;
              Cross-Attention 时 X_q 来自解码器, X_kv 来自编码器
        """
        Q = self.W_Q(X_q)     # (batch, n, d_k)
        K = self.W_K(X_kv)    # (batch, m, d_k)
        V = self.W_V(X_kv)    # (batch, m, d_v)

        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.d_k ** 0.5)
        if mask is not None:
            scores = scores.masked_fill(mask == True, float('-inf'))
        weights = torch.softmax(scores, dim=-1)
        context = torch.matmul(weights, V)

        return self.W_O(context), weights  # (batch, n, d_model)


# === 测试 ===
torch.manual_seed(42)
d_model, d_k, d_v = 32, 16, 16
attn = SingleHeadAttention(d_model, d_k, d_v)

# Self-Attention
X = torch.randn(2, 5, d_model)
out_self, w_self = attn(X, X)
print(f"Self-Attention:  输入 {X.shape} → 输出 {out_self.shape}")

# Cross-Attention
X_dec = torch.randn(2, 5, d_model)
X_enc = torch.randn(2, 8, d_model)
out_cross, w_cross = attn(X_dec, X_enc)
print(f"Cross-Attention: 解码器 {X_dec.shape} + 编码器 {X_enc.shape} → 输出 {out_cross.shape}")
print(f"  权重 shape: {w_cross.shape} (每个解码器位置对 8 个编码器位置的注意力)")

params = sum(p.numel() for p in attn.parameters())
print(f"\n参数量: {params} (W_Q:{d_model*d_k} + W_K:{d_model*d_k}"
      f" + W_V:{d_model*d_v} + W_O:{d_v*d_model})")
```

## 4. 工程师视角

### 4.1 Q/K/V 在不同场景下的物理含义

| 场景 | Query（Q） | Key（K） | Value（V） |
|------|-----------|---------|-----------|
| **Seq2Seq + Bahdanau** | 解码器当前隐藏状态 | 编码器所有隐藏状态 | = Key（K=V） |
| **Transformer Self-Attn** | 当前序列的投影 | 当前序列的投影 | 当前序列的投影（同源不同投影矩阵） |
| **Transformer Cross-Attn** | 解码器的投影 | 编码器输出的投影 | 编码器输出的投影 |
| **信息检索类比** | 搜索关键词 | 文档标题 | 文档内容 |

> **核心直觉**：Q 决定"要找什么"，K 决定"怎么被找到"，V 决定"找到后取什么"。Key 和 Value 的分离意味着"用于匹配的特征"和"匹配后提取的内容"不必是同一回事。

### 4.2 注意力的计算复杂度

缩放点积注意力的瓶颈在 $QK^T$。Self-Attention 中 $n = m$：

$$\text{时间复杂度} = O(n^2 \cdot d_k), \quad \text{空间复杂度} = O(n^2)$$

| 序列长度 $n$ | 注意力矩阵大小 | FP32 内存 |
|-------------|---------------|----------|
| 512 | 262 K | 1 MB |
| 2,048 | 4.2 M | 16 MB |
| 8,192 | 67 M | 256 MB |
| 32,768 | 1.07 G | 4 GB |
| 131,072 | 17.2 G | 64 GB |

$O(n^2)$ 增长极其恐怖，这就是早期 Transformer 上下文被限制在 512\~1024 的原因。

### 4.3 Flash Attention 简介

Flash Attention（Dao et al., 2022）是目前最重要的注意力加速技术，**不改变数学结果**，只优化内存访问模式。

**核心问题**：标准注意力的瓶颈不是计算量，而是内存带宽。每一步（$S = QK^T$、$P = \text{softmax}(S)$、$O = PV$）都要读写 GPU 主存（HBM），计算核心大部分时间在等数据。

**解法**：分块计算（tiling）+ 增量式 softmax（online softmax），让整个注意力计算在片上缓存（SRAM）中完成：

```
for 每块 Q_block:
    for 每块 K_block, V_block:
        在 SRAM 中计算小块的 scores、softmax、累加结果
    写回最终结果到 HBM

关键: 不需要物化完整的 (n, n) 注意力矩阵
```

| 对比 | 标准注意力 | Flash Attention |
|------|-----------|----------------|
| 数学结果 | $\text{softmax}(QK^T / \sqrt{d_k})V$ | **完全相同** |
| 内存 | $O(n^2)$ | $O(n)$ |
| 速度 | IO-bound | 接近 compute-bound，2\~4x 加速 |

PyTorch >= 2.0 中 `F.scaled_dot_product_attention` 自动选择最优后端（含 Flash Attention）：

```python
# 直接使用, PyTorch 自动选择最优内核
output = F.scaled_dot_product_attention(Q, K, V)
```

> **工程建议**：不要自己写注意力 CUDA kernel。Flash Attention 是"免费"加速——升级 PyTorch 即可获得。

### 4.4 常见陷阱

| 陷阱 | 说明 |
|------|------|
| **忘了缩放** | 不除以 $\sqrt{d_k}$，高维时 softmax 饱和，训练不动 |
| **softmax 维度搞错** | 应沿 key 维度（最后一维）做 softmax，搞成 query 维度语义完全错误 |
| **mask 布尔方向** | PyTorch `masked_fill` 中 True = 被屏蔽，有些实现相反，务必确认 |
| **数值溢出** | softmax 实现必须减去最大值（log-sum-exp trick），否则 `exp(大数) = inf` |

## 本节小结

| 概念 | 一句话总结 |
|------|-----------|
| 通用注意力 | $\text{Attention}(Q,K,V) = \text{softmax}(\text{score}(Q,K)) \cdot V$，可微的软寻址 |
| Query / Key / Value | Q 查询、K 匹配、V 提取；通过线性投影从输入得到 |
| 加性 score（Bahdanau） | $\mathbf{v}^T \tanh(W_q \mathbf{q} + W_k \mathbf{k})$，灵活但有额外参数 |
| 缩放点积 score（Transformer） | $\frac{\mathbf{q}^T \mathbf{k}}{\sqrt{d_k}}$，除以 $\sqrt{d_k}$ 稳定 softmax 梯度 |
| 线性投影 | $Q = XW^Q$，让模型学习"匹配用什么特征、输出取什么内容" |
| 计算复杂度 | $O(n^2 d)$，序列长度的二次方是核心瓶颈 |
| Flash Attention | 分块计算 + online softmax，不改变数学结果，2\~4x 加速 |

## 读完本章的自检清单

读完这一章你应该能做到以下三点，如果有任何一项感觉模糊，建议回到对应小节重新阅读：

1. **用自己的话解释 QKV**：Query 是"我想找什么"，Key 是"我有什么标签可以被匹配"，Value 是"匹配成功后实际取出的内容"。三者通过各自的线性投影矩阵从输入分化，分工不同但协同完成注意力计算。
2. **手写缩放点积注意力**：给定 Q、K、V 矩阵，能独立写出 $\text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V$ 的完整计算流程（包括 softmax 沿 key 维度归一化、数值稳定的 log-sum-exp 技巧）。
3. **理解 $\sqrt{d_k}$ 的作用**：点积的方差随维度 $d_k$ 线性增长，不缩放时 softmax 输入过大导致分布趋近 one-hot、梯度消失；除以 $\sqrt{d_k}$ 将方差稳定在 1，保证 softmax 输入在合理范围内，训练才能正常进行。
