## 1. 直觉与概述

### 1.1 从图像到视频：多了什么？

> **重要区分**：在进入视频多模态之前，请先明确两个完全不同的问题方向：
> - **视频理解**（输入视频 → 输出文字）：给模型一段视频，让它描述内容、回答问题、提取信息。核心挑战是时空建模——如何同时理解每帧的空间内容和帧间的时间动态。代表：VideoMAE、InternVideo、Video-LLaVA。
> - **视频生成**（输入文字 → 输出视频）：给模型一段文字描述，让它生成对应的视频。核心挑战是时间一致性——如何确保生成的每帧之间连贯、物体不突变。代表：Sora、CogVideoX、Wan。
>
> 这两个方向的技术栈几乎完全不同（理解侧基于视觉 Transformer，生成侧基于扩散模型），不要混为一谈。本节先讲视频理解（2.1），再讲视频生成（2.2），最后讲语音（2.3-2.4）。

前两节我们学了图像的多模态建模（ViT、CLIP、LLaVA）。视频在图像的基础上多了一个维度——**时间**。

| 模态 | 数据形状 | 本质 |
|------|---------|------|
| 图像 | (H, W, C) | 2D 空间信号 |
| 视频 | (T, H, W, C) | 2D 空间 + 1D 时间 = 3D 时空信号 |
| 语音 | (T, F) | 1D 时间 + 频率 = 2D 时频信号 |

视频理解的核心挑战是**时空建模**——不仅要理解每一帧"有什么"（空间），还要理解帧与帧之间"发生了什么"（时间）。

类比：图像理解是看一张照片，视频理解是看一段电影。你不仅要识别场景中的物体，还要理解动作、事件、因果关系。

### 1.2 语音的特殊性

语音与文本/图像有本质区别：

```
文本：  离散 token → 天然适合 Transformer
图像：  2D 像素 → ViT 切 patch → token 化
语音：  连续波形 → 需要先转为频谱图（Spectrogram）→ 再处理

语音波形 (1D, 16kHz)
    │
    ▼
短时傅里叶变换 (STFT)
    │
    ▼
梅尔频谱图 (2D: 时间 × 频率)  ← 类似于"图像"，可以用类似方法处理
    │
    ▼
Transformer 编码
```

## 2. 严谨定义与原理

### 2.1 视频理解：时空建模

#### 2.1.1 从图像 Transformer 到视频 Transformer

图像 ViT 处理的是 2D patch 序列。视频有额外的时间维度，需要处理 3D "时空管"（spatiotemporal tube）。

**视频 token 化**：

```
视频输入: (T, H, W, C)
    │
    ▼
切分为时空 patch (tubelet):
    每个 patch 覆盖 t × h × w 的时空区域
    │
    ▼
Token 数量: N = (T/t) × (H/h) × (W/h)
    │
    例如: 16 帧 × 224×224 图像, tubelet=2×16×16
    N = (16/2) × (224/16) × (224/16) = 8 × 14 × 14 = 1568 个 token
```

**核心问题**：1568 个 token 的全局注意力计算量是 $O(1568^2) \approx 250$ 万——对于更长的视频，这个数字会爆炸。

#### 2.1.2 时空注意力的三种策略

```
策略 1: 联合时空注意力（Joint Space-Time）
┌─────────────────────────┐
│ 所有时空 token 做全局注意力 │
│ 计算量: O((T×H×W)^2)     │  ← 最强但最贵
│ 代表: ViViT Model 1      │
└─────────────────────────┘

策略 2: 分解时空注意力（Divided Space-Time）
┌─────────────────────────┐
│ 先做空间注意力（每帧内部） │
│ 再做时间注意力（跨帧同位置）│
│ 计算量: O(T×(HW)^2 + HW×T^2) │  ← 大幅降低
│ 代表: TimeSformer          │
└─────────────────────────┘

策略 3: 窗口化注意力
┌─────────────────────────┐
│ 在局部时空窗口内做注意力   │
│ 类似 Swin Transformer    │
│ 计算量: O(T×H×W×w^3)     │  ← w 是窗口大小
│ 代表: Video Swin          │
└─────────────────────────┘
```

**TimeSformer 的分解注意力**：

```
输入: 视频 token 序列 (T 帧, 每帧 N 个空间 token)

每个 Transformer Block 内部:
┌──────────────────────────────────┐
│ Step 1: 时间注意力                 │
│   对每个空间位置，沿时间维度做注意力 │
│   (同一位置在不同帧之间交互)        │
│                                   │
│ Step 2: 空间注意力                 │
│   对每帧，在空间维度做注意力         │
│   (同一帧内的不同位置交互)          │
└──────────────────────────────────┘
```

形式化地，对位于时间 $t$、空间位置 $(i,j)$ 的 token $\mathbf{z}_{t,i,j}$：

$$\mathbf{z}_{t,i,j}' = \text{TemporalAttn}(\mathbf{z}_{t,i,j};\; \{\mathbf{z}_{t',i,j}\}_{t'=1}^{T}) \quad \text{(时间注意力：与同位置所有帧交互)}$$

$$\mathbf{z}_{t,i,j}'' = \text{SpatialAttn}(\mathbf{z}_{t,i,j}';\; \{\mathbf{z}_{t,i',j'}'\}_{i',j'}) \quad \text{(空间注意力：与同帧所有位置交互)}$$

#### 2.1.3 VideoMAE：视频自监督预训练

VideoMAE（Video Masked Autoencoder, 2022）将 MAE 的掩码预训练思路扩展到视频：

```
原始视频 → 切分为时空 patch → 随机掩码 90%~95% 的 patch
                                       │
                ┌──────────────────────┘
                │
        可见 patch (5%~10%)
                │
                ▼
          ViT Encoder
                │
                ▼
     Encoder 输出 + 掩码 token
                │
                ▼
          ViT Decoder
                │
                ▼
        重建被掩码的 patch
```

**关键发现**：视频中相邻帧高度相似（时间冗余），因此需要**极高的掩码率**（90-95%，而图像 MAE 只需 75%），迫使模型学习时间动态而非简单复制邻近帧。

### 2.2 Sora：视频生成的里程碑

#### 2.2.1 Sora 的技术架构

Sora（OpenAI, 2024.2 发布技术报告）将扩散模型和 Transformer 结合，实现高质量长视频生成。

```
文本 prompt: "一只猫在钢琴上弹奏"
        │
        ▼
  文本编码器 (推测: T5 或 CLIP)
        │
        ▼ 文本 embedding
        │
        ├──────────────────────────────┐
        │                              │
        ▼                              │
┌─────────────────────┐                │
│ 视频 VAE Encoder     │                │
│                     │                │
│ 视频 → 潜空间表示    │                │
│ (T,H,W,C) →        │                │
│ (t,h,w,c)           │  ← 大幅压缩    │
│                     │    时空分辨率    │
└────────┬────────────┘                │
         │                             │
         ▼                             │
┌─────────────────────┐                │
│ 时空 Patch 化        │                │
│                     │                │
│ 潜空间 → token 序列  │                │
│ (t,h,w,c) →        │                │
│ (N, d) 个 token     │                │
└────────┬────────────┘                │
         │                             │
         ▼                             │
┌──────────────────────────────────┐   │
│        DiT (Diffusion Transformer)│   │
│                                   │   │
│  输入: 噪声 token + 时间步 t       │   │
│  条件: 文本 embedding  ←──────────┼───┘
│                                   │
│  重复 T 步去噪:                    │
│  z_t → DiT → z_{t-1} → ... → z_0 │
│                                   │
└────────┬─────────────────────────┘
         │
         ▼
┌─────────────────────┐
│ 视频 VAE Decoder     │
│                     │
│ 潜空间 → 高清视频    │
│ (t,h,w,c) →        │
│ (T,H,W,C)           │
└─────────────────────┘
```

#### 2.2.2 Sora 的关键技术

| 技术 | 说明 |
|------|------|
| **时空 VAE** | 将高分辨率视频压缩到低维潜空间，同时在时间和空间维度压缩 |
| **时空 Patch** | 在潜空间中切 3D patch，而非在像素空间。极大减少 token 数 |
| **DiT 架构** | 用 Transformer 替代 U-Net 作为扩散模型的去噪骨干网络（参考模块 06） |
| **可变分辨率/时长** | 通过不固定 token 数量，支持任意分辨率和时长的视频生成 |
| **Scaling** | 推测使用了海量视频数据 + 大规模模型 + 长时间训练 |

#### 2.2.3 DiT 在视频生成中的应用

回顾 DiT（Diffusion Transformer，Peebles & Xie, 2023）的核心思想：

**图像 DiT**：

$$\mathbf{z}_{t-1} = \text{DiT}(\mathbf{z}_t, t, \mathbf{c})$$

其中 $\mathbf{z}_t$ 是第 $t$ 步的噪声潜表示，$t$ 是时间步，$\mathbf{c}$ 是文本条件。

**视频 DiT（Sora 的扩展）**：

$$\mathbf{Z}_{t-1} = \text{VideoDiT}(\mathbf{Z}_t, t, \mathbf{c})$$

其中 $\mathbf{Z}_t \in \mathbb{R}^{N_{st} \times d}$ 是时空 patch 的噪声序列，$N_{st} = N_t \times N_h \times N_w$ 是时空 token 总数。

VideoDiT 中的注意力可以是全局的（所有时空 token 互相关注）或分解的（先空间后时间），取决于视频长度和计算预算。

**条件注入**使用 AdaLN-Zero（Adaptive Layer Normalization）：

$$\gamma, \beta = \text{MLP}([\text{emb}(t),\; \mathbf{c}])$$
$$\hat{\mathbf{h}} = \gamma \odot \text{LayerNorm}(\mathbf{h}) + \beta$$

### 2.3 Whisper：通用语音识别

#### 2.3.1 Whisper 的架构

Whisper（Radford et al., 2022）是 OpenAI 发布的通用语音识别模型，采用标准的 Encoder-Decoder Transformer 架构：

```
语音输入 (16kHz 波形，最长 30 秒)
    │
    ▼
┌──────────────────────────┐
│ 预处理: 梅尔频谱图        │
│                          │
│ 波形 → STFT → 80 维梅尔  │
│ 频谱图 (80, T_mel)       │
│                          │
│ T_mel = 3000 (30秒音频)  │
└────────┬─────────────────┘
         │
         ▼
┌──────────────────────────┐
│ 2 层 1D 卷积              │
│ (stride=2, 下采样 2 倍)    │
│                          │
│ (80, 3000) → (D, 1500)   │
│ + 正弦位置编码             │
└────────┬─────────────────┘
         │
         ▼
┌──────────────────────────┐
│ Transformer Encoder       │
│ (N_enc 层)                │
│                          │
│ 输出: 音频特征序列         │
│ (1500, D)                │
└────────┬─────────────────┘
         │
         ▼
┌──────────────────────────┐
│ Transformer Decoder       │
│ (N_dec 层)                │
│                          │
│ 自回归生成文本 token       │
│ 支持多种任务:              │
│  - 语音转录                │
│  - 语音翻译                │
│  - 语言检测                │
│  - 时间戳标注              │
└──────────────────────────┘
```

#### 2.3.2 Whisper 的多任务设计

Whisper 的巧妙之处在于用**特殊 token 序列**指定不同任务：

```
Decoder 输入格式:

<|startoftranscript|>    ← 开始标记
<|en|>                   ← 语言标记（自动检测或指定）
<|transcribe|>           ← 任务标记（转录 or 翻译）
<|notimestamps|>         ← 是否输出时间戳
Hello, how are you?      ← 生成的文本
<|endoftext|>            ← 结束标记

任务切换示例:
  语音转录: <|zh|><|transcribe|> → 输出中文文本
  语音翻译: <|zh|><|translate|>  → 输出英文文本（将中文语音翻译为英文）
```

#### 2.3.3 Whisper 的训练数据

| 维度 | 数据 |
|------|------|
| 数据量 | 68 万小时 |
| 数据来源 | 互联网音频（自动字幕、播客等） |
| 语言数 | 99 种语言 |
| 标注方式 | 弱监督（使用已有的自动字幕作为标签） |
| 数据质量 | 远不如人工标注，但规模弥补质量 |

**关键洞察**：Whisper 证明了一个重要规律——**海量弱标注数据 > 少量精标注数据**。这与 CLIP 的思路一致（4 亿网络图文对 vs 人工标注的 ImageNet）。

### 2.4 语音 + LLM：端到端语音大模型

#### 2.4.1 传统级联方案 vs 端到端方案

```
传统级联方案（GPT-4o 之前）:
  语音 → ASR (Whisper) → 文本 → LLM → 文本 → TTS → 语音
                                        │
              延迟: 2~5 秒                │
              问题: 丢失语音中的情感/语气   │

端到端方案（GPT-4o 等）:
  语音 → 语音编码器 → 语音 token → LLM → 语音 token → 语音解码器 → 语音
                                        │
              延迟: ~320ms               │
              优势: 保留情感/语气，实时对话 │
```

#### 2.4.2 语音 token 化方案

将连续语音波形转化为离散 token 是端到端方案的关键：

| 方案 | 做法 | 代表 |
|------|------|------|
| **语义 token** | 用自监督模型（如 HuBERT）提取高层语义特征 → 量化 | GSLM、AudioPaLM |
| **声学 token** | 用神经编解码器（如 EnCodec）压缩为离散码本 | VALL-E、SoundStorm |
| **混合 token** | 语义 token + 声学 token 分层处理 | GPT-4o（推测）、CosyVoice |
| **连续特征** | 不做离散化，直接用连续特征接入 LLM | Qwen2-Audio |

#### 2.4.3 语音多模态模型全景（截至 2026.4）

| 模型 | 机构 | 输入 | 输出 | 特点 |
|------|------|------|------|------|
| GPT-4o | OpenAI | 文本+图像+语音 | 文本+语音 | 原生多模态，320ms 延迟 |
| Gemini 2.0 | Google | 文本+图像+视频+语音 | 文本+语音+图像 | 全模态输入输出 |
| Qwen2-Audio | 阿里 | 文本+语音 | 文本 | 语音理解（非生成） |
| SALMONN | 字节 | 文本+语音 | 文本 | 语音理解 |
| CosyVoice 2 | 阿里 | 文本 | 语音 | 高质量语音合成 |
| Fish Speech | 开源社区 | 文本 | 语音 | 开源 TTS |

## 3. Python 代码实战

### 3.1 使用 Whisper 进行语音识别

```python
"""
使用 Whisper 进行语音识别：从音频到文本。
依赖: pip install torch transformers datasets soundfile
"""
import torch
from transformers import WhisperProcessor, WhisperForConditionalGeneration

# 加载 Whisper 模型（small 版本，约 500MB）
model_name = "openai/whisper-small"
processor = WhisperProcessor.from_pretrained(model_name)
model = WhisperForConditionalGeneration.from_pretrained(model_name)
model.eval()

# ================================================================
# 方式 1：从 HuggingFace 数据集加载音频
# ================================================================

def transcribe_from_dataset():
    """使用 HuggingFace 数据集中的音频进行转录"""
    from datasets import load_dataset

    # 加载一个小型语音数据集
    dataset = load_dataset(
        "hf-internal-testing/librispeech_asr_dummy",
        "clean",
        split="validation",
        trust_remote_code=True,
    )
    sample = dataset[0]
    audio = sample["audio"]

    print(f"采样率: {audio['sampling_rate']} Hz")
    print(f"音频时长: {len(audio['array']) / audio['sampling_rate']:.2f} 秒")

    # 预处理
    input_features = processor(
        audio["array"],
        sampling_rate=audio["sampling_rate"],
        return_tensors="pt",
    ).input_features

    # 推理
    with torch.no_grad():
        predicted_ids = model.generate(input_features)

    # 解码
    transcription = processor.batch_decode(predicted_ids, skip_special_tokens=True)
    print(f"\n转录结果: {transcription[0]}")
    print(f"参考文本: {sample['text']}")


# ================================================================
# 方式 2：使用 pipeline 简化调用
# ================================================================

def transcribe_with_pipeline():
    """使用 transformers pipeline 简化 Whisper 调用"""
    from transformers import pipeline

    pipe = pipeline(
        "automatic-speech-recognition",
        model="openai/whisper-small",
        device="cpu",  # 或 "cuda:0"
    )

    # 从 URL 或本地文件转录
    # result = pipe("path/to/audio.wav")
    # result = pipe("https://example.com/audio.mp3")

    # 使用数据集样本
    from datasets import load_dataset
    dataset = load_dataset(
        "hf-internal-testing/librispeech_asr_dummy",
        "clean", split="validation",
        trust_remote_code=True,
    )

    result = pipe(dataset[0]["audio"]["array"])
    print(f"Pipeline 转录: {result['text']}")

    # 指定语言和任务
    result_zh = pipe(
        dataset[0]["audio"]["array"],
        generate_kwargs={
            "language": "english",
            "task": "transcribe",
        },
    )
    print(f"指定英语转录: {result_zh['text']}")


# ================================================================
# 方式 3：带时间戳的转录
# ================================================================

def transcribe_with_timestamps():
    """使用 Whisper 获取词级时间戳"""
    from transformers import pipeline

    pipe = pipeline(
        "automatic-speech-recognition",
        model="openai/whisper-small",
        device="cpu",
    )

    from datasets import load_dataset
    dataset = load_dataset(
        "hf-internal-testing/librispeech_asr_dummy",
        "clean", split="validation",
```

```python
"""
视频理解的基础：帧采样 + 时空特征提取。
依赖: pip install torch torchvision av
"""
import torch
import torch.nn as nn
import numpy as np


class VideoFeatureExtractor:
    """
    视频特征提取的基础流程：
    1. 帧采样（均匀/关键帧）
    2. 每帧用 ViT 提取空间特征
    3. 跨帧做时间聚合
    """

    def __init__(self, num_frames=8, image_size=224):
        self.num_frames = num_frames
        self.image_size = image_size

    def uniform_sample(self, total_frames: int) -> list[int]:
        """均匀帧采样：从 T 帧中均匀选取 N 帧"""
        if total_frames <= self.num_frames:
            return list(range(total_frames))
        indices = np.linspace(0, total_frames - 1, self.num_frames, dtype=int)
        return indices.tolist()

    def extract_features_per_frame(self, frames: torch.Tensor) -> torch.Tensor:
        """
        模拟: 对每帧用 ViT 提取特征。
        实际中会用预训练的 CLIP ViT 或 DINOv2。

        Args:
            frames: (N, C, H, W) 采样后的帧
        Returns:
            features: (N, D) 每帧的全局特征
        """
        # 模拟 ViT 编码（实际中用预训练模型）
        B = frames.shape[0]
        # 假设 ViT 输出 768 维特征
        features = torch.randn(B, 768)
        return features

    def temporal_aggregation(self, frame_features: torch.Tensor) -> torch.Tensor:
        """
        时间聚合：将多帧特征汇总为视频级特征。

        常用方法:
        1. 平均池化（最简单）
        2. 时间 Transformer（最强）
        3. 时间卷积（折中）
        """
        # 方法 1: 平均池化
        avg_feature = frame_features.mean(dim=0)  # (D,)

        return avg_feature

# ================================================================
# 分解时空注意力模块（TimeSformer 风格）
# ================================================================

class DividedSpaceTimeAttention(nn.Module):
    """
    TimeSformer 风格的分解时空注意力:
    先做时间注意力（跨帧），再做空间注意力（帧内）。
    """

    def __init__(self, dim=512, num_heads=8):
        super().__init__()
        self.temporal_attn = nn.MultiheadAttention(dim, num_heads, batch_first=True)
        self.spatial_attn = nn.MultiheadAttention(dim, num_heads, batch_first=True)
        self.norm1 = nn.LayerNorm(dim)
        self.norm2 = nn.LayerNorm(dim)

    def forward(self, x, T, S):
        """
        Args:
            x: (B, T*S, D) 时空 token 序列
            T: 时间维度大小（帧数）
            S: 空间维度大小（每帧 patch 数）
        """
        B, N, D = x.shape
        assert N == T * S

        # ---- 时间注意力 ----
        # 重排: (B, T, S, D) → (B*S, T, D)  每个空间位置跨时间做注意力
        x_t = x.view(B, T, S, D).permute(0, 2, 1, 3).reshape(B * S, T, D)
        x_t = self.norm1(x_t)
        attn_t, _ = self.temporal_attn(x_t, x_t, x_t)
        attn_t = attn_t.reshape(B, S, T, D).permute(0, 2, 1, 3).reshape(B, N, D)
        x = x + attn_t  # 残差连接

        # ---- 空间注意力 ----
        # 重排: (B, T, S, D) → (B*T, S, D)  每帧内做空间注意力
        x_s = x.view(B, T, S, D).reshape(B * T, S, D)
        x_s = self.norm2(x_s)
        attn_s, _ = self.spatial_attn(x_s, x_s, x_s)
        attn_s = attn_s.reshape(B, T, S, D).reshape(B, N, D)
        x = x + attn_s  # 残差连接

        return x


# 验证分解时空注意力
if __name__ == "__main__":
    print("\n=== 分解时空注意力 ===")
    T, S, D = 8, 196, 512  # 8 帧, 每帧 196 个 patch (14x14), 512 维
    x = torch.randn(2, T * S, D)  # batch_size=2

    module = DividedSpaceTimeAttention(dim=D, num_heads=8)
    output = module(x, T=T, S=S)

    print(f"输出: {output.shape}")
    print(f"时间维度: T={T} 帧")
    print(f"空间维度: S={S} patches/帧")
    print(f"总 token 数: {T * S}")
    print(f"全局注意力复杂度: O({(T*S)**2}) = O({(T*S)**2:,})")
    print(f"分解注意力复杂度: O({T * S**2} + {S * T**2}) = O({T * S**2 + S * T**2:,})")
    print(f"计算量降低: {(T*S)**2 / (T * S**2 + S * T**2):.1f}x")
```

### 3.3 视频扩散模型概念演示

```python
"""
视频扩散模型的核心概念演示：
展示从图像扩散到视频扩散的扩展思路。
依赖: pip install torch
"""
import torch
import torch.nn as nn


class SimplifiedVideoDiT(nn.Module):
    """
    简化的视频 DiT，演示 Sora 风格的视频生成核心逻辑。
    省略了大量细节，聚焦架构骨架。

    实际的 Sora 规模远大于此（推测数十亿参数）。
    """

    def __init__(self, hidden_dim=256, num_heads=4, num_layers=4, vocab_size=1000):
        super().__init__()
        self.hidden_dim = hidden_dim

        # 时间步嵌入（扩散过程的步数 t）
        self.time_embed = nn.Sequential(
            nn.Linear(1, hidden_dim),
            nn.SiLU(),
            nn.Linear(hidden_dim, hidden_dim),
        )

        # 文本条件嵌入（简化为 embedding lookup）
        self.text_embed = nn.Embedding(vocab_size, hidden_dim)

        # DiT blocks
        self.blocks = nn.ModuleList([
            DiTBlock(hidden_dim, num_heads) for _ in range(num_layers)
        ])

        # 输出头：预测噪声
        self.final_norm = nn.LayerNorm(hidden_dim)
        self.output_proj = nn.Linear(hidden_dim, hidden_dim)

    def forward(self, z_t, t, text_tokens):
        """
        Args:
            z_t: (B, N_st, D) 噪声时空 token
            t: (B,) 扩散时间步 (0~1)
            text_tokens: (B, L) 文本 token IDs
        """
        # 编码条件
        t_emb = self.time_embed(t.unsqueeze(-1))           # (B, D)
        c_emb = self.text_embed(text_tokens).mean(dim=1)    # (B, D) 简化：取平均
        condition = t_emb + c_emb                            # (B, D)

        # DiT blocks
        x = z_t
        for block in self.blocks:
            x = block(x, condition)

        # 预测噪声
        x = self.final_norm(x)
        noise_pred = self.output_proj(x)  # (B, N_st, D)

        return noise_pred


class DiTBlock(nn.Module):
    """
    DiT Block: Transformer + AdaLN 条件注入。
    """

    def __init__(self, dim, num_heads):
        super().__init__()
        self.attn = nn.MultiheadAttention(dim, num_heads, batch_first=True)
        self.ffn = nn.Sequential(
            nn.Linear(dim, dim * 4),
            nn.GELU(),
            nn.Linear(dim * 4, dim),
        )
        # AdaLN: 条件调节 LayerNorm 的 scale 和 shift
        self.adaLN_modulation = nn.Sequential(
            nn.SiLU(),
            nn.Linear(dim, 6 * dim),  # 6 个调节参数
        )
        self.norm1 = nn.LayerNorm(dim, elementwise_affine=False)
        self.norm2 = nn.LayerNorm(dim, elementwise_affine=False)

    def forward(self, x, condition):
        # AdaLN 参数
        params = self.adaLN_modulation(condition)  # (B, 6*D)
        gamma1, beta1, alpha1, gamma2, beta2, alpha2 = params.chunk(6, dim=-1)

        # Self-Attention with AdaLN
        h = self.norm1(x)
        h = h * (1 + gamma1.unsqueeze(1)) + beta1.unsqueeze(1)
        attn_out, _ = self.attn(h, h, h)
        x = x + alpha1.unsqueeze(1) * attn_out

        # FFN with AdaLN
        h = self.norm2(x)
        h = h * (1 + gamma2.unsqueeze(1)) + beta2.unsqueeze(1)
        ffn_out = self.ffn(h)
        x = x + alpha2.unsqueeze(1) * ffn_out

        return x


# ---------- 演示 ----------
if __name__ == "__main__":
    print("=== 简化版视频 DiT ===")

    # 模型配置
    model = SimplifiedVideoDiT(hidden_dim=256, num_heads=4, num_layers=4)

    # 模拟输入
    B = 2
    N_st = 8 * 14 * 14  # 8 帧 × 14×14 空间 patch = 1568 个时空 token
    D = 256

    z_t = torch.randn(B, N_st, D)          # 噪声时空 token
    t = torch.rand(B)                        # 扩散时间步
    text_tokens = torch.randint(0, 1000, (B, 10))  # 文本条件

    # 前向传播
    noise_pred = model(z_t, t, text_tokens)
    print(f"噪声输入: {z_t.shape}")
    print(f"噪声预测: {noise_pred.shape}")
    print(f"时间步: {t}")
    print(f"参数量: {sum(p.numel() for p in model.parameters()) / 1e6:.1f}M")

    # 模拟去噪过程
    print(f"\n=== 模拟去噪循环 ===")
    z = torch.randn(1, N_st, D)  # 初始纯噪声
    num_steps = 5  # 简化为 5 步
    text = torch.randint(0, 1000, (1, 10))

    for step in range(num_steps):
        t_val = torch.tensor([1.0 - step / num_steps])
        with torch.no_grad():
            noise_pred = model(z, t_val, text)
        # 简化的去噪步骤（实际用 DDPM/DDIM 采样器）
        z = z - noise_pred * (1.0 / num_steps)
        print(f"  Step {step+1}: 噪声范数 = {z.norm():.2f}")

    print(f"最终去噪结果: {z.shape}")
```

## 4. 工程师视角

### 4.1 视频理解的工程实践

#### 4.1.1 帧采样策略选择

| 策略 | 做法 | 适用场景 |
|------|------|---------|
| **均匀采样** | 等间隔取 N 帧 | 通用场景，最常用 |
| **关键帧采样** | 检测场景切换，取关键帧 | 长视频（电影/直播） |
| **密集采样** | 取所有帧或高频采样 | 精细动作识别（运动分析） |
| **自适应采样** | 模型决定采样哪些帧 | 最优但计算开销大 |

**实践建议**：
- 通用视频理解：均匀采样 8~32 帧
- 短视频（<30s）：8~16 帧通常足够
- 长视频（>1min）：结合关键帧检测 + 均匀采样
- 实时处理：滑动窗口 + 因果推理

#### 4.1.2 视频理解的 token 成本

| 视频参数 | 帧数 | 每帧 token | 总 token | 等价文本 token |
|---------|------|-----------|---------|---------------|
| 10s @8帧 | 8 | 576 | 4,608 | ~4.6K |
| 60s @16帧 | 16 | 576 | 9,216 | ~9.2K |
| 5min @32帧 | 32 | 576 | 18,432 | ~18.4K |
| 1h @64帧 | 64 | 576 | 36,864 | ~36.9K |

**关键**：视频的 token 量很容易超过 LLM 的上下文窗口。解决方案包括：
- 视觉 token 压缩（Q-Former、Perceiver）
- 帧级特征池化（减少每帧 token 数）
- 分段理解 + 汇总（长视频分段处理）

### 4.2 Sora 及视频生成的工程挑战

#### 4.3.2 实时流式语音识别

```
Whisper 原生不支持流式（设计为 30 秒段处理），但可以通过以下方式实现"准实时"：

方案 1: 滑动窗口
  音频流 → 每 5 秒取一段 → Whisper 处理 → 拼接结果

方案 3: 流式模型
  使用 Whisper-Streaming 或 Moonshine 等专门的流式模型
  延迟: ~1~2 秒
  适合: 实时字幕、同声传译
```

### 4.4 常见面试 / 系统设计问题

| 问题 | 核心要点 |
|------|---------|
| Sora 和 Stable Diffusion 的核心区别？ | Sora 在时空潜空间做扩散（3D），SD 在图像潜空间做扩散（2D）；Sora 用 DiT，SD 用 U-Net（SD3 后也转 DiT） |
| 为什么视频生成比图像生成难得多？ | 多了时间一致性要求（帧间连贯）、计算量暴增（token 数爆炸）、数据更稀缺（高质量视频+文本对少） |
| Whisper 为什么用 Encoder-Decoder 而不是 Decoder-only？ | 语音识别是序列到序列任务（音频序列→文本序列），Encoder-Decoder 天然适合；且音频和文本长度差异大 |
| 如何实现实时语音对话？ | 传统：Whisper + LLM + TTS 级联（延迟高）；前沿：端到端语音 LLM（GPT-4o），将语音直接 token 化输入 LLM |
| 如何处理超长视频？ | 分段采样 + 层级理解：先对每段生成摘要，再对摘要做全局推理 |

## 本节小结

| 概念 | 一句话总结 |
|------|-----------|
| **时空建模** | 视频 = 空间(图像) + 时间(帧序列)，需要同时建模两个维度 |
| **分解注意力** | 将全局时空注意力分解为"时间注意力+空间注意力"，大幅降低计算量 |
| **VideoMAE** | 90-95% 掩码率的自监督预训练，利用视频的时间冗余 |
| **Sora** | DiT + 时空 VAE + 大规模训练 = 高质量视频生成 |
| **Whisper** | 68 万小时弱监督数据 + Encoder-Decoder Transformer = 通用语音识别 |
| **语音 token 化** | 将连续语音转为离散 token，是端到端语音 LLM 的关键 |
| **端到端语音 LLM** | 跳过 ASR→LLM→TTS 级联，直接让 LLM 处理语音 token |
