# ViT 与 CLIP

> **前置知识**：[Transformer 架构](../AI学习_03_注意力机制与Transformer/3_Transformer架构.md)
>
> **本节目标**：理解图像如何被 Transformer 处理，以及 CLIP 如何把图片和文字对齐。

---

## 1. ViT：把图片切成小块来读

Transformer 原本处理的是 token。ViT 的想法很直接：

> 既然文字可以切成 token，图片也可以切成 patch。

一张图片会被切成很多小方块，每个小方块变成一个向量，像 token 一样送进 Transformer。

这样模型就能用注意力机制处理图像块之间的关系。

## 2. Patch 像图像版 token

文字 token 是一句话的基本片段。图像 patch 是图片的基本片段。

模型会学习：

- 哪些 patch 组成边缘。
- 哪些 patch 组成物体部件。
- 哪些远处 patch 有关系。
- 整张图的语义是什么。

和 CNN 不同，ViT 从一开始就可以让远处区域互相建立联系。

## 3. CLIP：把图片和文字放到同一张语义地图

CLIP 同时训练图像编码器和文本编码器。

它看大量“图片 + 描述文本”配对数据，目标是让匹配的图文向量靠近，不匹配的远离。

直白地说：

> 一张猫图和“a cat”应该在语义空间里靠近；和“a car”应该远离。

## 4. CLIP 能做什么

CLIP 的价值在于图文对齐。

它可以用于：

- 零样本图像分类。
- 图文检索。
- 图片语义打分。
- 作为多模态模型的视觉基础。
- 辅助生成模型理解 Prompt。

## 5. CLIP 不是图像生成模型

CLIP 本身主要负责理解和对齐图文，不负责生成图片。

在文生图系统里，它常扮演“把文字和图像语义联系起来”的角色，但真正生成图像还需要扩散模型等生成模块。

理解 ViT 和 CLIP，是理解多模态大模型的入口。
