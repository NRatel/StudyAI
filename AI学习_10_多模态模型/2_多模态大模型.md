## 1. 直觉与概述

### 1.1 核心问题：怎么让 LLM "看见"？

上一节我们学了 CLIP——它能把图像和文本编码到同一个向量空间。但 CLIP 只能做"匹配"（图文检索、零样本分类），无法做"理解"——它不能回答"图中的人在做什么？"这种开放式问题。

为什么？因为 CLIP 的输出是一个**固定长度的向量**（如 512 维），压缩了太多信息。你不能指望一个 512 维向量包含图中所有物体、它们的关系、场景语义等完整信息。

**多模态大模型的核心思路**：不只要一个向量，而是要一组"视觉 token"——把图像编码为一系列 token，作为 LLM 的输入上下文，让 LLM 基于这些视觉 token 来"推理"和"回答"。

类比：**CLIP 是看一眼照片然后给个标签；多模态大模型是仔细看照片然后跟你聊它看到了什么**。

### 1.2 多模态大模型的通用架构

大多数开源多模态大模型（LLaVA、InternVL、Qwen-VL 等）都遵循同一个架构范式（注：GPT-4o 是端到端训练的统一模型，不属于此类模块拼装架构，详见后文）：

```
                    输入图像
                       │
                       ▼
            ┌─────────────────┐
            │   视觉编码器      │  ← 通常是 CLIP ViT 或其变体
            │  (Visual Encoder) │     将图像编码为视觉特征序列
            └────────┬────────┘
                     │
                     ▼  视觉特征 (N_v 个向量)
            ┌─────────────────┐
            │   桥接模块        │  ← MLP / Q-Former / Perceiver
            │  (Projector)     │     将视觉特征"翻译"到 LLM 的语义空间
            └────────┬────────┘
                     │
                     ▼  视觉 token (N_t 个向量)
            ┌─────────────────────────────────────┐
            │        大语言模型（LLM）                │
            │                                      │
            │  输入 = [视觉token] + [文本token]      │
            │                                      │
            │  "这张图片里有一只橘猫躺在沙发上，      │
            │   看起来很放松..."                     │
            └──────────────────────────────────────┘
```

### 1.3 三个核心技术问题

| 问题 | 关键挑战 | 主流方案 |
|------|---------|---------|
| **用什么"看"？** | 选择哪个视觉编码器 | CLIP ViT-L/14、SigLIP、InternViT |
| **怎么"翻译"？** | 视觉特征如何映射到 LLM 空间 | MLP 投影、Q-Former、Perceiver Resampler |
| **怎么"训练"？** | 如何高效对齐视觉和语言模态 | 两阶段训练：预训练对齐 → 指令微调 |

## 2. 严谨定义与原理

### 2.1 GPT-4V / GPT-4o 的架构分析

OpenAI 没有公开 GPT-4V/4o 的详细架构，但基于公开信息和技术报告可以做合理推断：

#### GPT-4V（2023.9）

```
图像 → 视觉编码器（推测为改进的 ViT）
            │
            ▼
     视觉特征序列
            │
       适配层/投影
            │
            ▼
     视觉 token → 插入到 GPT-4 的输入序列中
            │
            ▼
  GPT-4 处理混合序列（视觉 token + 文本 token）
            │
            ▼
       文本输出
```

**关键技术推断**：
- 支持可变分辨率输入（通过自适应 patch 数量）
- 视觉 token 数量与图像分辨率正相关（高分辨率 → 更多 token）
- 使用"平铺"策略处理高分辨率图像：将大图切分为多个区块分别编码

#### GPT-4o（2024.5）

GPT-4o 的"o"代表"omni"（全能），其核心创新是**原生多模态**：

```
GPT-4V 的路线：         图像 → 视觉编码器 → 适配 → LLM → 文本
                        文本 ──────────────────→ LLM → 文本

GPT-4o 的路线：         图像 ─┐
                        文本 ─┼──→ 统一编码 → 统一模型 → 多模态输出
                        语音 ─┘                          (文本/语音/图像)
```

| 维度 | GPT-4V | GPT-4o |
|------|--------|--------|
| 输入模态 | 图像 + 文本 | 图像 + 文本 + 语音 |
| 输出模态 | 文本 | 文本 + 语音（+ 图像，2024 后期） |
| 架构 | 视觉编码器 + GPT-4 | 原生多模态统一模型 |
| 语音处理 | 语音→文本→LLM→文本→语音 | 端到端，不经过文本中间表示 |
| 延迟 | 语音约 2-5 秒 | 语音约 320ms（人类对话级） |

### 2.2 LLaVA 系列——开源多模态大模型的标杆

LLaVA（Large Language and Vision Assistant, Liu et al., 2023）是最重要的开源多模态大模型系列，因为：
1. **架构极简**——MLP 投影就够了，不需要复杂的桥接模块
2. **完全开源**——代码、数据、模型权重全部公开
3. **效果强大**——简单架构 + 好数据 = 惊人效果

#### 2.2.1 LLaVA 架构

```
输入图像 (例如 336×336)
    │
    ▼
┌────────────────────┐
│ CLIP ViT-L/14      │  ← 冻结的预训练 CLIP 视觉编码器
│ @336px             │
│                    │
│ 输出: 576 个视觉   │     576 = 24×24（336/14 = 24）
│ 特征向量 (576, D_v)│     D_v = 1024（ViT-L 的隐藏维度）
└────────┬───────────┘
         │
         ▼
┌────────────────────┐
│ MLP 投影层          │  ← 可训练
│ (线性 → GELU → 线性)│
│                    │
│ D_v → D_llm        │     将 1024 维映射到 LLM 的维度（如 4096）
│ 输出: (576, D_llm)  │
└────────┬───────────┘
         │
         ▼
┌─────────────────────────────────────────────┐
│            LLM (Vicuna / LLaMA)             │
│                                             │
│  输入序列 = [系统提示] + [视觉token×576]      │
│             + [用户问题文本token]              │
│                                             │
│  自回归生成回答                               │
└─────────────────────────────────────────────┘
```

**LLaVA 的核心洞察**：视觉编码器和 LLM 都已经很强了，关键是怎么"连接"它们。答案出人意料地简单——**两层 MLP 就够了**。

### 2.3 多模态对齐的三种范式

不同的模型使用不同的"桥接模块"来对齐视觉和语言特征空间：

#### 范式 1：线性/MLP 投影（LLaVA 路线）

```
视觉特征 (N, D_v) → Linear/MLP → 视觉token (N, D_llm)
```

- **优点**：简单高效，训练快
- **缺点**：视觉 token 数量 = 原始 patch 数量，高分辨率时 token 数爆炸
- **代表**：LLaVA、InternVL

#### 范式 2：Q-Former（BLIP-2 路线）

```
视觉特征 (N, D_v)
        │
        ▼
┌────────────────────────────┐
│ Q-Former                    │
│                            │
│ 可学习的查询向量 (M, D)     │  M << N（如 M=32 vs N=256）
│     │                      │
│     ├── Cross-Attention ←── 视觉特征
│     │                      │
│     └── Self-Attention      │
│                            │
│ 输出: (M, D_llm)           │  ← 将 N 个视觉特征压缩为 M 个
└────────────────────────────┘
```

- **优点**：固定数量的视觉 token（如 32 个），不随分辨率增长
- **缺点**：压缩可能丢失细节信息
- **代表**：BLIP-2、InstructBLIP

#### 范式 3：Perceiver Resampler（Flamingo 路线）

```
视觉特征 (N, D_v)
        │
        ▼
┌────────────────────────────┐
│ Perceiver Resampler         │
│                            │
│ 可学习的潜在向量 (M, D)     │
│     │                      │
│     └── Cross-Attention ←── 视觉特征（交替堆叠多层）
│                            │
│ 输出: (M, D_llm)           │
└────────────────────────────┘
```

- **优点**：类似 Q-Former 但更灵活，可处理可变长度输入
- **缺点**：训练复杂度较高
- **代表**：Flamingo、Emu

#### 三种范式对比

| 维度 | MLP 投影 | Q-Former | Perceiver |
|------|---------|----------|-----------|
| 复杂度 | 低 | 中 | 中 |
| 视觉 token 数 | N（与 patch 数相同） | M（固定，如 32） | M（固定） |
| 信息保留 | 高 | 中（有压缩损失） | 中 |
| 训练效率 | 高 | 中 | 中 |
| 2025 趋势 | 主流选择 | 逐渐式微 | 小众 |

**2024-2025 年的共识**：简单的 MLP 投影 + 高质量数据，效果不逊于甚至优于复杂的桥接模块。LLaVA 用实验证明了"简洁即正义"。

### 2.4 多模态对齐的数学理解

多模态对齐的本质是让两个异构特征空间产生映射关系。

设视觉编码器输出 $\mathbf{V} = [\mathbf{v}_1, \ldots, \mathbf{v}_N] \in \mathbb{R}^{N \times D_v}$，LLM 的嵌入空间维度为 $D_l$。

**MLP 投影的目标**：学习映射 $f: \mathbb{R}^{D_v} \to \mathbb{R}^{D_l}$，使得：

$$\mathbf{H}_v = f(\mathbf{V}) = \text{GELU}(\mathbf{V} \mathbf{W}_1 + \mathbf{b}_1) \mathbf{W}_2 + \mathbf{b}_2$$

其中 $\mathbf{W}_1 \in \mathbb{R}^{D_v \times D_h}$，$\mathbf{W}_2 \in \mathbb{R}^{D_h \times D_l}$。

投影后的视觉 token $\mathbf{H}_v$ 与文本 token $\mathbf{H}_t$ 拼接为 LLM 的输入序列：

$$\mathbf{H}_{\text{input}} = [\mathbf{H}_{\text{sys}},\; \mathbf{H}_v,\; \mathbf{H}_t]$$

LLM 在此混合序列上进行标准的因果语言建模（next token prediction）。

**对齐训练的损失函数**就是标准的语言建模交叉熵：

$$\mathcal{L} = -\sum_{i=1}^{L} \log P_\theta(w_i \mid \mathbf{H}_v, w_{<i})$$

即让模型在给定视觉 token 和之前的文本 token 的条件下，正确预测下一个文本 token。

## 3. Python 代码实战

### 3.1 简化版多模态模型架构

```python
"""
简化版多模态大模型架构：理解"视觉编码器 + 投影层 + LLM"的组合方式。
依赖: pip install torch transformers
"""
import torch
import torch.nn as nn


class SimpleMultimodalModel(nn.Module):
    """
    简化的多模态模型，展示 LLaVA 风格的架构。
    实际的 LLaVA 用预训练的 CLIP ViT + 预训练的 LLaMA，
    这里用简化组件演示原理。
    """

    def __init__(
        self,
        img_size=224,
        patch_size=16,
        vision_dim=768,      # 视觉编码器输出维度
        llm_dim=512,         # LLM 隐藏维度
        num_heads=8,
        num_layers=6,
        vocab_size=32000,
    ):
        super().__init__()

        self.num_patches = (img_size // patch_size) ** 2  # 196

        # ===== 1. 视觉编码器（简化的 ViT）=====
        self.patch_embed = nn.Conv2d(
            3, vision_dim, kernel_size=patch_size, stride=patch_size
        )
        self.vision_pos_embed = nn.Parameter(
            torch.randn(1, self.num_patches, vision_dim)
        )
        vision_encoder_layer = nn.TransformerEncoderLayer(
            d_model=vision_dim, nhead=num_heads,
            dim_feedforward=vision_dim * 4,
            batch_first=True, norm_first=True,
        )
        self.vision_encoder = nn.TransformerEncoder(
            vision_encoder_layer, num_layers=4  # 简化：4 层
        )

        # ===== 2. 桥接模块（MLP 投影）=====
        # 这就是 LLaVA 的核心创新——两层 MLP 足以对齐两个空间
        self.vision_projector = nn.Sequential(
            nn.Linear(vision_dim, llm_dim),
            nn.GELU(),
            nn.Linear(llm_dim, llm_dim),
        )

        # ===== 3. LLM（简化的 Transformer Decoder）=====
        self.text_embed = nn.Embedding(vocab_size, llm_dim)
        self.text_pos_embed = nn.Parameter(torch.randn(1, 512, llm_dim))
        decoder_layer = nn.TransformerDecoderLayer(
            d_model=llm_dim, nhead=num_heads,
            dim_feedforward=llm_dim * 4,
            batch_first=True, norm_first=True,
        )
        self.llm_decoder = nn.TransformerDecoder(
            decoder_layer, num_layers=num_layers
        )
        self.output_head = nn.Linear(llm_dim, vocab_size)

    def encode_image(self, images):
        """图像 → 视觉 token"""
        # Patch Embedding
        x = self.patch_embed(images)             # (B, vision_dim, H/P, W/P)
        x = x.flatten(2).transpose(1, 2)         # (B, num_patches, vision_dim)
        x = x + self.vision_pos_embed

        # 视觉 Transformer
        x = self.vision_encoder(x)               # (B, num_patches, vision_dim)

        # MLP 投影到 LLM 空间
        x = self.vision_projector(x)             # (B, num_patches, llm_dim)
        return x

    def forward(self, images, text_ids):
        """
        前向传播：图像 + 文本 → 预测下一个 token。

        Args:
            images: (B, 3, 224, 224) 输入图像
            text_ids: (B, L) 文本 token IDs
        """
        B, L = text_ids.shape

        # 1. 编码图像为视觉 token
        visual_tokens = self.encode_image(images)  # (B, N_v, llm_dim)
        N_v = visual_tokens.shape[1]

        # 2. 编码文本
        text_tokens = self.text_embed(text_ids)    # (B, L, llm_dim)
        text_tokens = text_tokens + self.text_pos_embed[:, :L, :]

        # 3. 拼接：[视觉token] + [文本token]
        combined = torch.cat([visual_tokens, text_tokens], dim=1)  # (B, N_v+L, llm_dim)

        # 4. 因果注意力掩码（只能看到自己和之前的 token）
        total_len = N_v + L
        causal_mask = nn.Transformer.generate_square_subsequent_mask(total_len)
        causal_mask = causal_mask.to(combined.device)

        # 5. Decoder（自回归生成）
        # 注意：这里简化了，实际的 LLM 是 decoder-only 架构
        memory = torch.zeros(B, 1, combined.shape[-1], device=combined.device)
        output = self.llm_decoder(combined, memory, tgt_mask=causal_mask)

        # 6. 只对文本部分计算 logits（视觉 token 不需要预测）
        text_output = output[:, N_v:, :]           # (B, L, llm_dim)
        logits = self.output_head(text_output)     # (B, L, vocab_size)

        return logits


# ---------- 验证 ----------
if __name__ == "__main__":
    model = SimpleMultimodalModel(
        img_size=224, patch_size=16,
        vision_dim=384, llm_dim=256,
        num_heads=4, num_layers=4, vocab_size=1000,
    )

    # 模拟输入
    images = torch.randn(2, 3, 224, 224)
    text_ids = torch.randint(0, 1000, (2, 20))

    logits = model(images, text_ids)
    print(f"图像输入: {images.shape}")
    print(f"文本输入: {text_ids.shape}")
    print(f"输出 logits: {logits.shape}")  # (2, 20, 1000)
    print(f"总参数量: {sum(p.numel() for p in model.parameters()) / 1e6:.1f}M")

    # 查看视觉 token
    visual_tokens = model.encode_image(images)
    print(f"\n视觉 token: {visual_tokens.shape}")  # (2, 196, 256)
    print(f"视觉 token 数 = patch 数 = (224/16)^2 = {model.num_patches}")
```

### 3.3 多模态对齐训练演示

```python
"""
模拟多模态对齐训练的核心流程。
展示"冻结视觉编码器和 LLM，只训练投影层"的训练范式。
"""
import torch
import torch.nn as nn
import torch.nn.functional as F


class AlignmentTrainingDemo:
    """
    演示 LLaVA 风格的两阶段训练流程。
    用小规模模拟组件展示训练逻辑。
    """

    def __init__(self):
        # 模拟组件（实际中这些是预训练好的大模型）
        self.vision_encoder = nn.Sequential(
            nn.Linear(768, 512),    # 模拟 CLIP ViT
            nn.ReLU(),
            nn.Linear(512, 512),
        )
        self.text_encoder = nn.Embedding(1000, 256)  # 模拟 LLM embedding
        self.llm_head = nn.Linear(256, 1000)          # 模拟 LLM 输出头

        # 投影层（这是需要训练的部分）
        self.projector = nn.Sequential(
            nn.Linear(512, 256),
            nn.GELU(),
            nn.Linear(256, 256),
        )

    def stage1_alignment(self, num_steps=100):
        """
        阶段 1：预训练对齐
        - 冻结视觉编码器和 LLM
        - 只训练投影层
        - 数据：图文描述对
        """
        # 冻结视觉编码器和 LLM
        for param in self.vision_encoder.parameters():
            param.requires_grad = False
        for param in self.text_encoder.parameters():
            param.requires_grad = False
        for param in self.llm_head.parameters():
            param.requires_grad = False

        # 只优化投影层
        optimizer = torch.optim.AdamW(
            self.projector.parameters(), lr=1e-3
        )

        print("=== 阶段 1：预训练对齐 ===")
        print("可训练参数: 投影层")
        trainable = sum(p.numel() for p in self.projector.parameters())
        total = sum(
            p.numel() for m in [self.vision_encoder, self.text_encoder,
                                  self.llm_head, self.projector]
            for p in m.parameters()
        )
        print(f"可训练/总参数: {trainable}/{total} ({trainable/total*100:.1f}%)\n")

        for step in range(num_steps):
            # 模拟数据：视觉特征 + 文本 token
            fake_visual_features = torch.randn(4, 10, 768)  # (B, N_patches, D_v)
            fake_text_ids = torch.randint(0, 1000, (4, 20))  # (B, L)
            fake_labels = torch.randint(0, 1000, (4, 20))

            # 前向传播
            with torch.no_grad():
                v_feat = self.vision_encoder(fake_visual_features)  # (4, 10, 512)
            v_tokens = self.projector(v_feat)  # (4, 10, 256)

            with torch.no_grad():
                t_tokens = self.text_encoder(fake_text_ids)  # (4, 20, 256)

            # 拼接并预测
            combined = torch.cat([v_tokens, t_tokens], dim=1)  # (4, 30, 256)
            logits = self.llm_head(combined[:, 10:, :])  # 只对文本部分预测

            loss = F.cross_entropy(logits.view(-1, 1000), fake_labels.view(-1))

            optimizer.zero_grad()
            loss.backward()
            optimizer.step()

            if (step + 1) % 25 == 0:
                print(f"  Step {step+1}/{num_steps}, Loss: {loss.item():.4f}")

    def stage2_instruction_tuning(self, num_steps=100):
        """
        阶段 2：指令微调
        - 冻结视觉编码器
        - 训练投影层 + LLM（全参或 LoRA）
        """
        # 解冻 LLM（模拟）
        for param in self.text_encoder.parameters():
            param.requires_grad = True
        for param in self.llm_head.parameters():
            param.requires_grad = True

        # 视觉编码器保持冻结
        for param in self.vision_encoder.parameters():
            param.requires_grad = False

        # 优化投影层 + LLM
        optimizer = torch.optim.AdamW([
            {"params": self.projector.parameters(), "lr": 2e-5},
            {"params": self.text_encoder.parameters(), "lr": 2e-5},
            {"params": self.llm_head.parameters(), "lr": 2e-5},
        ])

        print("\n=== 阶段 2：指令微调 ===")
        print("可训练参数: 投影层 + LLM")
        trainable = sum(
            p.numel() for m in [self.projector, self.text_encoder, self.llm_head]
            for p in m.parameters() if p.requires_grad
        )
        print(f"可训练参数量: {trainable}\n")

        for step in range(num_steps):
            fake_visual_features = torch.randn(4, 10, 768)
            fake_text_ids = torch.randint(0, 1000, (4, 20))
            fake_labels = torch.randint(0, 1000, (4, 20))

            with torch.no_grad():
                v_feat = self.vision_encoder(fake_visual_features)
            v_tokens = self.projector(v_feat)
            t_tokens = self.text_encoder(fake_text_ids)

            combined = torch.cat([v_tokens, t_tokens], dim=1)
            logits = self.llm_head(combined[:, 10:, :])

            loss = F.cross_entropy(logits.view(-1, 1000), fake_labels.view(-1))

            optimizer.zero_grad()
            loss.backward()
            optimizer.step()

            if (step + 1) % 25 == 0:
                print(f"  Step {step+1}/{num_steps}, Loss: {loss.item():.4f}")


# ---------- 运行 ----------
if __name__ == "__main__":
    demo = AlignmentTrainingDemo()
    demo.stage1_alignment(num_steps=100)
    demo.stage2_instruction_tuning(num_steps=100)
    print("\n两阶段训练演示完成！")
```

## 4. 工程师视角

### 4.1 多模态大模型的工程选型

#### 4.1.1 闭源 vs 开源

| 维度 | 闭源（GPT-4o / Gemini / Claude） | 开源（LLaVA / InternVL / Qwen-VL） |
|------|----------------------------------|--------------------------------------|
| 效果 | 最强（尤其复杂推理） | 接近闭源水平（2025 差距快速缩小） |
| 成本 | 按 token 计费，高分辨率图像较贵 | GPU 硬件成本，可批量推理 |
| 隐私 | 数据需发送到第三方 | 完全本地化 |
| 定制 | 无法微调（仅 prompt 工程） | 可全量/LoRA 微调 |
| 延迟 | 网络往返 + 排队 | 本地低延迟 |
| 适合 | 快速原型、要求最高质量 | 生产部署、隐私敏感、需要定制 |

#### 4.1.2 分辨率策略

高分辨率图像处理是多模态大模型的核心工程问题：

```
低分辨率策略（GPT-4o "low" 模式 / 早期 LLaVA）
  图像 → resize 到 224×224 或 336×336 → 编码
  优点：token 少（~200），速度快，成本低
  缺点：细节丢失，看不清小字/小物体

动态高分辨率策略（GPT-4o "high" 模式 / LLaVA-NeXT AnyRes）
  大图 → 切分为多个 336×336 的区块 → 分别编码 → 拼接

  ┌─────┬─────┐
  │ 区块1 │ 区块2 │    每个区块 → 576 个 token
  ├─────┼─────┤
  │ 区块3 │ 区块4 │    4 个区块 + 1 个全局缩略图 = 5 × 576 = 2880 个 token
  └─────┴─────┘
  + 全局缩略图（resize to 336×336）

  优点：保留细节
  缺点：token 数暴增（成本上升）
```

| 图像分辨率 | token 数（估算） | 适用场景 |
|-----------|-----------------|---------|
| 224×224 | ~196 | 简单分类 |
| 336×336 | ~576 | 一般理解 |
| 672×672 (AnyRes 4块) | ~2880 | 文档/图表理解 |
| 1344×1344 (AnyRes 16块) | ~9792 | OCR、精细分析 |

### 4.2 多模态数据工程

#### 4.2.1 数据质量 > 数据数量

LLaVA 的成功经验表明：**高质量的多模态指令数据比数量更重要**。

LLaVA 的数据构建流程：

```
Step 1: 收集图像 + 已有标注（如 COCO 的 caption 和 bbox）
                │
                ▼
Step 2: 用 GPT-4 生成多样化的指令数据
        输入：图像描述 + 物体标注
        输出：多轮对话、详细描述、复杂推理问答
                │
                ▼
Step 3: 质量过滤
        去除低质量/有害/重复样本
                │
                ▼
Step 4: 多样性平衡
        对话 : 描述 : 推理 ≈ 1 : 1 : 1
```

| 数据类型 | 示例 | 目的 |
|---------|------|------|
| **对话** | "图中有几个人？" → "图中有3个人..." | 基础视觉问答 |
| **详细描述** | "请详细描述这张图" → "这是一个公园场景..." | 丰富的视觉语言描述 |
| **复杂推理** | "根据图中的信息，这个实验可能说明什么？" → "..." | 视觉推理能力 |

#### 4.2.2 2025-2026 数据趋势

| 趋势 | 说明 |
|------|------|
| 合成数据 | 用强模型（GPT-4o）生成弱模型的训练数据 |
| 视频数据 | 从图文对扩展到视频+文本描述 |
| 多图交错 | 文档中的多图理解（如论文中的图表） |
| 多语言 | 中英日韩等多语言多模态数据 |

### 4.3 部署与推理优化

#### 4.3.1 多模态模型的推理瓶颈

```
推理延迟分解（以 LLaVA-1.5-13B 为例）：

视觉编码：  ~50ms    （ViT 前向传播，一次性）
投影层：    ~5ms     （MLP，几乎可忽略）
LLM prefill：~200ms  （处理 ~800 个 token，包含视觉 token）
LLM decode： ~500ms   （生成 ~100 个 token）
────────────────────
总计：       ~755ms

瓶颈分析：
1. LLM decode 占比最大 → 量化、投机解码等 LLM 推理优化直接适用
2. 视觉 token 数量增加 → prefill 时间线性增长
3. 高分辨率图像 → 视觉 token 暴增 → 成为新的瓶颈
```

#### 4.3.2 优化策略

| 策略 | 做法 | 效果 |
|------|------|------|
| **视觉 token 压缩** | 用 pooling/resampler 减少视觉 token 数 | 减少 LLM 的 token 处理量 |
| **模型量化** | INT4/INT8 量化 LLM 部分 | 显存减半，速度提升 |
| **KV Cache 优化** | 视觉 token 的 KV cache 可预计算 | 减少重复计算 |
| **批处理** | 同时处理多个请求 | 提升 GPU 利用率 |
| **流式输出** | 边生成边返回 | 降低用户感知延迟 |

### 4.4 常见面试 / 系统设计问题

| 问题 | 核心要点 |
|------|---------|
| LLaVA 为什么只用 MLP 就能对齐？ | CLIP ViT 已经学到了语义丰富的视觉表示，LLM 有强大的上下文理解能力；MLP 只需做"维度适配"而非"语义翻译" |
| Q-Former 和 MLP 投影怎么选？ | MLP 保留更多信息但 token 多；Q-Former 压缩 token 但可能丢信息；2024+ 趋势偏向 MLP |
| 多模态大模型能不能微调？ | 可以，通常只微调投影层 + LLM（LoRA），视觉编码器冻结；数据质量 > 数据数量 |
| 如何处理超高分辨率图像？ | 动态切块（AnyRes）：大图切成多个子图 + 一个缩略图，分别编码后拼接 |
| GPT-4o 的"原生多模态"意味着什么？ | 不再是"视觉编码器 + LLM"的拼接，而是从预训练阶段就同时训练多模态，实现更深度的跨模态理解 |

## 本节小结

| 概念 | 一句话总结 |
|------|-----------|
| **多模态大模型** | 视觉编码器 + 桥接模块 + LLM 的三段式架构，让 LLM 能"看图说话" |
| **GPT-4V/4o** | 闭源标杆；4o 实现原生多模态，语音延迟降至 320ms |
| **LLaVA** | 开源标杆；证明了 MLP 投影 + 好数据 = 强效果 |
| **两阶段训练** | 第一阶段对齐（训练投影层）→ 第二阶段指令微调（训练投影层 + LLM） |
| **MLP vs Q-Former** | MLP 简单信息保留多（主流），Q-Former 压缩 token 但有信息损失 |
| **多模态对齐** | 本质是将视觉特征映射到 LLM 的语义空间，让 LLM 能"理解"视觉 token |
| **高分辨率** | AnyRes 动态切块是处理高分辨率的主流方案 |
