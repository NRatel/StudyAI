# 多模态大模型

> **前置知识**：[ViT 与 CLIP](1_ViT与CLIP.md)
>
> **本节目标**：理解图像、文本等不同模态如何接入同一个大模型。

---

## 1. 多模态的关键是“翻译成模型能懂的表示”

语言模型本来只处理 token。图片、音频、视频不能直接丢给它。

多模态模型通常要先用专门编码器把这些输入变成向量，再接到语言模型里。

```text
图片 → 视觉编码器 → 视觉 token
文字 → Tokenizer → 文本 token
两者一起进入大模型
```

## 2. 视觉编码器：先看懂图片

视觉编码器负责把图片转换成一组表示。

这些表示可能对应图像 patch、区域、物体或整体语义。

语言模型不直接看像素，而是看视觉编码器整理后的信息。

## 3. 投影层：把两种语言接上

视觉编码器输出的向量，和语言模型内部的 token 表示不一定在同一个空间。

投影层像一个转接头，把视觉表示转换成语言模型能接收的格式。

这一步很关键：接得不好，语言模型就算再强，也无法正确利用视觉信息。

## 4. 多模态模型能做什么

典型能力包括：

- 看图回答。
- 理解图表。
- OCR 和文档问答。
- 视觉推理。
- 图像描述。
- 屏幕理解和 GUI 操作。

它的重点不是“看到图片”这么简单，而是把视觉信息和语言任务结合起来。

## 5. 多模态的边界

多模态模型仍可能：

- 看漏细节。
- 读错小字。
- 混淆空间关系。
- 对图表做错误推断。
- 把常识补成幻觉。

所以在高风险视觉任务中，仍需要原图核对、工具验证和人工审查。
