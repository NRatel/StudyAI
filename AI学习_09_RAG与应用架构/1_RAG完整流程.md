# RAG 完整流程

> **前置知识**：[预训练语言模型演进](../AI学习_04_预训练语言模型演进/README.md)
>
> **本节目标**：理解 RAG 为什么是“先查资料再回答”，以及完整链路由哪些步骤组成。

---

## 1. 为什么需要 RAG

大模型有三个天然问题：

- 训练后出现的新知识不知道。
- 容易把不确定的内容说得很像真的。
- 很难直接给出可追溯依据。

RAG 用外部资料补这几个短板。

{{img:ch09_01_rag_pipeline}}

## 2. 一次 RAG 请求怎么走

典型流程是：

1. 用户提出问题。
2. 系统把问题转换成检索请求。
3. 从文档库里找相关片段。
4. 把片段和问题一起放进 Prompt。
5. 模型基于这些材料回答。

这像开卷考试：模型不是闭着眼背答案，而是先翻资料。

## 3. 分块：文档要切成可检索的小片段

长文档不能一整本直接塞进检索系统。

通常要切成 chunk，也就是小片段。

切太大：信息混杂，检索不准。切太小：上下文不完整，模型看不懂。

好的分块要保留语义完整性。

## 4. 检索和重排

第一轮检索通常会找出一批候选片段。

然后重排模型再判断：哪些片段真的和问题最相关。

这像先粗筛简历，再细面试。只靠第一轮向量相似度，常常不够。

## 5. RAG 的失败常见在哪里

RAG 失败不一定是大模型错，也可能是前面步骤错：

- 文档没收进去。
- 分块切坏了。
- 检索没找到。
- 找到了但排序靠后。
- Prompt 没要求模型基于证据回答。
- 模型忽略了资料。

所以调 RAG，要看整条链路。
