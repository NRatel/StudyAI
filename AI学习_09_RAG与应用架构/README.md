# 09 RAG 与应用架构

RAG 的核心很朴素：模型回答前，先去外部资料里查一查，再带着资料回答。

## 先抓住这条线

```text
用户问题 → 检索相关资料 → 组装上下文 → 模型生成回答 → 给出依据
```

RAG 解决的不是模型不会说话，而是模型自身知识不够新、不够准、不够可追溯。
