# AI学习：09 RAG与应用架构

## 本模块导读

**学什么**：检索增强生成（Retrieval-Augmented Generation, RAG）——当前将大语言模型落地为实际产品的最核心工程架构。本模块从"为什么需要 RAG"出发，贯穿索引构建、向量检索、高级检索策略，直至完整的生产级 RAG 系统设计。

**RAG 与长上下文的关系**：RAG 与长上下文窗口不是替代关系，而是互补关系。RAG 负责从海量数据（可能是百万篇文档）中**检索**出少量相关内容，长上下文负责在这些检索结果之间进行**推理和综合**。即使模型支持 128K 甚至更长的上下文，也不可能把整个知识库塞进去——RAG 负责"找到"，长上下文负责"理解"，两者配合效果最好。

**为什么学**：大语言模型有三个根本性弱点——幻觉（编造不存在的事实）、时效性（训练数据有截止日期）、私有数据隔离（无法访问企业内部知识）。RAG 是工业界解决这三个问题的主流方案，也是当前 AI 应用最密集的工程方向。无论你要构建智能客服、企业知识库、代码助手还是研究工具，RAG 都是核心技术栈。

**学完能做什么**：
- 理解 RAG 的完整流程：文档索引构建 → 语义检索 → 增强生成
- 掌握 Embedding 模型选型、向量数据库部署、Chunking 策略设计
- 实现 Query 改写、HyDE、Reranker、混合检索等高级 RAG 技术
- 使用 RAGAS 等框架系统性评估 RAG 系统质量
- 设计生产级 RAG 架构，理解 Agentic RAG 的演进方向

**前置知识**：[04 预训练语言模型演进](../AI学习_04_预训练语言模型演进/README.md)（需理解 Encoder/Decoder 架构、Embedding 概念、预训练范式）

**预估学习时间**：10~14 小时

## 学习建议

按序号 1→3 顺序阅读，每个文件内部都是自包含的"直觉→原理→代码→工程视角"结构。

- **第 1 节**是全局地图——先建立对 RAG 全流程的整体认知
- **第 2 节**深入"检索"这一核心环节——Embedding、向量数据库、Chunking 是 RAG 效果的基石
- **第 3 节**是进阶——当 Naive RAG 效果不够时，如何通过各种高级技术持续提升

建议边学边动手：用 LangChain + FAISS 搭建一个本地知识库问答系统，然后逐步叠加高级技术观察效果变化。
