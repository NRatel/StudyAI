## 直觉与概述

### 核心问题

你已经学完了 BERT 和 GPT 的预训练过程。它们在海量文本上学会了语言的通用表示，但一个预训练好的模型并不直接等于一个能解决具体任务的工具。你需要回答：

1. 预训练模型如何适配到具体任务（情感分类、命名实体识别、问答）？
2. 必须更新所有参数吗？对于 175B 参数的 GPT-3，每个下游任务都存一份完整模型现实吗？
3. GPT-3 展示的 Few-shot 能力是怎么回事？为什么不微调也能做任务？
4. Encoder-only、Decoder-only、Encoder-Decoder 三种架构分别适合什么场景？

### 预训练 = 通识教育，微调 = 专业培训

**迁移学习（Transfer Learning）** 的核心思想：

```
大规模无标注数据          小规模标注数据
      │                        │
      ▼                        ▼
┌────────────┐          ┌────────────┐
│  预训练     │ ───────→ │  微调       │ ───→ 下游任务
│ (通识教育)  │  权重迁移  │ (专业培训)  │
└────────────┘          └────────────┘
```

关键洞察：语言中的通用模式（语法、语义、世界知识）可以从无标注文本中学到，这些模式对几乎所有 NLP 任务都有用。这就是为什么用 100 个标注样本微调 BERT，往往超过用 10000 个样本从零训练 LSTM。

### 从 Fine-tuning 到 Prompt 的演进

```
2018  BERT       → 全量微调（Full Fine-tuning）
2019  GPT-2      → Zero-shot（不微调，用提示）
2020  GPT-3      → Few-shot / In-Context Learning
2021  LoRA       → 低秩适配，高效微调
2023  QLoRA      → 量化 + LoRA，消费级 GPU 可用
```

趋势很清晰：**模型越大，我们越倾向于少改甚至不改模型参数**。

## 严谨定义与原理

### Full Fine-tuning（全量微调）

用预训练权重初始化模型，然后在下游任务的标注数据上继续训练**所有参数**。以 BERT 做文本分类为例：

```
输入: [CLS] 这 部 电 影 太 棒 了 [SEP]
        │
        ▼
┌──────────────────────┐
│  BERT Encoder × 12层  │  ← 预训练权重初始化，所有参数参与训练
└──────────┬───────────┘
    [CLS] 位置的输出向量 h ∈ R^768
           │
┌──────────────────────┐
│  线性分类头            │  ← 随机初始化，W ∈ R^{C×768}
└──────────┬───────────┘
      softmax → 类别概率
```

$$h_{\text{[CLS]}} = \text{BERT}(x_1, \ldots, x_n), \quad P(y|x) = \text{softmax}(W \cdot h_{\text{[CLS]}} + b)$$

#### 关键超参数

| 超参数 | 典型值 | 为什么 |
|--------|--------|--------|
| 学习率 | 2e-5 ~ 5e-5 | 比预训练小 10~100 倍，避免"灾难性遗忘" |
| Epoch 数 | 2 ~ 5 | 下游数据少，多了容易过拟合 |
| Batch size | 16 ~ 32 | 原始 BERT 论文推荐 |
| Warmup 比例 | 6% ~ 10% | 开始阶段缓慢提升学习率 |

**灾难性遗忘（Catastrophic Forgetting）**：学习率太大或训练太久，模型会"忘记"预训练阶段学到的通用知识，在下游任务上反而表现变差。

#### 不同任务的适配方式

| 任务类型 | 输入格式 | 输出方式 | 示例 |
|----------|----------|----------|------|
| 单句分类 | `[CLS] 句子 [SEP]` | [CLS] → 线性头 | 情感分析 |
| 句对分类 | `[CLS] 句A [SEP] 句B [SEP]` | [CLS] → 线性头 | 自然语言推理 |
| 序列标注 | `[CLS] 词1 词2 ... [SEP]` | 每个 token → 线性头 | 命名实体识别 |
| 抽取式问答 | `[CLS] 问题 [SEP] 段落 [SEP]` | 每个 token 预测起始/结束概率 | SQuAD |

### Parameter-Efficient Fine-tuning (PEFT)

#### 为什么需要 PEFT？

全量微调有一个致命问题——**规模不可持续**：

```
模型大小    下游任务数    需要存储的模型总量
─────────────────────────────────────────
BERT-base   10 个        110M × 10 = 1.1GB    ← 还可以接受
GPT-3       10 个        175B × 10 = 1.75TB   ← 完全不现实
LLaMA-70B   100 个       70B × 100 = 7TB      ← 荒谬
```

PEFT 的核心思想：**冻结大部分参数，只训练少量新增参数**。每个任务只存少量额外参数（不到原模型 1%），共享同一份预训练权重。

#### LoRA（Low-Rank Adaptation）

LoRA 是目前最主流的 PEFT 方法（Hu et al., 2021）。核心洞察：**微调时的权重更新矩阵是低秩的**。

对预训练权重 $W_0 \in \mathbb{R}^{d \times k}$，用低秩分解近似更新：

$$W = W_0 + \Delta W = W_0 + BA, \quad B \in \mathbb{R}^{d \times r}, A \in \mathbb{R}^{r \times k}, \quad r \ll \min(d, k)$$

```
        输入 x ∈ R^k
       ╱           ╲
   W₀ (冻结)      A: R^k → R^r  (降维)
   d × k            │
     │              B: R^r → R^d  (升维)
     ▼               ▼
   W₀·x      +    B·A·x     =    输出 h ∈ R^d
  (原始路径)    (LoRA 路径)
```

**参数量对比**（$d=k=4096, r=8$）：原始 $16M$ vs LoRA $65K$，压缩约 **250 倍**。

**初始化**：$A$ 高斯随机，$B$ 初始化为零。因此训练开始时 $\Delta W = BA = 0$，模型行为与预训练完全一致。输出乘以缩放因子 $\alpha / r$。

**应用位置**：通常对注意力的 $W_Q, W_V$ 加 LoRA 效果最好。

**独特优势**：推理时可把 $\Delta W = BA$ 合并回原权重 $W_0 + BA$，**零额外推理开销**。这是 LoRA 成为事实标准的关键原因。

#### Prefix Tuning / P-Tuning

**不修改模型参数，在输入前拼接可学习的"软提示"向量**：

```
原始输入：       [x₁, x₂, ..., xₙ]
Prefix Tuning：  [p₁, p₂, ..., pₘ, x₁, x₂, ..., xₙ]
                  └── 可学习 ──┘  └── 冻结 Embedding ──┘
```

$p_1, \ldots, p_m$ 不是真实 token 的 Embedding，而是直接优化的连续向量（"软提示"）。Prefix 长度 $m$ 通常 10~100，模型所有原始参数完全冻结。**P-Tuning v2** 在每一层都插入可学习前缀，效果接近全量微调。

#### Adapter

Adapter（Houlsby et al., 2019）在 Transformer 每层内部插入小型瓶颈网络：

```
Self-Attention → Add & Norm
        │
   ┌─────────┐
   │ Down: d→r│  ← 降维
   │ ReLU     │
   │ Up:  r→d │  ← 升维
   └────┬────┘
   (+) 残差连接
        │
      FFN → Add & Norm
        │
   ┌─────────┐
   │ Down: d→r│  ← 降维
   │ ReLU     │
   │ Up:  r→d │  ← 升维
   └────┬────┘
   (+) 残差连接
        │
      输出
```

每层两个 Adapter（Self-Attention 后和 FFN 后各一个），瓶颈维度 $r$ 通常取 64。参数量：$2L \times 2dr$（$L$ 层，每个 Adapter 含 Down 和 Up 两个矩阵）。

#### PEFT 方法对比

| 方法 | 新增参数位置 | 典型参数比例 | 推理额外开销 |
|------|-------------|-------------|-------------|
| LoRA | 注意力投影矩阵旁 | 0.1%~1% | **可合并为零** |
| Prefix Tuning | 输入序列前 | 0.1%~1% | 增加序列长度 |
| Adapter | 层内部 | 1%~5% | 增加前向传播计算 |
| P-Tuning v2 | 每层输入前 | 0.1%~3% | 增加序列长度 |

### Prompt 范式（无微调）

GPT-3（2020）的里程碑发现：**足够大的模型不需要微调，只需给出任务描述和少量示例，就能完成新任务**。

```
Zero-shot:   将以下句子翻译成法语：The cat sat on the mat →
Few-shot:    Hello → Bonjour | Good morning → Bonjour | The cat sat on the mat →
In-Context Learning:  ≈ Few-shot，强调模型通过上下文"临时学会"任务模式
```

**为什么大模型能做 In-Context Learning？** 这仍然是活跃的研究问题，主流解释包括：

1. **隐式贝叶斯推断**：预训练时模型见过各种"格式"的文本，few-shot 示例帮助模型定位到正确的"任务分布"
2. **隐式梯度下降**：有研究（Akyürek et al., 2023; Von Oswald et al., 2023）表明 Transformer 的前向传播可以模拟梯度下降——示例在 Attention 中的作用类似于训练样本
3. **任务识别**：模型在预训练中已经隐式学会了各种任务模式，示例只是帮助"识别"当前该执行哪个任务

一个重要的实验发现：即使把 few-shot 示例的标签打乱（比如正面样本标为"负面"），大模型的输出格式仍然正确，只是准确率下降。这说明模型更多是从示例中识别"任务格式"而非"学习映射"。

| 维度 | Full Fine-tuning | LoRA | Prompt (Few-shot) |
|------|-----------------|------|-------------------|
| 标注数据 | 数百~数万 | 数百~数千 | 0~几十 |
| 训练成本 | 高 | 中 | 零 |
| 存储成本 | 每任务一份完整模型 | 每任务存 LoRA 权重 | 每任务存一段 Prompt |
| 模型大小要求 | 任意 | 任意 | 通常 >10B |

### 三大架构路线总结

预训练语言模型的三条技术路线，核心区别在于**注意力掩码**和**训练目标**：

```
                   Encoder-only          Decoder-only         Encoder-Decoder
                   (BERT 系列)            (GPT 系列)           (T5 / BART)

注意力方向          双向                   单向（因果）          编码器双向 + 解码器单向

训练目标           MLM                     CLM                  Seq2Seq
                  (完形填空)               (下一词预测)          (输入→输出)

注意力掩码
  位置 i 可见：   全部位置                 位置 1..i             编码器全部 + 解码器 1..i

典型输入输出      [CLS]文本[SEP]          前文 → 续写            输入文本 → 输出文本
                 → 分类/标注               → 下一 token          → 目标序列
```

#### 详细对比表

| 维度 | Encoder-only (BERT) | Decoder-only (GPT) | Encoder-Decoder (T5) |
|------|-------------|-------------|----------------|
| **注意力** | 双向 | 因果（单向） | 编码器双向 + 解码器因果 |
| **训练目标** | MLM（完形填空） | CLM（下一词预测） | Seq2Seq（Span Corruption） |
| **擅长** | 理解：分类、NER | 生成：对话、续写 | 翻译、摘要 |
| **使用方式** | 加分类头微调 | Prompt / LoRA | 输入→模型→输出 |
| **扩展性** | 通常 <1B | 已验证数千亿 | 中等规模为主 |

**为什么 Decoder-only 成为主流？** (1) 所有任务可转化为文本生成；(2) Scaling Law 友好；(3) 大模型天然支持 In-Context Learning；(4) 架构工程简单。

## Python 代码示例

### 示例 1：PyTorch 从零实现 LoRA 层

```python
import torch
import torch.nn as nn
import math

class LoRALinear(nn.Module):
    """
    LoRA 适配层。h = W₀·x + (α/r) · B·A·x，W₀ 冻结，只训练 A 和 B。
    """
    def __init__(self, original_linear: nn.Linear, rank: int = 8, alpha: float = 16.0):
        super().__init__()
        self.in_features = original_linear.in_features
        self.out_features = original_linear.out_features
        self.scaling = alpha / rank

        # 冻结原始权重
        self.original_linear = original_linear
        for param in self.original_linear.parameters():
            param.requires_grad = False

        # LoRA 低秩矩阵：A 降维，B 升维
        self.lora_A = nn.Linear(self.in_features, rank, bias=False)
        self.lora_B = nn.Linear(rank, self.out_features, bias=False)

        # 初始化：A 高斯，B 为零 → 初始时 ΔW = 0
        nn.init.kaiming_uniform_(self.lora_A.weight, a=math.sqrt(5))
        nn.init.zeros_(self.lora_B.weight)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.original_linear(x) + self.lora_B(self.lora_A(x)) * self.scaling

    def merge_weights(self) -> nn.Linear:
        """推理时合并 LoRA 权重，消除额外计算：W = W₀ + (α/r)·B·A"""
        with torch.no_grad():
            merged = nn.Linear(self.in_features, self.out_features,
                               bias=(self.original_linear.bias is not None))
            delta_W = self.lora_B.weight @ self.lora_A.weight
            merged.weight.copy_(self.original_linear.weight + self.scaling * delta_W)
            if self.original_linear.bias is not None:
                merged.bias.copy_(self.original_linear.bias)
        return merged

# ── 验证 ──
torch.manual_seed(42)
pretrained = nn.Linear(768, 768)
lora = LoRALinear(pretrained, rank=8, alpha=16.0)

total = sum(p.numel() for p in lora.parameters())
trainable = sum(p.numel() for p in lora.parameters() if p.requires_grad)
print(f"总参数: {total:,} | 可训练: {trainable:,} ({trainable/total*100:.1f}%)")
# 总参数: 602,880 | 可训练: 12,288 (2.0%)   ← 768×8 + 8×768

x = torch.randn(2, 10, 768)
with torch.no_grad():
    print(f"初始差异: {(pretrained(x) - lora(x)).abs().max():.0e}")  # 0e+00，因为 B=0
```

### 示例 2：自动注入 LoRA（PEFT 库核心思路）

```python
def inject_lora(model: nn.Module, target_modules=('q_proj', 'v_proj'), rank=8, alpha=16.0):
    """扫描模型，把指定 nn.Linear 替换为 LoRALinear。这是 HuggingFace PEFT 的简化版。"""
    for name, module in model.named_modules():
        for child_name, child in module.named_children():
            if child_name in target_modules and isinstance(child, nn.Linear):
                setattr(module, child_name, LoRALinear(child, rank=rank, alpha=alpha))
                print(f"  [LoRA] {name}.{child_name}: {child.in_features}×{child.out_features}")
    total = sum(p.numel() for p in model.parameters())
    trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
    print(f"\n总参数: {total:,} | 可训练: {trainable:,} ({trainable/total*100:.2f}%)")
    return model

# ── 演示：给一个 6 层 Transformer 注入 LoRA ──
class SimpleTransformerLayer(nn.Module):
    def __init__(self, d=512):
        super().__init__()
        self.q_proj = nn.Linear(d, d)
        self.k_proj = nn.Linear(d, d)
        self.v_proj = nn.Linear(d, d)
        self.o_proj = nn.Linear(d, d)
        self.ffn_up = nn.Linear(d, d * 4)
        self.ffn_down = nn.Linear(d * 4, d)

class SimpleModel(nn.Module):
    def __init__(self, n_layers=6, d=512):
        super().__init__()
        self.layers = nn.ModuleList([SimpleTransformerLayer(d) for _ in range(n_layers)])

model = SimpleModel(n_layers=6, d=512)
model = inject_lora(model, target_modules=['q_proj', 'v_proj'], rank=8)
# 输出:
#   [LoRA] layers.0.q_proj: 512×512
#   [LoRA] layers.0.v_proj: 512×512
#   ...（共 12 个）
#   总参数: 9,447,424 | 可训练: 98,304 (1.04%)
```

### 示例 3：HuggingFace 微调 BERT（全量 vs LoRA）

```python
"""依赖: pip install transformers datasets peft torch"""
from transformers import BertTokenizer, BertForSequenceClassification, TrainingArguments, Trainer
from datasets import load_dataset
import numpy as np

model_name = "bert-base-uncased"
tokenizer = BertTokenizer.from_pretrained(model_name)

# ── 方式 A：全量微调 ──
model = BertForSequenceClassification.from_pretrained(model_name, num_labels=2)
dataset = load_dataset("imdb")
tokenized = dataset.map(lambda ex: tokenizer(ex["text"], truncation=True, padding="max_length",
                                              max_length=256), batched=True)
trainer = Trainer(
    model=model,
    args=TrainingArguments(
        output_dir="./results", num_train_epochs=3, learning_rate=2e-5,  # ← 小学习率！
        per_device_train_batch_size=16, warmup_ratio=0.1, weight_decay=0.01,
        eval_strategy="epoch", load_best_model_at_end=True,
    ),
    train_dataset=tokenized["train"], eval_dataset=tokenized["test"],
    compute_metrics=lambda p: {"acc": (np.argmax(p[0], -1) == p[1]).mean()},
)
trainer.train()  # IMDB 准确率 ~93-94%

# ── 方式 B：LoRA 微调 ──
from peft import LoraConfig, get_peft_model, TaskType

model = BertForSequenceClassification.from_pretrained(model_name, num_labels=2)
model = get_peft_model(model, LoraConfig(
    task_type=TaskType.SEQ_CLS, r=8, lora_alpha=16, lora_dropout=0.1,
    target_modules=["query", "value"],  # BERT 中的模块名；LLaMA 则是 "q_proj","v_proj"
))
model.print_trainable_parameters()
# trainable params: 294,912 || all params: 109,778,434 || trainable%: 0.27%
# 后续训练流程完全一致，省略

# 保存（~1MB vs 全量 ~440MB）→ 加载 → 合并
model.save_pretrained("./lora_ckpt")
from peft import PeftModel
merged = PeftModel.from_pretrained(
    BertForSequenceClassification.from_pretrained(model_name, num_labels=2), "./lora_ckpt"
).merge_and_unload()  # 合并后零额外开销
```

### 示例 4：三大架构 HuggingFace 使用对比

```python
"""一段代码看清 Encoder-only / Decoder-only / Encoder-Decoder 的区别。"""
from transformers import (BertTokenizer, BertForSequenceClassification,
                          GPT2Tokenizer, GPT2LMHeadModel,
                          T5Tokenizer, T5ForConditionalGeneration)
import torch

# ── 1. Encoder-only (BERT) → 理解任务（分类） ──
# 特点：双向注意力，通过 [CLS] 做分类，需要额外分类头
tok = BertTokenizer.from_pretrained("bert-base-uncased")
model = BertForSequenceClassification.from_pretrained("bert-base-uncased", num_labels=2)
logits = model(**tok("Great movie!", return_tensors="pt")).logits  # shape: (1, 2)
print(f"[BERT] 分类 logits: {logits}")

# ── 2. Decoder-only (GPT-2) → 生成任务 ──
# 特点：因果注意力，自回归逐 token 生成，通过 Prompt 描述任务
tok = GPT2Tokenizer.from_pretrained("gpt2")
model = GPT2LMHeadModel.from_pretrained("gpt2")
out = model.generate(**tok("The future of AI is", return_tensors="pt"),
                     max_new_tokens=50, temperature=0.7, do_sample=True)
print(f"[GPT-2] 生成: {tok.decode(out[0])}")

# ── 3. Encoder-Decoder (T5) → Seq2Seq 任务 ──
# 特点：编码器双向+解码器因果，所有任务统一为 text-to-text 格式
tok = T5Tokenizer.from_pretrained("t5-small")
model = T5ForConditionalGeneration.from_pretrained("t5-small")
out = model.generate(**tok("translate English to French: Hello", return_tensors="pt"))
print(f"[T5] 翻译: {tok.decode(out[0], skip_special_tokens=True)}")  # "Bonjour"

# ┌──────────┬──────────────┬───────────────┬──────────────────┐
# │          │ BERT(Enc)    │ GPT-2(Dec)    │ T5(Enc-Dec)      │
# ├──────────┼──────────────┼───────────────┼──────────────────┤
# │ 输入     │ [CLS]+文本   │ Prompt 前文    │ 前缀+输入文本    │
# │ 输出     │ 类别 logits  │ 续写 token     │ 目标文本         │
# │ 分类头   │ 需要         │ 不需要         │ 不需要           │
# │ 典型任务 │ 分类、NER    │ 对话、续写      │ 翻译、摘要       │
# └──────────┴──────────────┴───────────────┴──────────────────┘
```

## 工程师视角

### LoRA 实践经验

| 超参数 | 推荐值 | 说明 |
|--------|--------|------|
| rank (r) | 8~64 | 数据量大可用大 rank |
| alpha | 2×rank | 有些库默认 alpha=rank |
| target_modules | q_proj, v_proj | 也可加 k_proj, o_proj, FFN |
| dropout | 0.05~0.1 | 防止 LoRA 过拟合 |
| 学习率 | 1e-4 ~ 3e-4 | 比全量微调大（参数少） |

### QLoRA：消费级 GPU 微调大模型

QLoRA（Dettmers et al., 2023）= **4-bit 量化基座 + LoRA 微调**，让消费级 GPU 也能微调大模型。

```
                  全量微调              QLoRA
               ────────────────    ────────────────
 7B 模型       ~28GB (FP16)       ~5GB (4-bit)
 13B 模型      ~52GB (FP16)       ~10GB (4-bit)
 70B 模型      ~140GB (FP16)      ~35GB (4-bit)
 GPU 需求      多张 A100           单张 24GB 消费级 GPU（7B/13B）
```

三个关键技术：

1. **NF4 量化（NormalFloat 4-bit）**：将预训练权重量化为 4-bit，专门为正态分布权重设计，信息损失最小
2. **双重量化（Double Quantization）**：量化常数本身也被量化，进一步节省内存（每个参数再省约 0.37 bit）
3. **分页优化器（Paged Optimizers）**：利用 CPU-GPU 统一内存管理，当 GPU 内存不足时自动卸载到 CPU，防止 OOM

```python
from transformers import AutoModelForCausalLM, BitsAndBytesConfig
from peft import LoraConfig, get_peft_model, prepare_model_for_kbit_training

model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-2-7b-hf",
    quantization_config=BitsAndBytesConfig(
        load_in_4bit=True, bnb_4bit_quant_type="nf4",
        bnb_4bit_compute_dtype="bfloat16", bnb_4bit_use_double_quant=True),
    device_map="auto")
model = prepare_model_for_kbit_training(model)  # 冻结量化层 + LayerNorm→FP32
model = get_peft_model(model, LoraConfig(
    r=16, lora_alpha=32, lora_dropout=0.05, bias="none", task_type="CAUSAL_LM",
    target_modules=["q_proj","k_proj","v_proj","o_proj","gate_proj","up_proj","down_proj"]))
model.print_trainable_parameters()  # trainable%: 0.59%
```

### 全量微调 vs LoRA vs Prompt 的选择指南

**速记口诀**：数据多 + 模型小 → 全参数微调；数据少 + 模型大 → LoRA；无数据 → Prompting。

```
                      模型参数量
           小 (<1B)    中 (1B~10B)    大 (>10B)
        ┌──────────┬───────────┬──────────┐
 数据多  │ 全量微调  │ 全量/LoRA  │  QLoRA   │
(>10K)  │          │           │          │
        ├──────────┼───────────┼──────────┤
 数据中  │ 全量微调  │   LoRA    │  QLoRA   │
(1K~10K)│          │           │          │
        ├──────────┼───────────┼──────────┤
 数据少  │ LoRA     │  Few-shot  │ Few-shot │
(<1K)   │          │   LoRA    │ Prompt   │
        ├──────────┼───────────┼──────────┤
 无数据  │ 不适用    │ Zero-shot  │ Zero-shot│
        └──────────┴───────────┴──────────┘
```

**决策流程**：

```
有标注数据吗？
 ├─ 否 → 模型 >10B？ ── 是 → Zero-shot / Few-shot Prompt
 │                    └─ 否 → 考虑标注数据或换方案
 └─ 是 → 多少数据？
       ├─ <100 条  → Few-shot（模型够大时）或 LoRA（rank 小）
       ├─ 100~10K  → LoRA 首选；全量微调（模型 <1B）
       └─ >10K     → 全量微调（GPU 足够）/ LoRA / QLoRA（GPU 有限）

需要多任务部署？
 └─ 是 → LoRA（一个基座 + 多套 adapter，运行时动态切换）
```

### 常见陷阱

1. **学习率过大**：微调 BERT 用 2e-5 而非 1e-3。LoRA 参数少，学习率可稍大（1e-4~3e-4）
2. **LoRA rank 不是越大越好**：rank=8 通常够用，rank=256 浪费计算且易过拟合
3. **忽略数据质量**：100 条高质量标注 > 10000 条噪声标注
4. **部署前忘记合并 LoRA**：不合并则推理有额外计算开销
5. **Prompt 的不稳定性**：Few-shot 效果对示例选择和顺序敏感，准确率可能波动 5%~15%

## 本节总结

```
微调与迁移学习知识地图

预训练模型
    ├── 全量微调 (Full Fine-tuning)
    │     用预训练权重初始化，更新所有参数
    │     学习率小（2e-5）、Epoch 少（2~5）
    │
    ├── 参数高效微调 (PEFT)
    │     冻结大部分参数，只训练少量新增参数
    │     ├── LoRA ★ 事实标准（低秩分解，可合并，零额外开销）
    │     ├── Prefix Tuning / P-Tuning（可学习软提示）
    │     └── Adapter（层间瓶颈网络）
    │
    └── Prompt 范式（无微调）
          Zero-shot / Few-shot / In-Context Learning
          模型越大越有效，无需训练

三大架构路线
    ├── Encoder-only (BERT)   → 理解任务
    ├── Decoder-only (GPT)    → 生成任务（当前主流）
    └── Encoder-Decoder (T5)  → Seq2Seq 任务
```
