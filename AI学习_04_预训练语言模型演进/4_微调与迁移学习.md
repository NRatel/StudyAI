# 微调与迁移学习

> **前置知识**：[BERT 与 GPT](3_BERT与GPT.md)、[自监督预训练范式](2_自监督预训练范式.md)
>
> **本节目标**：理解预训练模型如何迁移到具体任务。重点掌握全量微调、参数高效微调、Prompt 与 In-Context Learning 的概念区别。

---

## 1. 为什么预训练模型还要适配任务

预训练模型像一个读过很多书的人：语法、常识、表达习惯都懂一些，但你不能直接把他扔到所有岗位上。

- 做客服，要学会客服话术。
- 做法律检索，要适应法律文本。
- 做情感分类，要知道输出“正面/负面”。

这就需要**迁移学习**：把预训练阶段学到的通用能力，迁移到具体任务。

可以把它理解为：

```text
预训练：通识教育
  在海量文本中学习语言规律和世界知识

微调：专业培训
  用较少任务数据，让模型适应具体任务

Prompt：现场说明
  不改模型，只告诉它这次要怎么回答
```

模型越大，直接“现场说明”的效果通常越好；任务越专业、越要求稳定，微调或参数高效微调就越有价值。

## 2. 微调、迁移与提示的主要方法

### 2.1 迁移学习

**迁移学习（Transfer Learning）**：先在一个大而通用的数据分布上学习，再把学到的知识迁移到较小、较具体的任务上。

在预训练语言模型中，常见流程是：

```text
海量无标注文本 → 自监督预训练 → 通用语言模型
                               ↓
                         下游任务适配
                               ↓
                        分类 / 问答 / 对话 / 摘要
```

迁移学习的核心收益：

- 减少对人工标注数据的依赖。
- 让小任务也能利用大语料中学到的语言知识。
- 比从零训练更稳定、更高效。

### 2.2 Full Fine-tuning：全量微调

**全量微调（Full Fine-tuning）**：用预训练权重初始化模型，然后在下游任务数据上继续训练，更新模型的全部参数。

以 BERT 文本分类为例：

```text
[CLS] 这部电影太棒了 [SEP]
        ↓
BERT Encoder
        ↓
[CLS] 位置表示
        ↓
分类头
        ↓
正面 / 负面
```

优点：

- 适配能力强。
- 对中小模型和有标注数据的任务很有效。
- 任务形式清晰，输出可控。

缺点：

- 每个任务都要训练并保存一份模型参数。
- 大模型全量微调成本很高。
- 数据少或学习率过大时，可能出现**灾难性遗忘**：模型过度适应小任务，损伤预训练阶段学到的通用能力。

### 2.3 PEFT：参数高效微调

**PEFT（Parameter-Efficient Fine-tuning）**：冻结大部分预训练参数，只训练少量新增参数或少量可调模块。

它解决的问题很直接：大模型太大，不能每个任务都复制和训练一整份。

```text
共享的大模型参数：冻结
任务专属的小参数：训练
```

常见方法包括：

| 方法 | 核心直觉 |
|------|----------|
| **LoRA** | 不直接大改原权重，只学习一个小的低秩更新 |
| **Prefix / Prompt Tuning** | 在输入或每层前加入可学习的“软提示” |
| **Adapter** | 在模型层中插入小型可训练模块 |

学习本章时不需要记住每种方法的结构细节。重要的是理解 PEFT 的共同思想：**保留大模型的通用能力，用很小的任务参数完成适配。**

### 2.4 LoRA：给大模型加一个小旋钮

LoRA 是最常见的 PEFT 方法之一。它的直觉是：微调并不一定需要改动完整的大矩阵，很多任务只需要一个低维方向上的小调整。

用公式表示：

$$W = W_0 + \Delta W,\quad \Delta W \approx BA$$

其中 $W_0$ 是冻结的原始权重，$BA$ 是低秩更新。可以把 LoRA 想成给大模型旁边加了一个“小旋钮”：主机器不拆，只调一个小部件。

LoRA 的优势：

- 训练参数少。
- 每个任务只保存小的适配参数。
- 通常能保持接近全量微调的效果。
- 在很多场景中可把更新合并回原权重，使用时更方便。

### 2.5 Prompt：不改模型，只改提问方式

**Prompting** 是另一种适配方式：不训练模型参数，而是通过输入文本告诉模型要做什么。

```text
Zero-shot：
请判断情感：这部电影太棒了。答案：

Few-shot：
句子：这部电影太棒了。情感：正面
句子：剧情很无聊。情感：负面
句子：演员表现不错。情感：
```

GPT-3 之后，人们发现大模型可以从 prompt 中临时识别任务格式。这种能力常被称为 **In-Context Learning（上下文学习）**。

注意：In-Context Learning 并不是像微调那样真正更新模型参数。它更像“看了几个例子后，在当前上下文里按这个模式继续写”。

### 2.6 什么时候用哪种方法

| 情况 | 更合适的方法 | 理由 |
|------|--------------|------|
| 中小模型，有标注数据 | 全量微调 | 模型不大，微调成本可接受 |
| 大模型，任务较专业 | LoRA / PEFT | 成本低，适配能力强 |
| 大模型，任务轻量或探索阶段 | Prompt / Few-shot | 不训练，试错最快 |
| 输出格式必须稳定 | 微调或 PEFT | 比纯 Prompt 更可控 |
| 数据极少且模型足够强 | Prompt / ICL | 可直接利用模型已有能力 |

实际使用时常见顺序是：先用 Prompt 验证任务是否可行；如果不稳定，再考虑 PEFT；只有确实必要时才做全量微调。

### 2.7 三大架构路线如何适配任务

| 路线 | 代表 | 常见适配方式 | 适合任务 |
|------|------|--------------|----------|
| **Encoder-only** | BERT | 加任务头微调 | 分类、NER、匹配、检索 |
| **Decoder-only** | GPT | Prompt、指令微调、LoRA | 对话、生成、问答、代码 |
| **Encoder-Decoder** | T5 / BART | 输入文本 → 输出文本 | 翻译、摘要、改写 |

这张表也解释了为什么前面要学 BERT 和 GPT 的差异：架构和预训练目标，会直接影响后续的适配方式。

## 本节小结

| 概念 | 一句话总结 |
|------|-----------|
| **迁移学习** | 把预训练学到的通用能力迁移到具体任务 |
| **全量微调** | 更新所有参数，适配强但成本高 |
| **灾难性遗忘** | 微调过猛导致模型丢失通用能力 |
| **PEFT** | 冻结大模型，只训练少量任务参数 |
| **LoRA** | 用低秩更新近似任务适配，是常见 PEFT 方法 |
| **Prompt** | 不改参数，通过输入说明任务 |
| **In-Context Learning** | 模型在当前上下文中根据示例临时对齐任务格式 |
