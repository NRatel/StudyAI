## 直觉与概述

### 核心问题

你已经学完了 Transformer 架构和词向量的演进。但此时仍有一个根本性的问题悬而未决：

**如何在没有人工标注数据的情况下，让模型学会"理解"语言？**

传统监督学习需要大量 (输入, 标签) 对——翻译需要平行语料，情感分析需要人工标注。但互联网上有万亿级别的**无标注文本**（维基百科、新闻、书籍、网页），这些数据里蕴含着丰富的语言知识，如何利用它们？

答案是**自监督学习（Self-Supervised Learning）**：不需要人工标注，而是从数据本身构造训练信号。具体来说，把一部分输入"藏起来"，让模型去预测被藏起来的部分。

### 两大范式

```
                    自监督预训练
                         │
          ┌──────────────┴──────────────┐
          ▼                              ▼
    完形填空（MLM）                  接龙（CLM）
    Masked Language Model           Causal Language Model
          │                              │
    遮住部分词，预测被遮词         给定前文，预测下一个词
          │                              │
      双向上下文                       单向上下文
    能同时看左边和右边             只能看左边（causal mask）
          │                              │
    代表：BERT (2018)              代表：GPT (2018)
    擅长：理解类任务               擅长：生成类任务
```

**类比**：
- **MLM 像完形填空**：一段话中间挖了几个空，你根据上下文填词。你可以同时看空格的左边和右边。
- **CLM 像接龙游戏**：给你一段话的开头，你续写后面的内容。写作时你只能利用已经写好的内容。

### 为什么预训练如此重要

```
旧范式：  标注数据  →  从零训练模型  →  单一任务
           (少)         (效果有限)

新范式：  海量无标注文本  →  预训练（学通用语言能力）→  少量标注数据微调  →  多种任务
           (多且免费)       (自监督，无需标注)        (高效迁移)
```

核心价值：用海量无标注文本学到的通用语言理解能力，可以迁移到下游任务上，大幅减少对标注数据的依赖。

## 严谨定义与原理

### 掩码语言模型 MLM（BERT 的预训练目标）

#### 基本原理

给定一个句子 $\mathbf{x} = (x_1, x_2, \ldots, x_n)$，随机选择约 15% 的 token 位置组成集合 $\mathcal{M}$，将这些位置"遮住"后得到损坏的句子 $\tilde{\mathbf{x}}$。模型的目标是预测被遮住位置的原始 token：

$$\mathcal{L}_{\text{MLM}} = -\sum_{i \in \mathcal{M}} \log P(x_i \mid \tilde{\mathbf{x}}; \theta)$$

关键点：$P(x_i \mid \tilde{\mathbf{x}})$ 利用的是**双向上下文**——位置 $i$ 左边和右边的所有可见 token 都参与预测。

#### 80/10/10 策略

直接把所有被选中的 token 替换为 `[MASK]` 会引入分布偏差：微调和推理时输入中不会出现 `[MASK]` 标记。BERT 的应对方案——对被选中的 15% token 按如下比例处理：

| 比例 | 操作 | 目的 |
|------|------|------|
| 80% | 替换为 `[MASK]` | 主要的学习信号：从上下文推断被遮住的词 |
| 10% | 替换为词表中的随机词 | 迫使模型真正理解每个位置的语义，而非仅靠 `[MASK]` 标记判断 |
| 10% | 保持原词不变 | 缓解分布偏差，让模型在正常输入下也能正确表示语义 |

```
原句:     The   cat   sat   on   the   mat   .
选中位置:              ✓              ✓
80% 概率: The   cat  [MASK]  on  [MASK]  mat   .
10% 概率: The   cat   dog    on   park   mat   .
10% 概率: The   cat   sat    on   the    mat   .
```

#### Next Sentence Prediction (NSP)

BERT 还引入了辅助任务——下一句预测：50% 样本是真正连续的两个句子（`IsNext`），50% 是随机拼接（`NotNext`）。但后续研究（RoBERTa, 2019）发现 NSP 对大多数下游任务贡献有限甚至有害，去掉后效果更好。原因：NSP 太简单，模型只需判断两句是否来自同一篇文章（主题匹配），而非真正理解逻辑关系。

#### MLM 的优势与局限

**优势**：双向上下文对理解类任务（分类、NER、阅读理解）非常有效。

**局限**：
- 每次训练只从 15% 的 token 获取 loss 信号，样本效率不如 CLM
- `[MASK]` 标记带来的预训练/微调分布偏差
- 天然不擅长生成任务——训练方式非自回归

### 因果语言模型 CLM（GPT 的预训练目标）

#### 基本原理

给定序列 $\mathbf{x} = (x_1, x_2, \ldots, x_n)$，CLM 最大化自回归概率：

$$\mathcal{L}_{\text{CLM}} = -\sum_{t=1}^{n} \log P(x_t \mid x_1, x_2, \ldots, x_{t-1}; \theta)$$

对每个位置，根据前面的所有 token 预测当前 token。这直接对应语言的自然生成过程。

#### 因果掩码实现单向上下文

单向性通过**因果掩码（causal mask）**实现：

```
注意力矩阵:  x₁    x₂    x₃    x₄
      x₁   [ 可以  -∞    -∞    -∞  ]   → 只能看自己
      x₂   [ 可以  可以  -∞    -∞  ]   → 能看 x₁, x₂
      x₃   [ 可以  可以  可以  -∞  ]   → 能看 x₁, x₂, x₃
      x₄   [ 可以  可以  可以  可以 ]   → 能看所有
```

训练时整个序列一次性输入，因果掩码保证每个位置 $t$ 看不到后续位置的信息。虽然训练是并行的，但每个位置的表示等价于只看了前面的上下文。

#### 为什么 CLM 天然适合生成任务

```
训练时:  给定 (x₁, ..., x_{t-1})  → 预测 x_t
推理时:  给定 (x₁, ..., x_{t-1})  → 生成 x_t → 追加到输入 → 继续生成 x_{t+1}
```

训练与推理完全一致：没有分布偏差，不需要 `[MASK]` 等特殊标记，且自然支持任意长度的文本生成。

#### CLM 的优势与局限

**优势**：每个 token 都贡献 loss（样本效率高）；预训练/推理无偏差；通过 prompt 统一处理多种任务。

**局限**：单向上下文在某些理解任务上不如双向模型；自回归推理速度受限于序列长度。

### MLM vs CLM 系统对比

| 维度 | MLM（BERT 路线） | CLM（GPT 路线） |
|------|-----------------|-----------------|
| 预训练目标 | 预测被遮住的 15% token | 预测每个位置的下一个 token |
| 上下文方向 | 双向（看左边和右边） | 单向（只看左边） |
| 训练信号效率 | 仅 15% token 提供 loss | 每个 token 都提供 loss |
| 架构 | Transformer Encoder | Transformer Decoder |
| 分布偏差 | 有（`[MASK]` 标记） | 无 |
| 擅长任务 | 理解类（分类、NER、相似度） | 生成类（续写、对话、翻译） |
| 代表模型 | BERT, RoBERTa, ALBERT | GPT, GPT-2/3/4, LLaMA |
| 扩展性 | 约 3.4 亿参数后收益递减 | 持续受益于规模扩大（Scaling Law） |

#### 为什么最终 CLM（GPT 路线）胜出

**1. 统一的生成范式**——CLM 把所有任务统一为"文本生成"：

```
分类:  "影评情感？太好看了！答案：" → 生成 "正面"
翻译:  "翻译成中文：Hello world。" → 生成 "你好世界"
问答:  "法国的首都？"              → 生成 "巴黎"
```

BERT 每个任务需要加特定的分类头并微调；CLM 无需架构修改，通过 prompt 切换任务。

**2. 规模扩展效益**——Scaling Law（Kaplan et al., 2020）表明 CLM 的 loss 随参数量、数据量呈幂律下降，千亿参数仍未饱和。BERT 类模型在 ~3.4 亿参数后提升变缓。

**3. In-Context Learning**——GPT-3 展示了无需微调、仅在 prompt 中给出几个示例就能学会新任务的能力。这种涌现能力只在足够大的 CLM 模型中出现。

**4. 工程简洁性**——Decoder-only 架构只需一种注意力，KV Cache 机制让推理效率远优于双向模型。

### Seq2Seq 预训练：T5 的 Span Corruption（简要提及）

Google 的 T5（2019）采用 **span corruption** 目标——遮蔽连续片段而非单个 token：

```
原文:     Thank you for inviting me to your party last week .
损坏:     Thank you <extra_id_0> me to your party <extra_id_1> week .
目标输出: <extra_id_0> for inviting <extra_id_1> last <extra_id_2>
```

编码器看到损坏序列，解码器生成被遮蔽片段。T5 的"Text-to-Text"框架把所有任务转化为文本到文本格式，但 Encoder-Decoder 架构的推理效率不如 Decoder-only，在大模型趋势下逐渐让位于 GPT 路线。

## Python 代码示例

### 示例 1：PyTorch 实现 MLM 预训练

```python
import torch
import torch.nn as nn
import math

# ============================================================
# 极简 Transformer Encoder + MLM 头
# ============================================================

class MLMModel(nn.Module):
    def __init__(self, vocab_size, d_model=128, n_heads=4, n_layers=2, d_ff=512):
        super().__init__()
        self.d_model = d_model
        self.token_embed = nn.Embedding(vocab_size, d_model)
        self.pos_embed = nn.Embedding(128, d_model)

        encoder_layer = nn.TransformerEncoderLayer(
            d_model=d_model, nhead=n_heads, dim_feedforward=d_ff,
            dropout=0.1, batch_first=True, activation="gelu",
        )
        self.encoder = nn.TransformerEncoder(encoder_layer, num_layers=n_layers)
        self.mlm_head = nn.Sequential(
            nn.Linear(d_model, d_model), nn.GELU(),
            nn.LayerNorm(d_model), nn.Linear(d_model, vocab_size),
        )

    def forward(self, input_ids):
        seq_len = input_ids.size(1)
        pos = torch.arange(seq_len, device=input_ids.device).unsqueeze(0)
        x = self.token_embed(input_ids) * math.sqrt(self.d_model) + self.pos_embed(pos)
        # 双向注意力：无因果掩码
        return self.mlm_head(self.encoder(x))

# ============================================================
# MLM 遮蔽策略：80/10/10
# ============================================================

VOCAB_SIZE, MASK_ID, MASK_PROB = 1000, 1, 0.15

def create_mlm_batch(batch_size=32, seq_len=32):
    input_ids = torch.randint(2, VOCAB_SIZE, (batch_size, seq_len))
    labels = input_ids.clone()

    # 15% 的位置被选中
    mask_positions = torch.rand(batch_size, seq_len) < MASK_PROB
    masked_input = input_ids.clone()
    strategy = torch.rand(batch_size, seq_len)

    # 80% → [MASK], 10% → 随机词, 10% → 保持不变
    masked_input[mask_positions & (strategy < 0.8)] = MASK_ID
    random_pos = mask_positions & (strategy >= 0.8) & (strategy < 0.9)
    masked_input[random_pos] = torch.randint(2, VOCAB_SIZE, (random_pos.sum(),))

    labels[~mask_positions] = -100  # 未遮蔽位置不计算 loss
    return masked_input, labels, mask_positions

# ============================================================
# 训练循环
# ============================================================

model = MLMModel(VOCAB_SIZE)
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3, weight_decay=0.01)
criterion = nn.CrossEntropyLoss(ignore_index=-100)
print(f"MLM 模型参数量: {sum(p.numel() for p in model.parameters()):,}\n")

model.train()
mlm_losses = []
for step in range(200):
    masked_input, labels, mask_pos = create_mlm_batch(64, 32)
    logits = model(masked_input)
    loss = criterion(logits.view(-1, VOCAB_SIZE), labels.view(-1))

    optimizer.zero_grad()
    loss.backward()
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()
    mlm_losses.append(loss.item())

    if (step + 1) % 50 == 0:
        with torch.no_grad():
            acc = ((logits.argmax(-1) == labels) & mask_pos).sum().float() / mask_pos.sum()
        print(f"Step {step+1:>3d} | Loss: {loss:.4f} | Masked Acc: {acc:.4f}")

print(f"\n初始 Loss: {mlm_losses[0]:.4f} → 最终: {mlm_losses[-1]:.4f}")
print(f"随机猜测期望 Loss: {math.log(VOCAB_SIZE):.4f}")
```

### 示例 2：PyTorch 实现 CLM 预训练

```python
import torch
import torch.nn as nn
import math

# ============================================================
# 极简 Transformer Decoder（Decoder-only CLM）
# ============================================================

class CLMModel(nn.Module):
    def __init__(self, vocab_size, d_model=128, n_heads=4, n_layers=2, d_ff=512):
        super().__init__()
        self.d_model = d_model
        self.token_embed = nn.Embedding(vocab_size, d_model)
        self.pos_embed = nn.Embedding(128, d_model)

        # Decoder-only：用 TransformerEncoderLayer + 因果掩码即可
        layer = nn.TransformerEncoderLayer(
            d_model=d_model, nhead=n_heads, dim_feedforward=d_ff,
            dropout=0.1, batch_first=True, activation="gelu",
        )
        self.decoder = nn.TransformerEncoder(layer, num_layers=n_layers)
        self.lm_head = nn.Linear(d_model, vocab_size, bias=False)
        self.lm_head.weight = self.token_embed.weight  # 权重共享

    def forward(self, input_ids):
        seq_len = input_ids.size(1)
        pos = torch.arange(seq_len, device=input_ids.device).unsqueeze(0)
        x = self.token_embed(input_ids) * math.sqrt(self.d_model) + self.pos_embed(pos)
        # 关键区别：因果掩码（上三角为 -inf）
        causal = torch.triu(torch.ones(seq_len, seq_len, device=x.device), diagonal=1)
        causal = causal.masked_fill(causal == 1, float('-inf'))
        return self.lm_head(self.decoder(x, mask=causal))

# ============================================================
# CLM 数据准备与训练
# ============================================================

VOCAB_SIZE = 1000

model = CLMModel(VOCAB_SIZE)
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3, weight_decay=0.01)
criterion = nn.CrossEntropyLoss()
print(f"CLM 模型参数量: {sum(p.numel() for p in model.parameters()):,}")
print(f"每个 token 都贡献 loss（不像 MLM 只有 15%）\n")

model.train()
clm_losses = []
for step in range(200):
    tokens = torch.randint(2, VOCAB_SIZE, (64, 33))
    input_ids, labels = tokens[:, :-1], tokens[:, 1:]  # 右移一位

    logits = model(input_ids)
    loss = criterion(logits.view(-1, VOCAB_SIZE), labels.view(-1))

    optimizer.zero_grad()
    loss.backward()
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()
    clm_losses.append(loss.item())

    if (step + 1) % 50 == 0:
        with torch.no_grad():
            acc = (logits.argmax(-1) == labels).float().mean()
        print(f"Step {step+1:>3d} | Loss: {loss:.4f} | Next-Token Acc: {acc:.4f}")

# --- 自回归生成演示 ---
model.eval()
prompt = torch.tensor([[42, 137, 256]])
generated = prompt.clone()
with torch.no_grad():
    for _ in range(10):
        next_token = model(generated)[:, -1, :].argmax(-1, keepdim=True)
        generated = torch.cat([generated, next_token], dim=1)
print(f"\nPrompt:    {prompt[0].tolist()}")
print(f"Generated: {generated[0].tolist()}")
print("（随机数据训练，生成结果无语义。真实语料训练后会生成有意义的文本。）")
```

## 工程师视角

### Tokenizer 在预训练中的角色：BPE 分词

预训练的第一步不是喂数据给模型，而是**分词**。BPE（Byte-Pair Encoding）是当前最主流的分词算法：

```
初始:     每个字符是一个 token
            "lower" → ['l', 'o', 'w', 'e', 'r']

迭代合并:  统计相邻 token 对频率，合并最高频的对
            ('e', 'r') 频率最高 → 合并为 'er' → ['l', 'o', 'w', 'er']
            ('l', 'o') 频率最高 → 合并为 'lo' → ['lo', 'w', 'er']
            ...重复直到达到目标词表大小（通常 32K ~ 128K）
```

**关键特性**：高频词成为独立 token；低频词被拆分为子词；未见词不会 OOV。

| 词表大小 | 序列长度 | Embedding 参数量 | 代表模型 |
|---------|---------|-----------------|---------|
| 30,522 | 较长 | 23.4M (768d) | BERT |
| 50,257 | 中等 | 38.6M (768d) | GPT-2 |
| 32,000 | 中等 | 131M (4096d) | LLaMA |
| 128,000 | 较短 | 524M (4096d) | GPT-4 |

### 预训练的计算规模

#### 训练数据量级

| 模型 | 训练数据量 | Token 数 |
|------|-----------|---------|
| BERT-Base (2018) | 16GB | ~33 亿 tokens |
| GPT-3 (2020) | 570GB | ~3000 亿 tokens |
| LLaMA-2 (2023) | ~2TB | 2 万亿 tokens |
| LLaMA-3 (2024) | ~15TB | 15 万亿 tokens |

#### 计算量估算

训练 FLOPs 可用 Chinchilla 公式近似：$C \approx 6 \times N \times D$（$N$=参数量，$D$=token 数）。

系数 6 的来源：前向传播 $\approx 2ND$（每个参数对每个 token 做一次乘加），反向传播 $\approx 4ND$（约为前向的 2 倍）。

| 模型 | 参数量 | Token 数 | 估算 FLOPs | 估算成本 |
|------|--------|---------|------------|---------|
| BERT-Base | 110M | 3.3B | 2.2e18 | ~$500 |
| GPT-3 | 175B | 300B | 3.1e23 | ~$5M |
| LLaMA-2-70B | 70B | 2T | 8.4e23 | ~$3M |

### 为什么 CLM 在大模型时代胜出

从工程角度的四个核心原因：

**1. 架构统一性**——一个模型通过 prompt 处理所有任务，无需为每个任务加分类头并微调。

**2. KV Cache 加速推理**——CLM 可以缓存已计算的 Key/Value，生成新 token 时只需增量计算。BERT 的双向注意力无法增量计算，因为每个 token 的表示依赖后续所有 token。

**3. 部署简洁性**——CLM 一个模型处理所有任务；MLM 每个任务一个微调模型，运维成本高。

**4. 数据飞轮**——CLM + Scaling Law 让"更多数据/更大模型 → 更好效果"持续有效，而 BERT 类模型的规模收益在 ~3.4 亿参数后趋于饱和。

---

## 本章小结

| 概念 | 一句话总结 |
|------|-----------|
| **自监督学习** | 从数据本身构造训练信号，无需人工标注 |
| **MLM** | 遮住 15% 的 token 让模型预测，利用双向上下文 |
| **80/10/10 策略** | 缓解 `[MASK]` 标记带来的预训练/微调分布偏差 |
| **NSP** | BERT 的辅助任务，后被证明贡献有限 |
| **CLM** | 自回归预测下一个 token，利用单向（左侧）上下文 |
| **因果掩码** | 上三角设为 $-\infty$，保证 CLM 的单向性 |
| **CLM 胜出原因** | 统一生成范式 + Scaling Law + In-Context Learning + KV Cache |
| **Span Corruption (T5)** | 遮蔽连续片段，Encoder-Decoder 架构的折中方案 |
| **BPE** | 子词分词算法，通过迭代合并高频字符对构建词表 |
| **$C \approx 6ND$** | 预训练计算量的快速估算公式 |
| **信号效率** | CLM 每个 token 都贡献 loss，MLM 只有 15% |
