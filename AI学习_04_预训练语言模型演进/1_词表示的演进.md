# 词表示的演进

> **前置知识**：[Transformer 架构](../AI学习_03_注意力机制与Transformer/3_Transformer架构.md)（已学完 Attention、Multi-Head、Transformer 全流程）
>
> **本节目标**：从最朴素的 One-Hot 编码出发，沿 Word2Vec → GloVe → ELMo 的路线，理解词表示如何从"静态、无语义"演进到"动态、上下文相关"。这是理解 BERT/GPT 等预训练模型的直接前置。

> **演进时间线**：One-Hot（传统）→ Word2Vec（2013）→ GloVe（2014）→ ELMo（2018）→ BERT/GPT（2018）

---

## 1. 直觉与概述

### 1.1 核心问题：文字怎么变成数字？

神经网络只能处理数值张量。自然语言是离散的符号序列——"猫"、"狗"、"bank" 对计算机来说只是字符串，没有任何数学结构。我们需要一种映射：

$$f: \text{词汇表} \to \mathbb{R}^d$$

把每个词变成一个 $d$ 维向量，使得**语义相近的词在向量空间中距离相近**。

### 1.2 演进路线一览

```
One-Hot 编码（1950s~）
  │  稀疏、高维、词与词正交，无任何语义信息
  ▼
Word2Vec（Mikolov et al., 2013）
  │  低维稠密向量，通过上下文预测学到语义
  │  king - man + woman ≈ queen
  ▼
GloVe（Pennington et al., 2014）
  │  全局共现统计 + 矩阵分解，显式利用语料统计
  ▼
ELMo（Peters et al., 2018）
  │  同一个词在不同句子中得到不同向量
  │  "bank"（银行）≠ "bank"（河岸）
  ▼
BERT / GPT（2018~）→ 后续章节
```

**关键跳跃**：
- One-Hot → Word2Vec/GloVe：从**无语义**到**静态语义**（每个词一个固定向量）
- Word2Vec/GloVe → ELMo：从**静态**到**动态**（同一词在不同上下文中有不同向量）
- ELMo → BERT/GPT：从**特征提取**到**微调**（后续章节）

## 2. 严谨定义与原理

### 2.1 One-Hot 编码

给定词汇表大小 $|V|$，每个词用一个 $|V|$ 维的向量表示，只有对应位置为 1，其余全为 0：

$$\mathbf{e}_{\text{cat}} = [0, 0, 1, 0, \ldots, 0]^T \in \mathbb{R}^{|V|}$$

**致命缺陷**：

| 问题 | 说明 |
|------|------|
| **维度灾难** | $|V|$ 通常为 3 万~10 万，向量极其稀疏 |
| **无语义关系** | $\mathbf{e}_{\text{cat}}^T \mathbf{e}_{\text{dog}} = 0$，猫和狗"完全无关" |
| **无法泛化** | 模型学到 "cat is cute" 的知识，完全无法迁移到 "dog is cute" |

One-Hot 仅仅是一种**索引**，不携带任何语义信息。

### 2.2 Word2Vec

**核心思想**（Distributional Hypothesis，Harris 1954）：

> 一个词的含义由它经常出现的上下文决定。"You shall know a word by the company it keeps."

#### 2.2.1 两种架构

**CBOW（Continuous Bag of Words）**：用上下文预测中心词。

```
上下文窗口（window=2）:
    "the  cat  [sat]  on  the"
     ↓    ↓          ↓    ↓
  平均上下文词向量 → 预测 "sat"
```

**Skip-gram**：用中心词预测上下文（方向相反）。

```
中心词:  "sat" → 预测 "the", "cat", "on", "the"
```

$$P(w_{t+j} | w_t) = \frac{\exp(\mathbf{u}_{w_{t+j}}^T \mathbf{v}_{w_t})}{\sum_{w=1}^{|V|} \exp(\mathbf{u}_w^T \mathbf{v}_{w_t})}$$

其中 $\mathbf{v}_{w_t}$ 是中心词的**输入向量**，$\mathbf{u}_{w_{t+j}}$ 是上下文词的**输出向量**。每个词维护两套向量（训练完毕后通常只保留 $\mathbf{v}$）。

| 对比 | CBOW | Skip-gram |
|------|------|-----------|
| 方向 | 上下文 → 中心词 | 中心词 → 上下文 |
| 速度 | 较快 | 较慢（多个预测） |
| 低频词 | 表现一般 | 表现更好 |

#### 2.2.2 训练目标与负采样

Skip-gram 的目标是最大化所有（中心词, 上下文词）对的对数概率：

$$J = \frac{1}{T} \sum_{t=1}^{T} \sum_{-c \le j \le c, j \neq 0} \log P(w_{t+j} | w_t)$$

问题：softmax 分母需要对整个词汇表求和（$|V|$ 可达 10 万），代价极大。

**负采样**把问题转化为**二分类**——区分真正的上下文词和随机噪声词。对正样本对 $(w, c)$，随机采 $k$ 个负样本：

$$J_{\text{neg}} = \log \sigma(\mathbf{u}_c^T \mathbf{v}_w) + \sum_{i=1}^{k} \log \sigma(-\mathbf{u}_{c_i^-}^T \mathbf{v}_w)$$

直觉：拉近真正的上下文词向量，推远随机采样的词向量。负样本按 $P_n(w) \propto f(w)^{3/4}$ 采样（平滑词频分布，给低频词更多机会）。

#### 2.2.3 词向量的神奇性质

训练完毕后，词向量空间中出现了线性的语义关系：

$$\mathbf{v}_{\text{king}} - \mathbf{v}_{\text{man}} + \mathbf{v}_{\text{woman}} \approx \mathbf{v}_{\text{queen}}$$

| 关系 | 类比 | 计算方式 |
|------|------|---------|
| 性别 | king : queen :: man : woman | $\mathbf{v}_{\text{king}} - \mathbf{v}_{\text{queen}} + \mathbf{v}_{\text{man}}$ |
| 国家-首都 | France : Paris :: Japan : Tokyo | $\mathbf{v}_{\text{France}} - \mathbf{v}_{\text{Paris}} + \mathbf{v}_{\text{Japan}}$ |
| 时态 | walk : walked :: swim : swam | $\mathbf{v}_{\text{walk}} - \mathbf{v}_{\text{walked}} + \mathbf{v}_{\text{swim}}$ |

### 2.3 GloVe（Global Vectors）

Word2Vec 的训练是**局部的**——每次只看一个窗口。GloVe 先统计**全局共现矩阵** $X_{ij}$（词 $i$ 和词 $j$ 在窗口内共现的次数），再做加权分解。

训练目标——让词向量的点积逼近共现概率的对数：

$$J = \sum_{i,j=1}^{|V|} f(X_{ij}) \left( \mathbf{w}_i^T \tilde{\mathbf{w}}_j + b_i + \tilde{b}_j - \log X_{ij} \right)^2$$

$f(x)$ 是权重函数，对高频共现做截断避免其主导训练。

**GloVe 与 Word2Vec 的关系**：

| 维度 | Word2Vec (Skip-gram) | GloVe |
|------|---------------------|-------|
| 训练方式 | 在线、逐窗口 | 离线、先统计再优化 |
| 利用的信息 | 局部上下文窗口 | 全局共现统计 |
| 本质 | 隐式矩阵分解（Levy & Goldberg, 2014 证明） | 显式加权矩阵分解 |
| 实际效果 | 大多数任务上表现接近 | 大多数任务上表现接近 |

### 2.4 静态词向量的根本局限

无论 Word2Vec 还是 GloVe，每个词只有**一个**固定向量。这对多义词是灾难性的：

```
"I went to the bank to deposit money."     → bank = 银行
"I sat on the bank of the river."          → bank = 河岸
```

两个 "bank" 的静态词向量**完全相同**，但语义截然不同。

### 2.5 ELMo（Embeddings from Language Models）

ELMo 的核心贡献：**用预训练的双向 LSTM 为每个词生成上下文相关的动态表示**。

#### 架构

```
            ┌─────────────────────────────────────────────┐
            │   最终表示 = γ(s_0·h_0 + s_1·h_1 + s_2·h_2) │
            └───────┬──────────┬──────────┬───────────────┘
                    ↑          ↑          ↑
层 2:  前向 LSTM₂ ──→ h₂ ←── 后向 LSTM₂    (语义层)
层 1:  前向 LSTM₁ ──→ h₁ ←── 后向 LSTM₁    (句法层)
层 0:  字符级 CNN  ──→ h₀                    (形态学层)
                    ↑
            输入: "The  cat  sat  on  the  bank"
```

#### 训练目标

双向语言模型——**分别**从左到右和从右到左预测下一个词：

$$J = \sum_{k=1}^{N} \left[ \log P(t_k | t_1, \ldots, t_{k-1}) + \log P(t_k | t_{k+1}, \ldots, t_N) \right]$$

前向和后向 LSTM 是**独立训练**的。这与 BERT 有本质区别——BERT 真正实现了同时看到左右上下文。

#### 使用方式：特征提取

ELMo 的范式是**预训练 + 冻结 + 特征提取**——LSTM 参数冻结，只有加权系数 $s_0, s_1, s_2$ 和缩放因子 $\gamma$ 是可学习的。后来 BERT 改变了这种范式——整个预训练模型都参与微调。

| 问题 | 静态词向量 | ELMo |
|------|-----------|------|
| "bank"（银行 vs 河岸） | 同一个向量 | 不同上下文 → 不同向量 |
| "play"（动词 vs 名词） | 同一个向量 | 根据句法角色变化 |
| OOV（未登录词） | 无法处理 | 字符级 CNN 可泛化 |

## 3. Python 代码示例

### 3.1 numpy 实现简化版 Skip-gram Word2Vec

```python
import numpy as np

class SimpleWord2Vec:
    """Skip-gram + Negative Sampling 的 numpy 实现（教学用）。"""

    def __init__(self, vocab_size, embed_dim=16, neg_samples=5, lr=0.01):
        self.vocab_size, self.embed_dim = vocab_size, embed_dim
        self.neg_samples, self.lr = neg_samples, lr
        scale = 0.5 / embed_dim
        self.W_in = np.random.uniform(-scale, scale, (vocab_size, embed_dim))
        self.W_out = np.random.uniform(-scale, scale, (vocab_size, embed_dim))

    def sigmoid(self, x):
        return np.where(x >= 0, 1 / (1 + np.exp(-x)), np.exp(x) / (1 + np.exp(x)))

    def train_pair(self, center, context, word_freq):
        """对一个（中心词, 上下文词）对做一步梯度更新。"""
        v, u_pos = self.W_in[center], self.W_out[context]

        # 正样本: dL/d(u·v) = σ(u·v) - 1
        grad_pos = self.sigmoid(u_pos @ v) - 1.0
        self.W_out[context] -= self.lr * grad_pos * v
        grad_v = grad_pos * u_pos

        # 负样本: dL/d(u·v) = σ(u·v)
        probs = word_freq ** 0.75
        probs[context] = 0
        probs /= probs.sum()
        for neg in np.random.choice(self.vocab_size, self.neg_samples, False, p=probs):
            u_neg = self.W_out[neg]
            grad_neg = self.sigmoid(u_neg @ v)
            self.W_out[neg] -= self.lr * grad_neg * v
            grad_v += grad_neg * u_neg

        self.W_in[center] -= self.lr * grad_v

    def train(self, indices, window=2, epochs=50):
        freq = np.maximum(np.bincount(indices, minlength=self.vocab_size).astype(float), 1e-8)
        for ep in range(epochs):
            pairs = 0
            for i in range(len(indices)):
                w = np.random.randint(1, window + 1)
                for j in range(max(0, i - w), min(len(indices), i + w + 1)):
                    if j != i:
                        self.train_pair(indices[i], indices[j], freq)
                        pairs += 1
            if (ep + 1) % 25 == 0:
                print(f"  Epoch {ep+1:>3d}, 训练词对数: {pairs}")

    def most_similar(self, idx, idx2word, topk=5):
        v = self.W_in[idx]
        norms = np.maximum(np.linalg.norm(self.W_in, axis=1), 1e-8)
        cos = (self.W_in @ v) / (norms * np.linalg.norm(v))
        return [(idx2word[i], cos[i]) for i in cos.argsort()[::-1][1:topk+1]]


# === 小语料训练 ===
corpus = [
    "the king is a strong man", "the queen is a wise woman",
    "the prince is a young man", "the princess is a young woman",
    "a man and a woman walked by the river",
    "the king and queen ruled the land",
    "the prince and princess played in the garden",
    "a wise man is also a strong man",
    "a wise woman is also a strong woman",
    "the king sat on the throne", "the queen sat beside the king",
]
all_words = ' '.join(corpus).split()
vocab = sorted(set(all_words))
w2i = {w: i for i, w in enumerate(vocab)}
i2w = {i: w for w, i in w2i.items()}
indices = [w2i[w] for w in all_words]

print(f"词汇表大小: {len(vocab)}, 语料总词数: {len(indices)}\n")

np.random.seed(42)
model = SimpleWord2Vec(len(vocab), embed_dim=16, neg_samples=3, lr=0.05)
model.train(indices, window=3, epochs=100)

# 相似词查询
print("\n=== 相似词查询 ===")
for q in ['king', 'queen', 'man', 'woman']:
    sims = model.most_similar(w2i[q], i2w, topk=3)
    print(f"  {q:>8s} → {', '.join(f'{w}({s:.3f})' for w, s in sims)}")

# 类比推理
v_res = model.W_in[w2i['king']] - model.W_in[w2i['man']] + model.W_in[w2i['woman']]
norms = np.linalg.norm(model.W_in, axis=1)
cos = (model.W_in @ v_res) / (norms * np.linalg.norm(v_res) + 1e-8)
for w in ['king', 'man', 'woman']:
    cos[w2i[w]] = -1
print(f"\n类比: king - man + woman ≈ {i2w[cos.argmax()]} (cos={cos.max():.3f})")
print("（小语料结果不稳定，大语料上效果显著）")
```

### 3.4 静态 vs 动态词表示（ELMo 思想演示）

```python
import numpy as np

# 模拟静态词向量
static = {
    "bank": np.array([0.5, 0.3, -0.2, 0.1, 0.4]),
    "river": np.array([0.1, -0.5, 0.8, 0.3, -0.1]),
    "money": np.array([0.6, 0.4, -0.3, 0.2, 0.5]),
    "deposit": np.array([0.7, 0.3, -0.1, 0.1, 0.6]),
    "water": np.array([0.0, -0.4, 0.9, 0.2, -0.2]),
}
cos = lambda a, b: a @ b / (np.linalg.norm(a) * np.linalg.norm(b))

# 问题：无论上下文如何，bank 的向量都一样
print("=== 静态词向量：bank 在两种语境下完全相同 ===")
print(f"  bank ↔ money: {cos(static['bank'], static['money']):.3f}")
print(f"  bank ↔ river: {cos(static['bank'], static['river']):.3f}\n")

# ELMo 思路：用上下文调整词向量（极度简化的模拟）
def elmo_simulate(word_vec, ctx_vecs, s=(0.1, 0.3, 0.6)):
    """模拟 ELMo 三层加权: h0=原始, h1=浅层融合, h2=深层融合"""
    ctx_mean = np.mean(ctx_vecs, axis=0)
    h0 = word_vec
    h1 = 0.6 * word_vec + 0.4 * ctx_mean   # 模拟第一层 LSTM
    h2 = 0.3 * word_vec + 0.7 * ctx_mean   # 模拟第二层 LSTM
    return s[0] * h0 + s[1] * h1 + s[2] * h2

bank_finance = elmo_simulate(static["bank"], [static["money"], static["deposit"]])
bank_nature = elmo_simulate(static["bank"], [static["river"], static["water"]])

print("=== ELMo 思路：上下文相关的词表示 ===")
print(f"  金融 bank ↔ money: {cos(bank_finance, static['money']):.3f}")
print(f"  金融 bank ↔ river: {cos(bank_finance, static['river']):.3f}")
print(f"  自然 bank ↔ money: {cos(bank_nature, static['money']):.3f}")
print(f"  自然 bank ↔ river: {cos(bank_nature, static['river']):.3f}")
print(f"  两种 bank 互相:     {cos(bank_finance, bank_nature):.3f} (远小于1)")
```

## 4. 工程师视角

### 4.1 现代模型中 Word2Vec 思想的延续

现代 Transformer 的第一层就是一个 `nn.Embedding(vocab_size, d_model)`——本质上和 Word2Vec 一样是可学习的查找表，区别在于这些向量会随整个模型端到端训练，而且 Transformer 后续层会将其转化为上下文相关的表示。

| 维度 | Word2Vec | 现代 LLM 的 Embedding |
|------|---------|---------------------|
| 训练方式 | 独立预训练 | 与模型端到端联合训练 |
| 输入单位 | 词（word） | 子词（subword token） |
| 输出 | 静态向量 | Transformer 进一步处理为上下文向量 |

### 4.2 Tokenizer：现代 LLM 的"词表示"入口

Word2Vec 时代以"词"为单位有两个严重问题：(1) OOV——未见过的词无法表示；(2) 词汇表太大——英语约 17 万词，加上多语言更是爆炸。现代 LLM 使用**子词分词（Subword Tokenization）**解决这些问题。

**BPE（Byte Pair Encoding）**核心算法：

```
1. 初始词汇表 = 所有单字符
2. 统计语料中所有相邻 token 对的频率
3. 合并频率最高的 token 对为新 token
4. 重复 2-3 直到达到目标词汇表大小

示例: "lower" = ["low", "er"],  "lowest" = ["low", "es", "t"]
```

**SentencePiece**（Google）把 BPE/Unigram 包装成语言无关的工具，直接在原始文本上训练。LLaMA、Mistral 等都使用它。

```python
# pip install tiktoken
import tiktoken

enc = tiktoken.get_encoding("cl100k_base")  # GPT-4 使用的编码
for text in ["Hello, world!", "supercalifragilisticexpialidocious", "Word2Vec 是词向量方法"]:
    tokens = enc.encode(text)
    decoded = [enc.decode([t]) for t in tokens]
    print(f"  '{text}' → {len(tokens)} tokens: {decoded}")
```

**关键洞察**：常见词是单个 token，罕见词被拆成子词，任何文本都能编码（彻底解决 OOV），词汇表大小通常 32K~100K。

### 4.3 选型指南

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 快速原型 / 小数据 | 预训练 GloVe/Word2Vec | 不需要 GPU，效果够用 |
| 搜索/推荐的召回层 | 静态词向量 + ANN 索引 | 速度快、存储小 |
| NLP 分类/NER/问答 | BERT/RoBERTa 微调 | 上下文表示 + 微调 |
| 文本生成/对话 | GPT/LLaMA | Decoder-only 适合生成 |

### 4.4 历史时间线

```
2003  Bengio - Neural Language Model（首次用神经网络学词向量）
2013  Mikolov - Word2Vec（让词向量走入实用）
2014  Pennington - GloVe（全局统计视角）
2014  Levy & Goldberg - 证明 Skip-gram ≈ PMI 矩阵分解
2015  Sennrich - BPE 用于 NMT（子词分词的崛起）
2018  Peters - ELMo（上下文相关的词表示）
2018  Devlin - BERT / Radford - GPT（Transformer + 微调/生成）
```

### 4.5 常见面试/设计问题

| 问题 | 要点 |
|------|------|
| Word2Vec 和 GloVe 的区别？ | 局部窗口 vs 全局共现；隐式 vs 显式矩阵分解；效果接近 |
| 为什么需要两套向量？ | 中心词和上下文词扮演不同角色；防止恒等映射 |
| 负采样的 3/4 次方有什么用？ | 平滑词频分布，让低频词有更多机会被采为负例 |
| ELMo 为什么不用 Transformer？ | 2018 年 Transformer 刚出，LSTM 是主流；BERT 随后就用了 Transformer |
| 为什么现代模型用子词？ | 解决 OOV、控制词汇表大小、平衡表达力和效率 |

## 本节小结

| 概念 | 一句话总结 |
|------|-----------|
| **One-Hot** | 稀疏高维索引，词间正交，无语义信息 |
| **Word2Vec** | 通过预测上下文（Skip-gram）或中心词（CBOW）学习稠密词向量 |
| **负采样** | 把 softmax 简化为二分类，极大加速训练 |
| **GloVe** | 对全局共现矩阵加权分解，本质与 Word2Vec 等价 |
| **静态词向量的局限** | 一词一向量，无法处理多义词 |
| **ELMo** | 双向 LSTM 语言模型，为每个词生成上下文相关的动态表示 |
| **特征提取 vs 微调** | ELMo 冻结参数做特征提取；BERT 开启了全参数微调的时代 |
| **BPE / SentencePiece** | 子词分词是现代 LLM 的标准入口，彻底解决 OOV |
