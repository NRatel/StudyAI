# StudyAI 项目执行计划

## 目标
为具备长期 Python 编程经验、高等数学基础有限、AI 使用经验丰富但原理了解较少的学习者，构建一套从基础到前沿的 AI 及大模型原理**自包含学习教材**。每份模块文档本身就是可直接学习的内容，重点是准确概念定义、直观生动的解释、核心原理与常见误区，而非推荐去哪里学的指引。代码示例只作为附录辅助理解，不占用主线篇幅；不再设置"工程师视角"固定章节。

## AI学习目录索引.md 排序规则

排序原则：
1. **学习目标优先**：先提供核心原理路径、完整原理路径、工程与前沿选读三条路径。
2. **依赖关系作为推荐前置**：除明显无法理解的概念外，不把所有前置关系写成硬阻塞；允许学习者先读主线，遇到断点再回补。
3. **高级内容后置**：13_长文本与高效架构虽然技术上依赖 03，但教学上建议放在 05/08 之后作为高级选读。

文档结构：
- **头部摘要**：主线总学习时间、附录额外时间、推荐学习节奏（如"每天 2h，约 X 周"）
- **模块列表**（按推荐阅读顺序排列），每条包含：
  - `顺序`：推荐阅读位置
  - `模块名`
  - `定位`（必修、补读、应用核心、工程选读、高级选读）
  - `推荐前置`
  - `读完能回答的问题`

## 验收标准

### 所有模块通用
- [ ] 文档本身是**自包含教材**：读完本文档即可掌握该模块内容，不依赖外部资源
- [ ] 有"本模块导读"（学什么、为什么学、学完能做什么）
- [ ] 每个知识点从具体问题或类比切入，随后讲清准确概念、核心原理与关键机制，最后澄清常见误区
- [ ] 主线正文优先服务于理解 AI 原理：定义准确、讲解直观、语言生动，避免被代码和实际应用细节稀释
- [ ] 主线正文不设置"Python 代码示例"固定章节；确有必要时，只允许极短伪代码或公式辅助理解
- [ ] 主线正文不设置"工程师视角"章节，也不做实际应用角度的长篇扩写
- [ ] 每个模块 README 提供"学完本模块，你应该能回答"自测问题，帮助学习者判断是否可以继续往后读
- [ ] 代码示例精选后放入 `附录_代码示例集.md`，按模块编号组织，避免附录变成第二套教材
- [ ] 每个模块至少列出 3 篇关键论文（作为延伸阅读，不替代正文讲解）
- [ ] 有"常见误区与FAQ"（至少 3 条）

### 前沿模块额外要求
- [ ] 明确标注内容时效（"截至 2026 年 4 月"）
- [ ] 包含 2025~2026 年的重要进展（论文/模型/技术）
- [ ] 在文档末尾设"时效性说明"段落，注明哪些内容可能快速过时

### 数学附录
- [ ] 每个数学概念配"一句话直觉"，必要时引用 `附录_代码示例集.md` 中的演示
- [ ] 标注哪些主线模块会用到该知识点

### 代码示例附录
- [ ] 代码按模块编号组织，作为辅助材料而非主线教材
- [ ] 每个示例服务于一个明确概念，代码短小、可读、少依赖
- [ ] 不要求覆盖每个核心概念，每个主线模块最多保留 1~3 个示例，只有在代码能显著帮助理解时才加入
