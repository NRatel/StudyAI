## 1. 直觉与概述

### 1.1 为什么需要条件生成

DDPM 是一个**无条件生成模型**——它从纯噪声出发去噪，生成来自 $p(x)$ 的随机图像，但我们无法指定"生成什么"。这就像给画家一张白纸，他会随机画一幅画，但你没办法告诉他"画一只戴帽子的猫"。

现实中我们需要**条件生成**——给定条件 $c$，从 $p(x|c)$ 中采样：

```
无条件生成：  噪声 → 去噪 → 随机图像
条件生成：    噪声 + 条件c → 引导去噪 → 符合条件的图像
```

| 条件类型 | 示例 | 应用 |
|---------|------|------|
| 类别标签 | "猫"、"狗" | ImageNet 条件生成 |
| 文本描述 | "一只戴墨镜的柴犬在沙滩上" | Stable Diffusion、DALL-E |
| 图像 | 线稿、深度图 | ControlNet |

### 1.2 引导的核心思想

**引导（Guidance）**= 在去噪采样过程中，注入条件信号偏移每一步的去噪方向。

类比：你在浓雾中走路（去噪过程）。无条件生成就是随机摸索。引导就是有人在目的地亮了一盏灯（条件信号），你每走一步都往灯的方向偏一点。

数学上，根据贝叶斯定理，条件 score function 可以分解为：

$$\nabla_{x_t} \log p(x_t | c) = \underbrace{\nabla_{x_t} \log p(x_t)}_{\text{无条件 score}} + \underbrace{\nabla_{x_t} \log p(c | x_t)}_{\text{引导信号}}$$

### 1.3 两种引导范式

| 方法 | Classifier Guidance | Classifier-Free Guidance |
|------|-------------------|------------------------|
| 提出 | 2021（Dhariwal & Nichol） | 2022（Ho & Salimans） |
| 思路 | 外部分类器梯度做引导 | 同一模型做条件/无条件对比 |
| 需额外模型 | 是 | 否 |
| 灵活性 | 受限于分类器类别 | 任意条件 |
| 实际使用 | 较少（历史价值） | **所有现代文生图模型的标配** |

## 2. Classifier Guidance（分类器引导）

### 2.1 原理与公式

训练一个分类器 $p_\phi(y | x_t)$（接受任意噪声水平的含噪图像），用它的梯度引导去噪：

$$\hat{\epsilon} = \epsilon_\theta(x_t, t) - s \cdot \sigma_t \nabla_{x_t} \log p_\phi(y | x_t)$$

- $\epsilon_\theta(x_t, t)$：无条件模型预测的噪声
- $\nabla_{x_t} \log p_\phi(y | x_t)$：分类器关于**输入 $x_t$** 的梯度（不是模型参数！）
- $s$：guidance scale，$s=0$ 退化为无条件，越大引导越强
- $\sigma_t$：噪声标准差，用于尺度匹配

### 2.2 推导

从条件 score 出发，加入 guidance scale $s$ 来控制引导强度：

$$\nabla_{x_t} \log p_s(x_t | y) = \nabla_{x_t} \log p(x_t) + s \cdot \nabla_{x_t} \log p(y | x_t)$$

利用 score function 与噪声预测的关系 $\nabla_{x_t} \log p(x_t) = -\epsilon_\theta(x_t, t) / \sigma_t$，代入并整理即得上述采样公式。

**直觉**：分类器梯度指向"让图像更像类别 $y$"的方向。减去这个梯度（乘以 $\sigma_t$ 转换到噪声空间），等于在噪声预测中加入一个偏置，让去噪结果偏向类别 $y$。$s$ 越大，偏移越强。

### 2.3 局限

| 局限 | 说明 |
|------|------|
| 需额外训练分类器 | 必须在各种噪声水平的图像上训练 |
| 条件受限 | 只能引导分类器认识的类别，无法处理开放文本 |
| 对抗性伪影 | 分类器梯度可能产生对抗样本式的伪影 |
| 计算开销 | 每步采样都要对 $x_t$ 做反向传播求梯度 |

## 3. Classifier-Free Guidance（无分类器引导）

### 3.1 核心创新

Ho & Salimans（2022）的洞察：分类器梯度可以用条件与无条件 score 的差来替代：

$$\nabla_{x_t} \log p(c | x_t) = \nabla_{x_t} \log p(x_t | c) - \nabla_{x_t} \log p(x_t)$$

既然扩散模型本身就在学 score，只要让**同一个模型同时学会条件和无条件的 score**，就不需要分类器了。

### 3.2 训练方法

训练时**以一定概率随机丢弃条件**，就这么简单：

```
对每个训练样本 (x, c):
    以概率 p_uncond（通常 10%~20%）:
        将条件 c 替换为空条件 ∅（零向量或特殊 token）
    正常训练: 最小化 ||ε - ε_θ(x_t, t, c)||²
```

一个模型身兼两职：$c = \emptyset$ 时做无条件预测，$c$ 为真实条件时做条件预测。

### 3.3 推理公式

$$\hat{\epsilon} = \underbrace{\epsilon_\theta(x_t, t, \emptyset)}_{\text{无条件预测}} + s \cdot \underbrace{\left[\epsilon_\theta(x_t, t, c) - \epsilon_\theta(x_t, t, \emptyset)\right]}_{\text{条件方向（被 s 放大）}}$$

| $s$ 值 | 效果 |
|--------|------|
| $s = 0$ | 纯无条件生成，完全忽略条件 |
| $s = 1$ | 标准条件生成，不做引导放大 |
| $s = 7 \sim 8$ | **Stable Diffusion 默认值**，质量与一致性最佳平衡 |
| $s > 15$ | 过度引导，图像过饱和、失真 |

**直觉**：$s > 1$ 等于放大条件方向，采样自"更尖锐"的条件分布——更贴合提示词，但多样性降低。类似于语言模型中降低采样温度。

### 3.4 CFG 与 Classifier Guidance 的等价性

将 CFG 公式用 score function 重写：

$$\hat{\epsilon} \propto -\nabla_{x_t} \left[\log p(x_t) + s \cdot \log p(c|x_t)\right]$$

这与 Classifier Guidance 的公式完全等价！分类器梯度被隐式地用两次前向传播的差来替代。

### 3.5 为什么 CFG 更好

| 维度 | Classifier Guidance | Classifier-Free Guidance |
|------|-------------------|------------------------|
| 训练复杂度 | 需训练扩散模型 + 分类器 | 只需训练一个模型 |
| 推理复杂度 | 每步需分类器前向 + 反向传播 | 每步两次前向传播（可 batch 并行） |
| 条件灵活性 | 受限于分类器类别 | 支持任意条件（文本、图像等） |
| 对抗性伪影 | 分类器梯度容易产生 | 不存在此问题 |
| 实际效果 | 良好 | 更好（FID/IS 指标更优） |
| 工程简洁性 | 需维护两个模型 | 一个模型搞定 |

CFG 的代价是每步采样需要两次前向传播（一次有条件，一次无条件），推理时间约翻倍。但实践中可以把两次调用 batch 在一起，GPU 并行计算，实际开销小于 2 倍。

## 4. 文本条件的实现

### 4.1 文本编码器

扩散模型无法直接理解文本，需要文本编码器将字符串转为向量序列 $c \in \mathbb{R}^{N \times D}$：

| 编码器 | 使用模型 | 输出维度 | 特点 |
|--------|---------|---------|------|
| CLIP Text Encoder | SD v1/v2 | 77 x 768 | 图文对齐能力强 |
| T5-XXL | Imagen、SD3、Flux | 可变长 x 4096 | 纯文本理解更强 |
| 双编码器（CLIP + T5） | SDXL、SD3 | 拼接 | 取长补短 |

### 4.2 Cross-Attention（最常用）

Q 来自图像特征，K/V 来自文本编码。图像每个空间位置"查询"文本中最相关的词，据此调整特征：

```
图像特征 z  →  Q = z · W_Q     (图像在问什么)
文本条件 c  →  K = c · W_K     (文本中有什么)
             →  V = c · W_V     (文本中的信息)

输出 = softmax(QK^T / √d) · V   → 融合了文本信息的图像特征
```

### 4.3 AdaLN（DiT 使用）

条件向量通过 MLP 生成 LayerNorm 的缩放和偏移参数，间接控制特征分布：

```python
# AdaLN: γ, β 由条件 c 动态生成
γ, β = MLP(c)
output = γ * LayerNorm(x) + β
```

| 注入方式 | 条件粒度 | 典型使用 |
|---------|---------|---------|
| Cross-Attention | 逐 token（精细） | SD v1/v2/XL 的 U-Net |
| AdaLN | 全局 | DiT、SD3、Flux |

## 5. Python 代码示例

### 5.1 CFG 训练：随机丢弃条件

```python
import torch
import torch.nn as nn

class SimpleCFGModel(nn.Module):
    """极简条件扩散模型，展示 CFG 训练核心逻辑"""
    def __init__(self, data_dim=784, cond_dim=128, hidden=512):
        super().__init__()
        self.time_mlp = nn.Sequential(nn.Linear(1, hidden), nn.SiLU(), nn.Linear(hidden, hidden))
        self.cond_mlp = nn.Sequential(nn.Linear(cond_dim, hidden), nn.SiLU(), nn.Linear(hidden, hidden))
        self.net = nn.Sequential(
            nn.Linear(data_dim + hidden * 2, hidden), nn.SiLU(),
            nn.Linear(hidden, hidden), nn.SiLU(),
            nn.Linear(hidden, data_dim),
        )

    def forward(self, x_t, t, c):
        """c 为零向量时等价于无条件预测"""
        return self.net(torch.cat([x_t, self.time_mlp(t), self.cond_mlp(c)], dim=-1))


def train_step_cfg(model, optimizer, x_0, c, p_uncond=0.1):
    """单步 CFG 训练"""
    B, device = x_0.shape[0], x_0.device

    # ---- CFG 核心：随机丢弃条件 ----
    drop_mask = (torch.rand(B, device=device) < p_uncond).float().unsqueeze(1)
    c_masked = c * (1 - drop_mask)  # 丢弃时变全零

    # ---- 标准扩散训练 ----
    t = torch.rand(B, 1, device=device)
    epsilon = torch.randn_like(x_0)
    alpha_bar = 1 - t  # 简化 schedule
    x_t = alpha_bar.sqrt() * x_0 + (1 - alpha_bar).sqrt() * epsilon

    loss = nn.functional.mse_loss(model(x_t, t, c_masked), epsilon)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    return loss.item()
```

### 5.2 CFG 采样：引导去噪

```python
@torch.no_grad()
def sample_cfg(model, shape, cond, guidance_scale=7.5, steps=50, device="cpu"):
    """Classifier-Free Guidance 采样"""
    B = shape[0]
    x_t = torch.randn(shape, device=device)
    cond_uncond = torch.zeros_like(cond)
    timesteps = torch.linspace(1.0, 0.0, steps + 1, device=device)

    for i in range(steps):
        t_now, t_next = timesteps[i], timesteps[i + 1]
        t_batch = torch.full((B, 1), t_now, device=device)

        # ---- CFG 核心：两次前向 + 放大条件方向 ----
        eps_uncond = model(x_t, t_batch, cond_uncond)
        eps_cond   = model(x_t, t_batch, cond)
        eps_guided = eps_uncond + guidance_scale * (eps_cond - eps_uncond)

        # 工程优化：可 batch 合并为一次前向
        # x2 = torch.cat([x_t, x_t]); c2 = torch.cat([cond_uncond, cond])
        # eps_u, eps_c = model(x2, t2, c2).chunk(2)

        # ---- 简化 DDIM 更新 ----
        a_now, a_next = 1 - t_now, 1 - t_next
        x0_pred = (x_t - (1 - a_now).sqrt() * eps_guided) / a_now.sqrt()
        x0_pred = x0_pred.clamp(-3, 3)
        x_t = a_next.sqrt() * x0_pred + (1 - a_next).sqrt() * eps_guided if t_next > 0 else x0_pred

    return x_t


# --- 不同 guidance scale 的效果 ---
if __name__ == "__main__":
    model = SimpleCFGModel().eval()
    cond = torch.randn(4, 128)
    for s in [1.0, 3.0, 7.5, 15.0]:
        out = sample_cfg(model, (4, 784), cond, guidance_scale=s, steps=20)
        print(f"s={s:5.1f} | range=[{out.min():.1f}, {out.max():.1f}] | std={out.std():.3f}")
```

### 5.3 Cross-Attention 实现

```python
class CrossAttention(nn.Module):
    """Q 来自图像特征，K/V 来自文本条件——SD U-Net 的核心条件注入机制"""
    def __init__(self, img_dim, cond_dim, head_dim=64, num_heads=8):
        super().__init__()
        inner = head_dim * num_heads
        self.num_heads, self.head_dim = num_heads, head_dim
        self.scale = head_dim ** -0.5
        self.to_q = nn.Linear(img_dim, inner, bias=False)
        self.to_k = nn.Linear(cond_dim, inner, bias=False)
        self.to_v = nn.Linear(cond_dim, inner, bias=False)
        self.to_out = nn.Linear(inner, img_dim)

    def forward(self, x, context):
        """x: (B, HW, img_dim), context: (B, N_tokens, cond_dim) → (B, HW, img_dim)"""
        B, HW, _ = x.shape
        H, d = self.num_heads, self.head_dim
        q = self.to_q(x).view(B, HW, H, d).transpose(1, 2)
        k = self.to_k(context).view(B, -1, H, d).transpose(1, 2)
        v = self.to_v(context).view(B, -1, H, d).transpose(1, 2)
        attn = (q @ k.transpose(-2, -1) * self.scale).softmax(dim=-1)
        out = (attn @ v).transpose(1, 2).reshape(B, HW, -1)
        return self.to_out(out)

# 验证
cross_attn = CrossAttention(img_dim=320, cond_dim=768)
img_feat = torch.randn(2, 64, 320)    # 8x8 空间位置，320 维特征
text_cond = torch.randn(2, 77, 768)   # 77 个 CLIP token
print(cross_attn(img_feat, text_cond).shape)  # (2, 64, 320) — 形状不变，但已融合文本信息
```

## 6. 工程师视角

### 6.1 Guidance Scale 的选择

```
guidance_scale 太小 (s < 3):
  ├── 图像多样性高，但不太遵循提示词
  └── 图像可能模糊、语义不明确

guidance_scale 适中 (s = 5 ~ 10):
  ├── 质量与一致性的最佳平衡
  └── s = 7.5 是 SD v1.5 经典值

guidance_scale 太大 (s > 15):
  ├── 非常严格遵循提示词
  ├── 但图像过饱和、对比度过高、细节丢失
  ├── 多样性极低（多次生成几乎相同）
  └── 本质原因：CFG 是线性外推（extrapolation），s 过大将采样点
      推离真实数据分布，进入"分布外"区域
```

不同模型的推荐值：

| 模型 | 推荐 $s$ | 原因 |
|------|---------|------|
| SD v1.5 | 7.0 ~ 8.5 | 经典值，社区广泛验证 |
| SDXL | 5.0 ~ 7.0 | 模型更强，不需要太高引导 |
| SD3 / Flux | 3.5 ~ 5.0 | 架构改进，低引导即可高质量 |

规律：**模型越强，需要的 guidance scale 越低**。

### 6.2 负提示词（Negative Prompt）

原理极其简单——把 CFG 中的**无条件预测替换为负条件预测**：

$$\hat{\epsilon} = \epsilon_\theta(x_t, t, c_{neg}) + s \cdot [\epsilon_\theta(x_t, t, c_{pos}) - \epsilon_\theta(x_t, t, c_{neg})]$$

标准 CFG 是"从随机出发，朝提示词走"。负提示词是"从负提示词的反方向出发，朝正提示词走"——同时"远离不想要的"和"靠近想要的"。

常见用法：

```
正提示词: "a beautiful landscape, high quality, 4k"
负提示词: "blurry, low quality, watermark, text, deformed"
```

代码层面只需一行改动：

```python
# 负提示词的实现：只需把 cond_uncond 换成 cond_neg
eps_neg = model(x_t, t, cond_neg)     # 替代原来的 eps_uncond
eps_pos = model(x_t, t, cond_pos)
eps_guided = eps_neg + s * (eps_pos - eps_neg)  # 公式完全一样
```

### 6.3 CFG 的计算优化

| 优化手段 | 做法 | 效果 |
|---------|------|------|
| Batch 合并 | 条件/无条件拼成一个 batch 一次前向 | 减少 kernel launch 开销 |
| Dynamic CFG | 前期强引导（定语义），后期弱引导（补细节） | 提升质量 |
| CFG Distillation | 训练学生模型直接输出引导后的结果 | 省去双重前向传播 |

### 6.4 CFG 在现代模型中的地位

CFG 是几乎所有文生图/文生视频模型的核心技术：Stable Diffusion 全系列、DALL-E 2/3、Imagen、Sora 均使用 CFG。最新研究开始探索消除 CFG 的方法（guidance distillation、直接对齐训练），但目前 CFG 仍是基石。

## 7. 常见误区

**"$s=1$ 是无条件生成"** —— 错。$s=1$ 时 CFG 公式化简为 $\hat\epsilon = \epsilon_\theta(x_t,t,c)$，这是标准条件生成。$s=0$ 才是无条件。

**"CFG 是线性插值"** —— $s > 1$ 时是**线性外推**（extrapolation），放大了条件方向。这就是为什么 $s$ 太大会导致过饱和。

**"$p_{uncond}$ 不重要"** —— 太小（1%）则无条件预测不准，引导方向偏差；太大（50%）则条件能力减弱。经验最优：10%~20%。

**"Cross-Attention 只用在文生图"** —— 它是通用的条件注入机制。图像 CLIP 编码（IP-Adapter）、音频特征、姿态编码等都可以通过 Cross-Attention 注入。

**"CFG 系数越大质量越高"** —— 错。CFG 系数（guidance scale）控制的是生成质量与 prompt 遵循度之间的 trade-off。$s$ 过大时会出现：(1) **过饱和**——颜色异常鲜艳、对比度过高；(2) **失真**——物体形状扭曲、细节崩坏；(3) **伪影**——出现不自然的纹理和重复结构。这是因为 CFG 本质上是线性外推，$s$ 过大会把采样点推出真实数据分布的边界。最佳 $s$ 值因模型而异（SD 1.5 约 7~8，SDXL 约 5~7，SD3/Flux 约 3.5~5），需要根据具体任务实验确定。
