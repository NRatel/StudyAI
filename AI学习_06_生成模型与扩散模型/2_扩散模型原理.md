## 一、直觉与概述

### 1.1 核心思想：噪声的"录像倒放"

扩散模型的核心思想：**逐步往图像上加噪声把它变成纯噪声（前向过程），然后训练一个神经网络学会逐步去噪（反向过程）**。

想象你拿一张清晰的照片，每秒往上面撒一点细沙。1000 秒后照片完全被沙子覆盖。如果你录下了整个过程，**倒放录像**就能看到沙子一粒粒飞走、照片逐渐清晰。扩散模型就是学会这个"倒放"过程的模型：

- **前向过程**（加噪）：已知的、固定的物理过程，不需要学习
- **反向过程**（去噪）：需要神经网络来学习的过程
- **生成新图像**：从纯噪声出发，执行反向过程，逐步去噪得到全新的图像

### 1.2 与 VAE/GAN 的直觉对比

| 模型 | 生成策略 | 类比 |
|------|---------|------|
| VAE | 一步到位：从隐变量 $z$ 直接解码出图像 | 画家一笔画完整幅画 |
| GAN | 一步到位：生成器从噪声直接映射到图像 | 造假者直接伪造一张画 |
| Diffusion | 逐步精修：从噪声出发，经过 $T$ 步逐渐去噪 | 雕塑家从石块逐步雕琢出作品 |

扩散模型的"逐步精修"策略带来两个关键优势：
- **训练更稳定**：每一步只需去掉一小部分噪声，任务更简单（对比 GAN 的模式崩塌）
- **生成质量更高**：多步精修天然适合产生高质量的细节（对比 VAE 的模糊输出）

### 1.3 扩散模型家族全景

```
扩散模型（Diffusion Models）
├── DDPM (2020, Ho et al.)          ← 本文重点，奠基性工作
├── DDIM (2020, Song et al.)        ← 本文重点，加速采样
├── Score-based Models (Song & Ermon) → SDE/ODE 统一框架
├── Latent Diffusion (2022)         → 在 VAE 潜空间做扩散 → Stable Diffusion
└── 条件扩散模型                     → Classifier-Free Guidance（后续章节）
```

## 二、前向过程（加噪）

### 2.1 单步加噪

前向过程的每一步：**把当前图像和一小份高斯噪声混合**。

$$q(x_t | x_{t-1}) = \mathcal{N}(x_t;\; \sqrt{1 - \beta_t}\, x_{t-1},\; \beta_t I)$$

其中 $\beta_t \in (0, 1)$ 是第 $t$ 步的**噪声调度系数**。等价的采样形式：

$$x_t = \sqrt{1 - \beta_t}\, x_{t-1} + \sqrt{\beta_t}\, \epsilon, \quad \epsilon \sim \mathcal{N}(0, I)$$

- $\sqrt{1 - \beta_t}\, x_{t-1}$：信号衰减（保留大部分原图）
- $\sqrt{\beta_t}\, \epsilon$：注入新噪声
- **方差保持**：若 $\text{Var}(x_{t-1}) = 1$，则 $\text{Var}(x_t) = (1 - \beta_t) + \beta_t = 1$，数值稳定

### 2.2 噪声调度 $\beta_t$

**线性调度**（DDPM 原文）：$\beta_t = \beta_{\min} + \frac{t-1}{T-1}(\beta_{\max} - \beta_{\min})$，其中 $\beta_1 = 10^{-4}$，$\beta_T = 0.02$，$T = 1000$。问题：后期信号已消失但仍在加噪，浪费"噪声预算"。

**余弦调度**（Improved DDPM）：

$$\bar\alpha_t = \frac{f(t)}{f(0)}, \quad f(t) = \cos\!\left(\frac{t/T + s}{1 + s} \cdot \frac{\pi}{2}\right)^2, \quad s = 0.008$$

噪声更均匀地分布在整个过程中，信噪比变化更平滑，多数任务上优于线性调度。

### 2.3 关键推导：从 $x_0$ 直接跳到任意 $x_t$

定义 $\alpha_t = 1 - \beta_t$，$\bar\alpha_t = \prod_{s=1}^{t} \alpha_s$（累积信号保留率）。

从单步公式 $x_t = \sqrt{\alpha_t}\, x_{t-1} + \sqrt{1 - \alpha_t}\, \epsilon_t$ 出发，递归展开 $x_{t-1}$：

$$x_t = \sqrt{\alpha_t \alpha_{t-1}}\, x_{t-2} + \underbrace{\sqrt{\alpha_t(1 - \alpha_{t-1})}\, \epsilon_{t-1} + \sqrt{1 - \alpha_t}\, \epsilon_t}_{\text{两个独立高斯，方差可合并}}$$

合并后方差为 $\alpha_t(1 - \alpha_{t-1}) + (1 - \alpha_t) = 1 - \alpha_t\alpha_{t-1}$。继续递归到 $x_0$：

$$\boxed{q(x_t | x_0) = \mathcal{N}(x_t;\; \sqrt{\bar\alpha_t}\, x_0,\; (1 - \bar\alpha_t) I)}$$

$$\boxed{x_t = \sqrt{\bar\alpha_t}\, x_0 + \sqrt{1 - \bar\alpha_t}\, \epsilon, \quad \epsilon \sim \mathcal{N}(0, I)}$$

**重要性**：训练时不需逐步加噪，随机采样 $t$ 后直接从 $x_0$ 跳到 $x_t$，大幅节省计算。$\sqrt{\bar\alpha_t}$ 是信号系数，$\sqrt{1 - \bar\alpha_t}$ 是噪声系数。$t$ 越大信号越弱。

### 2.4 信噪比（SNR）

$\text{SNR}(t) = \bar\alpha_t / (1 - \bar\alpha_t)$：

| 阶段 | $t$ 范围 | $\bar\alpha_t$ | 图像状态 |
|------|---------|---------------|---------|
| 早期 | 1~100 | 接近 1 | 几乎看不出噪声 |
| 中期 | 100~500 | 0.5~0.1 | 依稀可辨结构 |
| 后期 | 500~1000 | 接近 0 | 纯噪声 |

模型在中期步骤学到的"去噪"最有用——既有噪声需要去除，又有足够的结构信息可以利用。

**数值示例**（线性调度，$T=1000$）：

```
t = 100 :  ᾱ = 0.9802   信号系数 = 0.990   噪声系数 = 0.140   → 几乎全是原图
t = 300 :  ᾱ = 0.5932   信号系数 = 0.770   噪声系数 = 0.638   → 信号和噪声各半
t = 500 :  ᾱ = 0.0868   信号系数 = 0.295   噪声系数 = 0.956   → 基本看不出原图
t = 800 :  ᾱ = 0.0014   信号系数 = 0.037   噪声系数 = 0.999   → 完全是噪声
```

## 三、反向过程（去噪）

### 3.1 目标：学习 $p_\theta(x_{t-1}|x_t)$

由于每步加噪量很小，反向步骤也可以用高斯分布近似：

$$p_\theta(x_{t-1}|x_t) = \mathcal{N}(x_{t-1};\; \mu_\theta(x_t, t),\; \sigma_t^2 I)$$

核心问题：如何参数化 $\mu_\theta$？Ho et al. (2020) 发现——**预测噪声**比直接预测均值更好。

### 3.2 为什么预测噪声 $\epsilon$

利用贝叶斯公式，前向过程的后验分布 $q(x_{t-1}|x_t, x_0)$ 可以解析计算：

$$q(x_{t-1}|x_t, x_0) = \mathcal{N}(x_{t-1};\; \tilde\mu_t,\; \tilde\beta_t I)$$

$$\tilde\mu_t = \frac{\sqrt{\bar\alpha_{t-1}}\,\beta_t}{1 - \bar\alpha_t} x_0 + \frac{\sqrt{\alpha_t}(1 - \bar\alpha_{t-1})}{1 - \bar\alpha_t} x_t, \quad \tilde\beta_t = \frac{1 - \bar\alpha_{t-1}}{1 - \bar\alpha_t} \beta_t$$

$\tilde\mu_t$ 依赖 $x_0$，但生成时没有 $x_0$。利用 $x_0 = (x_t - \sqrt{1-\bar\alpha_t}\,\epsilon) / \sqrt{\bar\alpha_t}$ 代入：

$$\tilde\mu_t = \frac{1}{\sqrt{\alpha_t}} \left( x_t - \frac{\beta_t}{\sqrt{1 - \bar\alpha_t}} \epsilon \right)$$

因此让网络预测 $\epsilon_\theta(x_t, t)$ 即可计算 $\mu_\theta$：

$$\mu_\theta(x_t, t) = \frac{1}{\sqrt{\alpha_t}} \left( x_t - \frac{\beta_t}{\sqrt{1 - \bar\alpha_t}} \epsilon_\theta(x_t, t) \right)$$

**预测噪声的好处**：(1) $\epsilon \sim \mathcal{N}(0,I)$ 统计特性稳定，比预测自然图像更简单；(2) 所有时间步共享同一目标分布；(3) 实验中 FID 比预测 $x_0$ 或 $\mu$ 好 20%+。

### 3.3 方差选择

$\sigma_t^2 = \beta_t$（DDPM 原文，上界）或 $\sigma_t^2 = \tilde\beta_t$（后验方差，下界），实践中差别不大。Improved DDPM 让网络学习方差（在两个边界间插值），获得了更好的对数似然。

## 四、训练目标

### 4.1 从 ELBO 到简化损失

完整 ELBO 展开为 $T$ 个 KL 散度项。因为两侧都是高斯，KL 退化为均值的 MSE。用 $\epsilon_\theta$ 参数化后，每项变为：

$$D_{KL} \propto \frac{\beta_t^2}{2\sigma_t^2 \alpha_t (1 - \bar\alpha_t)} \|\epsilon - \epsilon_\theta(x_t, t)\|^2$$

**Ho et al. 的关键发现**：去掉随 $t$ 变化的权重系数，使用等权简化损失效果更好：

$$\boxed{L_{\text{simple}} = \mathbb{E}_{t,\, x_0,\, \epsilon} \left[ \| \epsilon - \epsilon_\theta(x_t, t) \|^2 \right]}$$

### 4.2 训练算法

```
算法：DDPM 训练
───────────────────────────────────────
重复直到收敛：
  1. 采样 x_0 ~ 数据集
  2. 采样 t ~ Uniform{1, ..., T}
  3. 采样 ε ~ N(0, I)
  4. 计算 x_t = √(ᾱ_t) * x_0 + √(1 - ᾱ_t) * ε
  5. 计算损失 L = ||ε - ε_θ(x_t, t)||²
  6. 反向传播，更新 θ
```

优雅之处：不需逐步加噪、不需执行反向采样链、损失就是 MSE——没有对抗训练，没有 KL 平衡项。

### 4.3 时间步嵌入

网络需要知道当前噪声水平。时间步 $t$ 通过正弦位置编码（与 Transformer 相同）转为高维向量，以加法或 scale-and-shift 方式注入 U-Net 每个残差块：

$$\text{PE}(t, 2i) = \sin(t / 10000^{2i/d}), \quad \text{PE}(t, 2i+1) = \cos(t / 10000^{2i/d})$$

## 五、DDPM 采样

```
算法：DDPM 采样
───────────────────────────────────────
1. x_T ~ N(0, I)
2. for t = T, T-1, ..., 1:
     ε̂ = ε_θ(x_t, t)
     z ~ N(0, I)（t=1 时 z=0）
     x_{t-1} = (1/√α_t)(x_t - β_t/√(1-ᾱ_t) * ε̂) + σ_t * z
3. 返回 x_0
```

确定性部分根据预测噪声计算去噪均值，随机部分 $\sigma_t z$ 维持分布正确性。最后一步不加噪声。

**缺点**：1000 步网络推理 = 极慢。以 256x256 图像为例：

- 每步 U-Net 推理约 50ms（A100 GPU）
- 1000 步 = 50 秒生成一张图
- 对比 GAN：一次前向传播 = 几十毫秒

这促成了 DDIM 和后续众多加速方法的提出。

## 六、DDIM 采样

### 6.1 核心思想

DDIM（Song et al., 2020）发现：**反向过程不一定需要随机性**。存在一族非马尔可夫前向过程，与 DDPM 共享 $q(x_t|x_0)$ 但反向可以确定性。

### 6.2 更新公式

$$x_{t-1} = \sqrt{\bar\alpha_{t-1}} \underbrace{\left( \frac{x_t - \sqrt{1 - \bar\alpha_t}\, \epsilon_\theta}{\sqrt{\bar\alpha_t}} \right)}_{\text{预测 } x_0} + \underbrace{\sqrt{1 - \bar\alpha_{t-1} - \sigma_t^2}\, \epsilon_\theta}_{\text{方向分量}} + \sigma_t z$$

- $\sigma_t = \sqrt{(1-\bar\alpha_{t-1})/(1-\bar\alpha_t)} \cdot \sqrt{\beta_t}$：退化为 DDPM
- $\sigma_t = 0$：完全确定性（纯 DDIM）

### 6.3 跳步采样

DDIM 可选时间步子序列，如 $\tau = [1000, 950, \ldots, 50, 1]$（20 步），公式中直接使用对应 $\bar\alpha$ 值。

| 方法 | 步数 | FID (CIFAR-10) | 加速 |
|------|------|----------------|------|
| DDPM | 1000 | 3.17 | 1x |
| DDIM | 100 | 4.67 | 10x |
| DDIM | 50 | 4.97 | 20x |
| DDIM | 20 | 6.84 | 50x |

50 步 DDIM 已接近 1000 步 DDPM 质量，速度快 20 倍。

### 6.4 确定性映射的额外特性

$\sigma_t = 0$ 时，同一噪声 $x_T$ 始终生成同一图像，支持：(1) **语义插值**——在两个 $x_T$ 间做 slerp 后去噪得到平滑过渡；(2) **图像编码**——将真实图像"反转"为 $x_T$，实现编辑。

### 7.1 NumPy 演示：前向加噪过程可视化

```python
"""前向扩散过程可视化：线性 vs 余弦调度对比。完整可运行。"""
import numpy as np
import matplotlib.pyplot as plt

def linear_schedule(T, beta_start=1e-4, beta_end=0.02):
    return np.linspace(beta_start, beta_end, T)

def cosine_schedule(T, s=0.008):
    steps = np.arange(T + 1, dtype=np.float64)
    f = np.cos((steps / T + s) / (1 + s) * np.pi / 2) ** 2
    alphas_cumprod = f / f[0]
    betas = 1 - alphas_cumprod[1:] / alphas_cumprod[:-1]
    return np.clip(betas, 0, 0.999)

def forward_diffusion(x_0, t, alpha_bars, noise=None):
    """x_t = sqrt(ᾱ_t) * x_0 + sqrt(1 - ᾱ_t) * ε"""
    ab = alpha_bars[t]
    if noise is None:
        noise = np.random.randn(*x_0.shape)
    return np.sqrt(ab) * x_0 + np.sqrt(1 - ab) * noise, noise

# 生成 64x64 彩色测试图案
np.random.seed(42)
H, W = 64, 64
y, x = np.mgrid[-1:1:complex(H), -1:1:complex(W)]
r = np.sqrt(x**2 + y**2)
img = np.stack([np.sin(r * 8), np.cos(x * 6), np.sin(y * 6)], axis=-1)
img = np.clip(img, -1, 1)

T = 1000
ab_lin = np.cumprod(1.0 - linear_schedule(T))
ab_cos = np.cumprod(1.0 - cosine_schedule(T))
noise = np.random.randn(*img.shape)

# ---- 可视化加噪过程 ----
steps = [0, 50, 100, 200, 400, 600, 800, 999]
fig, axes = plt.subplots(2, len(steps), figsize=(20, 6))
fig.suptitle("Forward Diffusion: Linear vs Cosine Schedule", fontsize=14)

for i, t in enumerate(steps):
    for row, ab in enumerate([ab_lin, ab_cos]):
        noisy = img if t == 0 else forward_diffusion(img, t - 1, ab, noise)[0]
        axes[row, i].imshow(np.clip((noisy + 1) / 2, 0, 1))
        axes[row, i].set_title(f"t={t}", fontsize=10); axes[row, i].axis("off")

axes[0, 0].set_ylabel("Linear", fontsize=12, rotation=0, labelpad=50)
axes[1, 0].set_ylabel("Cosine", fontsize=12, rotation=0, labelpad=50)
plt.tight_layout(rect=[0, 0, 1, 0.93])
plt.savefig("forward_diffusion_visualization.png", dpi=150, bbox_inches="tight")
plt.show()

# ---- alpha_bar 和 SNR 曲线 ----
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(14, 5))
t_ax = np.arange(T)
ax1.plot(t_ax, ab_lin, label="Linear", lw=2, color="#e74c3c")
ax1.plot(t_ax, ab_cos, label="Cosine", lw=2, color="#3498db")
ax1.set(xlabel="t", ylabel=r"$\bar{\alpha}_t$", title=r"$\bar{\alpha}_t$ over Time")
ax1.legend(); ax1.grid(alpha=0.3)

ax2.semilogy(t_ax, ab_lin / (1 - ab_lin + 1e-8), label="Linear", lw=2, color="#e74c3c")
ax2.semilogy(t_ax, ab_cos / (1 - ab_cos + 1e-8), label="Cosine", lw=2, color="#3498db")
ax2.set(xlabel="t", ylabel="SNR (log)", title="Signal-to-Noise Ratio")
ax2.legend(); ax2.grid(alpha=0.3)
plt.tight_layout()
plt.savefig("noise_schedule_comparison.png", dpi=150, bbox_inches="tight")
plt.show()

print(f"线性 t=500: ᾱ={ab_lin[499]:.4f} | 余弦 t=500: ᾱ={ab_cos[499]:.4f}")
print("余弦调度更均匀地利用了整个时间范围")
```

### 7.2 PyTorch 简化版 DDPM 训练（MNIST）

```python
"""简化版 DDPM：在 MNIST 上训练小型 U-Net。完整可运行，GPU 约 5 分钟。"""
import torch, torch.nn as nn, torch.nn.functional as F
from torch.utils.data import DataLoader
from torchvision import datasets, transforms
import numpy as np, matplotlib.pyplot as plt

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

# ---- 噪声调度 ----
class NoiseSchedule:
    def __init__(self, T=1000, beta_start=1e-4, beta_end=0.02, device="cpu"):
        self.T, self.device = T, device
        self.betas = torch.linspace(beta_start, beta_end, T, device=device)
        self.alphas = 1.0 - self.betas
        self.alpha_bars = torch.cumprod(self.alphas, dim=0)
        self.sqrt_ab = torch.sqrt(self.alpha_bars)
        self.sqrt_1m_ab = torch.sqrt(1.0 - self.alpha_bars)

    def forward(self, x_0, t, noise=None):
        if noise is None: noise = torch.randn_like(x_0)
        s = self.sqrt_ab[t].view(-1,1,1,1)
        n = self.sqrt_1m_ab[t].view(-1,1,1,1)
        return s * x_0 + n * noise, noise

# ---- 简化 U-Net ----
class SinPosEmb(nn.Module):
    def __init__(self, d):
        super().__init__(); self.d = d
    def forward(self, t):
        half = self.d // 2
        e = torch.exp(torch.arange(half, device=t.device) * -(np.log(10000) / (half - 1)))
        e = t.float().unsqueeze(1) * e.unsqueeze(0)
        return torch.cat([torch.sin(e), torch.cos(e)], dim=1)

class ResBlock(nn.Module):
    def __init__(self, c_in, c_out, t_dim):
        super().__init__()
        self.conv1 = nn.Conv2d(c_in, c_out, 3, padding=1)
        self.conv2 = nn.Conv2d(c_out, c_out, 3, padding=1)
        self.t_mlp = nn.Linear(t_dim, c_out)
        self.bn1, self.bn2 = nn.BatchNorm2d(c_out), nn.BatchNorm2d(c_out)
        self.skip = nn.Conv2d(c_in, c_out, 1) if c_in != c_out else nn.Identity()

    def forward(self, x, t):
        h = F.gelu(self.bn1(self.conv1(x)))
        h = h + F.gelu(self.t_mlp(t))[:, :, None, None]
        return F.gelu(self.bn2(self.conv2(h))) + self.skip(x)

class SimpleUNet(nn.Module):
    """编码器→中间层→解码器 + 跳跃连接，适配 MNIST 28x28"""
    def __init__(self, ch=64, t_dim=128):
        super().__init__()
        self.t_emb = nn.Sequential(SinPosEmb(t_dim), nn.Linear(t_dim,t_dim),
                                   nn.GELU(), nn.Linear(t_dim,t_dim))
        self.enc1 = ResBlock(1, ch, t_dim)
        self.down1 = nn.Conv2d(ch, ch, 3, stride=2, padding=1)        # 28→14
        self.enc2 = ResBlock(ch, ch*2, t_dim)
        self.down2 = nn.Conv2d(ch*2, ch*2, 3, stride=2, padding=1)    # 14→7
        self.mid  = ResBlock(ch*2, ch*2, t_dim)
        self.up2  = nn.ConvTranspose2d(ch*2, ch*2, 4, stride=2, padding=1)  # 7→14
        self.dec2 = ResBlock(ch*4, ch, t_dim)   # concat 后通道翻倍
        self.up1  = nn.ConvTranspose2d(ch, ch, 4, stride=2, padding=1)      # 14→28
        self.dec1 = ResBlock(ch*2, ch, t_dim)
        self.out  = nn.Conv2d(ch, 1, 1)

    def forward(self, x, t):
        te = self.t_emb(t)
        e1 = self.enc1(x, te)
        e2 = self.enc2(self.down1(e1), te)
        m  = self.mid(self.down2(e2), te)
        d2 = self.dec2(torch.cat([self.up2(m), e2], 1), te)
        d1 = self.dec1(torch.cat([self.up1(d2), e1], 1), te)
        return self.out(d1)

# ---- 训练 ----
def train_ddpm(epochs=10, bs=128, lr=2e-4, T=1000):
    tf = transforms.Compose([transforms.ToTensor(), transforms.Normalize([.5],[.5])])
    loader = DataLoader(datasets.MNIST("./data", train=True, download=True, transform=tf),
                        batch_size=bs, shuffle=True, num_workers=2, drop_last=True)
    sched = NoiseSchedule(T, device=device)
    model = SimpleUNet().to(device)
    opt = torch.optim.Adam(model.parameters(), lr=lr)
    print(f"参数量: {sum(p.numel() for p in model.parameters()):,}")
    losses = []
    for ep in range(epochs):
        model.train(); ep_loss = 0
        for x0, _ in loader:
            x0 = x0.to(device)
            t = torch.randint(0, T, (x0.shape[0],), device=device)
            xt, noise = sched.forward(x0, t)
            loss = F.mse_loss(model(xt, t), noise)
            opt.zero_grad(); loss.backward(); opt.step()
            ep_loss += loss.item()
        losses.append(ep_loss / len(loader))
        print(f"Epoch {ep+1}/{epochs} | Loss: {losses[-1]:.6f}")
    return model, sched, losses

model, schedule, losses = train_ddpm(epochs=10)
```

### 7.3 DDPM vs DDIM 采样对比

```python
"""DDPM 与 DDIM 采样实现及对比可视化。"""

@torch.no_grad()
def sample_ddpm(model, sc, n=16, shape=(1,28,28)):
    """DDPM：完整 T 步随机去噪"""
    model.eval(); x = torch.randn(n, *shape, device=sc.device); intermediates = []
    for t in reversed(range(sc.T)):
        tb = torch.full((n,), t, device=sc.device, dtype=torch.long)
        eps = model(x, tb)
        mean = (1/torch.sqrt(sc.alphas[t])) * (
            x - sc.betas[t]/torch.sqrt(1-sc.alpha_bars[t]) * eps)
        x = mean + (torch.sqrt(sc.betas[t]) * torch.randn_like(x) if t > 0 else 0)
        if t in [999,700,500,300,100,50,0]:
            intermediates.append((t, x.cpu().clone()))
    return x.cpu(), intermediates

@torch.no_grad()
def sample_ddim(model, sc, n=16, shape=(1,28,28), steps=50, eta=0.0):
    """DDIM：确定性（eta=0）或随机采样，支持跳步"""
    model.eval()
    ts = list(reversed(list(range(0, sc.T, sc.T // steps))[:steps]))
    x = torch.randn(n, *shape, device=sc.device)
    for i, t in enumerate(ts):
        tb = torch.full((n,), t, device=sc.device, dtype=torch.long)
        eps = model(x, tb)
        ab = sc.alpha_bars[t]
        # 最后一步约定 α̅_prev = 1.0（即 t=0 时信号完全恢复），这是 DDIM 论文的标准约定
        ab_prev = sc.alpha_bars[ts[i+1]] if i < len(ts)-1 else torch.tensor(1.0, device=sc.device)
        x0 = torch.clamp((x - torch.sqrt(1-ab)*eps) / torch.sqrt(ab), -1, 1)
        sigma = eta * torch.sqrt((1-ab_prev)/(1-ab) * (1-ab/ab_prev))
        d = torch.sqrt(1-ab_prev-sigma**2) * eps
        z = torch.randn_like(x) if i < len(ts)-1 else 0
        x = torch.sqrt(ab_prev) * x0 + d + sigma * z
    return x.cpu()

# ---- 采样并可视化 ----
print("DDPM (1000 步)..."); ddpm_out, intermed = sample_ddpm(model, schedule)
print("DDIM (50 步)...");   ddim50 = sample_ddim(model, schedule, steps=50)
print("DDIM (20 步)...");   ddim20 = sample_ddim(model, schedule, steps=20)

fig, axes = plt.subplots(3, 16, figsize=(20, 5))

axes[0,0].set_ylabel("DDPM\n1000", rotation=0, labelpad=50, fontsize=10)
```

## 八、工程师视角

### 8.1 噪声调度的实际影响

```python
betas = np.linspace(1e-4, 0.02, 1000)
ab = np.cumprod(1 - betas)
# t=100: ᾱ≈0.98（几乎没加噪）  t=500: ᾱ≈0.09（信号很弱）  t=700: ᾱ≈0.003（纯噪声）
```

t=500 后信号已消失，线性调度浪费了 500 步。余弦调度让 alpha_bar 更平滑衰减。工程建议：低分辨率用余弦，高分辨率用 scaled linear（Stable Diffusion）。

### 8.2 采样速度 vs 质量权衡

| 场景 | 步数 | 方法 | 延迟 |
|------|------|------|------|
| 实时预览 | 5~10 | DPM-Solver | <1s |
| 生产级生成 | 20~50 | DPM-Solver++ | 2~5s |
| 最高质量 | 50~100 | DDIM / Euler | 5~10s |
| 科研评测 | 200~1000 | DDPM | >10s |

### 8.3 训练 T=1000 但推理 20 步？

$T=1000$ 定义的是噪声调度的精细程度，而非推理必须走的步数。模型学会了"在任意噪声水平下预测噪声"的通用能力，DDIM 只需选择一个子集即可。$T=1000$ 是标准选择，更大极少需要，更小可能使每步噪声变化过大。

### 8.4 实用训练技巧

**EMA（必须用）**：采样时使用指数移动平均权重（decay=0.9999），FID 提升 5~15%，几乎免费。

```python
class EMA:
    def __init__(self, model, decay=0.9999):
        self.decay = decay
        self.shadow = {n: p.data.clone() for n, p in model.named_parameters() if p.requires_grad}
        self.backup = {}  # 用于 restore
    def update(self, model):
        for n, p in model.named_parameters():
            if p.requires_grad:
                self.shadow[n] = self.decay * self.shadow[n] + (1 - self.decay) * p.data
    def apply(self, model):
        """切换到 EMA 权重（采样/评估时调用）"""
        self.backup = {n: p.data.clone() for n, p in model.named_parameters() if p.requires_grad}
        for n, p in model.named_parameters():
            if p.requires_grad: p.data.copy_(self.shadow[n])
    def restore(self, model):
        """恢复训练权重（评估结束后调用，否则训练权重会被覆盖）"""
        for n, p in model.named_parameters():
            if p.requires_grad: p.data.copy_(self.backup[n])
        self.backup = {}
# 使用模式：训练中 ema.update(model) → 评估时 ema.apply(model) → 评估后 ema.restore(model)
```

**梯度裁剪**：`torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)`，不同时间步梯度量级差异大。

**归一化到 [-1, 1]**：零中心数据与高斯噪声混合更自然（不是 [0, 1]）。

### 8.5 常见误区

**误区 1："扩散模型很慢所以不实用"**

早期的 DDPM 确实慢（1000 步），但现代采样器（DPM-Solver++、Consistency Models）已将步数降到 1~4 步，生成一张 512x512 图像只需不到 1 秒。Stable Diffusion XL Turbo 甚至可以实时生成。

**误区 2："预测噪声是唯一的参数化方式"**

除了预测 $\epsilon$，还有：
- **预测 $x_0$**：直接预测原始图像
- **预测 $v = \sqrt{\bar\alpha_t}\epsilon - \sqrt{1-\bar\alpha_t}x_0$**（v-prediction，Stable Diffusion 2.x 使用）
- **预测 score** $\nabla_{x_t} \log p(x_t)$（score matching 视角）

这些参数化在数学上等价，但在训练动态和数值稳定性上各有优劣。

**误区 3："时间步 $t$ 是连续的"**

标准 DDPM 中 $t$ 是离散的（1 到 $T$）。Score-based models 将其推广为连续时间 SDE 框架，但这是更高级的主题。

**误区 4："扩散模型需要巨大的数据集"**

扩散模型的数据效率并不差。在 MNIST 这样的小数据集上也能训练出不错的模型（如本文代码所示）。数据量越大效果越好——但这对所有生成模型都一样。

## 九、总结与速查表

### 核心公式

| 公式 | 含义 |
|------|------|
| $x_t = \sqrt{\bar\alpha_t}\, x_0 + \sqrt{1 - \bar\alpha_t}\, \epsilon$ | 前向加噪 |
| $L = \|\epsilon - \epsilon_\theta(x_t, t)\|^2$ | 训练损失 |
| $x_{t-1} = \frac{1}{\sqrt{\alpha_t}}(x_t - \frac{\beta_t}{\sqrt{1-\bar\alpha_t}}\epsilon_\theta) + \sigma_t z$ | DDPM 采样 |
| $x_{t-1} = \sqrt{\bar\alpha_{t-1}}\hat{x}_0 + \sqrt{1-\bar\alpha_{t-1}}\,\epsilon_\theta$ | DDIM 采样 ($\eta=0$) |

### DDPM vs DDIM

| 特性 | DDPM | DDIM |
|------|------|------|
| 采样类型 | 随机 | 确定性 ($\eta=0$) 或随机 |
| 步数 | 必须 $T$ 步 | 可任意少 |
| 同一噪声同一结果 | 否 | 是 ($\eta=0$) |
| 典型用法 | 科研评测 | 实际部署 |

### 核心要点

1. **扩散模型 = 前向加噪 + 反向去噪**。前向固定，反向由神经网络学习。
2. **关键数学**：$x_t = \sqrt{\bar\alpha_t}\, x_0 + \sqrt{1-\bar\alpha_t}\, \epsilon$，训练时不需逐步加噪。
3. **预测噪声 $\epsilon$** 优于预测 $x_0$ 或 $\mu$，这是 DDPM 的核心工程洞察。
4. **训练极简**：随机采时间步、加噪、预测噪声、MSE。没有对抗训练，没有 KL 平衡。
5. **DDIM 跳步加速**：推理 20~50 步，速度提升 20~50 倍，质量损失很小。
6. **噪声调度很重要**：余弦通常优于线性。
7. **EMA 是免费午餐**：所有扩散模型都应使用 EMA 权重推理。
