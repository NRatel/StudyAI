## 1. 扩散模型的骨干网络
### 1.1 扩散模型的"大脑"是什么？

上一节我们学了扩散模型的数学：给定含噪图像 $\mathbf{x}_t$ 和时间步 $t$，模型需要预测噪声 $\boldsymbol{\epsilon}_\theta(\mathbf{x}_t, t)$。但具体用什么网络来实现这个 $\boldsymbol{\epsilon}_\theta$？

答案经历了一条清晰的演进路线：

```
U-Net（2015, 最初用于医学图像分割）
  │
  ├── DDPM 采用 U-Net 做去噪骨干（2020）
  │     加入时间步嵌入 + 注意力层
  │
  ├── Stable Diffusion 在潜空间用 U-Net（2022）
  │     加入 Cross-Attention 接收文本条件
  │
  ▼
DiT（Diffusion Transformer, 2023）
  │
  ├── 用 Transformer 完全替代 U-Net
  ├── 与 Scaling Laws 更兼容
  │
  ▼
Sora 等视频生成模型（2024）
      DiT 路线扩展到视频
```

### 1.2 为什么不直接在像素空间做扩散？

一张 $512 \times 512$ 的 RGB 图像有 $512 \times 512 \times 3 = 786{,}432$ 个数值。让网络在这么高维的空间里一步步去噪，计算量巨大，训练一张卡跑不动。

**Latent Diffusion**（潜空间扩散）的核心思路：先用一个预训练的 VAE 编码器把图像压缩到低维潜空间（比如 $64 \times 64 \times 4$），在潜空间做扩散，最后再用 VAE 解码器还原。维度从 786,432 降到 $64 \times 64 \times 4 = 16{,}384$——减少了约 **48 倍**。

### 1.3 Stable Diffusion 的全景图

```
               Stable Diffusion 完整管线
               ========================

"a photo of          ┌──────────────┐     条件向量 c
 a cat on       ──→  │ CLIP 文本    │  ──→  (77, 768)
 the moon"           │ 编码器       │
                     └──────────────┘

                     ┌──────────────┐     干净潜变量 z₀
 随机噪声 z_T   ──→  │ U-Net 去噪   │  ──→  (4, 64, 64)
 (4, 64, 64)         │（T 步迭代）   │
                     │ + CFG 引导   │
                     └──────────────┘
                            ↑
                        Cross-Attention
                        接收条件 c

                     ┌──────────────┐     最终图像
 干净潜变量 z₀  ──→  │ VAE 解码器   │  ──→  (3, 512, 512)
                     └──────────────┘
```

一句话总结：**Stable Diffusion = CLIP（理解文本）+ U-Net（潜空间去噪）+ VAE（还原图像）。**

## 2. U-Net、DiT 与潜空间扩散
### 2.1 U-Net 骨干

#### 2.1.1 原始 U-Net 回顾

U-Net 最初由 Ronneberger 等人（2015）为医学图像分割设计。其核心结构是**对称的编码器-解码器**加上**跳跃连接**：

```
输入图像                                                输出（与输入同尺寸）
   │                                                        ▲
   ▼                                                        │
┌──────┐  skip connection  ┌──────┐
│ Conv │ ─────────────────→│ Conv │    上采样路径（解码器）
│ 64ch │                   │ 64ch │
└──┬───┘                   └──▲───┘
   │ 下采样                    │ 上采样
   ▼                          │
┌──────┐  skip connection  ┌──────┐
│ Conv │ ─────────────────→│ Conv │
│128ch │                   │128ch │
└──┬───┘                   └──▲───┘
   │ 下采样                    │ 上采样
   ▼                          │
┌──────┐  skip connection  ┌──────┐
│ Conv │ ─────────────────→│ Conv │
│256ch │                   │256ch │
└──┬───┘                   └──▲───┘
   │ 下采样                    │ 上采样
   ▼                          │
   ┌────────────────────────────┐
   │      瓶颈层 (512ch)        │
   └────────────────────────────┘
```

关键设计：

| 组件 | 作用 |
|------|------|
| **编码器**（下采样路径） | 逐步降低空间分辨率、增加通道数，提取多尺度特征 |
| **解码器**（上采样路径） | 逐步恢复空间分辨率、减少通道数 |
| **跳跃连接** | 把编码器每一层的特征直接拼接到解码器对应层，保留细节信息 |
| **瓶颈层** | 分辨率最低、通道数最多的中间层，捕获全局语义 |

为什么 U-Net 适合做去噪？因为去噪需要**同时理解全局语义和局部细节**。编码器路径捕获"图里有什么"（语义），跳跃连接保留"东西在哪里"（细节），解码器路径把两者融合。

#### 2.1.2 扩散模型对 U-Net 的改造

DDPM（Ho et al., 2020）将 U-Net 适配为去噪网络，做了三项关键改造：

**改造 1：时间步嵌入（Timestep Embedding）**

去噪网络需要知道当前处于哪个时间步 $t$——$t$ 接近 $T$ 时图像几乎是纯噪声，$t$ 接近 $0$ 时图像接近干净。不同时间步的去噪策略完全不同。

时间步嵌入的做法借鉴了 Transformer 的正弦位置编码：

$$\text{PE}(t, 2i) = \sin\left(\frac{t}{10000^{2i/d}}\right), \quad \text{PE}(t, 2i+1) = \cos\left(\frac{t}{10000^{2i/d}}\right)$$

其中 $d$ 是嵌入维度，$i$ 是维度索引。这个正弦编码先把标量 $t$ 映射为 $d$ 维向量，再通过两层 MLP 投影后注入到 U-Net 的每一层：

$$\mathbf{t}_{\text{emb}} = \text{MLP}\big(\text{SinusoidalPE}(t)\big)$$

注入方式通常是加法或缩放：在 ResBlock 中，$\mathbf{t}_{\text{emb}}$ 通过线性层映射到与当前特征同维度，然后与特征相加。

**改造 2：Self-Attention 层**

在 U-Net 的中低分辨率层（如 $16 \times 16$ 和 $8 \times 8$）加入 Self-Attention，让网络能建模长距离依赖关系。高分辨率层（如 $64 \times 64$）不加 Attention 是因为计算量太大（Attention 的复杂度是 $O(n^2)$，$n=64 \times 64 = 4096$ 已经很大了）。

**改造 3：Cross-Attention 接收条件**

对于条件扩散模型（如文生图），需要把文本条件 $\mathbf{c}$ 注入网络。做法是在 U-Net 中加入 **Cross-Attention** 层：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

其中：
- $Q$ 来自图像特征（U-Net 当前层的空间特征展平后投影）
- $K, V$ 来自文本条件（CLIP 编码器输出的 token 序列投影）

直觉：图像中的每个空间位置去"查询"文本中的每个 token，找出与自己最相关的文本信息，然后融合进来。比如图像中"天空"区域会更关注 "moon" 这个 token，而"动物"区域会更关注 "cat"。

#### 2.1.3 DDPM U-Net 中一个 ResBlock 的完整流程

```
输入特征 h
  │
  ├── GroupNorm → SiLU → Conv3x3 ──→ h'
  │
  ├── 时间步嵌入 t_emb → Linear ──→ 与 h' 相加
  │
  ├── GroupNorm → SiLU → Dropout → Conv3x3 ──→ h''
  │
  └── 残差连接：h_out = h'' + shortcut(h)
```

每个 ResBlock 之后，可选地加入 Self-Attention 或 Cross-Attention 层。

### 2.2 DiT（Diffusion Transformer）

#### 2.2.1 动机：U-Net 的局限

U-Net 在扩散模型中表现出色，但有几个问题：

1. **架构设计依赖人工经验**：下采样多少层、每层多少通道、哪里放 Attention——这些超参数需要反复调。
2. **扩展性不够好**：增加参数时，性能提升不如 Transformer 可预测。Transformer 有 Scaling Laws（参数量、数据量、计算量的幂律关系），U-Net 没有类似的规律。
3. **与 NLP 基础设施不兼容**：NLP 领域积累了大量 Transformer 的优化技术（FlashAttention、张量并行、混合精度训练等），U-Net 无法直接受益。

#### 2.2.2 DiT 的核心设计

Peebles & Xie（2023）提出 DiT，用 Transformer 完全替代 U-Net。核心流程：

```
输入潜变量 z (C, H, W)
         │
         ▼
  ┌──────────────┐
  │  Patchify    │    把 z 分成 p×p 的 patch，展平为 token 序列
  │  (类似 ViT)  │    (H/p × W/p) 个 token，每个 token 维度 = p²·C
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │ 线性嵌入 +   │    每个 patch token 投影到 d 维
  │ 位置编码     │    加上可学习的位置嵌入
  └──────┬───────┘
         │
         ▼
  ┌──────────────────────────────────────────────────┐
  │              DiT Block × N                       │
  │  ┌────────────────────────────────────────────┐  │
  │  │ AdaLN-Zero(condition) → Self-Attention     │  │
  │  │ AdaLN-Zero(condition) → FFN (MLP)          │  │
  │  └────────────────────────────────────────────┘  │
  └──────────────────┬───────────────────────────────┘
                     │
                     ▼
              ┌──────────────┐
              │   线性解码    │    投影回 p²·C 维
              │ + Unpatchify  │    重组为 (C, H, W)
              └──────────────┘
                     │
                     ▼
             预测的噪声 ε (C, H, W)
```

**Patchify**：和 ViT（Vision Transformer）一样，把输入切成不重叠的 patch。比如 $32 \times 32$ 的潜变量用 $2 \times 2$ 的 patch，得到 $16 \times 16 = 256$ 个 token。patch 越小，token 越多，精度越高但计算量也越大。

**AdaLN-Zero（自适应层归一化）**：这是 DiT 最关键的条件注入方式。

标准 Layer Norm 的公式：

$$\text{LN}(\mathbf{x}) = \gamma \odot \frac{\mathbf{x} - \mu}{\sigma} + \beta$$

其中 $\gamma$（缩放）和 $\beta$（偏移）是可学习参数。AdaLN 把 $\gamma$ 和 $\beta$ 变成**条件的函数**：

$$\gamma, \beta, \alpha = \text{MLP}(\mathbf{c} + \mathbf{t}_{\text{emb}})$$

$$\text{AdaLN}(\mathbf{x}) = \gamma(\mathbf{c}, t) \odot \frac{\mathbf{x} - \mu}{\sigma} + \beta(\mathbf{c}, t)$$

其中 $\alpha$ 是残差连接之前的缩放因子（"Zero" 的含义是训练开始时 $\alpha$ 初始化为 0，使得网络初始行为接近恒等映射，有利于训练稳定性）。

这样，条件信息（文本、类别标签、时间步）通过**调制归一化层的参数**来影响每一层的计算，而不是通过 Cross-Attention。

#### 2.2.3 DiT vs U-Net 对比

| 维度 | U-Net | DiT |
|------|-------|-----|
| 基础架构 | 编码器-解码器 + 跳跃连接 | 纯 Transformer（无层级结构） |
| 输入表示 | 像素/潜变量直接作为特征图 | 图像切成 patch token（类似 ViT） |
| 条件注入 | Cross-Attention + 时间步嵌入 | AdaLN-Zero（统一注入时间步和条件） |
| 分辨率处理 | 多尺度（下采样/上采样） | 单尺度（所有 token 在同一分辨率处理） |
| 整体风格 | CNN 风格（卷积 + 局部感受野） | Transformer 风格（全局自注意力） |
| 参数扩展 | 增加通道数或层数，效果不可预测 | 增加 depth/width，遵循 Scaling Laws |
| 训练效率 | 中等 | 与 NLP Transformer 共享优化技术 |
| 代表模型 | Stable Diffusion 1.5/2.0 | Stable Diffusion 3.0、Sora |

> **U-Net 到 DiT 的变化不只是骨干替换**，还包括三个层面的范式转变：(1) **输入表示**从像素/特征图变为 patch token——图像被切成不重叠的小块，每块展平为一个 token，与 NLP 的 tokenization 思路一致；(2) **条件注入**从 Cross-Attention 变为 AdaLN——不再让图像"查询"文本 token，而是用条件信息直接调制每一层的归一化参数，更高效也更统一；(3) **整体架构**从 CNN 风格（多尺度、局部卷积、跳跃连接）变为 Transformer 风格（单尺度、全局注意力、无层级结构）。这三重变化使得 DiT 能无缝继承 Transformer 生态的 Scaling Laws 和工程优化（FlashAttention、张量并行等），这才是工业界转向 DiT 的根本原因。

#### 2.2.4 DiT 的 Scaling Laws

DiT 论文的核心发现：在 ImageNet 类条件生成上，DiT 的性能（FID 分数）随模型参数量的增加呈现**可预测的幂律下降**，与 LLM 的 Scaling Laws 类似。

```
模型大小     参数量       GFLOPs    FID (越低越好)
DiT-S/2      33M          5.5       68.4
DiT-B/2      130M         22.7      43.5
DiT-L/2      458M         80.3      23.3
DiT-XL/2     675M         118.6      9.6
```

这意味着：只要有更多算力和数据，性能提升是可预测的。这正是 U-Net 所缺乏的性质，也是工业界转向 DiT 的核心原因。

### 2.3 Latent Diffusion（潜空间扩散）

#### 2.3.1 问题：像素空间扩散的计算瓶颈

假设要在 $256 \times 256 \times 3$ 的像素空间做扩散。U-Net 的每一步去噪都要处理这么大的张量，典型训练配置：

- 单张图像的特征图：数百 MB 显存
- Batch size 受限，训练速度慢
- Self-Attention 在 $256 \times 256$ 分辨率上的计算量：$O((256 \times 256)^2) = O(4.3 \times 10^9)$——不现实

#### 2.3.2 解决方案：两阶段流程

Rombach et al.（2022）提出 Latent Diffusion Model（LDM），把扩散过程转移到潜空间：

**第一阶段：训练自编码器（离线，一次性）**

$$\mathbf{z} = \mathcal{E}(\mathbf{x}), \quad \hat{\mathbf{x}} = \mathcal{D}(\mathbf{z})$$

- $\mathcal{E}$：编码器，将图像 $\mathbf{x} \in \mathbb{R}^{H \times W \times 3}$ 压缩为潜变量 $\mathbf{z} \in \mathbb{R}^{h \times w \times c}$
- $\mathcal{D}$：解码器，将潜变量还原为图像
- 下采样倍率 $f = H / h$，Stable Diffusion 使用 $f = 8$（即 $512 \times 512 \to 64 \times 64$）
- 这个自编码器通常是 KL 正则化的 VAE（带 KL 散度损失 + 感知损失 + 对抗损失）

**第二阶段：在潜空间训练扩散模型**

$$\mathcal{L}_{\text{LDM}} = \mathbb{E}_{\mathbf{z}_0, \boldsymbol{\epsilon}, t} \left[ \left\| \boldsymbol{\epsilon} - \boldsymbol{\epsilon}_\theta(\mathbf{z}_t, t, \mathbf{c}) \right\|_2^2 \right]$$

和原始 DDPM 损失完全一样，只不过 $\mathbf{z}_t$ 是潜变量而非像素。

#### 2.3.3 压缩比与信息保留

| 空间 | 尺寸 | 数据量 | 占比 |
|------|------|--------|------|
| 像素空间 | $512 \times 512 \times 3$ | 786,432 | 100% |
| 潜空间 | $64 \times 64 \times 4$ | 16,384 | 2.1% |

只用原始数据量 2% 的表示就能基本无损重建图像。这得益于自然图像的高冗余性——相邻像素高度相关，大量信息可以被压缩。

关键洞察：**感知上重要的信息**（形状、纹理、颜色分布）可以用很少的维度表示，被压缩掉的主要是**高频细节**（像素级噪声、微小纹理变化）。扩散模型不需要建模这些高频细节，VAE 解码器在最后一步补回来就行。

### 2.4 Stable Diffusion 完整管线

#### 2.4.1 三大组件详解

**组件一：CLIP 文本编码器**

CLIP（Contrastive Language-Image Pre-training, Radford et al., 2021）是在 4 亿个图文对上训练的多模态模型。Stable Diffusion 只用其文本编码器部分：

- 输入：文本 prompt（如 "a photo of a cat on the moon"）
- Tokenize：将文本分词为 token 序列，最长 77 个 token
- 输出：token 级别的特征序列 $\mathbf{c} \in \mathbb{R}^{77 \times 768}$（SD 1.x 用 CLIP ViT-L/14）

这个特征序列通过 Cross-Attention 注入 U-Net 的每一层。

**组件二：U-Net 去噪网络**

Stable Diffusion 1.5 的 U-Net 结构细节：

```
编码器:
  64×64, 320ch  →  32×32, 640ch  →  16×16, 1280ch  →  8×8, 1280ch

瓶颈:
  8×8, 1280ch (Self-Attention + Cross-Attention)

解码器:
  8×8, 1280ch  →  16×16, 1280ch  →  32×32, 640ch  →  64×64, 320ch
```

每个阶段包含 ResBlock + Attention Block。Attention Block 内部结构：

```
输入特征 h (B, C, H, W)
  │
  ├── reshape 为 (B, H*W, C) —— 空间维度展平为序列
  │
  ├── Self-Attention：图像特征自己互相关注
  │     Q = h @ W_q,  K = h @ W_k,  V = h @ W_v
  │
  ├── Cross-Attention：图像特征关注文本条件
  │     Q = h @ W_q,  K = c @ W_k,  V = c @ W_v
  │                    ^文本条件
  │
  ├── FFN (Linear → GEGLU → Linear)
  │
  └── reshape 回 (B, C, H, W)
```

**组件三：VAE 解码器**

- 输入：干净潜变量 $\mathbf{z}_0 \in \mathbb{R}^{4 \times 64 \times 64}$
- 通过卷积上采样网络，逐步恢复分辨率：$64 \to 128 \to 256 \to 512$
- 输出：RGB 图像 $\hat{\mathbf{x}} \in \mathbb{R}^{3 \times 512 \times 512}$

#### 2.4.2 推理流程（文生图）

完整的推理流程分为四步：

**Step 1：文本编码**

$$\mathbf{c} = \text{CLIP}_{\text{text}}(\text{prompt}) \in \mathbb{R}^{77 \times 768}$$

同时编码空文本（用于 Classifier-Free Guidance）：

$$\mathbf{c}_\varnothing = \text{CLIP}_{\text{text}}(\text{""}) \in \mathbb{R}^{77 \times 768}$$

**Step 2：初始化噪声**

$$\mathbf{z}_T \sim \mathcal{N}(\mathbf{0}, \mathbf{I}), \quad \mathbf{z}_T \in \mathbb{R}^{4 \times 64 \times 64}$$

**Step 3：迭代去噪（T 步）**

对于每个时间步 $t = T, T-1, \dots, 1$：

1. 分别预测有条件和无条件噪声：
   $$\boldsymbol{\epsilon}_{\text{cond}} = \boldsymbol{\epsilon}_\theta(\mathbf{z}_t, t, \mathbf{c}), \quad \boldsymbol{\epsilon}_{\text{uncond}} = \boldsymbol{\epsilon}_\theta(\mathbf{z}_t, t, \mathbf{c}_\varnothing)$$

2. 用 CFG 引导公式混合：
   $$\hat{\boldsymbol{\epsilon}} = \boldsymbol{\epsilon}_{\text{uncond}} + w \cdot (\boldsymbol{\epsilon}_{\text{cond}} - \boldsymbol{\epsilon}_{\text{uncond}})$$
   其中 $w$ 是引导强度（guidance scale），SD 默认 $w = 7.5$。

3. 用调度器（如 DDIM）从 $\mathbf{z}_t$ 计算 $\mathbf{z}_{t-1}$。

注意：CFG 需要对每个时间步做**两次**前向传播（有条件和无条件），所以实际计算量翻倍。实践中把两次的输入拼成一个 batch 一起算。

**Step 4：VAE 解码**

$$\hat{\mathbf{x}} = \mathcal{D}(\mathbf{z}_0)$$

#### 2.4.3 各组件参数量

以 Stable Diffusion 1.5 为例：

| 组件 | 参数量 | 训练状态 | 作用 |
|------|--------|----------|------|
| CLIP 文本编码器 | ~123M | 冻结（直接用预训练权重） | 将文本转化为语义向量 |
| U-Net | ~860M | 训练 | 在潜空间去噪（核心组件） |
| VAE | ~84M | 冻结或微调 | 像素空间 ↔ 潜空间转换 |
| **总计** | **~1.07B** | | |

U-Net 占了大部分参数，它是真正在训练中学习"如何去噪"的组件。CLIP 和 VAE 通常使用预训练权重，不参与扩散模型的训练。

### 2.5 与 Transformer 路线的融合

#### 2.5.1 趋势：两条技术路线正在合流

AI 领域存在两条主要的技术路线：

```
路线一：自回归 Transformer（语言模型路线）
  GPT → GPT-4 → ...
  特点：逐 token 预测下一个 token，用于文本生成

路线二：扩散模型（生成模型路线）
  DDPM → Stable Diffusion → ...
  特点：迭代去噪，用于图像/视频生成
```

DiT 是两条路线的交汇点——它用 Transformer 架构来做扩散生成。

#### 2.5.2 Sora：DiT 用于视频生成

OpenAI 的 Sora（2024）将 DiT 从图像扩展到视频。核心思路：

- 把视频看作"三维数据"：$(T_{\text{frames}}, H, W)$
- 用 3D 的 Patchify：在时间和空间维度都做分块，得到一组 3D patch token
- 用 Transformer 在这些 token 上做扩散去噪
- 文本条件通过 Cross-Attention 注入

视频生成的关键挑战是**时空一致性**——不仅每一帧要好看，帧与帧之间还要连贯。Transformer 的全局注意力天然适合建模长距离时空依赖，这是 U-Net 难以做到的。

#### 2.5.3 融合趋势的更多证据

| 模型 | 基础架构 | 生成方式 | 年份 |
|------|----------|----------|------|
| DALL-E 1 | Transformer | 自回归 | 2021 |
| Stable Diffusion 1.x/2.x | U-Net | 扩散 | 2022 |
| DALL-E 2 | U-Net + CLIP | 扩散 | 2022 |
| DALL-E 3 | U-Net | 扩散 | 2023 |
| DiT | Transformer | 扩散 | 2023 |
| Stable Diffusion 3.0 | MM-DiT (Transformer) | 扩散 + Flow Matching | 2024 |
| Sora | DiT (Transformer) | 扩散 | 2024 |

趋势非常清晰：**骨干网络从 U-Net 转向 Transformer**，而生成范式保持扩散（或其变体 Flow Matching）。Transformer 正在成为统一的基础架构——无论是理解（语言模型）还是生成（图像/视频），底层都是 Transformer。

## 本节小结

| 概念 | 一句话总结 |
|------|-----------|
| U-Net 骨干 | 编码器-解码器 + 跳跃连接，通过时间步嵌入和 Cross-Attention 适配为扩散去噪网络 |
| DiT | 用 Transformer 替代 U-Net，通过 Patchify + AdaLN-Zero 实现，与 Scaling Laws 兼容 |
| Latent Diffusion | 在 VAE 压缩后的潜空间做扩散，维度降低约 48 倍，使高分辨率生成成为可能 |
| Stable Diffusion | CLIP（文本编码）+ U-Net/DiT（潜空间去噪）+ VAE（图像还原）的三件套 |
| ControlNet | 通过冻结原始 U-Net + 训练控制分支，实现边缘/深度/姿态等精细空间控制 |
| LoRA | 低秩分解的轻量微调方法，用不到 1% 的参数实现风格定制 |
| Diffusers | Hugging Face 的扩散模型标准库，提供 Pipeline/Model/Scheduler 三层抽象 |
