## 1. 直觉与概述

### 1.1 扩散模型的"大脑"是什么？

上一节我们学了扩散模型的数学：给定含噪图像 $\mathbf{x}_t$ 和时间步 $t$，模型需要预测噪声 $\boldsymbol{\epsilon}_\theta(\mathbf{x}_t, t)$。但具体用什么网络来实现这个 $\boldsymbol{\epsilon}_\theta$？

答案经历了一条清晰的演进路线：

```
U-Net（2015, 最初用于医学图像分割）
  │
  ├── DDPM 采用 U-Net 做去噪骨干（2020）
  │     加入时间步嵌入 + 注意力层
  │
  ├── Stable Diffusion 在潜空间用 U-Net（2022）
  │     加入 Cross-Attention 接收文本条件
  │
  ▼
DiT（Diffusion Transformer, 2023）
  │
  ├── 用 Transformer 完全替代 U-Net
  ├── 与 Scaling Laws 更兼容
  │
  ▼
Sora 等视频生成模型（2024）
      DiT 路线扩展到视频
```

### 1.2 为什么不直接在像素空间做扩散？

一张 $512 \times 512$ 的 RGB 图像有 $512 \times 512 \times 3 = 786{,}432$ 个数值。让网络在这么高维的空间里一步步去噪，计算量巨大，训练一张卡跑不动。

**Latent Diffusion**（潜空间扩散）的核心思路：先用一个预训练的 VAE 编码器把图像压缩到低维潜空间（比如 $64 \times 64 \times 4$），在潜空间做扩散，最后再用 VAE 解码器还原。维度从 786,432 降到 $64 \times 64 \times 4 = 16{,}384$——减少了约 **48 倍**。

### 1.3 Stable Diffusion 的全景图

```
               Stable Diffusion 完整管线
               ========================

"a photo of          ┌──────────────┐     条件向量 c
 a cat on       ──→  │ CLIP 文本    │  ──→  (77, 768)
 the moon"           │ 编码器       │
                     └──────────────┘

                     ┌──────────────┐     干净潜变量 z₀
 随机噪声 z_T   ──→  │ U-Net 去噪   │  ──→  (4, 64, 64)
 (4, 64, 64)         │（T 步迭代）   │
                     │ + CFG 引导   │
                     └──────────────┘
                            ↑
                        Cross-Attention
                        接收条件 c

                     ┌──────────────┐     最终图像
 干净潜变量 z₀  ──→  │ VAE 解码器   │  ──→  (3, 512, 512)
                     └──────────────┘
```

一句话总结：**Stable Diffusion = CLIP（理解文本）+ U-Net（潜空间去噪）+ VAE（还原图像）。**

## 2. 严谨定义与原理

### 2.1 U-Net 骨干

#### 2.1.1 原始 U-Net 回顾

U-Net 最初由 Ronneberger 等人（2015）为医学图像分割设计。其核心结构是**对称的编码器-解码器**加上**跳跃连接**：

```
输入图像                                                输出（与输入同尺寸）
   │                                                        ▲
   ▼                                                        │
┌──────┐  skip connection  ┌──────┐
│ Conv │ ─────────────────→│ Conv │    上采样路径（解码器）
│ 64ch │                   │ 64ch │
└──┬───┘                   └──▲───┘
   │ 下采样                    │ 上采样
   ▼                          │
┌──────┐  skip connection  ┌──────┐
│ Conv │ ─────────────────→│ Conv │
│128ch │                   │128ch │
└──┬───┘                   └──▲───┘
   │ 下采样                    │ 上采样
   ▼                          │
┌──────┐  skip connection  ┌──────┐
│ Conv │ ─────────────────→│ Conv │
│256ch │                   │256ch │
└──┬───┘                   └──▲───┘
   │ 下采样                    │ 上采样
   ▼                          │
   ┌────────────────────────────┐
   │      瓶颈层 (512ch)        │
   └────────────────────────────┘
```

关键设计：

| 组件 | 作用 |
|------|------|
| **编码器**（下采样路径） | 逐步降低空间分辨率、增加通道数，提取多尺度特征 |
| **解码器**（上采样路径） | 逐步恢复空间分辨率、减少通道数 |
| **跳跃连接** | 把编码器每一层的特征直接拼接到解码器对应层，保留细节信息 |
| **瓶颈层** | 分辨率最低、通道数最多的中间层，捕获全局语义 |

为什么 U-Net 适合做去噪？因为去噪需要**同时理解全局语义和局部细节**。编码器路径捕获"图里有什么"（语义），跳跃连接保留"东西在哪里"（细节），解码器路径把两者融合。

#### 2.1.2 扩散模型对 U-Net 的改造

DDPM（Ho et al., 2020）将 U-Net 适配为去噪网络，做了三项关键改造：

**改造 1：时间步嵌入（Timestep Embedding）**

去噪网络需要知道当前处于哪个时间步 $t$——$t$ 接近 $T$ 时图像几乎是纯噪声，$t$ 接近 $0$ 时图像接近干净。不同时间步的去噪策略完全不同。

时间步嵌入的做法借鉴了 Transformer 的正弦位置编码：

$$\text{PE}(t, 2i) = \sin\left(\frac{t}{10000^{2i/d}}\right), \quad \text{PE}(t, 2i+1) = \cos\left(\frac{t}{10000^{2i/d}}\right)$$

其中 $d$ 是嵌入维度，$i$ 是维度索引。这个正弦编码先把标量 $t$ 映射为 $d$ 维向量，再通过两层 MLP 投影后注入到 U-Net 的每一层：

$$\mathbf{t}_{\text{emb}} = \text{MLP}\big(\text{SinusoidalPE}(t)\big)$$

注入方式通常是加法或缩放：在 ResBlock 中，$\mathbf{t}_{\text{emb}}$ 通过线性层映射到与当前特征同维度，然后与特征相加。

**改造 2：Self-Attention 层**

在 U-Net 的中低分辨率层（如 $16 \times 16$ 和 $8 \times 8$）加入 Self-Attention，让网络能建模长距离依赖关系。高分辨率层（如 $64 \times 64$）不加 Attention 是因为计算量太大（Attention 的复杂度是 $O(n^2)$，$n=64 \times 64 = 4096$ 已经很大了）。

**改造 3：Cross-Attention 接收条件**

对于条件扩散模型（如文生图），需要把文本条件 $\mathbf{c}$ 注入网络。做法是在 U-Net 中加入 **Cross-Attention** 层：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

其中：
- $Q$ 来自图像特征（U-Net 当前层的空间特征展平后投影）
- $K, V$ 来自文本条件（CLIP 编码器输出的 token 序列投影）

直觉：图像中的每个空间位置去"查询"文本中的每个 token，找出与自己最相关的文本信息，然后融合进来。比如图像中"天空"区域会更关注 "moon" 这个 token，而"动物"区域会更关注 "cat"。

#### 2.1.3 DDPM U-Net 中一个 ResBlock 的完整流程

```
输入特征 h
  │
  ├── GroupNorm → SiLU → Conv3x3 ──→ h'
  │
  ├── 时间步嵌入 t_emb → Linear ──→ 与 h' 相加
  │
  ├── GroupNorm → SiLU → Dropout → Conv3x3 ──→ h''
  │
  └── 残差连接：h_out = h'' + shortcut(h)
```

每个 ResBlock 之后，可选地加入 Self-Attention 或 Cross-Attention 层。

### 2.2 DiT（Diffusion Transformer）

#### 2.2.1 动机：U-Net 的局限

U-Net 在扩散模型中表现出色，但有几个问题：

1. **架构设计依赖人工经验**：下采样多少层、每层多少通道、哪里放 Attention——这些超参数需要反复调。
2. **扩展性不够好**：增加参数时，性能提升不如 Transformer 可预测。Transformer 有 Scaling Laws（参数量、数据量、计算量的幂律关系），U-Net 没有类似的规律。
3. **与 NLP 基础设施不兼容**：NLP 领域积累了大量 Transformer 的优化技术（FlashAttention、张量并行、混合精度训练等），U-Net 无法直接受益。

#### 2.2.2 DiT 的核心设计

Peebles & Xie（2023）提出 DiT，用 Transformer 完全替代 U-Net。核心流程：

```
输入潜变量 z (C, H, W)
         │
         ▼
  ┌──────────────┐
  │  Patchify    │    把 z 分成 p×p 的 patch，展平为 token 序列
  │  (类似 ViT)  │    (H/p × W/p) 个 token，每个 token 维度 = p²·C
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │ 线性嵌入 +   │    每个 patch token 投影到 d 维
  │ 位置编码     │    加上可学习的位置嵌入
  └──────┬───────┘
         │
         ▼
  ┌──────────────────────────────────────────────────┐
  │              DiT Block × N                       │
  │  ┌────────────────────────────────────────────┐  │
  │  │ AdaLN-Zero(condition) → Self-Attention     │  │
  │  │ AdaLN-Zero(condition) → FFN (MLP)          │  │
  │  └────────────────────────────────────────────┘  │
  └──────────────────┬───────────────────────────────┘
                     │
                     ▼
              ┌──────────────┐
              │   线性解码    │    投影回 p²·C 维
              │ + Unpatchify  │    重组为 (C, H, W)
              └──────────────┘
                     │
                     ▼
             预测的噪声 ε (C, H, W)
```

**Patchify**：和 ViT（Vision Transformer）一样，把输入切成不重叠的 patch。比如 $32 \times 32$ 的潜变量用 $2 \times 2$ 的 patch，得到 $16 \times 16 = 256$ 个 token。patch 越小，token 越多，精度越高但计算量也越大。

**AdaLN-Zero（自适应层归一化）**：这是 DiT 最关键的条件注入方式。

标准 Layer Norm 的公式：

$$\text{LN}(\mathbf{x}) = \gamma \odot \frac{\mathbf{x} - \mu}{\sigma} + \beta$$

其中 $\gamma$（缩放）和 $\beta$（偏移）是可学习参数。AdaLN 把 $\gamma$ 和 $\beta$ 变成**条件的函数**：

$$\gamma, \beta, \alpha = \text{MLP}(\mathbf{c} + \mathbf{t}_{\text{emb}})$$

$$\text{AdaLN}(\mathbf{x}) = \gamma(\mathbf{c}, t) \odot \frac{\mathbf{x} - \mu}{\sigma} + \beta(\mathbf{c}, t)$$

其中 $\alpha$ 是残差连接之前的缩放因子（"Zero" 的含义是训练开始时 $\alpha$ 初始化为 0，使得网络初始行为接近恒等映射，有利于训练稳定性）。

这样，条件信息（文本、类别标签、时间步）通过**调制归一化层的参数**来影响每一层的计算，而不是通过 Cross-Attention。

#### 2.2.3 DiT vs U-Net 对比

| 维度 | U-Net | DiT |
|------|-------|-----|
| 基础架构 | 编码器-解码器 + 跳跃连接 | 纯 Transformer（无层级结构） |
| 输入表示 | 像素/潜变量直接作为特征图 | 图像切成 patch token（类似 ViT） |
| 条件注入 | Cross-Attention + 时间步嵌入 | AdaLN-Zero（统一注入时间步和条件） |
| 分辨率处理 | 多尺度（下采样/上采样） | 单尺度（所有 token 在同一分辨率处理） |
| 整体风格 | CNN 风格（卷积 + 局部感受野） | Transformer 风格（全局自注意力） |
| 参数扩展 | 增加通道数或层数，效果不可预测 | 增加 depth/width，遵循 Scaling Laws |
| 训练效率 | 中等 | 与 NLP Transformer 共享优化技术 |
| 代表模型 | Stable Diffusion 1.5/2.0 | Stable Diffusion 3.0、Sora |

> **U-Net 到 DiT 的变化不只是骨干替换**，还包括三个层面的范式转变：(1) **输入表示**从像素/特征图变为 patch token——图像被切成不重叠的小块，每块展平为一个 token，与 NLP 的 tokenization 思路一致；(2) **条件注入**从 Cross-Attention 变为 AdaLN——不再让图像"查询"文本 token，而是用条件信息直接调制每一层的归一化参数，更高效也更统一；(3) **整体架构**从 CNN 风格（多尺度、局部卷积、跳跃连接）变为 Transformer 风格（单尺度、全局注意力、无层级结构）。这三重变化使得 DiT 能无缝继承 Transformer 生态的 Scaling Laws 和工程优化（FlashAttention、张量并行等），这才是工业界转向 DiT 的根本原因。

#### 2.2.4 DiT 的 Scaling Laws

DiT 论文的核心发现：在 ImageNet 类条件生成上，DiT 的性能（FID 分数）随模型参数量的增加呈现**可预测的幂律下降**，与 LLM 的 Scaling Laws 类似。

```
模型大小     参数量       GFLOPs    FID (越低越好)
DiT-S/2      33M          5.5       68.4
DiT-B/2      130M         22.7      43.5
DiT-L/2      458M         80.3      23.3
DiT-XL/2     675M         118.6      9.6
```

这意味着：只要有更多算力和数据，性能提升是可预测的。这正是 U-Net 所缺乏的性质，也是工业界转向 DiT 的核心原因。

### 2.3 Latent Diffusion（潜空间扩散）

#### 2.3.1 问题：像素空间扩散的计算瓶颈

假设要在 $256 \times 256 \times 3$ 的像素空间做扩散。U-Net 的每一步去噪都要处理这么大的张量，典型训练配置：

- 单张图像的特征图：数百 MB 显存
- Batch size 受限，训练速度慢
- Self-Attention 在 $256 \times 256$ 分辨率上的计算量：$O((256 \times 256)^2) = O(4.3 \times 10^9)$——不现实

#### 2.3.2 解决方案：两阶段流程

Rombach et al.（2022）提出 Latent Diffusion Model（LDM），把扩散过程转移到潜空间：

**第一阶段：训练自编码器（离线，一次性）**

$$\mathbf{z} = \mathcal{E}(\mathbf{x}), \quad \hat{\mathbf{x}} = \mathcal{D}(\mathbf{z})$$

- $\mathcal{E}$：编码器，将图像 $\mathbf{x} \in \mathbb{R}^{H \times W \times 3}$ 压缩为潜变量 $\mathbf{z} \in \mathbb{R}^{h \times w \times c}$
- $\mathcal{D}$：解码器，将潜变量还原为图像
- 下采样倍率 $f = H / h$，Stable Diffusion 使用 $f = 8$（即 $512 \times 512 \to 64 \times 64$）
- 这个自编码器通常是 KL 正则化的 VAE（带 KL 散度损失 + 感知损失 + 对抗损失）

**第二阶段：在潜空间训练扩散模型**

$$\mathcal{L}_{\text{LDM}} = \mathbb{E}_{\mathbf{z}_0, \boldsymbol{\epsilon}, t} \left[ \left\| \boldsymbol{\epsilon} - \boldsymbol{\epsilon}_\theta(\mathbf{z}_t, t, \mathbf{c}) \right\|_2^2 \right]$$

和原始 DDPM 损失完全一样，只不过 $\mathbf{z}_t$ 是潜变量而非像素。

#### 2.3.3 压缩比与信息保留

| 空间 | 尺寸 | 数据量 | 占比 |
|------|------|--------|------|
| 像素空间 | $512 \times 512 \times 3$ | 786,432 | 100% |
| 潜空间 | $64 \times 64 \times 4$ | 16,384 | 2.1% |

只用原始数据量 2% 的表示就能基本无损重建图像。这得益于自然图像的高冗余性——相邻像素高度相关，大量信息可以被压缩。

关键洞察：**感知上重要的信息**（形状、纹理、颜色分布）可以用很少的维度表示，被压缩掉的主要是**高频细节**（像素级噪声、微小纹理变化）。扩散模型不需要建模这些高频细节，VAE 解码器在最后一步补回来就行。

### 2.4 Stable Diffusion 完整管线

#### 2.4.1 三大组件详解

**组件一：CLIP 文本编码器**

CLIP（Contrastive Language-Image Pre-training, Radford et al., 2021）是在 4 亿个图文对上训练的多模态模型。Stable Diffusion 只用其文本编码器部分：

- 输入：文本 prompt（如 "a photo of a cat on the moon"）
- Tokenize：将文本分词为 token 序列，最长 77 个 token
- 输出：token 级别的特征序列 $\mathbf{c} \in \mathbb{R}^{77 \times 768}$（SD 1.x 用 CLIP ViT-L/14）

这个特征序列通过 Cross-Attention 注入 U-Net 的每一层。

**组件二：U-Net 去噪网络**

Stable Diffusion 1.5 的 U-Net 结构细节：

```
编码器:
  64×64, 320ch  →  32×32, 640ch  →  16×16, 1280ch  →  8×8, 1280ch

瓶颈:
  8×8, 1280ch (Self-Attention + Cross-Attention)

解码器:
  8×8, 1280ch  →  16×16, 1280ch  →  32×32, 640ch  →  64×64, 320ch
```

每个阶段包含 ResBlock + Attention Block。Attention Block 内部结构：

```
输入特征 h (B, C, H, W)
  │
  ├── reshape 为 (B, H*W, C) —— 空间维度展平为序列
  │
  ├── Self-Attention：图像特征自己互相关注
  │     Q = h @ W_q,  K = h @ W_k,  V = h @ W_v
  │
  ├── Cross-Attention：图像特征关注文本条件
  │     Q = h @ W_q,  K = c @ W_k,  V = c @ W_v
  │                    ^文本条件
  │
  ├── FFN (Linear → GEGLU → Linear)
  │
  └── reshape 回 (B, C, H, W)
```

**组件三：VAE 解码器**

- 输入：干净潜变量 $\mathbf{z}_0 \in \mathbb{R}^{4 \times 64 \times 64}$
- 通过卷积上采样网络，逐步恢复分辨率：$64 \to 128 \to 256 \to 512$
- 输出：RGB 图像 $\hat{\mathbf{x}} \in \mathbb{R}^{3 \times 512 \times 512}$

#### 2.4.2 推理流程（文生图）

完整的推理流程分为四步：

**Step 1：文本编码**

$$\mathbf{c} = \text{CLIP}_{\text{text}}(\text{prompt}) \in \mathbb{R}^{77 \times 768}$$

同时编码空文本（用于 Classifier-Free Guidance）：

$$\mathbf{c}_\varnothing = \text{CLIP}_{\text{text}}(\text{""}) \in \mathbb{R}^{77 \times 768}$$

**Step 2：初始化噪声**

$$\mathbf{z}_T \sim \mathcal{N}(\mathbf{0}, \mathbf{I}), \quad \mathbf{z}_T \in \mathbb{R}^{4 \times 64 \times 64}$$

**Step 3：迭代去噪（T 步）**

对于每个时间步 $t = T, T-1, \dots, 1$：

1. 分别预测有条件和无条件噪声：
   $$\boldsymbol{\epsilon}_{\text{cond}} = \boldsymbol{\epsilon}_\theta(\mathbf{z}_t, t, \mathbf{c}), \quad \boldsymbol{\epsilon}_{\text{uncond}} = \boldsymbol{\epsilon}_\theta(\mathbf{z}_t, t, \mathbf{c}_\varnothing)$$

2. 用 CFG 引导公式混合：
   $$\hat{\boldsymbol{\epsilon}} = \boldsymbol{\epsilon}_{\text{uncond}} + w \cdot (\boldsymbol{\epsilon}_{\text{cond}} - \boldsymbol{\epsilon}_{\text{uncond}})$$
   其中 $w$ 是引导强度（guidance scale），SD 默认 $w = 7.5$。

3. 用调度器（如 DDIM）从 $\mathbf{z}_t$ 计算 $\mathbf{z}_{t-1}$。

注意：CFG 需要对每个时间步做**两次**前向传播（有条件和无条件），所以实际计算量翻倍。实践中把两次的输入拼成一个 batch 一起算。

**Step 4：VAE 解码**

$$\hat{\mathbf{x}} = \mathcal{D}(\mathbf{z}_0)$$

#### 2.4.3 各组件参数量

以 Stable Diffusion 1.5 为例：

| 组件 | 参数量 | 训练状态 | 作用 |
|------|--------|----------|------|
| CLIP 文本编码器 | ~123M | 冻结（直接用预训练权重） | 将文本转化为语义向量 |
| U-Net | ~860M | 训练 | 在潜空间去噪（核心组件） |
| VAE | ~84M | 冻结或微调 | 像素空间 ↔ 潜空间转换 |
| **总计** | **~1.07B** | | |

U-Net 占了大部分参数，它是真正在训练中学习"如何去噪"的组件。CLIP 和 VAE 通常使用预训练权重，不参与扩散模型的训练。

### 2.5 与 Transformer 路线的融合

#### 2.5.1 趋势：两条技术路线正在合流

AI 领域存在两条主要的技术路线：

```
路线一：自回归 Transformer（语言模型路线）
  GPT → GPT-4 → ...
  特点：逐 token 预测下一个 token，用于文本生成

路线二：扩散模型（生成模型路线）
  DDPM → Stable Diffusion → ...
  特点：迭代去噪，用于图像/视频生成
```

DiT 是两条路线的交汇点——它用 Transformer 架构来做扩散生成。

#### 2.5.2 Sora：DiT 用于视频生成

OpenAI 的 Sora（2024）将 DiT 从图像扩展到视频。核心思路：

- 把视频看作"三维数据"：$(T_{\text{frames}}, H, W)$
- 用 3D 的 Patchify：在时间和空间维度都做分块，得到一组 3D patch token
- 用 Transformer 在这些 token 上做扩散去噪
- 文本条件通过 Cross-Attention 注入

视频生成的关键挑战是**时空一致性**——不仅每一帧要好看，帧与帧之间还要连贯。Transformer 的全局注意力天然适合建模长距离时空依赖，这是 U-Net 难以做到的。

#### 2.5.3 融合趋势的更多证据

| 模型 | 基础架构 | 生成方式 | 年份 |
|------|----------|----------|------|
| DALL-E 1 | Transformer | 自回归 | 2021 |
| Stable Diffusion 1.x/2.x | U-Net | 扩散 | 2022 |
| DALL-E 2 | U-Net + CLIP | 扩散 | 2022 |
| DALL-E 3 | U-Net | 扩散 | 2023 |
| DiT | Transformer | 扩散 | 2023 |
| Stable Diffusion 3.0 | MM-DiT (Transformer) | 扩散 + Flow Matching | 2024 |
| Sora | DiT (Transformer) | 扩散 | 2024 |

趋势非常清晰：**骨干网络从 U-Net 转向 Transformer**，而生成范式保持扩散（或其变体 Flow Matching）。Transformer 正在成为统一的基础架构——无论是理解（语言模型）还是生成（图像/视频），底层都是 Transformer。

## 3. Python 代码示例

### 3.1 简化版扩散 U-Net（含时间步嵌入和 Cross-Attention）

以下代码展示一个**结构清晰、可运行**的简化 U-Net。为了可读性，省略了 GroupNorm 的 num_groups 等细节参数，重点展示时间步嵌入和 Cross-Attention 的注入方式。

```python
import torch
import torch.nn as nn
import math

# ============================================================
# 简化版扩散 U-Net
# 结构：编码器(下采样) → 瓶颈 → 解码器(上采样)
# 核心改造：时间步嵌入 + Cross-Attention 条件注入
# ============================================================


class SinusoidalPosEmb(nn.Module):
    """
    正弦时间步嵌入：把标量 t 编码为 d 维向量。
    和 Transformer 的位置编码原理一样——用不同频率的正弦/余弦
    让模型能区分不同的时间步。
    """
    def __init__(self, dim):
        super().__init__()
        self.dim = dim

    def forward(self, t):
        """
        Args:
            t: (B,) 时间步张量，值为 0 ~ T 的整数
        Returns:
            (B, dim) 时间步嵌入
        """
        device = t.device
        half_dim = self.dim // 2
        # 频率从低到高：10000^(0/d) 到 10000^(1)
        emb = math.log(10000) / (half_dim - 1)
        emb = torch.exp(torch.arange(half_dim, device=device) * -emb)
        # 外积：每个时间步 × 每个频率
        emb = t[:, None].float() * emb[None, :]  # (B, half_dim)
        # 拼接 sin 和 cos
        emb = torch.cat([emb.sin(), emb.cos()], dim=-1)  # (B, dim)
        return emb


class TimeEmbMLP(nn.Module):
    """
    时间步嵌入的 MLP：正弦编码 → 线性 → SiLU → 线性
    把原始的正弦编码投影到更有表达力的空间。
    """
    def __init__(self, dim):
        super().__init__()
        self.sinusoidal = SinusoidalPosEmb(dim)
        self.mlp = nn.Sequential(
            nn.Linear(dim, dim * 4),
            nn.SiLU(),                  # SiLU = x * sigmoid(x)，比 ReLU 更平滑
            nn.Linear(dim * 4, dim),
        )

    def forward(self, t):
        """t: (B,) → (B, dim)"""
        return self.mlp(self.sinusoidal(t))


class CrossAttention(nn.Module):
    """
    Cross-Attention：图像特征 (Q) 关注文本条件 (K, V)。
    这就是文本信息注入扩散 U-Net 的方式。

    和 Self-Attention 唯一的区别：K 和 V 来自另一个序列（文本），
    而不是和 Q 同一个序列（图像）。
    """
    def __init__(self, dim, context_dim, n_heads=4):
        super().__init__()
        self.n_heads = n_heads
        self.head_dim = dim // n_heads
        self.scale = self.head_dim ** -0.5

        self.to_q = nn.Linear(dim, dim, bias=False)
        self.to_k = nn.Linear(context_dim, dim, bias=False)
        self.to_v = nn.Linear(context_dim, dim, bias=False)
        self.out_proj = nn.Linear(dim, dim)
        self.norm = nn.LayerNorm(dim)

    def forward(self, x, context):
        """
        Args:
            x:       (B, L, dim)         图像特征序列 (空间位置展平)
            context: (B, S, context_dim)  文本特征序列 (CLIP 输出)
        Returns:
            (B, L, dim) 融合了文本信息的图像特征
        """
        x = self.norm(x)
        B, L, _ = x.shape

        # Q 来自图像，K/V 来自文本
        q = self.to_q(x)        # (B, L, dim)
        k = self.to_k(context)  # (B, S, dim)
        v = self.to_v(context)  # (B, S, dim)

        # 拆成多头: (B, n_heads, seq_len, head_dim)
        def reshape_heads(t, seq_len):
            return t.view(B, seq_len, self.n_heads, self.head_dim).transpose(1, 2)

        q = reshape_heads(q, L)       # (B, n_heads, L, head_dim)
        k = reshape_heads(k, k.size(1))
        v = reshape_heads(v, v.size(1))

        # 缩放点积注意力
        attn = (q @ k.transpose(-2, -1)) * self.scale  # (B, n_heads, L, S)
        attn = attn.softmax(dim=-1)
        out = attn @ v                                  # (B, n_heads, L, head_dim)

        # 合并多头
        out = out.transpose(1, 2).contiguous().view(B, L, -1)
        return self.out_proj(out)


class ResBlock(nn.Module):
    """
    残差块 + 时间步嵌入注入。
    这是扩散 U-Net 的基本构建单元。

    流程：GroupNorm → SiLU → Conv → 加入时间步嵌入 → GroupNorm → SiLU → Conv → 残差连接
    """
    def __init__(self, in_ch, out_ch, time_dim):
        super().__init__()
        self.conv1 = nn.Sequential(
            nn.GroupNorm(8, in_ch),
            nn.SiLU(),
            nn.Conv2d(in_ch, out_ch, 3, padding=1),
        )
        # 时间步嵌入投影到 out_ch 维，然后加到特征上
        self.time_proj = nn.Sequential(
            nn.SiLU(),
            nn.Linear(time_dim, out_ch),
        )
        self.conv2 = nn.Sequential(
            nn.GroupNorm(8, out_ch),
            nn.SiLU(),
            nn.Conv2d(out_ch, out_ch, 3, padding=1),
        )
        # 如果输入输出通道数不同，需要一个 1x1 卷积做残差连接
        self.shortcut = (
            nn.Conv2d(in_ch, out_ch, 1) if in_ch != out_ch else nn.Identity()
        )

    def forward(self, x, t_emb):
        """
        Args:
            x:     (B, in_ch, H, W)  输入特征图
            t_emb: (B, time_dim)     时间步嵌入
        Returns:
            (B, out_ch, H, W)
        """
        h = self.conv1(x)                             # (B, out_ch, H, W)
        # 时间步嵌入：投影后扩展空间维度，加到特征上
        t = self.time_proj(t_emb)[:, :, None, None]   # (B, out_ch, 1, 1)
        h = h + t                                      # 广播相加
        h = self.conv2(h)                              # (B, out_ch, H, W)
        return h + self.shortcut(x)                    # 残差连接


class SimpleDiffusionUNet(nn.Module):
    """
    简化版扩散 U-Net，展示核心结构：

    编码器（下采样）: 64ch → 128ch → 256ch
    瓶颈:             256ch (带 Cross-Attention)
    解码器（上采样）: 256ch → 128ch → 64ch

    每一步都注入时间步嵌入。
    瓶颈和解码器加入 Cross-Attention 接收文本条件。

    注意：这是教学用的简化版本。实际的 Stable Diffusion U-Net
    有更多层、更多通道、更多 Attention 层，但核心结构与此一致。
    """
    def __init__(self, in_ch=4, base_ch=64, time_dim=256, context_dim=768):
        """
        Args:
            in_ch:       输入通道数（潜空间通常是 4）
            base_ch:     基础通道数
            time_dim:    时间步嵌入维度
            context_dim: 文本条件维度（CLIP 输出通常是 768）
        """
        super().__init__()

        # --- 时间步嵌入 ---
        self.time_emb = TimeEmbMLP(time_dim)

        # --- 编码器（下采样路径）---
        self.enc1 = ResBlock(in_ch, base_ch, time_dim)        # 64×64 → 64×64, 64ch
        self.down1 = nn.Conv2d(base_ch, base_ch, 3, stride=2, padding=1)  # → 32×32

        self.enc2 = ResBlock(base_ch, base_ch * 2, time_dim)  # 32×32, 128ch
        self.down2 = nn.Conv2d(base_ch * 2, base_ch * 2, 3, stride=2, padding=1)  # → 16×16

        self.enc3 = ResBlock(base_ch * 2, base_ch * 4, time_dim)  # 16×16, 256ch
        self.down3 = nn.Conv2d(base_ch * 4, base_ch * 4, 3, stride=2, padding=1)  # → 8×8

        # --- 瓶颈 ---
        self.bottleneck = ResBlock(base_ch * 4, base_ch * 4, time_dim)  # 8×8, 256ch
        # 瓶颈处加 Cross-Attention
        self.bottleneck_attn = CrossAttention(base_ch * 4, context_dim)

        # --- 解码器（上采样路径）---
        self.up3 = nn.ConvTranspose2d(base_ch * 4, base_ch * 4, 2, stride=2)  # 8→16
        # 跳跃连接拼接后通道翻倍: 256 + 256 = 512 → 256
        self.dec3 = ResBlock(base_ch * 8, base_ch * 4, time_dim)
        self.dec3_attn = CrossAttention(base_ch * 4, context_dim)

        self.up2 = nn.ConvTranspose2d(base_ch * 4, base_ch * 2, 2, stride=2)  # 16→32
        # 跳跃连接: 128 + 128 = 256 → 128
        self.dec2 = ResBlock(base_ch * 4, base_ch * 2, time_dim)

        self.up1 = nn.ConvTranspose2d(base_ch * 2, base_ch, 2, stride=2)  # 32→64
        # 跳跃连接: 64 + 64 = 128 → 64
        self.dec1 = ResBlock(base_ch * 2, base_ch, time_dim)

        # --- 输出层 ---
        self.out = nn.Sequential(
            nn.GroupNorm(8, base_ch),
            nn.SiLU(),
            nn.Conv2d(base_ch, in_ch, 1),  # 投影回输入通道数
        )

    def forward(self, x, t, context):
        """
        Args:
            x:       (B, 4, 64, 64)    含噪潜变量
            t:       (B,)              时间步
            context: (B, 77, 768)      文本条件 (CLIP 输出)
        Returns:
            (B, 4, 64, 64)            预测的噪声
        """
        # 时间步嵌入
        t_emb = self.time_emb(t)  # (B, time_dim)

        # --- 编码器 ---
        e1 = self.enc1(x, t_emb)          # (B, 64, 64, 64)
        e2 = self.enc2(self.down1(e1), t_emb)   # (B, 128, 32, 32)
        e3 = self.enc3(self.down2(e2), t_emb)   # (B, 256, 16, 16)

        # --- 瓶颈 ---
        b = self.bottleneck(self.down3(e3), t_emb)  # (B, 256, 8, 8)
        # Cross-Attention: 把空间维度展平为序列
        B, C, H, W = b.shape
        b_flat = b.view(B, C, H * W).transpose(1, 2)  # (B, 64, 256)
        b_flat = b_flat + self.bottleneck_attn(b_flat, context)  # 残差
        b = b_flat.transpose(1, 2).view(B, C, H, W)   # 恢复形状

        # --- 解码器 + 跳跃连接 ---
        d3 = self.up3(b)                          # (B, 256, 16, 16)
        d3 = torch.cat([d3, e3], dim=1)           # (B, 512, 16, 16) 跳跃连接
        d3 = self.dec3(d3, t_emb)                 # (B, 256, 16, 16)
        # Cross-Attention
        B, C, H, W = d3.shape
        d3_flat = d3.view(B, C, H * W).transpose(1, 2)
        d3_flat = d3_flat + self.dec3_attn(d3_flat, context)
        d3 = d3_flat.transpose(1, 2).view(B, C, H, W)

        d2 = self.up2(d3)                         # (B, 128, 32, 32)
        d2 = torch.cat([d2, e2], dim=1)           # (B, 256, 32, 32)
        d2 = self.dec2(d2, t_emb)                 # (B, 128, 32, 32)

        d1 = self.up1(d2)                         # (B, 64, 64, 64)
        d1 = torch.cat([d1, e1], dim=1)           # (B, 128, 64, 64)
        d1 = self.dec1(d1, t_emb)                 # (B, 64, 64, 64)

        return self.out(d1)                        # (B, 4, 64, 64)


# --- 验证形状 ---
if __name__ == "__main__":

    # 模型参数量: ~12,000,000 (简化版，实际 SD 的 U-Net 约 860M)
```

**关键观察**：

1. **输入输出形状相同**：$(\text{B}, 4, 64, 64) \to (\text{B}, 4, 64, 64)$。去噪网络的任务是预测噪声，噪声和输入形状完全相同。
2. **时间步嵌入注入到每一层**：每个 ResBlock 都接收 $t_{\text{emb}}$，让网络知道当前在哪个去噪阶段。
3. **Cross-Attention 只在低分辨率层**：瓶颈（$8 \times 8$）和解码器（$16 \times 16$）有 Cross-Attention，高分辨率层没有——这是计算量和效果的折中。

### 3.2 简化版 DiT Block

```python
import torch
import torch.nn as nn

# ============================================================
# 简化版 DiT Block
# 展示 AdaLN-Zero 条件注入机制
# ============================================================


class AdaLNZero(nn.Module):
    """
    AdaLN-Zero: 用条件信息调制 Layer Norm 的参数。

    标准 LayerNorm: γ * (x - μ) / σ + β    (γ, β 是可学习参数)
    AdaLN:         γ(c) * (x - μ) / σ + β(c) (γ, β 是条件的函数)

    "Zero" 意味着残差连接前的缩放 α 初始化为 0，
    使训练初期网络行为接近恒等映射，提升训练稳定性。
    """
    def __init__(self, dim, cond_dim):
        super().__init__()
        self.norm = nn.LayerNorm(dim, elementwise_affine=False)
        # 一个 MLP 输出 6 个调制参数:
        # (γ1, β1, α1) 用于 Attention, (γ2, β2, α2) 用于 FFN
        self.proj = nn.Sequential(
            nn.SiLU(),
            nn.Linear(cond_dim, dim * 6),
        )
        # 初始化为 0，实现 "Zero" 的效果
        nn.init.zeros_(self.proj[-1].weight)
        nn.init.zeros_(self.proj[-1].bias)

    def forward(self, c):
        """
        Args:
            c: (B, cond_dim) 条件嵌入（时间步 + 类别标签等）
        Returns:
            六个调制参数，每个形状 (B, 1, dim)
        """
        params = self.proj(c)             # (B, dim*6)
        params = params.unsqueeze(1)      # (B, 1, dim*6)  方便广播到 token 序列
        return params.chunk(6, dim=-1)    # 6 个 (B, 1, dim)


class DiTBlock(nn.Module):
    """
    DiT Block = AdaLN-Zero 调制的 Transformer Block。

    和标准 Transformer Block 的区别：
    1. 没有 Cross-Attention（条件通过 AdaLN-Zero 注入）
    2. Layer Norm 的 γ/β 不是固定参数，而是条件的函数
    3. 残差连接前有可学习的缩放因子 α
    """
    def __init__(self, dim, n_heads, cond_dim, mlp_ratio=4):
        super().__init__()
        self.adaln = AdaLNZero(dim, cond_dim)
        self.norm = nn.LayerNorm(dim, elementwise_affine=False)

        # Multi-Head Self-Attention
        self.attn = nn.MultiheadAttention(dim, n_heads, batch_first=True)

        # FFN (和标准 Transformer 一样)
        self.ffn = nn.Sequential(
            nn.Linear(dim, dim * mlp_ratio),
            nn.GELU(),
            nn.Linear(dim * mlp_ratio, dim),
        )

    def forward(self, x, c):
        """
        Args:
            x: (B, N, dim)     token 序列 (N = patch 数量)
            c: (B, cond_dim)   条件嵌入
        Returns:
            (B, N, dim)
        """
        # 获取 6 个调制参数
        gamma1, beta1, alpha1, gamma2, beta2, alpha2 = self.adaln(c)

        # --- Attention 分支 ---
        # AdaLN: 用条件调制归一化后的特征
        h = self.norm(x) * (1 + gamma1) + beta1          # 调制后的特征
        attn_out, _ = self.attn(h, h, h)                  # Self-Attention
        x = x + alpha1 * attn_out                         # 残差连接 (α1 控制缩放)

        # --- FFN 分支 ---
        h = self.norm(x) * (1 + gamma2) + beta2
        ffn_out = self.ffn(h)
        x = x + alpha2 * ffn_out

        return x


class SimpleDiT(nn.Module):
    """
    简化版 DiT：Patchify → DiT Blocks → Unpatchify

    对比 U-Net：
    - 没有编码器-解码器结构，没有跳跃连接
    - 所有 token 在同一分辨率下处理
    - 条件通过 AdaLN-Zero 注入，不需要 Cross-Attention
    """
    def __init__(
        self,
        in_ch=4,
        patch_size=2,
        dim=512,
        depth=6,
        n_heads=8,
        num_classes=1000,
    ):
        super().__init__()
        self.patch_size = patch_size
        self.dim = dim
        patch_dim = in_ch * patch_size * patch_size  # 每个 patch 展平后的维度

        # Patchify: 用卷积实现 patch 嵌入
        self.patch_embed = nn.Conv2d(
            in_ch, dim, kernel_size=patch_size, stride=patch_size
        )

        # 条件嵌入：时间步 + 类别标签
        self.time_emb = TimeEmbMLP(dim)
        self.class_emb = nn.Embedding(num_classes, dim)

        # 可学习的位置嵌入
        # 假设输入 32×32，patch_size=2 → 16×16=256 个 token
        self.pos_emb = nn.Parameter(torch.zeros(1, 256, dim))

        # DiT Blocks
        self.blocks = nn.ModuleList([
            DiTBlock(dim, n_heads, cond_dim=dim) for _ in range(depth)
        ])

        # 最终归一化和投影
        self.final_norm = nn.LayerNorm(dim, elementwise_affine=False)
        self.final_adaln = nn.Sequential(
            nn.SiLU(),
            nn.Linear(dim, dim * 2),
        )
        nn.init.zeros_(self.final_adaln[-1].weight)
        nn.init.zeros_(self.final_adaln[-1].bias)

        # 投影回 patch 空间
        self.out_proj = nn.Linear(dim, patch_dim)

    def forward(self, x, t, y):
        """
        Args:
            x: (B, 4, 32, 32)   含噪潜变量
            t: (B,)             时间步
            y: (B,)             类别标签
        Returns:
            (B, 4, 32, 32)     预测的噪声
        """
        B, C, H, W = x.shape
        p = self.patch_size

        # 1. Patchify：把图像切成 patch 并嵌入
        tokens = self.patch_embed(x)            # (B, dim, H/p, W/p)
        h_p, w_p = tokens.shape[2], tokens.shape[3]
        tokens = tokens.flatten(2).transpose(1, 2)  # (B, N, dim), N = (H/p)*(W/p)

        # 2. 加上位置嵌入
        tokens = tokens + self.pos_emb[:, :tokens.size(1)]

        # 3. 条件嵌入：时间步 + 类别标签相加
        c = self.time_emb(t) + self.class_emb(y)  # (B, dim)

        # 4. 通过 DiT Blocks
        for block in self.blocks:
            tokens = block(tokens, c)

        # 5. 最终层：AdaLN → 线性投影
        gamma, beta = self.final_adaln(c).unsqueeze(1).chunk(2, dim=-1)
        tokens = self.final_norm(tokens) * (1 + gamma) + beta
        tokens = self.out_proj(tokens)          # (B, N, patch_dim)

        # 6. Unpatchify：把 token 序列重组为图像
        tokens = tokens.transpose(1, 2)         # (B, patch_dim, N)
        out = tokens.view(B, C, p, p, h_p, w_p)
        out = out.permute(0, 1, 4, 2, 5, 3).contiguous()
        out = out.view(B, C, H, W)              # (B, 4, 32, 32)

        return out


# --- 验证形状 ---
if __name__ == "__main__":
    model = SimpleDiT(in_ch=4, patch_size=2, dim=512, depth=6, n_heads=8)

    B = 2
    x = torch.randn(B, 4, 32, 32)
    t = torch.randint(0, 1000, (B,))
    y = torch.randint(0, 1000, (B,))

    out = model(x, t, y)
    print(f"输入形状: {x.shape}")
    print(f"输出形状: {out.shape}")
    print(f"参数量:   {sum(p.numel() for p in model.parameters()):,}")

    # 预期输出:
    # 输入形状: torch.Size([2, 4, 32, 32])
    # 输出形状: torch.Size([2, 4, 32, 32])
    # 参数量:   ~19,000,000
```

### 3.3 Stable Diffusion 推理管线伪代码

以下代码展示用 Hugging Face `diffusers` 库进行文生图推理的**实际可运行代码**，以及等价的手动实现伪代码：

```python
# ============================================================
# 方式一：使用 diffusers 库（实际工程中的做法）
# pip install diffusers transformers accelerate torch
# ============================================================

from diffusers import StableDiffusionPipeline
import torch

# 加载预训练模型（首次运行会下载约 4GB 权重）
pipe = StableDiffusionPipeline.from_pretrained(
    "runwayml/stable-diffusion-v1-5",
    torch_dtype=torch.float16,       # 半精度，省显存
)
pipe = pipe.to("cuda")

# 生成图像（一行代码！）
image = pipe(
    prompt="a photo of a cat sitting on the moon, highly detailed",
    num_inference_steps=50,          # 去噪步数
    guidance_scale=7.5,              # CFG 引导强度
).images[0]

image.save("cat_on_moon.png")


# ============================================================
# 方式二：手动拆解管线（理解每一步在做什么）
# 以下是伪代码，展示 pipe(...) 内部的完整流程
# ============================================================

import torch
from transformers import CLIPTextModel, CLIPTokenizer
from diffusers import AutoencoderKL, UNet2DConditionModel, DDIMScheduler

# --- 加载各组件 ---
tokenizer = CLIPTokenizer.from_pretrained("openai/clip-vit-large-patch14")
text_encoder = CLIPTextModel.from_pretrained("openai/clip-vit-large-patch14")
vae = AutoencoderKL.from_pretrained("runwayml/stable-diffusion-v1-5", subfolder="vae")
unet = UNet2DConditionModel.from_pretrained("runwayml/stable-diffusion-v1-5", subfolder="unet")
scheduler = DDIMScheduler.from_pretrained("runwayml/stable-diffusion-v1-5", subfolder="scheduler")

device = "cuda"
text_encoder = text_encoder.to(device)
vae = vae.to(device)
unet = unet.to(device)

# --- Step 1: 文本编码 ---
prompt = "a photo of a cat sitting on the moon, highly detailed"
guidance_scale = 7.5
num_steps = 50

# Tokenize
tokens = tokenizer(
    prompt,
    padding="max_length",
    max_length=77,
    return_tensors="pt",
).input_ids.to(device)                               # (1, 77)

# CLIP 编码
with torch.no_grad():
    text_embeddings = text_encoder(tokens).last_hidden_state  # (1, 77, 768)

# 空文本编码（用于 CFG）
uncond_tokens = tokenizer(
    "",
    padding="max_length",
    max_length=77,
    return_tensors="pt",
).input_ids.to(device)

with torch.no_grad():
    uncond_embeddings = text_encoder(uncond_tokens).last_hidden_state  # (1, 77, 768)

# 拼成一个 batch，一次前向传播同时算有条件和无条件
text_embeddings = torch.cat([uncond_embeddings, text_embeddings])  # (2, 77, 768)

# --- Step 2: 初始化噪声 ---
latents = torch.randn(1, 4, 64, 64, device=device)  # 随机噪声

# 设置调度器
scheduler.set_timesteps(num_steps)

# --- Step 3: 迭代去噪 ---
for t in scheduler.timesteps:
    # 把潜变量复制一份（有条件 + 无条件）
    latent_input = torch.cat([latents] * 2)           # (2, 4, 64, 64)

    with torch.no_grad():
        # U-Net 预测噪声
        noise_pred = unet(
            latent_input,
            t,
            encoder_hidden_states=text_embeddings,     # 文本条件通过 Cross-Attention 注入
        ).sample                                       # (2, 4, 64, 64)

    # CFG: 分离有条件和无条件预测，然后混合
    noise_uncond, noise_cond = noise_pred.chunk(2)
    noise_pred = noise_uncond + guidance_scale * (noise_cond - noise_uncond)

    # 调度器：根据预测噪声更新潜变量
    latents = scheduler.step(noise_pred, t, latents).prev_sample

# --- Step 4: VAE 解码 ---
with torch.no_grad():
    # VAE 解码器把潜变量还原为图像
    latents = latents / vae.config.scaling_factor      # 缩放因子
    image = vae.decode(latents).sample                 # (1, 3, 512, 512)

# 后处理：[-1, 1] → [0, 255]
image = (image / 2 + 0.5).clamp(0, 1)
image = image.permute(0, 2, 3, 1).cpu().numpy()
image = (image[0] * 255).astype("uint8")

from PIL import Image
Image.fromarray(image).save("cat_on_moon_manual.png")
```

**关键观察**：

1. `pipe(...)` 一行代码内部做的事情，就是 Step 1~4 的完整流程。
2. CFG 的实现方式是把有条件和无条件输入拼成一个 batch 一起算，而不是分两次调用。
3. `scheduler.step()` 封装了 DDIM/DDPM 的更新公式。
4. VAE 解码是最后一步，把 $4 \times 64 \times 64$ 的潜变量变成 $3 \times 512 \times 512$ 的图像。

## 4. 工程师视角

### 4.1 Stable Diffusion 版本演进

| 版本 | 发布时间 | 骨干网络 | 文本编码器 | 分辨率 | 关键改进 |
|------|----------|----------|-----------|--------|----------|
| SD 1.5 | 2022.10 | U-Net | CLIP ViT-L/14 (768d) | 512 | 开源标杆 |
| SD 2.0 | 2022.11 | U-Net | OpenCLIP ViT-H/14 (1024d) | 512/768 | 更大文本编码器 |
| SDXL | 2023.07 | U-Net (更大) | CLIP ViT-L + OpenCLIP ViT-bigG (双编码器) | 1024 | 双文本编码器、Refiner 二阶段 |
| SD 3.0 | 2024.02 | MM-DiT (Transformer) | 三个文本编码器 (CLIP×2 + T5-XXL) | 1024 | 改用 Transformer 骨干、Flow Matching |

**版本选择建议**：

- **入门学习**：SD 1.5，社区资源最丰富，LoRA/ControlNet 生态最完善
- **生产质量**：SDXL 或 SD 3.0，图像质量更高
- **理解前沿**：SD 3.0 代表了当前主流的 DiT + Flow Matching 技术路线

### 4.2 ControlNet：精细控制图像生成

ControlNet（Zhang et al., 2023）解决的问题：文本 prompt 只能给出粗略的语义描述（"一只猫"），无法精细控制姿态、构图、边缘等空间结构。

**核心思路**：复制 U-Net 编码器的权重作为"控制分支"，输入额外的控制信号（边缘图、深度图、姿态骨架等），通过 zero convolution（初始化为零的 $1 \times 1$ 卷积）把控制分支的输出注入到原始 U-Net 中。

```
           控制信号（如 Canny 边缘图）
                    │
                    ▼
        ┌─────────────────────┐
        │  U-Net 编码器副本    │  ← 从预训练 U-Net 拷贝权重
        │  （可训练）          │
        └─────────┬───────────┘
                  │ zero convolution
                  ▼
        ┌─────────────────────┐
        │  原始 U-Net          │  ← 权重冻结
        │  （文本 prompt 驱动） │
        └─────────────────────┘
                  │
                  ▼
              生成的图像
```

**工程优势**：

- 原始 U-Net 权重冻结，不破坏已有的文本理解能力
- Zero convolution 确保训练初期控制分支的输出为零，网络退化为原始 SD，训练非常稳定
- 一个 ControlNet 只需训练一种控制类型（边缘/深度/姿态），可以自由组合

### 4.3 LoRA：轻量级微调

LoRA（Low-Rank Adaptation, Hu et al., 2022）最初为 LLM 微调设计，后来广泛用于 Stable Diffusion 的风格微调。

**核心思路**：不修改原始权重 $W_0$，而是添加一个低秩分解的增量 $\Delta W = BA$：

$$W = W_0 + \alpha \cdot BA$$

其中 $B \in \mathbb{R}^{d \times r}$，$A \in \mathbb{R}^{r \times k}$，秩 $r \ll \min(d, k)$。

**参数量对比**：

| 方法 | U-Net 860M 参数中需要训练的 |
|------|---------------------------|
| 全量微调 | 860M（全部参数） |
| LoRA (rank=4) | ~1-2M（不到原参数的 0.2%） |
| LoRA (rank=64) | ~10-20M（约 2%） |

LoRA 文件通常只有几十 MB，方便分享和切换。社区平台（如 Civitai）上有大量用户训练的 LoRA，涵盖各种绘画风格、IP 角色和场景类型。

### 4.4 开源生态：Diffusers 库

Hugging Face 的 `diffusers` 库是扩散模型的 PyTorch 标准库，类似于 NLP 的 `transformers`。

**核心抽象**：

```python
from diffusers import (
    StableDiffusionPipeline,          # 高级接口：一行代码文生图
    UNet2DConditionModel,             # 去噪骨干网络
    AutoencoderKL,                    # VAE
    DDIMScheduler,                    # 噪声调度器
    DDPMScheduler,
    EulerDiscreteScheduler,
)
```

| 抽象层 | 类 | 作用 |
|--------|-----|------|
| **Pipeline** | `StableDiffusionPipeline` | 封装完整管线，一行代码生成图像 |
| **Model** | `UNet2DConditionModel` | 去噪网络，可替换为 DiT |
| **Scheduler** | `DDIMScheduler` 等 | 噪声调度策略，决定如何从 $\mathbf{z}_t$ 到 $\mathbf{z}_{t-1}$ |
| **VAE** | `AutoencoderKL` | 像素空间 ↔ 潜空间转换 |

**常用操作示例**：

```python
# 切换调度器（不需要重新训练模型！）
from diffusers import EulerDiscreteScheduler

pipe.scheduler = EulerDiscreteScheduler.from_config(pipe.scheduler.config)

# 加载 LoRA 权重
pipe.load_lora_weights("path/to/lora.safetensors")

# 加载 ControlNet
from diffusers import ControlNetModel, StableDiffusionControlNetPipeline

controlnet = ControlNetModel.from_pretrained("lllyasviel/sd-controlnet-canny")
pipe = StableDiffusionControlNetPipeline.from_pretrained(
    "runwayml/stable-diffusion-v1-5",
    controlnet=controlnet,
)
```

### 4.5 性能优化技巧

| 技巧 | 效果 | 实现 |
|------|------|------|
| **半精度推理** (float16) | 显存减半，速度提升 | `torch_dtype=torch.float16` |
| **xFormers** | Attention 显存优化 | `pipe.enable_xformers_memory_efficient_attention()` |
| **VAE 切片** | 大图生成时避免 OOM | `pipe.enable_vae_slicing()` |
| **减少去噪步数** | 速度提升（50→20 步） | `num_inference_steps=20` |
| **SDXL Turbo / LCM** | 1-4 步生成 | 蒸馏技术，牺牲少量质量换极快速度 |

### 4.6 常见陷阱

1. **CFG 值太高**：`guidance_scale` 超过 15 容易出现过饱和、失真。默认 7.5 是个好起点。
2. **忽略负面 prompt**：Stable Diffusion 支持 `negative_prompt`（描述不想要的内容），合理使用可以显著提升质量。
3. **分辨率不匹配**：SD 1.5 训练分辨率是 512，用 768 生成容易出现重复结构（如两个头）。SDXL 训练分辨率是 1024，更灵活。
4. **随机种子的重要性**：同一 prompt + 同一种子 = 完全相同的图像。调试时固定种子（`generator=torch.manual_seed(42)`），生产时随机。
5. **VAE 解码瓶颈**：VAE 解码虽然只跑一次，但在大分辨率下可能成为显存瓶颈。用 `enable_vae_tiling()` 分块解码解决。

## 本节小结

| 概念 | 一句话总结 |
|------|-----------|
| U-Net 骨干 | 编码器-解码器 + 跳跃连接，通过时间步嵌入和 Cross-Attention 适配为扩散去噪网络 |
| DiT | 用 Transformer 替代 U-Net，通过 Patchify + AdaLN-Zero 实现，与 Scaling Laws 兼容 |
| Latent Diffusion | 在 VAE 压缩后的潜空间做扩散，维度降低约 48 倍，使高分辨率生成成为可能 |
| Stable Diffusion | CLIP（文本编码）+ U-Net/DiT（潜空间去噪）+ VAE（图像还原）的三件套 |
| ControlNet | 通过冻结原始 U-Net + 训练控制分支，实现边缘/深度/姿态等精细空间控制 |
| LoRA | 低秩分解的轻量微调方法，用不到 1% 的参数实现风格定制 |
| Diffusers | Hugging Face 的扩散模型标准库，提供 Pipeline/Model/Scheduler 三层抽象 |
