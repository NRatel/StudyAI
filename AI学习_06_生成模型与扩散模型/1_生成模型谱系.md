> **前置知识**：CNN 基础（卷积、转置卷积）、神经网络训练流程（前向传播、反向传播、损失函数）。概率论基础可查阅 [附录：数学基础速览](../附录_数学基础速览.md)。
>
> **本节目标**：理解三大经典生成模型——VAE、GAN、Flow——的核心思想、数学原理与工程特点，为后续深入扩散模型奠定基础。

## 1. 直觉与概述

### 1.1 判别模型 vs 生成模型

你训练过的大多数模型（图像分类、情感分析等）都是 **判别模型（Discriminative Model）**：给定输入 $x$，预测标签 $y$，学习条件概率 $P(y|x)$。

**生成模型（Generative Model）** 走的是另一条路：它学习数据本身的分布 $P(x)$。一旦学会了"这个世界的数据长什么样"，就能从中采样，"生成"出新的、从未见过的数据。

```
判别模型：   图片  ──→  "这是猫"         学 P(y|x)
生成模型：   随机噪声  ──→  一张猫的图片    学 P(x)，然后从中采样
```

类比：**判别模型**像鉴定师——给一幅画，告诉你是哪个画家的风格；**生成模型**像画家——学会某种风格后，自己能画出新的画。

### 1.2 为什么生成模型重要？

| 能力 | 应用 |
|------|------|
| 生成新样本 | 图像生成（Stable Diffusion）、文本生成（GPT）、音频生成 |
| 数据增强 | 训练数据不足时，用生成模型合成更多训练样本 |
| 异常检测 | 学会正常数据的分布后，偏离分布的就是异常 |
| 表示学习 | 学到的潜空间（latent space）可以用于下游任务 |
| 缺失数据填补 | 知道数据分布后，可以推断缺失部分（图像修复） |

### 1.3 四大生成模型流派

```
生成模型
├── VAE（变分自编码器, 2013）
│     编码到潜空间分布，再从分布中采样解码
│
├── GAN（生成对抗网络, 2014）
│     生成器和判别器对抗博弈
│
├── Flow（流模型, 2014-2018）
│     通过可逆变换直接计算精确似然
│
└── Diffusion（扩散模型, 2015/2020）
      逐步加噪再逐步去噪（下一节详细展开）
```

本节聚焦 VAE、GAN、Flow 三大经典模型。扩散模型将在第 2 节独立展开。

## 2. 严谨定义与原理

### 2.1 VAE（变分自编码器）

#### 2.1.1 从自编码器说起

```
输入 x ──→ [编码器] ──→ 潜变量 z ──→ [解码器] ──→ 重建 x̂
                         (瓶颈层)
```

自编码器让输出 $\hat{x}$ 尽量等于输入 $x$，中间的 $z$ 是低维压缩表示。但 $z$ 的分布没有约束——随机采样一个点去解码，大概率生成的是"垃圾"。**普通自编码器不是生成模型**——它能压缩数据，但不能生成新数据。

#### 2.1.2 VAE 的核心改进

VAE（Kingma & Welling, 2013）对自编码器做了关键改造：**编码器输出概率分布的参数（均值 $\mu$ 和方差 $\sigma^2$），而不是固定的点**。

```
输入 x ──→ [编码器] ──→ μ, σ² ──→ z ~ N(μ, σ²) ──→ [解码器] ──→ 重建 x̂
                         │                │
                         └── KL 正则化 ──→ 约束分布接近 N(0, I)
```

- **编码器**（推断网络）：$q_\phi(z|x) = \mathcal{N}(\mu_\phi(x), \sigma^2_\phi(x))$
- **解码器**（生成网络）：$p_\theta(x|z)$
- **先验分布**：$p(z) = \mathcal{N}(0, I)$（标准正态分布）

#### 2.1.3 ELBO：VAE 的训练目标

直接计算 $p_\theta(x) = \int p_\theta(x|z) p(z) dz$ 需要对所有 $z$ 积分，高维空间中不可行。用**变分推断**推导出可优化的下界——**ELBO（Evidence Lower Bound）**：

$$\log p_\theta(x) \geq \underbrace{\mathbb{E}_{q_\phi(z|x)}[\log p_\theta(x|z)]}_{\text{重建项}} - \underbrace{D_{KL}(q_\phi(z|x) \| p(z))}_{\text{KL 正则项}} = \text{ELBO}$$

| 项 | 直觉 |
|---|---|
| 重建项 | "解码器能从 $z$ 还原出原图吗？"越像越好 |
| KL 正则项 | "潜空间要接近标准正态"，这样才能从标准正态采样来生成 |

实际计算中，重建项用 MSE 或 BCE，KL 项对高斯有**解析解**：

$$D_{KL} = -\frac{1}{2} \sum_{j=1}^{d} \left(1 + \log \sigma_j^2 - \mu_j^2 - \sigma_j^2\right)$$

#### 2.1.4 重参数化技巧

采样操作 $z \sim \mathcal{N}(\mu, \sigma^2)$ 是随机的，梯度无法通过随机节点反向传播。**重参数化技巧**将随机性分离出来：

$$z = \mu + \sigma \odot \epsilon, \quad \epsilon \sim \mathcal{N}(0, I)$$

$\epsilon$ 不依赖模型参数，$z$ 对 $\mu$ 和 $\sigma$ 的梯度可以正常计算，让 VAE 能够端到端训练。

#### 2.1.5 VAE 的优缺点

| 优点 | 缺点 |
|------|------|
| 理论框架优雅，有明确的概率解释 | 生成图像模糊（重建损失倾向于取平均） |
| 潜空间平滑、连续，支持插值和算术 | KL 项和重建项的平衡需要仔细调节 |
| 训练稳定 | 生成质量不如 GAN |

### 2.2 GAN（生成对抗网络）

#### 2.2.1 核心思想：对抗博弈

GAN（Goodfellow et al., 2014）：一个**伪造者**（生成器 G）和一个**鉴定师**（判别器 D）博弈——伪造者越来越厉害，鉴定师也越来越精准——最终伪造者能生成以假乱真的作品。

```
随机噪声 z ~ N(0, I)                  真实数据 x
        │                                  │
        ▼                                  ▼
    [生成器 G] ──→ 生成数据 G(z) ──→ [判别器 D] ←── 真实数据 x
                                        │
                                        ▼
                                 真/假？ D(·) ∈ [0, 1]
```

#### 2.2.2 GAN 的目标函数

$$\min_G \max_D \; V(D, G) = \mathbb{E}_{x \sim p_{\text{data}}}[\log D(x)] + \mathbb{E}_{z \sim p_z}[\log(1 - D(G(z)))]$$

- **D 最大化 $V$**：真实数据上 $D(x) \to 1$（判真），生成数据上 $D(G(z)) \to 0$（判假）
- **G 最小化 $V$**：让 $D(G(z)) \to 1$（骗过判别器）

理论上，当 $p_G = p_{\text{data}}$ 时达到 Nash 均衡，$D(x) = 0.5$（完全分不出真假）。

> **实践技巧**：G 的损失常用 $-\log D(G(z))$ 代替 $\log(1 - D(G(z)))$，因为前者在训练初期梯度更强。

#### 2.2.3 GAN 的训练困难

**模式坍塌（Mode Collapse）**：生成器"偷懒"，只学会生成几种样本，忽略多样性。类比：伪造者发现一种"万能赝品"就能骗过鉴定师，不再尝试其他风格。

**训练不收敛**：D 太强，G 学不到有效梯度；D 太弱，G 得不到有效反馈。需要仔细平衡两者的学习速率和更新频率。

#### 2.2.4 GAN 的优缺点

| 优点 | 缺点 |
|------|------|
| 生成质量高，图像锐利 | 训练不稳定（模式坍塌、不收敛） |
| 生成速度快（只需一次前向传播） | 没有显式的密度估计 |
| 激发大量后续工作（DCGAN, WGAN, StyleGAN） | 超参数敏感，调参困难 |

---

### 2.3 Flow（流模型）

#### 2.3.1 核心思想：可逆变换

找到一个**可逆变换** $f$，把简单分布（标准正态）直接映射到复杂的数据分布：

- **生成**：$z \sim \mathcal{N}(0, I)$，通过 $x = f(z)$ 得到数据
- **推断**：$z = f^{-1}(x)$
- **密度估计**：通过变量替换公式精确计算 $p(x)$

#### 2.3.2 变量替换公式

$$\log p(x) = \log p(z) + \log \left| \det \frac{\partial f^{-1}(x)}{\partial x} \right|$$

Jacobian 行列式衡量变换对局部体积的拉伸/压缩率。直觉：概率密度在变换下守恒，拉伸的区域密度变低，压缩的区域密度变高。

实践中 $f$ 由多个简单可逆变换串联：$x = f_K \circ f_{K-1} \circ \cdots \circ f_1(z)$。每个 $f_k$ 需要：可逆 + Jacobian 行列式可高效计算。

代表方法：**NICE**（2014，加性耦合层）、**RealNVP**（2016，仿射耦合层）、**Glow**（2018，可逆 1x1 卷积）。

#### 2.3.3 Flow 的优缺点

| 优点 | 缺点 |
|------|------|
| 精确的对数似然，训练稳定 | 可逆性约束限制架构（输入输出维度必须相同） |
| 推断和生成都是精确的 | 计算成本高，生成质量通常不如 GAN |

---

### 2.4 对比总结表

| 维度 | VAE | GAN | Flow |
|------|-----|-----|------|
| **核心思想** | 编码到分布 + 解码重建 | 生成器与判别器对抗 | 可逆变换 + 精确似然 |
| **训练目标** | 最大化 ELBO | minimax 博弈 | 最大化精确对数似然 |
| **训练稳定性** | 稳定 | 不稳定 | 稳定 |
| **生成质量** | 模糊 | 锐利、逼真 | 中等 |
| **显式密度** | 近似下界 | 无 | 精确 |
| **生成速度** | 快 | 快 | 快 |
| **代表作** | VAE, beta-VAE, VQ-VAE | DCGAN, WGAN, StyleGAN | NICE, RealNVP, Glow |

> **一句话**：VAE 稳定但模糊，GAN 锐利但难训，Flow 精确但受限。扩散模型兼具训练稳定和高质量，成为当前主流。

## 3. Python 代码示例

### 3.1 PyTorch 实现简单 VAE（MNIST）

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader
from torchvision import datasets, transforms
import matplotlib
matplotlib.use("Agg")
import matplotlib.pyplot as plt

# ============================================================
# 超参数
# ============================================================
LATENT_DIM = 20       # 潜空间维度
HIDDEN_DIM = 400      # 隐藏层维度
BATCH_SIZE = 128
EPOCHS     = 10
LR         = 1e-3
DEVICE = torch.device("cuda" if torch.cuda.is_available() else "cpu")

# ============================================================
# 数据加载
# ============================================================
train_loader = DataLoader(
    datasets.MNIST("./data", train=True, download=True,
                   transform=transforms.ToTensor()),
    batch_size=BATCH_SIZE, shuffle=True,
)

# ============================================================
# VAE 模型
# ============================================================
class VAE(nn.Module):
    """编码器: 784→400→(μ,logσ²) 各20维 | 解码器: 20→400→784"""
    def __init__(self):
        super().__init__()
        # 编码器
        self.encoder_fc = nn.Linear(784, HIDDEN_DIM)
        self.fc_mu      = nn.Linear(HIDDEN_DIM, LATENT_DIM)   # 均值 μ
        self.fc_logvar  = nn.Linear(HIDDEN_DIM, LATENT_DIM)   # log(σ²)
        # 为什么输出 log(σ²)？方差必须>0，输出 log 值范围 (-∞,+∞) 更好学
        # 解码器
        self.decoder_fc1 = nn.Linear(LATENT_DIM, HIDDEN_DIM)
        self.decoder_fc2 = nn.Linear(HIDDEN_DIM, 784)

    def encode(self, x):
        h = torch.relu(self.encoder_fc(x))
        return self.fc_mu(h), self.fc_logvar(h)

    def reparameterize(self, mu, logvar):
        """重参数化: z = μ + σ*ε, ε~N(0,I)"""
        std = torch.exp(0.5 * logvar)
        return mu + std * torch.randn_like(std)

    def decode(self, z):
        h = torch.relu(self.decoder_fc1(z))
        return torch.sigmoid(self.decoder_fc2(h))

    def forward(self, x):
        mu, logvar = self.encode(x)
        z = self.reparameterize(mu, logvar)
        return self.decode(z), mu, logvar

# ============================================================
# 损失函数：负 ELBO = 重建损失(BCE) + KL 散度(解析解)
# ============================================================
def vae_loss(x_recon, x, mu, logvar):
    recon = nn.functional.binary_cross_entropy(x_recon, x, reduction="sum")
    kl    = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return (recon + kl) / x.size(0)

# ============================================================
# 训练
# ============================================================
model = VAE().to(DEVICE)
optimizer = optim.Adam(model.parameters(), lr=LR)
print(f"设备: {DEVICE} | 参数量: {sum(p.numel() for p in model.parameters()):,}\n")

for epoch in range(1, EPOCHS + 1):
    model.train()
    total_loss = 0.0
    for images, _ in train_loader:
        images = images.view(-1, 784).to(DEVICE)
        x_recon, mu, logvar = model(images)
        loss = vae_loss(x_recon, images, mu, logvar)
        optimizer.zero_grad(); loss.backward(); optimizer.step()
        total_loss += loss.item()
    print(f"Epoch [{epoch}/{EPOCHS}]  Loss: {total_loss / len(train_loader):.2f}")

# ============================================================
# 从标准正态采样 z，通过解码器生成新图像
# ============================================================
model.eval()
with torch.no_grad():
    z = torch.randn(16, LATENT_DIM).to(DEVICE)
    generated = model.decode(z).view(-1, 1, 28, 28).cpu()

fig, axes = plt.subplots(2, 8, figsize=(12, 3))
for i, ax in enumerate(axes.flat):
    ax.imshow(generated[i, 0], cmap="gray"); ax.axis("off")
fig.suptitle("VAE Generated Digits")
plt.tight_layout(); plt.savefig("vae_generated.png", dpi=100)
print("生成结果已保存到 vae_generated.png")
```

**预期输出**（数值因随机种子略有不同）：

```
设备: cuda | 参数量: 646,420

Epoch [1/10]   Loss: 163.28
Epoch [2/10]   Loss: 120.43
...
Epoch [10/10]  Loss: 104.52
生成结果已保存到 vae_generated.png
```

**代码要点**：

1. **编码器输出 $\mu$ 和 $\log\sigma^2$**（两个分支），而不是一个固定的 $z$
2. **重参数化**：`z = mu + std * eps`，让梯度能穿过采样操作
3. **损失 = BCE（重建）+ KL（正则）**
4. **生成**：只用解码器——从 $\mathcal{N}(0, I)$ 采样 $z$，送入 `decode(z)`

### 3.2 PyTorch 实现简单 GAN（MNIST）

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader
from torchvision import datasets, transforms
import matplotlib
matplotlib.use("Agg")
import matplotlib.pyplot as plt

# ============================================================
# 超参数
# ============================================================
NOISE_DIM  = 100
BATCH_SIZE = 128
EPOCHS     = 50
LR         = 2e-4
DEVICE = torch.device("cuda" if torch.cuda.is_available() else "cpu")

# ============================================================
# 数据：归一化到 [-1,1]，匹配 G 最后的 Tanh 输出范围
# ============================================================
train_loader = DataLoader(
    datasets.MNIST("./data", train=True, download=True,
                   transform=transforms.Compose([
                       transforms.ToTensor(),
                       transforms.Normalize((0.5,), (0.5,)),
                   ])),
    batch_size=BATCH_SIZE, shuffle=True, drop_last=True,
)

# ============================================================
# 生成器：噪声(100) → 256 → 512 → 1024 → 784
# ============================================================
class Generator(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(NOISE_DIM, 256),  nn.LeakyReLU(0.2), nn.BatchNorm1d(256),
            nn.Linear(256, 512),        nn.LeakyReLU(0.2), nn.BatchNorm1d(512),
            nn.Linear(512, 1024),       nn.LeakyReLU(0.2), nn.BatchNorm1d(1024),
            nn.Linear(1024, 784),       nn.Tanh(),  # 输出 [-1, 1]
        )
    def forward(self, z):
        return self.net(z)

# ============================================================
# 判别器：图像(784) → 1024 → 512 → 256 → 1
# ============================================================
class Discriminator(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(784, 1024),  nn.LeakyReLU(0.2), nn.Dropout(0.3),
            nn.Linear(1024, 512),  nn.LeakyReLU(0.2), nn.Dropout(0.3),
            nn.Linear(512, 256),   nn.LeakyReLU(0.2), nn.Dropout(0.3),
            nn.Linear(256, 1),     nn.Sigmoid(),  # 输出概率 [0, 1]
        )
    def forward(self, x):
        return self.net(x)

# ============================================================
# 初始化
# ============================================================
G = Generator().to(DEVICE)
D = Discriminator().to(DEVICE)
# beta1=0.5 是 GAN 的经典设置，降低动量，适应交替训练
opt_G = optim.Adam(G.parameters(), lr=LR, betas=(0.5, 0.999))
opt_D = optim.Adam(D.parameters(), lr=LR, betas=(0.5, 0.999))
criterion = nn.BCELoss()

print(f"G 参数量: {sum(p.numel() for p in G.parameters()):,}")
print(f"D 参数量: {sum(p.numel() for p in D.parameters()):,}\n")

# ============================================================
# 训练：每个 batch 先更新 D，再更新 G
# ============================================================
for epoch in range(1, EPOCHS + 1):
    epoch_loss_G, epoch_loss_D = 0.0, 0.0
    for real_images, _ in train_loader:
        real_images = real_images.view(-1, 784).to(DEVICE)
        bs = real_images.size(0)
        ones  = torch.ones(bs, 1, device=DEVICE)
        zeros = torch.zeros(bs, 1, device=DEVICE)

        # ---- 训练 D：真实→1，生成→0 ----
        z = torch.randn(bs, NOISE_DIM, device=DEVICE)
        fake = G(z).detach()  # detach：不让 D 的梯度回流到 G
        loss_D = criterion(D(real_images), ones) + criterion(D(fake), zeros)
        opt_D.zero_grad(); loss_D.backward(); opt_D.step()

        # ---- 训练 G：让 D 把生成数据判为 1 ----
        z = torch.randn(bs, NOISE_DIM, device=DEVICE)
        loss_G = criterion(D(G(z)), ones)  # -log(D(G(z)))
        opt_G.zero_grad(); loss_G.backward(); opt_G.step()

        epoch_loss_D += loss_D.item(); epoch_loss_G += loss_G.item()

    n = len(train_loader)
    if epoch % 10 == 0 or epoch == 1:
        print(f"Epoch [{epoch}/{EPOCHS}]  "
              f"Loss_D: {epoch_loss_D/n:.4f}  Loss_G: {epoch_loss_G/n:.4f}")

# ============================================================
# 生成新图像
# ============================================================
G.eval()
with torch.no_grad():
    z = torch.randn(16, NOISE_DIM, device=DEVICE)
    generated = (G(z).view(-1, 1, 28, 28).cpu() + 1) / 2  # [-1,1]→[0,1]

fig, axes = plt.subplots(2, 8, figsize=(12, 3))
for i, ax in enumerate(axes.flat):
    ax.imshow(generated[i, 0], cmap="gray"); ax.axis("off")
fig.suptitle("GAN Generated Digits")
plt.tight_layout(); plt.savefig("gan_generated.png", dpi=100)
print("生成结果已保存到 gan_generated.png")
```

**预期输出**：

```
G 参数量: 1,052,432
D 参数量: 1,052,929

Epoch [1/50]   Loss_D: 0.5124  Loss_G: 2.3456
Epoch [10/50]  Loss_D: 0.8932  Loss_G: 1.2134
...
Epoch [50/50]  Loss_D: 1.2567  Loss_G: 0.7890
生成结果已保存到 gan_generated.png
```

**代码要点**：

1. **交替训练**：每个 batch 先更新 D 再更新 G
2. **`.detach()` 是关键**：训练 D 时防止梯度回流到 G
3. **G 的损失用 `ones`**：即 $-\log D(G(z))$，比理论上的 $\log(1-D(G(z)))$ 梯度更强
4. **数据归一化到 [-1,1]**：匹配 G 最后 Tanh 的输出范围

### 3.3 VAE vs GAN 对比实验观察

| 观察点 | VAE | GAN |
|--------|-----|-----|
| 图像清晰度 | 数字轮廓偏模糊 | 笔画更锐利 |
| 多样性 | 0~9 各数字都能生成 | 可能偏向某几个数字 |
| 训练稳定性 | 损失单调下降 | D 和 G 的损失互相拉扯 |
| 训练速度 | 10 epoch 可见效果 | 需要 30~50 epoch |
| 潜空间 | 可在两个 z 间插值，平滑过渡 | 无结构化潜空间 |

## 4. 工程师视角

### 4.1 为什么扩散模型最终胜出

| 模型 | 致命弱点 |
|------|---------|
| VAE | 重建损失取平均 → 图像模糊（数学结构性缺陷，调参无法解决） |
| GAN | 训练不稳定 → 工业产品需要可靠训练流程，调参成本太高 |
| Flow | 可逆性约束 → 模型效率低，难以扩展到高分辨率 |

**扩散模型**（2020 DDPM 之后）的优势：

1. **训练简单**：本质就是训练去噪网络，用 MSE 即可
2. **训练极其稳定**：没有对抗博弈，不会模式坍塌
3. **生成质量极高**：逐步去噪允许每一步做精细调整
4. **理论严谨**：有严格的概率论基础
5. **可扩展性好**：轻松扩展到高分辨率、条件生成、视频生成

唯一代价：**采样慢**（需 20~1000 步去噪），但 DDIM、Consistency Model 等正在解决。

```
       训练稳定性       生成质量        理论基础
VAE:     好              差(模糊)        好
GAN:     差(不稳定)       好              差(无密度)
Flow:    好              中等            好
Diff:    好              好              好
```

### 4.2 VAE 在 Stable Diffusion 中的角色

VAE 生成模糊，但在 Stable Diffusion 中**不做生成，做压缩**：

```
文本 "a cat on the beach"
    │
    ▼
[CLIP 文本编码器] → 文本嵌入
    │
    ▼
[U-Net 扩散模型]   在"潜空间"中去噪 (64×64，不是 512×512)
    │
    ▼
[VAE 解码器]       潜变量 → 高分辨率图像 (512×512)
```

VAE 编码器把 512x512 图像压缩到 64x64 潜变量（约 48 倍），扩散模型在低维空间操作——这就是 **Latent Diffusion Model (LDM)** 的核心思想。计算量减少约 48 倍，消费级 GPU 也能跑。

VAE 的"模糊"在这里不是问题：扩散模型负责创造力，VAE 只负责维度转换。

> **VQ-VAE**：把连续潜空间替换为离散"码本"，DALL-E 第一版使用了这个变体。它使图像生成可以转化为序列预测问题（Transformer 逐个预测离散 token）。

### 4.3 选型指南

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 高质量图像生成 | 扩散模型 | 当前最佳质量 |
| 实时生成（低延迟） | GAN 或蒸馏后的扩散模型 | 单次前向，延迟低 |
| 结构化潜空间（插值、编辑） | VAE 或扩散模型 | 平滑潜空间支持语义操控 |
| 密度估计（异常检测） | Flow 模型 | 精确似然值 |
| 工业部署、预算有限 | 微调预训练扩散模型 | 不需从头训练 |

### 4.4 里程碑时间线

```
2013  VAE (Kingma & Welling)        变分生成模型的基础
2014  GAN (Goodfellow et al.)       对抗训练的开创性工作
2014  NICE (Dinh et al.)            第一个实用 Flow 模型
2015  DCGAN (Radford et al.)        GAN + CNN，质量大幅提升
2016  RealNVP (Dinh et al.)         仿射耦合层，Flow 代表作
2017  VQ-VAE (van den Oord et al.)  离散潜空间，连接 VAE 和自回归
2017  WGAN (Arjovsky et al.)        Wasserstein 距离改善 GAN 训练
2018  Glow (Kingma & Dhariwal)      可逆卷积，Flow 的高点
2018  StyleGAN (Karras, NVIDIA)     高分辨率人脸生成里程碑
2020  DDPM (Ho et al.)              扩散模型实用化 → 开启统治时代
2021  DALL-E (OpenAI)               VQ-VAE + Transformer 文生图
2022  Stable Diffusion              LDM: VAE编解码 + U-Net扩散 + CLIP
2023  Consistency Models            解决扩散模型采样慢的问题
2024  SD3 / FLUX                    DiT (Diffusion Transformer) 替代 U-Net
```

## 本节小结

| 概念 | 一句话总结 |
|------|-----------|
| 判别 vs 生成模型 | 判别学 $P(y\|x)$（分类），生成学 $P(x)$（生成新数据） |
| VAE | 编码到潜空间分布，用 ELBO 训练，稳定但模糊 |
| 重参数化技巧 | $z = \mu + \sigma \odot \epsilon$，让采样可微分 |
| GAN | 生成器与判别器对抗博弈，锐利但训练不稳定 |
| 模式坍塌 | GAN 典型问题——只学会几种样本，丧失多样性 |
| Flow | 可逆变换 + 变量替换，精确似然但架构受限 |
| 扩散模型定位 | 兼具训练稳定、高质量、理论严谨，当前主流 |
| VAE 在 SD 中 | 不做生成，做维度压缩——图像压缩到潜空间给扩散模型 |
