# 论文与 FAQ

这个模块建议把论文当成 GPT 能力来源的线索：架构、规模、数据、对齐分别贡献了什么。

## 推荐了解的经典工作

| 主题 | 可以关注什么 |
|------|--------------|
| GPT 系列论文 | Decoder-only 语言模型如何扩展 |
| Scaling Laws | 模型、数据、算力如何影响损失 |
| InstructGPT / RLHF | 预训练模型如何变得更像助手 |
| DPO | 偏好学习如何简化 |
| 多模态与工具使用相关工作 | 模型如何走向系统能力 |

## FAQ

### GPT 为什么能做不是“续写”的任务？

很多任务都能包装成文本续写。问题后面续写答案，代码注释后面续写代码，指令后面续写执行结果。

### 涌现是不是说明模型有意识？

不是。涌现表示某些能力在规模到达一定程度后表现明显，不等于意识或真正理解。

### 对齐会让模型更真实吗？

对齐能改善回答行为，但不能保证事实完全正确。需要 RAG、工具、引用和验证来补可靠性。

### 更大的模型一定更好吗？

不一定。还要看数据质量、训练方法、推理成本、应用场景和安全要求。
