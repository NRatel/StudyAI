## 直觉与概述

### 核心问题

GPT-2 的上下文窗口是 1024 个 token。GPT-4 Turbo 扩展到 128K。Gemini 1.5 Pro 支持 1M+。从 1K 到 1M，增长了 1000 倍。这不仅仅是"把数字调大"——长文本面临两个根本性挑战。

> **关键认知前置：长上下文支持 ≠ 长上下文有效利用。** 模型宣称支持 128K 输入，不代表它能有效利用 128K 中每个位置的信息。研究发现（Liu et al., 2023, "Lost in the Middle"），多数模型对序列**开头和末尾**的信息利用率高，对**中间位置**的信息容易"遗忘"。也就是说，即使你把关键信息放在 128K 上下文的中间区域，模型也可能无法检索到它。2025-2026 年的模型（如 Claude 3.5、GPT-4 Turbo、Llama 3.1）已通过专门的训练策略大幅缓解此问题，但在多针测试（Multi-Needle NIAH）中仍有挑战。因此，"支持 128K"是必要条件，"有效利用 128K"才是真正的能力。

长文本面临两个根本性挑战：

1. **计算与内存的二次方瓶颈**：Self-Attention 的 $QK^T$ 矩阵大小为 $n \times n$，计算量和内存占用都是 $O(n^2)$。序列长度翻倍，计算量翻四倍。
2. **位置编码的外推难题**：模型在 4K 长度上训练，推理时面对 32K 甚至 128K 的序列，位置编码能否正常工作？

### 为什么 O(n^2) 是瓶颈

Self-Attention 的核心是计算 $QK^T$，其中 $Q, K \in \mathbb{R}^{n \times d}$，结果是 $n \times n$ 的注意力矩阵。让我们看看具体数字：

```
序列长度 n    注意力矩阵元素数    FP16 内存占用（单层单头）
    1K         1,000,000               2 MB
    4K        16,000,000              32 MB
   32K     1,024,000,000               2 GB
  128K    16,384,000,000              32 GB        ← 已超过单卡显存
    1M   1,000,000,000,000          2,000 GB       ← 完全不可行
```

实际情况更严重：多头注意力有 $h$ 个头，模型有 $L$ 层，训练还需要保存梯度。即使 FlashAttention 可以将**内存**从 $O(n^2)$ 降到 $O(n)$（不具化注意力矩阵），**计算量**仍然是 $O(n^2 \cdot d)$，绕不过去。

### 三条技术路线总览

| 路线 | 核心思路 | 代表方案 | 本节内容 |
|------|----------|----------|----------|
| 位置编码外推 | 不改模型结构，让 RoPE 支持更长位置 | PI / NTK-Aware / YaRN | 第一部分 |
| 序列并行 | 把长序列切块，分布到多 GPU 并行计算全注意力 | Ring Attention | 第二部分 |
| 压缩记忆 | 将历史信息压缩到固定大小的记忆向量 | Infini-Attention | 第三部分 |

## 严谨定义与原理

### 一、位置编码外推——RoPE 扩展

#### 回顾：RoPE 的核心思想

RoPE（详见 [位置编码](../AI学习_03_注意力机制与Transformer/4_位置编码.md)）对位置 $m$ 处的向量，将每对维度旋转角度 $m \theta_i$：

$$\theta_i = \text{base}^{-2i/d}, \quad \text{其中 } \text{base} = 10000$$

关键性质：位置 $m$ 和 $n$ 的 query-key 内积只依赖相对距离 $m - n$，这就是 RoPE 编码相对位置的方式。

#### 外推失败的原因

训练时模型只见过 $m \in [0, L_{\text{train}})$ 的位置。当推理长度超过训练长度时：

- **高频维度**（$\theta_i$ 较大，$i$ 较小）的旋转角度 $m \theta_i$ 远超训练时见过的范围
- 注意力分数的分布变得异常——模型对这些"从未见过的旋转角度"完全无法正确处理
- **实验表现**：超过训练长度 1.5~2 倍后，困惑度（perplexity）急剧上升

#### 方法 1：位置插值 PI（Position Interpolation, Meta 2023）

**核心思想极其简单**：不让位置 ID 超出训练范围，把更长序列的位置**线性压缩**回训练范围。

如果训练长度 $L = 4096$，要扩展到 $L' = 32768$（8 倍），则将位置 $m$ 替换为：

$$m' = m \cdot \frac{L}{L'} = m / 8$$

位置 0 ~ 32767 被映射到 0 ~ 4095.875，全部落在训练时见过的范围内。

```
原始位置: 0, 1, 2, 3, ..., 32767
PI 映射:  0, 0.125, 0.25, 0.375, ..., 4095.875
                                        ↑
                                   不超过训练范围 4096
```

**优点**：实现极简（一行代码修改位置 ID），效果好，只需 ~1000 步微调即可恢复性能。

**缺点**：所有频率维度被**均匀**压缩。高频维度原本编码相邻位置的细微差别，压缩后分辨率下降——位置 0 和位置 1 的区分度变成了原来的 1/8。

#### 方法 2：NTK-Aware 缩放（Bloc97 2023, 社区发现）

**核心思想**：高频维度编码局部位置，不需要太多缩放；低频维度编码远距离位置，需要大幅拉伸。应该**不均匀缩放**。

实现方式：修改 RoPE 的 base 常数。将 $\text{base} = 10000$ 替换为：

$$\text{base}' = \text{base} \cdot \alpha^{d/(d-2)}, \quad \alpha = L'/L$$

这个修改的效果：
- 高频维度（$i$ 小）：$\theta_i$ 本身就大，base 的变化对它影响小，缩放因子接近 1
- 低频维度（$i$ 大）：$\theta_i$ 本身就小，base 的变化对它影响大，缩放因子接近 $\alpha$

**优点**：无需任何微调即可获得不错的外推效果（"零样本外推"）。
**缺点**：对极长序列（>8 倍训练长度）精度仍有损失。

#### 方法 3：YaRN（Yet another RoPE extensioN, 2023）

**核心思想**：综合 PI 和 NTK-Aware 的优势，并加入注意力温度修正。

YaRN 将频率维度按波长与训练长度的关系分为三组：

| 频率段 | 判断标准 | 处理方式 |
|--------|----------|----------|
| 高频（波长 < 训练长度） | 训练时已见过完整周期 | 不缩放，保持原样 |
| 低频（波长 >> 训练长度） | 训练时远未见过完整周期 | 完全使用 PI 线性插值 |
| 中频（过渡区域） | 介于两者之间 | PI 和 NTK 缩放的线性混合 |

同时对注意力 logits 乘以温度因子 $\sqrt{1/t}$ 来补偿插值导致的注意力分布变化。

**效果**：截至 2025 年，YaRN 是最广泛使用的 RoPE 扩展方法之一。在 Llama、Qwen、Mistral 等模型上都有成功应用。

#### 2025-2026 进展补充

- **LongRoPE (Microsoft, 2024)**：为每个维度搜索最优的独立缩放因子，比 YaRN 更灵活
- **ABF (Adjusted Base Frequency, 2024-2025)**：在预训练阶段直接使用更大的 base（如 500000 而非 10000），从源头减少外推需求。Llama 3.1/3.3、Qwen-2.5 均采用此策略
- **Dynamic NTK (2024-2025)**：推理时根据实际输入长度动态选择缩放因子，短序列不缩放、长序列自适应缩放

### 二、Ring Attention（序列并行）

#### 问题：单 GPU 放不下的长序列

即使位置编码能正确外推，128K 序列的 $QK^T$ 计算量仍然巨大。需要把长序列**分布到多个 GPU 上**。

#### 核心思想（Liu et al., UC Berkeley, 2023）

将长度 $N$ 的序列均匀切成 $P$ 块，$P$ 个 GPU 各持有一块。通过"环形传递"KV 块，让每个 GPU 的 Q 块都与所有 KV 块完成注意力计算：

```
Step 0: 每个 GPU 持有本地 Q/K/V 块
  GPU 0: [Q₀, K₀, V₀]    GPU 1: [Q₁, K₁, V₁]
  GPU 2: [Q₂, K₂, V₂]    GPU 3: [Q₃, K₃, V₃]

Step 1: 每个 GPU 计算本地注意力 Attention(Qᵢ, Kᵢ, Vᵢ)
        同时，将 KV 块沿环传递：GPU 0→1, 1→2, 2→3, 3→0

Step 2: GPU 0 收到 K₃,V₃，计算 Attention(Q₀, K₃, V₃)
        同时继续传递 KV 块

Step 3: 类似地继续...

经过 P 步后: 每个 GPU 完成了 Qᵢ 与全部 K,V 的注意力计算
```

**关键工程技巧**：

1. **通信与计算重叠**：在传递第 $j$ 块 KV 的同时，计算第 $j-1$ 块的注意力。只要计算时间 >= 通信时间，通信开销就被完全隐藏。
2. **Online Softmax**：由于 softmax 需要全局归一化，但每步只看到部分 KV，需要使用在线 softmax 技巧（维护运行中的最大值和指数和），逐步更新结果。
3. **精确性**：数学上与标准全注意力**完全等价**，不是近似。

**效果**：$P$ 个 GPU 可以处理 $P$ 倍长度的序列，每个 GPU 的内存需求降到 $O(N^2/P^2)$（注意力矩阵部分）或更优。

### 三、Infini-Attention（压缩记忆）

#### 问题：序列真的无限长怎么办？

Ring Attention 需要 GPU 数量与序列长度成正比。对于持续对话、流式处理等场景，序列长度没有上界。能否用**固定大小的记忆**处理无限长序列？

#### 核心思想（Munkhdalai et al., Google, 2024）

在标准注意力中嵌入一个**压缩记忆模块**。输入被切成等长的段（segment），每段内做标准注意力，段间通过压缩记忆传递信息：

```
输入序列: [seg₁] [seg₂] [seg₃] ...

处理 seg_t 时:
  ① 段内标准注意力 → A_local         （局部精确）
  ② 用 Q_t 查询压缩记忆 M → A_memory  （全局近似）
  ③ 门控融合: output = β·A_memory + (1-β)·A_local
  ④ 用 K_t, V_t 更新压缩记忆 M        （增量式更新）
```

压缩记忆 $M$ 是一个 $d_{\text{key}} \times d_{\text{value}}$ 的矩阵（大小固定，与序列长度无关）：

- **检索**：$A_{\text{memory}} = \frac{Q_t M_{t-1}}{Q_t z_{t-1}}$，其中 $z$ 是归一化项
- **更新**：$M_t = M_{t-1} + K_t^T V_t$（线性注意力风格的增量更新）
- **门控**：可学习参数 $\beta$ 控制"用局部注意力还是全局记忆"

**本质**：Infini-Attention 在段内保留了完整注意力的精度，段间用线性注意力的形式做近似压缩——是精度与效率的折中。

| 特性 | 值 |
|------|-----|
| 段内复杂度 | $O(s^2)$，$s$ 为段长度 |
| 段间记忆大小 | $O(d^2)$，固定不变 |
| 总复杂度 | $O(n \cdot s)$，$n$ 为总长度 |
| 信息保真度 | 段内精确，段间有损压缩 |

### 示例 1：RoPE 扩展方案的对比

```python
import torch
import math

def rope_frequencies(dim: int, seq_len: int, base: float = 10000.0):
    """标准 RoPE：计算每个 (位置, 维度对) 的旋转角度。"""
    freqs = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
    positions = torch.arange(seq_len).float()
    angles = torch.outer(positions, freqs)  # (seq_len, dim//2)
    return angles

def pi_frequencies(dim, target_len, train_len=4096, base=10000.0):
    """Position Interpolation：线性压缩位置索引。"""
    freqs = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
    scale = train_len / target_len
    positions = torch.arange(target_len).float() * scale  # 压缩！
    return torch.outer(positions, freqs)

def ntk_aware_frequencies(dim, target_len, train_len=4096, base=10000.0):
    """NTK-Aware：修改 base 实现不均匀缩放。"""
    alpha = target_len / train_len
    new_base = base * alpha ** (dim / (dim - 2))
    freqs = 1.0 / (new_base ** (torch.arange(0, dim, 2).float() / dim))
    positions = torch.arange(target_len).float()
    return torch.outer(positions, freqs)

# --- 对比分析 ---
dim = 64
train_len = 4096
target_len = 32768  # 8 倍扩展

angles_standard = rope_frequencies(dim, target_len)
angles_pi = pi_frequencies(dim, target_len, train_len)
angles_ntk = ntk_aware_frequencies(dim, target_len, train_len)

# 训练范围内的最大旋转角度（作为参考基线）
angles_train = rope_frequencies(dim, train_len)
max_train_high = angles_train[-1, 0].item()   # 高频维度
max_train_low = angles_train[-1, -1].item()   # 低频维度

print(f"训练长度: {train_len}, 目标长度: {target_len} (扩展 {target_len/train_len:.0f} 倍)")
print(f"\n训练范围内最大旋转角度:")
print(f"  高频维度(dim 0): {max_train_high:.2f} rad")
print(f"  低频维度(dim {dim//2-1}): {max_train_low:.4f} rad")

print(f"\n位置 {target_len-1} 处各方案的旋转角度:")
print(f"{'方案':<20} {'高频(dim 0)':<15} {'低频(dim last)':<15} {'高频是否越界'}")
print("-" * 70)

for name, angles in [
    ("直接外推(会崩)", angles_standard),
    ("PI 插值", angles_pi),
    ("NTK-Aware", angles_ntk),
]:
    high = angles[-1, 0].item()
    low = angles[-1, -1].item()
    oob = "!!! 越界" if high > max_train_high * 1.1 else "安全"
    print(f"{name:<20} {high:<15.2f} {low:<15.4f} {oob}")

# 可视化不同方案在各维度上的缩放效果
print(f"\n各维度的缩放因子 (target/train 最大旋转角度之比):")
print(f"{'维度对':<8} {'直接外推':<12} {'PI':<12} {'NTK-Aware':<12}")
for idx in [0, dim//8, dim//4, 3*dim//8, dim//2-1]:
    ratio_std = angles_standard[-1, idx].item() / angles_train[-1, idx].item()
    ratio_pi = angles_pi[-1, idx].item() / angles_train[-1, idx].item()
    ratio_ntk = angles_ntk[-1, idx].item() / angles_train[-1, idx].item()
    print(f"  {idx:<6} {ratio_std:<12.2f} {ratio_pi:<12.2f} {ratio_ntk:<12.2f}")
```

### 示例 2：Ring Attention 的简化模拟

```python
import torch
import torch.nn.functional as F
import math

def standard_attention(Q, K, V):
    """标准全注意力（用于验证正确性）。"""
    d_k = Q.shape[-1]
    scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)
    return F.softmax(scores, dim=-1) @ V

def ring_attention_simulate(Q_blocks, K_blocks, V_blocks):
    """
    模拟 Ring Attention 的环形计算过程。

    真实实现中：
    - 每个块在不同 GPU 上
    - KV 传递与下一轮计算重叠执行（隐藏通信开销）
    - 使用 FlashAttention 作为块内注意力的计算内核

    这里在 CPU 上串行模拟，重点展示 online softmax 的正确性。
    """
    num_devices = len(Q_blocks)
    outputs = []

    for i in range(num_devices):
        Q_i = Q_blocks[i]
        d_k = Q_i.shape[-1]
        block_len = Q_i.shape[0]

        # Online softmax 状态
        running_max = torch.full((block_len, 1), float('-inf'))
        running_sum = torch.zeros(block_len, 1)
        running_output = torch.zeros_like(Q_i)

        for step in range(num_devices):
            # 环形传递：第 i 个 GPU 在第 step 步收到第 (i+step)%P 块
            src = (i + step) % num_devices
            K_j, V_j = K_blocks[src], V_blocks[src]

            # 局部注意力分数
            scores = Q_i @ K_j.transpose(-2, -1) / math.sqrt(d_k)

            # Online softmax 更新
            block_max = scores.max(dim=-1, keepdim=True).values
            new_max = torch.maximum(running_max, block_max)

            # 修正历史累积值
            old_scale = torch.exp(running_max - new_max)
            running_output = running_output * old_scale
            running_sum = running_sum * old_scale

            # 加入当前块
            exp_scores = torch.exp(scores - new_max)
            running_output = running_output + exp_scores @ V_j
            running_sum = running_sum + exp_scores.sum(dim=-1, keepdim=True)
            running_max = new_max

        outputs.append(running_output / running_sum)

    return torch.cat(outputs, dim=0)

# --- 验证 ---
torch.manual_seed(42)
total_len, dim, num_gpus = 32, 16, 4
block_size = total_len // num_gpus

Q = torch.randn(total_len, dim)
K = torch.randn(total_len, dim)
V = torch.randn(total_len, dim)

out_std = standard_attention(
    Q.unsqueeze(0), K.unsqueeze(0), V.unsqueeze(0)
).squeeze(0)

Q_b = [Q[i*block_size:(i+1)*block_size] for i in range(num_gpus)]
K_b = [K[i*block_size:(i+1)*block_size] for i in range(num_gpus)]
V_b = [V[i*block_size:(i+1)*block_size] for i in range(num_gpus)]
out_ring = ring_attention_simulate(Q_b, K_b, V_b)

max_err = (out_std - out_ring).abs().max().item()
print(f"序列长度: {total_len}, 分成 {num_gpus} 块 (每块 {block_size})")
print(f"Ring Attention vs 标准 Attention 最大误差: {max_err:.2e}")
print(f"数学上精确等价: {max_err < 1e-5}")

print(f"\n实际场景示例:")
print(f"  序列 128K, 8 GPU: 每 GPU 处理 16K token")
print(f"  每 GPU 注意力矩阵: 16K x 16K = 256M (而非 128K x 128K = 16G)")
print(f"  显存减少: {(128*128) / (16*16):.0f} 倍")
```

### 示例 3：Infini-Attention 的压缩记忆

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class InfiniAttentionLayer(nn.Module):
    """
    Infini-Attention 的简化实现。
    段内做标准注意力，段间通过压缩记忆传递信息。
    """
    def __init__(self, dim, segment_len=64):
        super().__init__()
        self.dim = dim
        self.segment_len = segment_len

        self.W_q = nn.Linear(dim, dim, bias=False)
        self.W_k = nn.Linear(dim, dim, bias=False)
        self.W_v = nn.Linear(dim, dim, bias=False)
        self.W_o = nn.Linear(dim, dim, bias=False)

        # 门控参数：学习 "局部注意力 vs 记忆检索" 的比例
        self.gate = nn.Parameter(torch.tensor(0.0))  # sigmoid(0)=0.5

    def forward(self, x):
        B, N, D = x.shape
        Q, K, V = self.W_q(x), self.W_k(x), self.W_v(x)

        # 初始化压缩记忆 M 和归一化项 z
        M = torch.zeros(B, D, D, device=x.device)
        z = torch.zeros(B, D, 1, device=x.device)

        outputs = []
        num_segs = math.ceil(N / self.segment_len)

        for s in range(num_segs):
            lo = s * self.segment_len
            hi = min(lo + self.segment_len, N)
            Qs, Ks, Vs = Q[:, lo:hi], K[:, lo:hi], V[:, lo:hi]

            # ① 段内标准注意力
            scores = Qs @ Ks.transpose(-2, -1) / math.sqrt(D)
            A_local = F.softmax(scores, dim=-1) @ Vs

            # ② 从压缩记忆检索（线性注意力风格）
            A_mem = (Qs @ M) / (Qs @ z + 1e-6)

            # ③ 门控融合
            beta = torch.sigmoid(self.gate)
            seg_out = beta * A_mem + (1 - beta) * A_local
            outputs.append(seg_out)

            # ④ 更新压缩记忆
            M = M + Ks.transpose(-2, -1) @ Vs
            z = z + Ks.sum(dim=1).unsqueeze(-1)

        return self.W_o(torch.cat(outputs, dim=1))

# --- 演示 ---
torch.manual_seed(42)
dim, seq_len, seg_len = 32, 512, 64

model = InfiniAttentionLayer(dim=dim, segment_len=seg_len)
x = torch.randn(2, seq_len, dim)
out = model(x)

print(f"输入形状: {list(x.shape)}")
print(f"输出形状: {list(out.shape)}")
print(f"段长: {seg_len}, 总段数: {seq_len // seg_len}")
print(f"门控 beta = {torch.sigmoid(model.gate).item():.3f}")
print(f"\n压缩记忆大小: {dim} x {dim} = {dim**2} 个参数 (固定)")
print(f"标准注意力矩阵: {seq_len} x {seq_len} = {seq_len**2} 个元素")
print(f"内存节省比: {seq_len**2 / dim**2:.0f} 倍")
print(f"\n无论序列多长，记忆大小始终是 {dim}x{dim}——这就是'Infini'的含义")
```

## 工程师视角

### 各方案选型指南

| 场景 | 推荐方案 | 理由 |
|------|----------|------|
| 已有模型，需推理时支持更长输入 | YaRN / Dynamic NTK | 免训练或轻量微调，最低成本方案 |
| 预训练时就规划长上下文 | 大 base RoPE + Ring Attention | 从训练阶段原生支持，效果最好 |
| 持续对话 / 流式场景 | Infini-Attention / 滑动窗口 + 摘要 | 固定显存，理论上无限长度 |
| 超长文档（>128K）精确检索 | 长上下文 + RAG 互补 | 纯长上下文有"大海捞针"退化 |

### 长上下文 vs RAG：互补而非替代

"有了 128K 甚至 1M 上下文窗口，还需要 RAG 吗？"答案是**需要**。

| 维度 | 长上下文 | RAG |
|------|----------|-----|
| 信息利用方式 | 端到端注意力，模型自行发现相关信息 | 依赖外部检索器，可能遗漏 |
| 推理成本 | 输入越长，成本越高（$O(n^2)$ 或 $O(n)$） | 只输入相关片段，成本可控 |
| 精确检索能力 | 128K+ 时中间区域容易被忽略 | 向量检索精度独立于文档总长 |
| 知识更新 | 需要全部重新输入 | 只需更新向量库 |

**最佳实践**：RAG 负责"找到最相关的信息"，长上下文负责"在相关信息之间做推理"。

### "大海捞针" (Needle In A Haystack) 测试

这是评估长文本能力的标准基准：在长文档的随机位置插入一条信息，测试模型能否检索到。

截至 2026 年的关键发现：
- 多数模型在**序列开头和末尾**表现最好，**中间区域**有明显退化（"Lost in the Middle"）
- 专门训练过长文本的模型（Claude 3/3.5、GPT-4 Turbo、Llama 3.1）此问题已大幅缓解
- **多针测试**（Multi-Needle NIAH，同时插入多条信息）中仍有挑战
- 2025 年的 Qwen-2.5、Llama 3.3 在 128K 单针测试上接近满分

### RoPE 扩展方案的实用选择

```
需要扩展上下文？
├─ 能否微调（哪怕 1000 步）？
│   ├─ 是 → YaRN（效果最好）或 PI（最简单）
│   │       用扩展后长度的数据微调 1000-2000 步
│   └─ 否（纯推理时扩展）
│       └─ Dynamic NTK（根据实际长度自适应缩放）
└─ 从头预训练？
    └─ 直接用大 base（如 500000）+ 长序列数据
       Llama 3.1 / Qwen-2.5 / DeepSeek-V3 均采用此路线
```

## 本章小结

| 概念 | 一句话总结 |
|------|-----------|
| **O(n^2) 瓶颈** | Self-Attention 的 $QK^T$ 大小与序列长度平方成正比，128K 时已不可承受 |
| **RoPE 外推失败** | 训练时未见过的旋转角度导致注意力分数异常、困惑度飙升 |
| **PI（位置插值）** | 线性压缩位置索引到训练范围内，简单有效，需轻量微调 |
| **NTK-Aware** | 修改 RoPE base 实现不均匀缩放，高频少动低频多拉，免训练 |
| **YaRN** | 分频段混合 PI + NTK + 温度修正，综合效果最佳 |
| **Ring Attention** | 将序列分块到多 GPU，环形传递 KV，数学上与全注意力精确等价 |
| **Infini-Attention** | 段内全注意力 + 段间固定大小压缩记忆，理论上支持无限长序列 |
| **长上下文 vs RAG** | 互补关系：RAG 负责检索，长上下文负责推理 |
