# 高效 Transformer

> **前置知识**：[长文本技术](1_长文本技术.md)
>
> **本节目标**：理解高效 Transformer 和新架构主要在优化计算、显存和长序列处理。

---

## 1. Transformer 强，但不便宜

Transformer 的 Self-Attention 很强，因为每个 token 可以看其他 token。

但序列一长，计算和显存都会变贵。

高效架构的目标就是：

> 尽量保留建模能力，同时减少计算和内存成本。

## 2. FlashAttention：把注意力算得更省

FlashAttention 不改变注意力的数学结果，主要改变计算方式。

它通过更好地利用 GPU 内存层次，减少显存读写，从而让注意力计算更快、更省。

直白地说：不是换题，而是把同一道题算得更聪明。

## 3. 稀疏注意力：不是每个人都看所有人

稀疏注意力会限制 token 之间的连接。

比如只看附近窗口，或只让部分关键 token 全局可见。

这样能减少计算，但也可能漏掉远距离关系。

## 4. 线性注意力和状态空间模型

一些方法试图避免标准注意力的二次成本。

线性注意力、SSM、Mamba 等路线都在探索：能不能用更低成本处理长序列。

它们各有优势和限制，不是简单替代 Transformer，而是在特定场景中提供更高效选择。

## 5. 高效不等于免费

所有高效方法都在做取舍：

- 更快可能牺牲部分表达能力。
- 更省显存可能增加实现复杂度。
- 更适合长序列不一定适合所有任务。

理解这些方法时，重点看它们优化的是长度、速度、显存，还是模型容量。
