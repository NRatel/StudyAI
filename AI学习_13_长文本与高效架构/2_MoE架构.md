# MoE 架构

> **前置知识**：[GPT 系列深度解析](../AI学习_05_GPT系列深度解析/README.md)
>
> **本节目标**：理解 MoE 为什么可以增加模型容量，同时不让每次推理都计算全部参数。

---

## 1. MoE：很多专家，但每次只请少数几个

MoE 是 Mixture of Experts，混合专家。

它的核心想法是：

> 模型里有很多专家网络，但每个 token 只激活其中少数几个。

这样模型总容量很大，但单次计算成本不必跟着全量增长。

{{img:ch13_02_moe_router_experts}}

## 2. Router：谁来决定找哪个专家

Router 负责给每个 token 分配专家。

比如某些 token 更适合语言专家，某些更适合代码专家，某些更适合数学模式。

真实模型里的专家不一定能这么清晰命名，但直觉上可以理解为不同子网络处理不同模式。

## 3. 为什么 MoE 省计算

普通密集模型每次都会用到所有参数。

MoE 虽然总参数很多，但每个 token 只走少数专家。

这叫**稀疏激活**：参数总量大，实际计算少。

## 4. MoE 的难点

MoE 不是白捡容量。

它会带来：

- 路由训练不稳定。
- 专家负载不均衡。
- 分布式通信复杂。
- 部署和推理调度更难。

如果很多 token 都挤向同一个专家，系统效率会下降。

## 5. 怎么理解 MoE 的价值

MoE 的价值是扩展模型容量，同时控制计算成本。

它适合大规模模型，但也让训练和部署更像复杂系统工程。

所以看 MoE，不要只看参数总量，还要看激活参数、推理成本和路由效果。
